# 제텔카스텐 노트 품질 검토

작성된 제텔카스텐 노트(단일 파일 또는 특정 폴더 내의 전체 노트)가 기술사 시험 기반지식 노트로서 적절히 작성되었는지 검토합니다.

## 사용법

```
/check-note $ARGUMENTS
```

`$ARGUMENTS`에 검토할 노트의 **파일 경로** 또는 **폴더 경로**를 입력하세요. 경로를 지정하지 않으면 현재 열려있는(Active) 문서를 대상으로 실행합니다.

예시:
- `/check-note content/docs/peim/01_InfoStrategy/04_IT_Audit/` (해당 폴더 내 모든 노트 일괄 검토)
- `/check-note content/docs/peim/01_InfoStrategy/04_IT_Audit/pmo.md` (단일 노트 검토)
- `/check-note` (현재 열린 문서 검토)

---

## 실행 지침

### 1단계: 대상 파일 파악

- 입력된 경로가 **폴더**인 경우: 해당 폴더 및 하위 폴더의 모든 `.md` 파일 목록을 수집합니다.
- 입력된 경로가 **파일**이거나 **생략**된 경우: 지정된 파일(또는 현재 활성화된 파일)만 대상으로 삼습니다.
- 대상이 여러 개일 경우 2~3단계를 각 파일마다 반복합니다.
- 대상 파일 수에 관계없이 모든 파일을 검사합니다. 파일이 많아 한 번에 처리하기 어려운 경우, 폴더 단위로 나누어 순차적으로 처리하되 중간 진행 상황을 보고합니다.

### 2단계: 노트 품질 검토

각 파일을 읽고 아래 4가지 기준으로 평가합니다.

**① 구조 완전성**
표준 템플릿 섹션(Frontmatter·개요·핵심 내용·실무 적용 예시·출제 포인트·연관 개념·참고 기출)이 모두 존재하는지 확인합니다. 누락된 섹션을 명시합니다.

**② 개념의 깊이와 정확성**
아래 체크리스트를 기준으로 확인합니다.
- 정의·특징·구성요소가 기술사 수준의 깊이로 기술되어 있는가
- 파일이 위치한 폴더(주요항목/세부항목)가 `make-note` 스킬의 출제기준 폴더 매핑표와 일치하는가
- 표·비교 항목에 오류나 누락된 주요 항목이 없는가
- 최신 표준·버전이 반영되어 있는가 (예: TLS 1.3, OWASP 2025 등)

**③ 출제 포인트 적절성**
`출제 포인트` 섹션이 실제 기술사 시험에서 어떻게 출제될 수 있는지를 구체적으로 안내하는지 확인합니다. "개념을 설명하시오" 수준의 단순 반복이 아니라, 비교·계산·설계·대응방안 등 논술 포인트가 담겨있는지 평가합니다.

**④ 보강 필요 사항**
핵심 키워드 누락, 설명이 피상적인 항목, 실무 사례의 적절성 등 구체적인 개선점을 도출합니다.

### 3단계: 결과 보고
