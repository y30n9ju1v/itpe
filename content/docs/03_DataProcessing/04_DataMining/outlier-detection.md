## 개요

아웃라이어(이상치)는 데이터 집합에서 다른 데이터와 현저히 다른 패턴을 보이는 관측값이다. 데이터 마이닝·머신러닝에서 아웃라이어 처리는 모델 성능에 직접 영향을 미치며, 금융 사기 탐지·장비 이상 감지 등에서 아웃라이어 자체가 분석 목표가 된다.

## 핵심 내용

**아웃라이어 개념**
- 통계적 정의: 평균 ± 2σ(또는 3σ) 범위 밖에 있는 값
- 데이터마이닝 정의: 나머지 데이터 대비 메커니즘이 다른 관측값

**아웃라이어 vs 노이즈(Noise)**

| 구분 | 아웃라이어 | 노이즈 |
|------|-----------|--------|
| 의미 | 정상 범위를 벗어난 의미 있는 관측값 | 측정 오차, 무작위 변동 |
| 분석 가치 | 높음 (이상 탐지의 목표) | 낮음 (제거 대상) |
| 발생 원인 | 사기·장비 고장·실제 이상 현상 | 센서 오류·입력 실수 |
| 처리 방향 | 보존하여 분석 또는 별도 모델링 | 전처리 단계에서 제거 |

**아웃라이어 3가지 유형**

| 유형 | 설명 | 예시 |
|------|------|------|
| 전역 이상치 (Global Outlier) | 전체 데이터셋 관점에서 명백히 다른 값 | 정상인 체온 데이터에서 60°C 값 |
| 맥락 이상치 (Contextual Outlier) | 특정 맥락(시간·위치)에서만 이상한 값 | 여름철 기온 데이터에서 30°C는 정상, 겨울에는 이상 |
| 군집 이상치 (Collective Outlier) | 개별로는 정상이나 집합적으로 이상한 값 | ECG 데이터에서 개별 파형은 정상이나 패턴이 비정상 |

**탐지 기법**
- 통계 기반: IQR(사분위수 범위), Z-score
- 거리 기반: KNN 이상치 탐지
- 밀도 기반: LOF(Local Outlier Factor)
- 모델 기반: Isolation Forest, AutoEncoder

## 실무 적용 예시

신용카드 이상 거래 탐지에서 평소와 다른 지역·시간대·금액 조합(맥락 이상치)을 LOF 알고리즘으로 실시간 탐지하여 사기 거래를 차단한다.

## 출제 포인트
