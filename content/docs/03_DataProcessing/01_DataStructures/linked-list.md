## 개요

연결 리스트(Linked List)는 데이터 요소(노드)가 메모리 상의 비연속적인 위치에 저장되고, 각 노드가 다음 노드의 포인터(주소)를 보유하여 논리적으로 연결되는 동적 자료구조다. 삽입·삭제가 O(1)이지만 임의 접근은 O(n)이다.

## 핵심 내용

### 연결 리스트 개념 및 적용 분야

**노드 구조**:
```
┌──────────┬──────────┐
│   Data   │  Next →  │
└──────────┴──────────┘
```

**특성**:
- 동적 크기 조정 (런타임 노드 추가/제거)
- 메모리 비연속 배치 (포인터로 연결)
- 삽입·삭제: O(1) (포인터만 변경)
- 임의 접근: O(n) (순차 탐색 필요)

**적용 분야**:
- OS 프로세스 관리 (Ready Queue, Wait Queue)
- 메모리 관리 (Free List)
- 그래프 인접 리스트 표현
- LRU 캐시 구현 (이중 연결 리스트)
- 파일 시스템 (FAT: File Allocation Table)

### 연결 리스트 구현 방법 (3가지 종류)

| 종류 | 구조 | 특징 |
|------|------|------|
| **단순 연결 리스트** | Data \| Next → | 순방향만 이동 |
| **이중 연결 리스트** | ← Prev \| Data \| Next → | 양방향 이동, 메모리 추가 |
| **원형 연결 리스트** | 마지막 노드 → 첫 노드 | 순환 구조, 큐·라운드로빈에 적합 |

**단순 연결 리스트 삽입 (의사 코드)**:
```
insertAfter(node, newNode):
  newNode.next = node.next
  node.next = newNode
```

**이중 연결 리스트 삭제**:
```
delete(node):
  node.prev.next = node.next
  node.next.prev = node.prev
```

### 배열 리스트(Array List) vs 연결 리스트 비교

| 항목 | 배열 리스트 | 연결 리스트 |
|------|------------|-------------|
| **메모리 배치** | 연속 (Contiguous) | 비연속 (Non-contiguous) |
| **임의 접근** | O(1) (인덱스 직접 접근) | O(n) (순차 탐색) |
| **삽입·삭제** | O(n) (이동 필요) | O(1) (포인터 변경) |
| **메모리 효율** | 낮음 (크기 예약) | 높음 (동적 할당) |
| **캐시 지역성** | 높음 (연속 메모리) | 낮음 (분산 메모리) |
| **크기 변경** | 재할당 필요 | 동적 확장 |
| **적합 사용** | 읽기·검색 빈번 | 삽입·삭제 빈번 |

## 출제 포인트

- 3가지 종류: 단순·이중·원형 연결 리스트 구조와 특성
- 시간 복잡도 비교: 접근 O(n), 삽입·삭제 O(1) vs 배열 O(1)/O(n)
- 실제 적용: OS Ready Queue, LRU 캐시, FAT 파일시스템
- 이중 연결 리스트의 장점: 역방향 탐색 가능, 삭제 효율적
