## 개요

제4정규형(4NF)은 BCNF를 만족하면서 다치종속(Multi-Valued Dependency, MVD)을 제거한 정규형이다. 하나의 릴레이션에서 두 개 이상의 다치종속이 존재하면 삽입·삭제·수정 이상이 발생하며, 이를 분리함으로써 해결한다.

## 핵심 내용

**정규화 단계 순서**
1NF → 2NF → 3NF → BCNF → **4NF** → 5NF

**다치종속 (Multi-Valued Dependency)**
- 정의: 속성 A → → B (A가 B를 다치 결정)
  - A의 값 하나에 B의 값이 여러 개 독립적으로 대응
  - A의 값 하나에 C의 값이 여러 개 독립적으로 대응
  - B와 C가 서로 독립적일 때 다치종속 발생
- 예시: 교수(A)가 여러 과목(B)을 담당하고, 여러 연구실(C)을 사용하는 경우
  - 과목과 연구실이 서로 무관하면 모든 조합을 저장해야 하는 불필요한 중복 발생

**4NF 위반 예시 → 분해**

위반 테이블: `강의(교수, 과목, 연구실)`
- 교수 → → 과목 (다치종속)
- 교수 → → 연구실 (다치종속)

분해 후:
- `교수_과목(교수, 과목)` — 교수가 담당하는 과목
- `교수_연구실(교수, 연구실)` — 교수가 사용하는 연구실

**BCNF vs 4NF**

| 구분 | BCNF | 4NF |
|------|------|-----|
| 제거 대상 | 함수적 종속의 비정규화 | 다치종속 |
| 만족 조건 | 모든 결정자가 후보키 | BCNF + 다치종속 없음 |
