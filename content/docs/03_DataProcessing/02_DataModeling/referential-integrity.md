## 개요

참조 무결성(Referential Integrity)은 관계형 데이터베이스에서 외래 키(Foreign Key)가 참조하는 부모 테이블의 기본 키 값이 반드시 존재해야 한다는 데이터 일관성 제약이다. 고아 레코드(Orphan Record)의 발생을 방지한다.

## 핵심 내용

### 무결성 제약의 종류

| 유형 | 내용 |
|------|------|
| **개체 무결성** | 기본 키는 NULL 불가, 유일해야 함 |
| **참조 무결성** | 외래 키 값은 부모 테이블의 기본 키에 존재하거나 NULL이어야 함 |
| **도메인 무결성** | 속성 값은 정의된 도메인(데이터 타입, 범위) 내에 있어야 함 |
| **사용자 정의 무결성** | 비즈니스 규칙 기반의 추가 제약 |

### 참조 무결성 규칙

**삭제 규칙 (Delete Rule)**
- `CASCADE`: 부모 삭제 시 자식 레코드도 자동 삭제
- `RESTRICT/NO ACTION`: 자식이 존재하면 부모 삭제 거부
- `SET NULL`: 자식의 외래 키를 NULL로 변경
- `SET DEFAULT`: 자식의 외래 키를 기본값으로 변경

**갱신 규칙 (Update Rule)**
- 위와 동일한 옵션 적용

### 정보모델링에서의 참조 무결성

ERD 설계 시 관계(Relationship)의 `식별 관계 vs 비식별 관계`에 따라 참조 무결성 적용 방식 차이:
- **식별 관계**: 자식의 PK에 부모 FK 포함 → 부모 없이 자식 존재 불가
- **비식별 관계**: 자식의 일반 속성으로 FK → NULL 허용 가능
