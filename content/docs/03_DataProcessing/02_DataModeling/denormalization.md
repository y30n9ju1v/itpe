## 개요

반정규화(Denormalization)는 정규화된 논리 데이터 모델을 물리 데이터 모델로 전환할 때, 성능 향상을 위해 의도적으로 중복·통합·분리를 허용하는 설계 기법이다. 조회 성능과 데이터 무결성 간 트레이드오프가 핵심이다.

## 핵심 내용

### 반정규화 절차

1. **반정규화 대상 조사**
   - 범위 처리가 많은 테이블, 대량 데이터 조회 빈번
   - 조인이 많고 조회 성능 저하 발생

2. **다른 방법 검토**
   - 인덱스 추가, 뷰(View) 활용, 파티셔닝 먼저 검토
   - 반정규화는 최후 수단

3. **반정규화 적용**
   - 테이블 통합/분리/추가, 컬럼 중복/파생, 관계 중복

4. **무결성 유지 방안 수립**
   - 트리거, 애플리케이션 로직으로 중복 데이터 동기화

### 반정규화 유형

**테이블 반정규화**

| 유형 | 설명 | 적용 상황 |
|------|------|---------|
| 테이블 병합 | 1:1 또는 1:M 관계 테이블 통합 | 항상 함께 조회되는 테이블 |
| 테이블 분할 | 수직/수평 분할로 분리 | 컬럼 수 과다, 데이터량 과다 |
| 테이블 추가 | 집계·이력·중복 테이블 신설 | 집계 연산 빈번, 이력 관리 필요 |

**컬럼 반정규화**

| 유형 | 설명 |
|------|------|
| 컬럼 중복 | 조인 줄이기 위해 자주 사용하는 컬럼을 타 테이블에 복사 |
| 파생 컬럼 | 계산 결과를 미리 저장 (예: 주문 합계) |
| 이력 컬럼 | 최근 이력을 기본 테이블에 포함 |

**관계 반정규화**

- 중복 관계 추가: 빠른 접근 경로 확보를 위해 이미 존재하는 관계를 추가로 표현

### 반정규화 시 고려사항

1. **데이터 무결성**: 중복 데이터 불일치 위험 → 동기화 메커니즘 필수
2. **DML 성능 저하**: INSERT/UPDATE/DELETE 시 중복 갱신 부담
3. **저장 공간**: 중복으로 인한 공간 증가
4. **유지보수 복잡성**: 스키마 변경 시 영향 범위 확대
5. **선택 기준**: 읽기(SELECT) 빈도 >> 쓰기(DML) 빈도인 경우 적합

## 출제 포인트

- 반정규화 4단계 절차 (조사 → 대안 검토 → 적용 → 무결성 유지)
- 3가지 유형: 테이블·컬럼·관계 반정규화
- 고려사항: 무결성 vs 성능 트레이드오프
