## 해설

### 출제 배경 및 의도

ISMS-P는 국내 정보보안 인증 체계의 핵심으로, 인증 대상·인증 체계·심사 절차·통제 항목을 체계적으로 서술해야 한다. ISMS(정보보호 관리체계)와 ISMS-P(개인정보 추가)의 차이도 명확히 설명해야 한다.

### 1. ISMS-P 개요

정 의  • 조직의 정보자산 보호를 위한 정보보호 관리체계(ISMS)와 개인정보보호 관리체계(PIMS)를 통합한 국내 인증 제도
- **법적 근거**: 정보통신망법 제47조, 개인정보보호법 제32조의2
- **주관**: 과학기술정보통신부·개인정보보호위원회(정책) + 한국인터넷진흥원 KISA(운영)

### 2. ISMS vs ISMS-P 비교

| 항목 | ISMS | ISMS-P |
|------|------|--------|
| 대상 | 정보보호 관리체계 | 정보보호 + 개인정보보호 관리체계 |
| 의무 대상 | ISP, IDC, 매출 100억 이상 정보통신서비스 등 | ISMS 의무 대상 중 개인정보 처리량 많은 기관 |
| 인증 통제 항목 | 80개 항목 | 102개 항목 (ISMS 80 + 개인정보 22) ※ 2023년 개정 후 일부 항목 통합·분리 |

### 3. ISMS-P 인증 체계

| 영역 | 통제 분야 | 항목 수 |
|------|-----------|---------|
| **1. 관리체계 수립 및 운영** | 관리체계 기반 마련, PDCA | 16개 |
| **2. 보호대책 요구사항** | 인증보호, 위험관리, 물리보안, 운영보안, 보안사고 | 64개 |
| **3. 개인정보 처리 단계별 요구사항** | 수집·이용·제공·위탁·파기 | 22개 |

### 4. ISMS-P 인증 절차

```
① 신청: 인증 신청서 제출 (KISA)
② 심사 준비: 자체 점검, 문서 준비
③ 예비 심사 (선택): 갭 분석
④ 본 심사: 서면 심사 + 현장 심사 (2~3주)
⑤ 결함 조치: 발견 결함 시정 (3개월 이내)
⑥ 인증서 발급: 3년 유효
⑦ 사후 심사: 1년마다 유지 심사
⑧ 갱신 심사: 3년마다 전체 재심사
```

### 5. ISMS-P 의무 인증 대상

| 대상 | 기준 |
|------|------|
| 정보통신서비스 제공자 | 매출 100억 이상 또는 일평균 이용자 100만명 이상 |
| 집적정보통신시설(IDC) | IDC 사업자 |
| 의료기관 | 상급 종합병원 |
| 교육기관 | 특정 대학 |

### 6. 2023년 ISMS-P 주요 개정 내용

| 구분 | 변경 내용 |
|------|-----------|
| **클라우드 보안** | 클라우드 서비스 이용 시 보안 요구사항 명확화 |
| **공급망 보안** | 소프트웨어·서비스 공급망 위험관리 항목 신설 |
| **원격근무 보안** | 재택·원격근무 환경 보안통제 요구사항 구체화 |
| **개인정보 국외이전** | 개정 개인정보보호법(2023.9.) 반영, 국외이전 통제 강화 |
| **인증서 유효기간** | 2025년부터 신규 발급 인증서 유효기간 단계적 단축 |

- ISMS-P 인증을 통해 정보보호 관리체계의 체계적인 수립·운영·개선이 가능하며, 인증 취득 자체보다 지속적 개선(PDCA) 사이클이 실질적 보안 수준 향상의 핵심임.  "끝"

### 실무 제언

**ISMS-P와 ISO 27001 통합 운영**
- **챌린지**: 글로벌 기업은 국내 ISMS-P와 해외 ISO 27001을 별도로 준비하면 중복 심사로 비용이 두 배 든다.
- **제언**: ISMS-P의 102개 통제 항목과 ISO 27001:2022의 93개 통제 항목을 매핑 분석하면 약 70%가 중복되므로, 통합 보안 관리 체계를 구축하고 심사 스케줄을 연계하여 효율적으로 운영해야 한다.
