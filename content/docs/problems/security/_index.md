## Topic 1. 정보보안 목적

### 1-1. CIA 트라이어드 — 정보보안의 세 기둥

정보보안의 가장 기본이 되는 목표는 **CIA 트라이어드(Triad)**입니다. 이 세 가지 목표 중 하나라도 위협받으면 보안 사고가 발생합니다.

**기밀성(Confidentiality)**: 인가된 사용자만 정보에 접근할 수 있어야 합니다. 암호화, 접근통제, VPN이 대표적인 기밀성 보호 수단입니다.

**무결성(Integrity)**: 정보가 허가 없이 변경되지 않아야 합니다. 해시 함수, 디지털 서명, 체크섬이 무결성을 보장합니다.

**가용성(Availability)**: 인가된 사용자가 필요할 때 정보에 접근할 수 있어야 합니다. DDoS 공격은 가용성을 위협하는 대표적인 공격입니다.

CIA 외에 **부인방지(Non-repudiation)**, **인증(Authentication)**, **책임추적성(Accountability)**도 현대 정보보안의 핵심 목표로 추가됩니다.

## Topic 2. 암호화 알고리즘

### 2-1. 대칭키 vs 비대칭키 암호화

암호화의 핵심은 누가 키를 갖느냐입니다.

**대칭키 암호화**는 암호화와 복호화에 같은 키를 사용합니다. 빠르지만 키를 안전하게 전달하는 것이 문제입니다. AES(128/256비트), DES, 3DES가 대표적입니다.

**비대칭키 암호화**는 공개키(암호화)와 개인키(복호화)가 쌍으로 존재합니다. 키 배포 문제를 해결하지만 느립니다. RSA, ECC, ElGamal이 대표적입니다.

실제 시스템에서는 **하이브리드 방식**을 사용합니다. 비대칭키로 대칭키를 안전하게 교환(TLS Handshake)한 뒤, 실제 데이터는 빠른 대칭키로 암호화하는 것입니다.

## Topic 7. 양자암호

### 7-3. 양자내성암호(PQC) — 미래의 위협에 대비하는 암호

양자컴퓨터가 실용화되면 현재 RSA와 ECC 암호는 Shor 알고리즘으로 수초 내에 풀릴 수 있습니다. NIST는 이에 대비해 2024년 **PQC(Post-Quantum Cryptography)** 국제 표준을 발표했습니다.

- **CRYSTALS-Kyber**: 키 캡슐화 메커니즘(KEM) — TLS 키 교환 대체
- **CRYSTALS-Dilithium**: 디지털 서명 — PKI 서명 대체
- **FALCON**, **SPHINCS+**: 추가 서명 알고리즘

현재 당장 위협은 없어 보이지만, **"지금 수집해서 나중에 복호화(Harvest Now, Decrypt Later)"** 전략으로 적이 현재 암호화된 통신을 저장해두고 있을 수 있습니다. 금융권·정부 기관은 PQC 전환 로드맵을 지금부터 수립해야 합니다.

## Topic 11. APT

### 11-1. APT — 멈추지 않는 표적 공격

**APT(Advanced Persistent Threat)**는 특정 조직을 표적으로 삼아 장기간(수개월~수년) 은밀하게 침투를 유지하는 고도화된 사이버 공격입니다.

APT는 킬 체인의 7단계와 유사한 흐름으로 진행됩니다. 정찰 → 스피어피싱/워터링홀로 초기 침투 → 내부망 이동(Lateral Movement) → 권한 상승 → 데이터 수집 → 유출.

특징은 **제로데이 취약점 활용**, **맞춤형 악성코드**, **정상 관리 도구 악용(LOLBAS)**입니다. 서명 기반 탐지로는 잡히지 않아, **행동 기반 탐지(EDR, UEBA)**와 **위협 헌팅(Threat Hunting)**이 필수입니다.

## Topic 18. 개인정보보호

### 18-4. 비식별화 3단계 모델

개인 데이터를 활용하면서도 프라이버시를 보호하는 핵심 기법들입니다.

- **k-익명성(k-Anonymity)**: 동일한 준식별자(나이, 성별, 지역) 조합을 가진 레코드가 최소 k개 이상 존재하도록 만들어 특정 개인을 식별하지 못하게 합니다.
- **l-다양성(l-Diversity)**: k-익명성 그룹 내에서 민감 속성 값이 최소 l개 이상 다양하게 분포하도록 합니다. (k-익명성 그룹에 모두 같은 병명이면 유추 가능)
- **t-근접성(t-Closeness)**: 그룹 내 민감 속성 분포가 전체 데이터셋 분포와 통계적으로 t 이내로 유사해야 합니다.

## Topic 24. 최신 보안 기술

### 24-1. 제로 트러스트 — "절대 신뢰하지 않고, 항상 검증하라"

원격근무 확산과 클라우드 전환으로 전통적인 **경계 기반 보안(Perimeter Security)**의 "내부망은 안전하다"는 전제가 무너졌습니다. 내부망에 침투한 공격자가 측면 이동(Lateral Movement)으로 핵심 자산까지 도달하는 APT 공격이 증가했기 때문입니다.

**NIST SP 800-207** 기반 제로 트러스트의 핵심은 세 가지입니다.
1. **신원 검증(Identity)**: 위치와 무관하게 항상 MFA로 검증
2. **최소 권한(Least Privilege)**: 세션 단위로 필요한 최소 권한만 부여
3. **지속 모니터링(Monitoring)**: 모든 트래픽 로깅, 이상 감지 자동화

### 24-7. 사이버 킬 체인 — 공격 체인을 끊어라

Lockheed Martin의 킬 체인 모델은 APT 공격을 7단계(정찰→무기화→전달→익스플로잇→설치→C2 통신→목표 달성)로 모델화합니다. **어느 한 단계에서든 체인을 끊으면 공격이 실패**한다는 것이 핵심 전략입니다.

**MITRE ATT&CK**은 킬 체인보다 훨씬 세밀한 300개 이상의 공격 기법(TTP)을 정의합니다. 킬 체인으로 공격 흐름을 파악하고, ATT&CK으로 세부 탐지 룰을 설계하는 것이 현대 위협 대응의 표준 접근입니다.

## Topic 25. API·AIoT 보안

### 25-1. OWASP API Security Top 10 — API가 새로운 공격 표면

마이크로서비스와 클라우드 전환으로 API가 폭발적으로 증가했습니다. API는 기능과 데이터를 직접 노출하기 때문에 웹 방화벽(WAF)만으로는 방어할 수 없는 **비즈니스 로직 공격**이 가능합니다.

가장 위험한 취약점은 **BOLA(Broken Object Level Authorization, API1)** — URL의 ID를 조작하여 다른 사용자의 데이터에 접근하는 것입니다. 예를 들어 `/api/orders/12345`를 `/api/orders/12346`으로 바꾸는 것만으로 타인의 주문 정보에 접근할 수 있다면 BOLA 취약점입니다.

### 25-3. AIoT 보안 — AI와 IoT가 만나는 새로운 위협

AIoT 환경에서는 기존 IT 보안 위협에 **OT(물리 제어) 보안**과 **AI 모델 공격**이 더해집니다. 특히 **적대적 예제(Adversarial Examples)** — AI 카메라를 속이는 특수 스티커 — 와 같은 새로운 공격이 자율주행·스마트팩토리에서 물리적 피해를 유발할 수 있습니다.

디바이스 보안의 핵심은 **Secure Boot + OTA 업데이트 + 고유 자격증명**입니다. 기기 생산 단계에서 보안이 내재화(Security by Design)되지 않으면 운영 단계에서 수백만 대 기기에 동시에 패치하기가 사실상 불가능합니다.

---

> 보안은 "공격자의 눈으로 시스템을 보는" 역발상의 학문입니다. 방어자는 모든 취약점을 막아야 하지만, 공격자는 단 하나의 취약점만 찾으면 됩니다. 이 불균형을 인식하고 **위협 모델링(Threat Modeling)**, **심층 방어(Defense in Depth)**, **제로 트러스트**로 대응하는 것이 현대 정보보안의 핵심 전략입니다.
