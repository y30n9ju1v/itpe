## 해설

### 1. 사이버 디셉션 개요

정 의  • 허위 자산(Decoy)·함정(Trap)을 배치하여 공격자를 속이고, 공격자의 TTPs(전술·기법·절차)를 수집·분석하여 탐지 및 방어 능력을 향상시키는 능동적 보안 전략
- **핵심 목표**: 탐지 시간 단축, 공격자 시간·리소스 낭비, 위협 인텔리전스 수집

### 2. 허니팟 vs 허니넷

| 항목 | 허니팟(Honeypot) | 허니넷(Honeynet) |
|------|----------------|----------------|
| **구성** | 단일 기만 시스템 | 여러 허니팟의 네트워크 |
| **목적** | 단순 공격 유인·탐지 | 공격자 행동 심층 분석 |
| **가시성** | 낮음 | 높음(공격 전 과정 관찰) |
| **운영 복잡도** | 낮음 | 높음 |
| **종류** | 저상호작용/고상호작용 | 가상/물리 혼합 |

### 3. 사이버 디셉션 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **허니파일(Honeyfile)** | 공격자가 탐낼 만한 가짜 파일 | 가짜 비밀번호.txt, 신용카드DB.csv |
| **허니토큰(Honeytoken)** | 사용 시 경보가 울리는 가짜 자격증명 | 가짜 API 키, 가짜 계정 |
| **허니유저(Honeyuser)** | 실제처럼 보이는 가짜 AD 계정 | 가짜 관리자 계정 |
| **허니서버** | 공격자를 유인하는 가짜 서버 | 가짜 DB 서버, 가짜 내부 API |
| **DNS 싱크홀** | 악성 도메인 요청을 기만 서버로 리다이렉션 | C2 통신 차단 |

### 4. 사이버 디셉션 기술

```
[기만 배치 계층]
  실제 환경과 동일하게 Decoy 자산 배치
  (실제 시스템 옆에 허니팟을 자연스럽게 삽입)

[감지 계층]
  Decoy 자산 접근 시 즉시 경보 (False Positive 거의 없음)

[분석 계층]
  공격자 행동 로깅 → SIEM·TIP 연동
  TTPs 추출, IOC 수집

[대응 계층]
  공격자 격리, 실제 자산 자동 보호 강화
```

### 5. 도입 고려사항 및 기대효과

**고려사항**

| 항목 | 내용 |
|------|------|
| **법적 이슈** | 기만 기술의 법적 허용 범위 (적극적 역공격 금지) |
| **현실감** | Decoy가 실제처럼 보이지 않으면 공격자가 회피 |
| **운영 부하** | 대규모 허니팟 운영 시 모니터링 부담 |
| **격리** | 기만 환경과 실제 환경 완전 분리 필요 |

**기대효과**

| 효과 | 내용 |
|------|------|
| **탐지 정확도 향상** | Decoy 접근 = 100% 공격 행위 (오탐 없음) |
| **조기 탐지** | 내부 침투 후 측면 이동 즉시 탐지 |
| **위협 인텔리전스** | 공격자 TTPs 수집으로 방어 체계 강화 |
| **공격 지연** | 공격자가 가짜 자산 분석하는 시간 낭비 유도 |


- 사이버 디셉션은 허니팟·허니넷·가짜 자산으로 공격자를 유인해 TTP를 수집하고 공격 시간을 지연시키는 능동 방어 전략으로, 탐지 회피형 APT에 대한 위협 인텔리전스 수집과 조기 경보 체계 강화에 효과적임.  "끝"

### 실무 제언

**랜섬웨어 공격 조기 탐지를 위한 허니파일 배치**
- **챌린지**: 랜섬웨어는 실제 파일 암호화가 시작된 후에야 EDR·백신이 탐지하기 때문에 이미 수십~수천 개 파일이 암호화된 후 대응이 시작된다.
- **제언**: 각 공유 폴더의 첫 번째 파일로 허니파일(AAA_decoy.docx 등)을 배치하고, 해당 파일 접근 또는 변경 시 즉각 경보·격리 스크립트를 자동 실행하여 랜섬웨어의 파일 암호화 진행을 1~2개 파일 피해에서 차단하는 조기 대응 체계를 구축해야 한다.
