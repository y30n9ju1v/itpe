## 해설

### 출제 배경 및 의도

RSA(공개키 암호), SHA-2(해시), AES(대칭키 블록 암호)는 현대 보안 인프라의 핵심 알고리즘이다. 각 알고리즘의 원리·특징·취약점·현재 표준 위상을 비교 서술하고, 실제 SSL/TLS 등에서의 결합 사용 사례를 제시해야 한다.

### 1. 암호화 알고리즘 개요

```
보안 목적별 알고리즘 분류:
┌─────────────────────────────────────────────────┐
│ 기밀성 보장: AES(대칭키), RSA(공개키)             │
│ 무결성 보장: SHA-2(해시), HMAC                   │
│ 인증·부인방지: RSA(디지털서명), ECDSA            │
│ 키 교환: Diffie-Hellman, ECDH                   │
└─────────────────────────────────────────────────┘
```

### 2. RSA (Rivest, Shamir, Adleman)

| 항목 | 내용 |
|------|------|
| 제안 | 1977년 Rivest, Shamir, Adleman |
| 원리 | 큰 소수의 곱을 인수분해하기 어렵다는 수학적 난제 기반 |
| 키 길이 | 2048bit 이상 권고(4096bit 고안전성) |
| 용도 | 공개키 암호화, 디지털 서명, 키 교환 |
| 장점 | 공개키 배포 용이, 부인방지 제공 |
| 단점 | 속도 느림, 양자 컴퓨터 공격에 취약(Shor's Algorithm) |

```
RSA 키 생성:
① 두 소수 p, q 선택 → n = p × q
② φ(n) = (p-1)(q-1)
③ gcd(e, φ(n)) = 1인 e 선택 (공개키: (e, n))
④ e × d ≡ 1 (mod φ(n))인 d 계산 (개인키: (d, n))

암호화: C = M^e mod n
복호화: M = C^d mod n
```

### 3. SHA-2 (Secure Hash Algorithm 2)

| 항목 | 내용 |
|------|------|
| 제안 | NSA 설계, NIST 표준(FIPS 180-4) |
| 종류 | SHA-224, SHA-256, SHA-384, SHA-512 |
| 특성 | 단방향(역산 불가), 충돌 저항성, 눈사태 효과 |
| 용도 | 파일 무결성 검증, 디지털 서명, 패스워드 해시 |
| 현황 | SHA-256이 표준으로 가장 널리 사용 |
| 취약점 | MD5·SHA-1은 충돌 취약(폐기), SHA-2는 현재 안전 |

- **SHA-3(Keccak)**: SHA-2의 후계 표준으로 구조적 다양성 확보, Sponge 구조 기반

### 4. AES (Advanced Encryption Standard)

| 항목 | 내용 |
|------|------|
| 제안 | 벨기에 암호학자 Daemen, Rijmen이 설계(Rijndael) |
| NIST 표준 | 2001년 FIPS 197로 채택 |
| 블록 크기 | 128bit |
| 키 길이 | 128/192/256bit (라운드 수: 10/12/14) |
| 구조 | SPN(Substitution-Permutation Network) |
| 안전성 | 현재까지 실용적 공격 없음, 가장 안전한 대칭키 암호 |

```
AES 처리 흐름(128bit 기준, 10라운드):
초기 AddRoundKey
  ↓
[SubBytes → ShiftRows → MixColumns → AddRoundKey] × 9
  ↓
[SubBytes → ShiftRows → AddRoundKey] (최종 라운드)
  ↓
암호문
```

### 5. 세 알고리즘 비교 요약

| 항목 | RSA | SHA-2 | AES |
|------|-----|-------|-----|
| 분류 | 비대칭키(공개키) | 해시함수 | 대칭키 블록 암호 |
| 목적 | 암호화·디지털서명 | 무결성·인증 | 기밀성 |
| 키 필요 | 공개키+개인키 | 없음(단방향) | 비밀키 |
| 속도 | 느림 | 빠름 | 매우 빠름 |
| 역연산 | 가능(개인키 필요) | 불가(단방향) | 가능(동일 키) |
| 현재 표준 | 2048bit 이상 권고 | SHA-256/512 권고 | AES-128 이상 |


- RSA는 소인수분해 난이도 기반 비대칭 암호로 전자서명·키 교환의 근간이며, SHA-2/SHA-3은 무결성 검증 표준이고 AES는 대칭키 암호화 표준으로, 세 알고리즘은 TLS·PKI·데이터 보호 전반에 걸쳐 상호 보완적으로 적용되므로 양자컴퓨팅 위협에 대비한 PQC 전환 로드맵 수립이 시급함.  "끝"

### 실무 제언

**TLS 1.3에서의 결합 활용**
- **챌린지**: 단일 알고리즘으로는 기밀성·무결성·인증을 모두 보장할 수 없다.
- **제언**: TLS 1.3은 ECDHE(키 교환) + AES-256-GCM(기밀성) + SHA-384(무결성) + ECDSA/RSA(인증)를 결합하여 모든 보안 요구사항을 계층적으로 충족한다. PFS(Perfect Forward Secrecy) 보장을 위해 ECDHE 키 교환을 필수 적용해야 한다.
