## 해설

### 출제 배경 및 의도

DRM과 DLP는 모두 정보 유출 방지를 위한 기술이지만 접근 방식이 다르다. DRM은 콘텐츠 자체를 보호하고, DLP는 데이터 이동을 제어한다. 두 기술의 개념·구성요소·장단점·차이점을 명확히 비교하고 결합 사용 사례를 제시해야 한다.

### 1. DRM(Digital Rights Management) 개요

정 의  • 디지털 콘텐츠의 저작권을 보호하고, 허가된 사용자만 정해진 방식으로 사용할 수 있도록 암호화·라이선스로 제어하는 기술
       - 저작권 보호, 불법 복제 방지, 사용 이력 추적

1) DRM 구성요소

| 구성요소 | 역할 |
|----------|------|
| 콘텐츠 패키저 | 콘텐츠 암호화 및 메타데이터 부착 |
| 라이선스 서버 | 사용 권한(횟수·기간·기기 제한) 발급 |
| DRM 클라이언트 | 라이선스 검증 후 콘텐츠 복호화·재생 |
| 워터마킹 | 사용자 추적을 위한 은닉 정보 삽입 |

### 2. DLP(Data Leakage/Loss Prevention) 개요

정 의  • 조직 내 민감 데이터(개인정보, 영업비밀 등)가 비인가 경로(이메일, USB, 클라우드)로 외부에 유출되는 것을 탐지·차단하는 기술
       - 내부 정보 유출 방지, 컴플라이언스 준수

1) DLP 구성요소

| 구성요소 | 설명 |
|----------|------|
| 네트워크 DLP | 이메일·웹·FTP 트래픽 내 민감 데이터 탐지 |
| 엔드포인트 DLP | PC 단에서 USB·프린터·클립보드 제어 |
| 스토리지 DLP | 데이터 보관 위치의 민감 데이터 검색·분류 |
| 클라우드 DLP | CASB와 연계하여 클라우드 서비스 내 데이터 제어 |

### 3. DRM vs DLP 비교

| 항목 | DRM | DLP |
|------|-----|-----|
| 보호 대상 | 디지털 콘텐츠(미디어, 문서) | 조직 내 민감 데이터 |
| 접근 방식 | 콘텐츠 자체 암호화·라이선스 | 데이터 이동 모니터링·차단 |
| 적용 위치 | 콘텐츠(파일) 레벨 | 네트워크·엔드포인트·스토리지 |
| 주요 기술 | 암호화, 디지털 서명, 워터마킹 | 콘텐츠 검사, 정책 엔진, 분류 |
| 사용자 경험 | 사용 제약(횟수·기간 제한) | 투명(정책 위반 시만 개입) |
| 주요 용도 | 저작권 보호, e-Book, 음원 | 기업 기밀 유출 방지, 개인정보 보호 |
| 적용 사례 | Netflix, Spotify, Adobe DRM | Symantec DLP, Forcepoint, Microsoft Purview |

### 4. 결합 사용 사례

- **기업 내부 문서 보호**: IRM(Information Rights Management) = DRM을 기업 문서에 적용
  - 문서 암호화(DRM) + 이메일 첨부파일 검사(DLP) 결합
- **공공기관 개인정보 보호**: 개인정보가 포함된 문서에 DRM 적용, 외부 전송 시 DLP로 차단.  "끝"

### 실무 제언

**Microsoft Purview 통합 적용**
- **챌린지**: DRM과 DLP를 별도 솔루션으로 운영하면 정책 일관성과 운영 비용 문제가 발생한다.
- **제언**: Microsoft Purview에서 Information Protection(MIP 레이블 = IRM/DRM)과 DLP 정책을 통합 관리하면, 민감도 레이블이 부착된 문서에 자동으로 DLP 정책이 적용되는 일관된 데이터 보호 체계를 구축할 수 있다.
