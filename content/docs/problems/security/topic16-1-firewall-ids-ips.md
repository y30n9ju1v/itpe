## 해설

### 출제 배경 및 의도

네트워크 보안 장비의 4대 축(방화벽, IDS, IPS, WAF)의 개념·기능·차이점을 명확히 서술하고, 각 장비의 위치(DMZ 구성)와 탐지 방식을 비교해야 한다.

### 1. 방화벽(Firewall)

정 의  • 내부 네트워크와 외부 네트워크 사이에서 미리 정의된 보안 규칙(ACL)에 따라 패킷을 허용·차단하는 장비
- **위치**: 인터넷과 내부망의 경계(경계 보안)

| 유형 | 동작 방식 | 특징 |
|------|-----------|------|
| **패킷 필터링** | IP/Port 기반 허용·차단 | 빠름, L3/L4, 상태 미확인 |
| **상태 검사(Stateful Inspection)** | 세션 상태 추적 | 정상 세션만 허용 |
| **애플리케이션 게이트웨이(프록시)** | L7 내용 검사 | 깊은 검사, 느림 |
| **NGFW(차세대 방화벽)** | DPI + URL 필터링 + 앱 인식 | 통합 기능 |

### 2. IDS(Intrusion Detection System)

정 의  • 네트워크·호스트의 트래픽을 모니터링하여 침입 시도를 탐지하고 경보를 발생시키는 시스템
       - 탐지만 하고 차단은 하지 않음(수동적)

| 탐지 방식 | 설명 | 특징 |
|-----------|------|------|
| **시그니처 기반(오용 탐지)** | 알려진 공격 패턴과 비교 | 낮은 오탐, 신규 공격 탐지 불가 |
| **이상 탐지(Anomaly Detection)** | 정상 행위 기준선 대비 이상 탐지 | 신규 공격 탐지 가능, 오탐 높음 |

| 유형 | 설명 |
|------|------|
| **NIDS**(Network IDS) | 네트워크 트래픽 전체 모니터링 |
| **HIDS**(Host IDS) | 개별 호스트의 로그·파일·프로세스 모니터링 |

### 3. IPS(Intrusion Prevention System)

정 의  • IDS 기능에 능동적 차단 기능을 추가한 시스템
       - 인라인(In-line) 방식으로 패킷 경로에 직접 삽입하여 실시간 차단

| 구분 | IDS | IPS |
|------|-----|-----|
| 배치 방식 | 미러링(Out-of-Band) | 인라인(In-line) |
| 동작 | 탐지·알림 | 탐지·차단·알림 |
| 영향 | 지연 없음 | 지연 가능 |
| 오탐 영향 | 경보만 | 정상 트래픽도 차단 위험 |

### 4. WAF(Web Application Firewall, 웹방화벽)

정 의  • 웹 애플리케이션 레이어(L7)에서 SQL Injection, XSS 등 웹 공격을 탐지·차단하는 방화벽
       - HTTP/HTTPS 트래픽을 검사, OWASP Top 10 공격 방어

| 구분 | 방화벽 | WAF |
|------|--------|-----|
| 계층 | L3/L4 | L7 |
| 검사 대상 | IP/Port | HTTP 내용(헤더, 페이로드) |
| 방어 대상 | 포트 스캔, IP 차단 | SQL Injection, XSS, CSRF |

### 5. 네트워크 보안 장비 배치 (DMZ 구성)

```
인터넷 ──[방화벽1]── DMZ(WAF, 웹서버) ──[방화벽2]── 내부망
                    [IDS/IPS 모니터링]
```


- 최근 방화벽, IDS, IPS, 웹방화벽 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**NGFW + EDR + SIEM 통합 보안 아키텍처**
- **챌린지**: 방화벽·IDS·IPS·WAF를 독립 운영하면 보안 이벤트 간 상관 분석이 어렵다.
- **제언**: Palo Alto NGFW, CrowdStrike EDR, Splunk SIEM을 연동하여 네트워크·엔드포인트·로그를 통합 분석하는 XDR(Extended Detection & Response) 아키텍처를 구축하고, SOAR로 탐지-분석-대응을 자동화해야 한다.
