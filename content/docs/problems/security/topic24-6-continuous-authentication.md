## 문제

최근 IoT·모바일 환경에서 사용자 편의성을 높이면서도 지속적으로 신원을 검증하는 무자각 지속인증(Transparent Continuous Authentication)이 주목받고 있다. 다음에 대해 설명하시오.
가. 무자각 지속인증의 개념 및 필요성
나. 무자각 지속인증 기법의 종류
다. 무자각 지속인증의 보안 강화 방안

## 해설

### 1. 무자각 지속인증 개요

정 의  • 사용자가 별도의 인증 행위 없이 시스템이 지속적으로 사용자 행동·생체·환경 데이터를 분석하여 신원을 자동 검증하는 인증 방식
- **필요성**: 세션 하이재킹, 내부자 위협, 공유 디바이스 악용 등 로그인 이후 발생하는 보안 위협 대응

| 기존 인증 방식 | 무자각 지속인증 |
|---------------|----------------|
| 최초 로그인 시 1회 인증 | 세션 전체 지속 인증 |
| 사용자 명시적 행위 필요 | 사용자 개입 없음 |
| 인증 후 신원 변경 감지 불가 | 인증 우회 즉시 감지 |

### 2. 무자각 지속인증 기법

| 기법 | 방법 | 특징 |
|------|------|------|
| **행동 생체인식** | 타이핑 패턴(리듬·압력), 마우스 동작 분석 | 의식 없이 수집, 개인별 고유 패턴 |
| **보행 패턴** | 스마트폰 가속도계·자이로센서로 걸음걸이 분석 | 모바일 환경 적합 |
| **음성 패턴** | 화자 인식, 지속 음성 패턴 분석 | 원격 환경 적합 |
| **얼굴 인식** | 카메라 기반 주기적 안면 재인증 | 자연스러운 지속 검증 |
| **위치·환경** | GPS 위치, Wi-Fi SSID, 접속 패턴 분석 | 이상 위치 탐지 |
| **UEBA** | 사용자·엔티티 행동 분석 (접근 패턴, 접속 시간) | 내부자 위협 탐지 |

### 3. 무자각 지속인증 시스템 구조

```
[데이터 수집 계층]
  키보드·마우스·센서·카메라·GPS 데이터 지속 수집

       ↓

[행동 프로파일링]
  초기 학습기간(2-4주) 동안 사용자 기준 패턴 수립

       ↓

[실시간 분석 엔진]
  수집 데이터 vs 기준 프로파일 지속 비교
  신뢰 점수(Trust Score) 계산

       ↓

[대응 모듈]
  고위험(점수 임계값 이하) → 추가 인증 요구 또는 세션 종료
  중위험 → 행동 모니터링 강화
  정상 → 투명하게 허용
```

### 4. 보안 강화 방안

| 방안 | 내용 |
|------|------|
| **멀티모달 융합** | 단일 기법 오인식 보완, 2개 이상 기법 복합 사용 |
| **적응형 임계값** | 업무 중요도·위험 수준에 따라 신뢰 점수 기준 동적 조정 |
| **프라이버시 보호** | 행동 데이터 로컬 처리(Edge AI), 개인정보보호법 준수 |
| **학습 모델 보호** | 모델 역공격(Model Inversion Attack) 방어 |
| **Zero Trust 연계** | 신뢰 점수를 PEP(정책 시행 지점)에 실시간 전달 |


- 무자각 지속인증은 행동 바이오메트릭(타이핑 패턴·마우스 동작·걸음걸이)을 세션 내내 분석해 사용자 경험 저하 없이 비정상 접근을 실시간 차단하며, Zero Trust 아키텍처의 지속적 검증 원칙을 사용자 친화적으로 구현하는 핵심 기술임.  "끝"

### 실무 제언

**금융권 비대면 서비스 무자각 지속인증 적용**
- **챌린지**: 모바일 뱅킹에서 계정 탈취 후 다른 사람이 사용하더라도, 최초 로그인 인증만 통과하면 지속적으로 서비스를 이용할 수 있는 세션 공유 취약점이 존재한다.
- **제언**: 타이핑 패턴·스크롤 속도·디바이스 기울기 등 행동 생체인식을 통해 5~10분 주기로 신뢰 점수를 재계산하고, 점수가 임계값 이하로 떨어지면 자동으로 OTP 추가 인증을 요구하는 Continuous Risk-Based Authentication을 고액 거래 시나리오에 우선 적용해야 한다.
