## 해설

### 출제 배경 및 의도

이중서명(Dual Signature)은 전자상거래 프로토콜 SET(Secure Electronic Transaction)에서 개발된 기법으로, 상점과 은행이 각각 필요한 정보만 보면서도 전체 거래의 무결성을 검증할 수 있다. 구체적인 암·복호화 절차를 그림과 함께 서술해야 한다.

### 1. 이중서명 개요

정 의  • 고객이 주문 정보와 결제 정보를 각각 서명하되, 두 서명을 연결하여 상점·은행 각자가 필요한 정보만 접근하도록 하는 기법
       - 
  - 상점: 주문 정보는 알지만 결제(카드) 정보는 알 수 없음
  - 은행: 결제 금액은 알지만 구체적인 주문 내용은 알 수 없음
  - 전체 무결성: 두 정보가 연결되어 있어 변조 불가

### 2. 주요 기술 및 수행 절차

```
고객(Customer)의 이중서명 생성:

① 주문정보 해시: H(OI) = OIMD
② 결제정보 해시: H(PI) = PIMD
③ 연결 해시: POMD = H(OIMD || PIMD)
④ 이중서명: DS = E(POMD, 고객 개인키)   ← 고객의 디지털 서명
⑤ 전송:
   ─ 상점에게: OI + PIMD + DS
   ─ 은행에게: PI + OIMD + DS
```

### 3. 이중서명 검증 절차

**상점의 검증:**
```
수신: OI + PIMD + DS
① H(OI) → OIMD'
② H(OIMD' || PIMD) → POMD'
③ D(DS, 고객 공개키) → POMD
④ POMD == POMD' 이면 유효
```

**은행의 검증:**
```
수신: PI + OIMD + DS
① H(PI) → PIMD'
② H(OIMD || PIMD') → POMD'
③ D(DS, 고객 공개키) → POMD
④ POMD == POMD' 이면 유효
```

### 4. 이중서명의 특징

| 특징 | 내용 |
|------|------|
| 프라이버시 보호 | 상점은 결제정보, 은행은 주문정보 접근 불가 |
| 무결성 보장 | POMD가 두 정보를 연결하여 위변조 탐지 |
| 부인방지 | 고객 개인키로 서명, 추후 부인 불가 |
| 활용 | SET(Secure Electronic Transaction) 프로토콜 |


- 이중서명은 주문 정보와 결제 정보를 연결하면서도 상인과 은행이 각각 필요한 정보만 검증할 수 있게 하는 SET 프로토콜의 핵심 기술로, 고객 프라이버시 보호와 거래 무결성을 동시에 달성하는 구조임.  "끝"

### 실무 제언

**전자금융 프라이버시 강화**
- **챌린지**: 온라인 결제에서 상점이 고객의 카드 정보를 수집·저장하면 보안 사고 위험이 높아진다.
- **제언**: 이중서명 원리를 응용하여 결제 대행사(PG)가 카드 정보를 관리하고 상점은 거래 토큰만 수신하는 토큰화(Tokenization) 방식과 결합하면 데이터 최소화 원칙과 보안을 동시에 달성할 수 있다.
