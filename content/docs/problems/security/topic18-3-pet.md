## 해설

### 출제 배경 및 의도

PET는 개인정보를 기술적으로 보호하는 다양한 방법론의 총칭이다. 가명처리·익명화·차분 프라이버시·연합학습 등 현대적 PET 기법을 사례와 함께 체계적으로 서술해야 한다.

### 1. PET 개념

- **정의**: 개인정보 처리 시 정보주체의 프라이버시를 보호하면서도 데이터의 유용성을 유지하는 기술적 방법론의 총칭
- **목적**: 개인정보보호법 준수 + 데이터 활용 가치 보존의 균형
- **원칙**: PBD(Privacy by Design)의 기술적 구현 수단

### 2. PET 주요 유형

#### 가. 식별 제거 기술

| 기술 | 설명 | 특징 |
|------|------|------|
| **가명처리(Pseudonymisation)** | 직접 식별자를 가명으로 대체 | 재식별 가능(Key 관리 필요), GDPR 허용 |
| **익명화(Anonymisation)** | 재식별 불가 수준으로 변환 | 개인정보보호법 적용 제외 |
| **마스킹(Masking)** | 일부 정보 부분 삭제·대체 (예: 주민번호 뒤 7자리 *처리) | 간단, 제한적 보호 |
| **일반화(Generalisation)** | 구체 값을 범위로 대체 (예: 1980 → 1980년대) | 분석 정확도 감소 |
| **데이터 교환(Swapping)** | 레코드 간 값 교환 | 통계적 특성 유지 |

#### 나. 암호화 기반 기술

| 기술 | 설명 | 적용 |
|------|------|------|
| **동형 암호(HE)** | 암호화 상태에서 연산 | AI 분석, 클라우드 |
| **영지식 증명(ZKP)** | 실제 값 노출 없이 사실 증명 | 연령 확인, 자격 증명 |
| **안전한 다자간 계산(MPC)** | 여러 당사자가 합산 값만 공유 | 분산 AI 학습 |

#### 다. 통계적 프라이버시 기술

| 기술 | 설명 | 적용 |
|------|------|------|
| **차분 프라이버시(Differential Privacy)** | 개인 데이터에 수학적 노이즈 추가 | 구글 RAPPOR, Apple |
| **k-익명성(k-Anonymity)** | 최소 k명이 같은 속성값을 가지도록 일반화 | 의료 데이터 공개 |
| **l-다양성, t-근접성** | k-익명성의 확장(민감 속성 다양성 보장) | 통계 공개 |

#### 라. 분산 처리 기술

| 기술 | 설명 | 적용 |
|------|------|------|
| **연합학습(Federated Learning)** | 원본 데이터 공유 없이 모델만 집계 | AI 모델 학습 |
| **데이터 금고(Data Vault)** | 개인이 직접 데이터를 저장·통제 | 마이데이터 |

### 3. PET 적용 사례

| 사례 | 적용 기술 | 내용 |
|------|-----------|------|
| **COVID-19 접촉 추적** | 블루투스 임시 ID + 암호화 | Apple/Google 프레임워크: 중앙 서버에 개인 정보 미전송 |
| **의료 데이터 연구** | 차분 프라이버시 + k-익명성 | 환자 개인정보 보호하면서 임상 연구 데이터 공유 |
| **AI 학습** | 연합학습 | Google Gboard: 사용자 기기에서 학습, 서버에 원본 미전송 |
| **금융 이상 탐지** | MPC + HE | 여러 금융기관이 데이터 공유 없이 공동 모델 구축 |
| **통계 공개** | k-익명성 + 마스킹 | 통계청 마이크로데이터 공개 |

### 실무 제언

**마이데이터(MyData) 서비스 PET 설계**
- **챌린지**: 마이데이터 서비스에서 개인 금융·의료 정보를 집계·분석할 때 정보 유출 위험이 크다.
- **제언**: 가명처리(ISMS-P 기준)로 식별자를 제거하고, 연합학습으로 개인 원본 데이터를 마이데이터 서비스에 전송하지 않으며, 동형암호로 집계 분석을 수행하는 PET 3층 설계를 적용하여 데이터 유용성과 프라이버시를 동시에 달성해야 한다.
