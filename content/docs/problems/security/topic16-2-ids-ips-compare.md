## 해설

### 출제 배경 및 의도

IDS와 IPS는 침입 탐지·방지의 핵심 보안 장비로, 배치 방식·동작 원리·탐지 방법·오탐 영향을 중심으로 심층 비교해야 한다. topic16-1에서 4가지 장비를 개괄했다면, 이 문제는 IDS/IPS에 집중하여 더 깊이 다룬다.

### 1. IDS(Intrusion Detection System) 개요

- **정의**: 네트워크·시스템의 이상 징후와 침입 시도를 모니터링·탐지하여 관리자에게 경보를 발생시키는 수동적 보안 시스템
- **배치**: 네트워크 스위치의 포트 미러링(SPAN)으로 트래픽 복사 수신

```
[정상 트래픽 경로]
인터넷 ──스위치── 서버
           │
           └──(미러링)──▶ IDS (모니터링·탐지·경보)
```

### 2. IPS(Intrusion Prevention System) 개요

- **정의**: IDS의 탐지 기능에 능동적 차단·응답 기능을 더한 인라인 보안 시스템
- **배치**: 트래픽 경로에 직접 삽입(인라인)하여 악성 트래픽 실시간 차단

```
[정상 트래픽 경로]
인터넷 ──[IPS]── 서버
           │
           └── 악성 트래픽 탐지 즉시 DROP
```

### 3. IDS vs IPS 상세 비교

| 항목 | IDS | IPS |
|------|-----|-----|
| 배치 방식 | Out-of-Band(미러링) | In-line(트래픽 경로에 직접) |
| 동작 방식 | 수동적(탐지·알림) | 능동적(탐지·차단·알림) |
| 응답 속도 | 사후 경보 | 실시간 차단 |
| 오탐 영향 | 경보 과다(False Positive) | 정상 트래픽 차단(서비스 장애 가능) |
| 지연 시간 | 없음(트래픽 경로 외) | 있음(패킷 검사 지연) |
| 가용성 영향 | 장애 시 모니터링만 중단 | 장애 시 네트워크 단절 위험(Fail-Open/Close) |
| 운영 복잡도 | 낮음 | 높음(튜닝 필요) |

### 4. 탐지 방식 비교

| 탐지 방식 | 설명 | IDS/IPS 적용 |
|-----------|------|--------------|
| **시그니처(Signature)** | 알려진 공격 패턴 DB 비교 | 정확도 높음, 제로데이 취약 |
| **이상 탐지(Anomaly)** | 정상 기준선 대비 편차 탐지 | 신규 공격 탐지, 오탐 많음 |
| **행위 기반(Behavioral)** | 프로세스·네트워크 행위 패턴 분석 | AI/ML 기반, 지능형 탐지 |
| **프로토콜 분석** | 프로토콜 규격 준수 여부 검사 | 변형 공격 탐지 |

### 5. Fail-Open vs Fail-Close

IPS 장애 시 동작 방식 (가용성 vs 보안 트레이드오프):
- **Fail-Open**: 장애 시 모든 트래픽 통과 → 가용성 우선, 보안 취약
- **Fail-Close**: 장애 시 모든 트래픽 차단 → 보안 우선, 서비스 중단

### 실무 제언

**IPS 오탐율 최소화 전략**
- **챌린지**: IPS를 처음 도입하면 오탐으로 인한 정상 트래픽 차단이 빈번하여 서비스 장애를 유발한다.
- **제언**: 초기에는 IDS 모드(탐지만)로 2~4주간 운영하며 오탐 패턴을 분석하고 화이트리스트를 구성한 후, IPS 차단 모드로 전환하는 점진적 배포 전략을 적용해야 한다. AI 기반 UEBA와 연동하여 시그니처 기반 탐지의 제로데이 취약점을 보완해야 한다.
