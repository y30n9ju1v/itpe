## 해설

### 출제 배경 및 의도

암호화는 정보보안의 기술적 핵심 수단이다. 대칭키·비대칭키의 원리와 차이, 블록/스트림 암호화의 특성을 체계적으로 비교 서술할 수 있는지를 검증한다. 각 알고리즘 유형별 대표 알고리즘(AES, RSA, SHA 등)을 함께 제시해야 고득점을 얻을 수 있다.

### 1. 암호화 알고리즘 원리와 특징

정 의  • 평문(Plaintext)을 키(Key)와 암호 알고리즘을 사용해 암호문(Ciphertext)으로 변환하고, 복호화 키로 원문을 복원하는 기술

```
평문(P) ──[암호화 키 K1 + 알고리즘]──▶ 암호문(C)
암호문(C) ──[복호화 키 K2 + 알고리즘]──▶ 평문(P)
```

- **Kerckhoffs의 원리**: 알고리즘은 공개되어도 되지만, 키는 비밀을 유지해야 한다.
- 암호 강도는 키 길이(비트 수)와 알고리즘 복잡도에 비례한다.

### 2. 대칭키 vs 비대칭키 알고리즘 비교

| 구분 | 대칭키(Symmetric) | 비대칭키(Asymmetric) |
|------|-------------------|----------------------|
| 키 구조 | 암호화·복호화 키가 동일 | 공개키(암호화) + 개인키(복호화) |
| 키 배포 | 사전에 안전한 채널로 키 공유 필요(키 배포 문제) | 공개키는 자유 배포, 개인키는 보관 |
| 속도 | 빠름(하드웨어 구현 용이) | 느림(수학적 복잡도 높음) |
| 보안성 | 키 노출 시 전체 통신 노출 | 개인키 보호만 하면 됨 |
| 키 개수 | N명 통신 시 N(N-1)/2개 | N명 통신 시 2N개 |
| 주요 알고리즘 | DES, 3DES, AES, SEED, ARIA | RSA, ECC, Diffie-Hellman, ElGamal |
| 용도 | 대용량 데이터 암호화 | 키 교환, 디지털 서명, 인증 |

- **하이브리드 암호**: 비대칭키로 세션 키를 교환하고, 대칭키로 데이터를 암호화하는 방식(SSL/TLS)

### 3. 블록 암호화 vs 스트림 암호화 비교

| 구분 | 블록 암호화(Block Cipher) | 스트림 암호화(Stream Cipher) |
|------|--------------------------|------------------------------|
| 처리 단위 | 고정 크기 블록(64bit, 128bit 등) | 비트/바이트 단위 순차 처리 |
| 구현 | 복잡, 소프트웨어 구현 | 단순, 하드웨어 구현 용이 |
| 속도 | 상대적으로 느림 | 빠름(실시간 처리 적합) |
| 오류 전파 | 블록 단위로 오류 전파 가능 | 해당 비트/바이트만 영향 |
| 운용 모드 | ECB, CBC, CFB, OFB, CTR | A5/1(이동통신), ChaCha20 |
| 주요 알고리즘 | AES, DES, 3DES, SEED, ARIA | ChaCha20(현용), RC4(폐기) |
| 용도 | 파일 암호화, 데이터베이스, 네트워크 | 무선 통신, 실시간 스트리밍 |

- **RC4 폐기**: RC4는 통계적 편향·키스트림 예측 취약점으로 TLS 1.3에서 완전 제거됨(RFC 7465). 현용 스트림 암호 표준은 **ChaCha20-Poly1305**(TLS 1.3, QUIC).
- 현대 표준: NIST가 선정한 **AES-128/192/256**이 블록 암호화의 표준으로 채택되어 가장 널리 사용됨.  "끝"

### 실무 제언

**클라우드 데이터 보호**
- **챌린지**: 클라우드 환경에서 대용량 데이터 암호화 시 성능 저하와 키 관리 복잡도가 증가한다.
- **제언**: AES-256 대칭키로 데이터를 암호화하고, 키 관리는 전용 KMS(Key Management Service)를 통해 중앙 집중식으로 관리하되, HSM(Hardware Security Module)으로 마스터 키를 보호해야 한다.

**전자서명과 암호화 혼합 적용**
- **챌린지**: 기밀성과 무결성·인증을 동시에 보장해야 하는 시나리오에서 알고리즘 선택이 복잡하다.
- **제언**: 데이터 암호화에는 AES(대칭키), 키 교환에는 RSA/ECDH(비대칭키), 무결성 검증에는 SHA-256(해시)을 계층적으로 적용하는 하이브리드 방식을 채택해야 한다.
