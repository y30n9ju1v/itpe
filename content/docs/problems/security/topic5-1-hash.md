## 해설

### 출제 배경 및 의도

해시함수는 무결성 검증·패스워드 저장·디지털 서명 등 보안의 핵심 기반 기술이다. 암호학적 3대 특성(충돌 저항성·역상 저항성·제2역상 저항성)과 함께 DB에서의 해시 인덱스 활용까지 연결하여 서술할 수 있어야 한다.

### 1. 해시 알고리즘 개요

정 의  • 임의 길이의 입력 메시지를 고정 길이의 해시값(다이제스트)으로 변환하는 단방향 함수
       - 결정론적(동일 입력 → 동일 출력), 빠른 계산, 역산 불가

```
입력 M (임의 길이) ──▶ [Hash Function] ──▶ H(M) 고정 길이(예: 256bit)
```

### 2. 보안 해시 알고리즘 분류

| 분류 | 알고리즘 | 출력 길이 | 현황 |
|------|----------|-----------|------|
| MD 계열 | MD5 | 128bit | 충돌 취약, 폐기 권고 |
| SHA-1 | SHA-1 | 160bit | 충돌 취약, 폐기 |
| SHA-2 계열 | SHA-224/256/384/512 | 224~512bit | 현재 표준 |
| SHA-3 계열 | SHA3-256/512 | 256~512bit | 차세대 표준(Keccak) |
| HMAC | HMAC-SHA256 등 | 가변 | 메시지 인증 코드 |
| 국내 표준 | HAS-160, LSH | 160~512bit | KCMVP 인증 |

### 3. 암호학적 3가지 기준

| 기준 | 정의 | 의미 |
|------|------|------|
| **역상 저항성** (Preimage Resistance) | H(x) = y가 주어질 때 x를 찾는 것이 계산적으로 불가능 | 단방향성 보장 |
| **제2역상 저항성** (Second Preimage Resistance) | x가 주어질 때 H(x) = H(x')인 다른 x'를 찾기 불가능 | 특정 입력에 대한 충돌 방지 |
| **충돌 저항성** (Collision Resistance) | H(x) = H(x')인 임의의 두 x, x'를 찾기 불가능 | 일반적 충돌 방지 |

- 강도: 충돌 저항성 > 제2역상 저항성 > 역상 저항성

### 4. 데이터베이스에서의 해시 활용

| 활용 | 설명 | 예시 |
|------|------|------|
| **해시 인덱스** | 해시 버킷에 레코드 주소 저장, O(1) 검색 | 메모리 DB, 동등 조건 검색 |
| **패스워드 저장** | 패스워드를 해시(+ Salt)하여 저장 | bcrypt, Argon2 |
| **무결성 검증** | 데이터 변조 감지 | Checksum, HMAC |
| **파티셔닝** | 해시 기반 수평 분할 | DB 샤딩(Sharding) |

```
패스워드 안전 저장:
평문 PW + 랜덤 Salt ──▶ Hash(PW || Salt) ──▶ DB 저장
→ Rainbow Table 공격 방어, 동일 PW도 다른 해시값 생성
```


- 해시 함수는 일방향성·충돌저항성·역상저항성을 갖춰 무결성 검증·전자서명·패스워드 저장의 핵심 기반이 되며, MD5·SHA-1은 충돌 취약점으로 폐기되고 SHA-256 이상이 현행 표준임을 숙지해야 함.  "끝"

### 실무 제언

**안전한 패스워드 해싱**
- **챌린지**: MD5·SHA-1로 해시된 패스워드는 Rainbow Table 공격으로 수초 내 역산 가능하다.
- **제언**: Argon2(메모리 집약적), bcrypt(적응형 비용), PBKDF2(NIST 권고) 등 패스워드 전용 KDF(Key Derivation Function)를 사용하고, 랜덤 Salt를 사용자별로 개별 적용해야 한다.
