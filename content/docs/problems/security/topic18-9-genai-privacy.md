## 해설

### 1. 생성형 AI 개발·활용 단계

개인정보보호위원회 「생성형 AI 개인정보 처리 안내서(2025.8.)」 기준:

| 단계 | 내용 |
|------|------|
| **① 데이터 수집** | 학습용 데이터 수집(공개 웹데이터, 라이선스 데이터, 자체 생성) |
| **② 전처리·가공** | 개인정보 필터링, 가명처리, 품질 정제 |
| **③ AI 모델 학습** | 사전학습(Pre-training), 파인튜닝(Fine-tuning), RLHF |
| **④ AI 서비스 배포** | API 서비스, 챗봇, 에이전트 형태로 제공 |
| **⑤ 사용자 인터랙션** | 프롬프트 입력 → AI 응답 → 대화 이력 처리 |
| **⑥ 지속 학습·개선** | 사용자 피드백 기반 모델 업데이트 |

### 2. 단계별 개인정보 처리 고려사항

#### 가. 데이터 수집 단계

| 고려사항 | 내용 |
|----------|------|
| 합법적 수집 근거 | 공개 데이터도 저작권·개인정보 포함 여부 확인 |
| 크롤링 제한 | robots.txt 준수, 개인정보 포함 데이터 크롤링 제한 |
| 동의 확인 | 개인정보 포함 시 정보주체 동의 또는 법적 근거 확보 |

#### 나. 전처리·가공 단계

| 고려사항 | 내용 |
|----------|------|
| 개인정보 제거 | 이름, 연락처, 주소 등 자동 식별·삭제(Named Entity Recognition) |
| 가명처리 | 제거 불가한 경우 가명처리 후 학습 |
| 데이터 최소화 | 학습 목적에 필요한 최소 정보만 포함 |

#### 다. AI 모델 학습 단계

| 고려사항 | 내용 |
|----------|------|
| 학습 데이터 보안 | 학습 데이터 접근 권한 최소화, 암호화 저장 |
| 멤버십 추론 공격 방어 | 차분 프라이버시 적용으로 원본 데이터 역추론 방지 |
| 학습 데이터 목록 보관 | 데이터 출처 및 처리 내역 기록(데이터 출처 추적성) |

#### 라. AI 서비스 배포·운영 단계

| 고려사항 | 내용 |
|----------|------|
| 개인정보처리방침 공개 | AI 서비스 내 개인정보 수집·이용·보관 기준 공개 |
| 프롬프트 내 개인정보 | 사용자 입력 개인정보 최소 보관·즉시 파기 정책 |
| AI 생성물 유출 방지 | 모델이 학습 데이터 개인정보를 생성·노출하지 않도록 가드레일 적용 |
| 정보주체 권리 보장 | AI 서비스의 개인정보 열람·삭제 요청 처리 절차 |
| 아동 보호 | 14세 미만 대상 서비스 시 법정대리인 동의 |

#### 마. 지속 학습·개선 단계

| 고려사항 | 내용 |
|----------|------|
| 사용자 피드백 활용 | 명시적 동의 없이 대화 내용을 모델 학습에 재사용 금지 |
| 모델 평가 시 개인정보 | 평가용 데이터셋의 개인정보 포함 여부 확인 |

### 실무 제언

**RAG 기반 기업 AI 챗봇 개인정보 설계**
- **챌린지**: 기업 내부 문서를 RAG(Retrieval-Augmented Generation) 방식으로 AI 챗봇에 연결할 때, 직원 개인정보가 포함된 문서가 무분별하게 노출될 수 있다.
- **제언**: RAG 검색 범위를 역할 기반 접근 통제(RBAC)로 제한하고, 개인정보 포함 청크(Chunk)에 민감도 레이블을 부착하여 일반 사용자 쿼리에서는 마스킹 처리 후 응답하며, 모든 AI 응답의 개인정보 포함 여부를 사후 감사하는 로깅 체계를 구축해야 한다.
