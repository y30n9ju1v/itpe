## 해설

### 1. 가명처리 vs 익명처리 비교

| 항목 | 가명처리(Pseudonymisation) | 익명처리(Anonymisation) |
|------|---------------------------|------------------------|
| 정의 | 직접 식별자를 가명으로 대체(재식별 키 별도 관리) | 재식별이 불가능한 수준으로 변환 |
| 재식별 가능성 | 가능(키를 통해) | 불가 |
| 개인정보보호법 적용 | 개인정보로 취급(보호 적용) | 개인정보에서 제외 |
| GDPR | 제4조 5항 정의, 제25조 기술적 조치로 권고 | 적용 제외 |
| 데이터 유용성 | 높음(원본 속성 유지) | 낮음(일부 정보 손실) |
| 활용 | 가명정보 결합·분석(개정 개인정보보호법) | 통계 공개, 연구 |

### 2. 개인정보 비식별 기술

| 기법 | 방법 | 예시 |
|------|------|------|
| **가명화** | 식별자를 의미없는 값으로 대체 | 홍길동 → ID_A001 |
| **총계처리** | 개인정보를 집계값으로 대체 | 개별 소득 → 평균 소득 |
| **데이터 삭제** | 식별 가능 필드 제거 | 주민번호 컬럼 삭제 |
| **데이터 마스킹** | 특정 필드 일부를 * 또는 임의값으로 대체 | 010-****-5678 |
| **일반화** | 구체 값을 범주/범위로 대체 | 35세 → 30대 |
| **데이터 범주화** | 연속값을 범주로 분류 | 소득 500만원 → 고소득 |
| **랜덤 노이즈 추가** | 수치 데이터에 랜덤값 추가 | 차분 프라이버시 |
| **데이터 교환** | 레코드 간 특정 속성 값을 교환 | 두 레코드의 나이 교환 |

### 3. 익명 결합 모델

- **개요**: 서로 다른 기관이 보유한 데이터를 익명으로 결합하여 분석하는 방법
- **법적 근거**: 개정 개인정보보호법 제28조의3 (가명정보 결합)
- **절차**:

```
기관 A의 가명정보 ──▶ [결합 전문기관] ──▶ 결합 데이터 ──▶ 기관 A/B 분석
기관 B의 가명정보 ──▶ [결합 키 관리]
```

| 단계 | 내용 |
|------|------|
| 결합 신청 | 데이터 보유 기관이 결합 전문기관에 신청 |
| 결합 키 생성 | 결합 전문기관이 익명 결합 키 발급 |
| 가명 처리 | 각 기관이 자체적으로 식별자를 가명처리 |
| 데이터 결합 | 결합 전문기관에서 안전하게 결합 |
| 반환 | 결합된 가명 데이터 반환, 재식별 금지 |


- 가명처리는 재식별 가능성을 잔존시켜 연구·통계 목적 활용을 허용하고, 익명처리는 재식별을 불가능하게 만들어 개인정보 보호법 적용을 면제하며, 전문기관을 통한 가명 결합으로 의료·금융 분야의 데이터 활용 가치를 극대화할 수 있음.  "끝"

### 실무 제언

**금융-의료 데이터 결합 분석**
- **챌린지**: 금융기관과 의료기관이 각자 가진 데이터를 결합하면 암 진단→신용도 변화 등 유용한 인사이트를 얻을 수 있지만, 직접 결합하면 심각한 개인정보 침해가 발생한다.
- **제언**: 개인정보보호위원회 지정 결합 전문기관을 통한 가명정보 결합을 활용하고, 결합 후 k-익명성(k≥5) 검증 및 이상징후 모니터링 체계를 구축하여 분석 가치와 프라이버시를 동시에 보장해야 한다.
