## 해설

### 출제 배경 및 의도

부채널 공격은 암호 알고리즘 자체가 아닌 구현 과정에서 발생하는 물리적 정보(시간, 전력, 전자기파 등)를 이용해 비밀키를 유출하는 공격이다. FIPS 140-3에서도 부채널 공격 방어가 검증 항목으로 포함되어 있어, 공격 유형과 대응 방안을 체계적으로 서술해야 한다.

### 1. 부채널 공격 개요

정 의  • 암호 시스템의 물리적 구현에서 발생하는 부가 정보(Side Information)를 분석하여 비밀 정보(키, 평문)를 추출하는 공격
       - 수학적으로 완벽한 암호 알고리즘도 구현 취약점으로 공격 가능

### 2. 부채널 공격 유형

| 유형 | 공격 대상 정보 | 원리 |
|------|----------------|------|
| **타이밍 공격(Timing Attack)** | 연산 처리 시간 | 비트 값에 따라 연산 시간이 달라지는 점 악용 |
| **전력 분석(Power Analysis)** | 소비 전력 파형 | SPA(단순), DPA(차등) 전력 분석 |
| **전자기 공격(EM Attack)** | 전자기 방출 | 장치에서 방출되는 전자기 신호 분석 |
| **캐시 타이밍(Cache-Timing)** | CPU 캐시 접근 시간 | 캐시 히트/미스 시간 차로 키 추론 |
| **음향 분석(Acoustic Attack)** | 소리 | 연산 중 발생하는 음향 파형 분석 |
| **오류 주입(Fault Injection)** | 오류 발생 시 출력 | 의도적 오류로 잘못된 출력 분석 |
| **Spectre/Meltdown** | 투기적 실행 캐시 | CPU 투기 실행 후 캐시 상태로 메모리 유출 |

### 3. 주요 공격 상세

#### 타이밍 공격 (RSA 예시):
```
RSA 서명에서 모듈러 연산 시간이 비밀키 비트에 따라 다름:
비트 = 0: 제곱만 수행 → 짧은 시간
비트 = 1: 제곱 + 곱셈 수행 → 긴 시간
→ 수천 번 측정으로 키 비트 추론 가능
```

#### Spectre 공격:
```
① 분기 예측기 오훈련
② 투기적 실행으로 권한 외 메모리 접근
③ 캐시 히트 타이밍으로 데이터 유출
```

### 4. 대응 방안

| 공격 유형 | 대응 방안 |
|-----------|-----------|
| 타이밍 공격 | 상수 시간(Constant-Time) 구현, 연산 시간 균일화 |
| 전력 분석 | 랜덤 마스킹, 전력 소비 균일화 회로 |
| 전자기 공격 | 전자기 차폐(Faraday cage), 금속 케이싱 |
| 캐시 타이밍 | 캐시 플러시, 프로세스 격리(Intel SGX) |
| 오류 주입 | 중복 계산·검증, 오류 감지 코드 |
| Spectre/Meltdown | 마이크로코드 패치, 커널 페이지 테이블 격리(KPTI) |


- 최근 부채널 공격(Side 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**암호 라이브러리 안전 구현**
- **챌린지**: OpenSSL 등 암호 라이브러리에서도 타이밍 공격 취약점이 발견된다.
- **제언**: 상수 시간 암호 구현을 검증하는 ctgrind·dudect 도구를 CI/CD 파이프라인에 통합하고, FIPS 140-3 Level 3 이상 인증을 받은 HSM에서 키 연산을 수행하여 부채널 공격 노출을 최소화해야 한다.
