## 해설

### 1. SOAR 개요

정 의  • 보안 툴·프로세스를 통합 오케스트레이션하고 위협 탐지·대응을 자동화하는 플랫폼 (Gartner 정의)
- **등장 배경**: 보안 경고(Alert)의 폭증, 보안 인력 부족, 평균 대응시간(MTTR) 단축 필요

### 2. SOAR 구성요소

| 구성요소 | 역할 |
|----------|------|
| **오케스트레이션(Orchestration)** | 이기종 보안 툴(SIEM, EDR, FW, TIP) 통합 연계 |
| **자동화(Automation)** | Playbook 기반 반복 작업 자동 수행 |
| **대응(Response)** | 이상 탐지 시 자동 격리·차단·티켓 생성 |
| **Playbook** | 위협 유형별 표준화된 대응 절차(YAML/JSON) |
| **Case Management** | 보안 사고 이력 관리, 협업 도구 |
| **TIP 연계** | 위협 인텔리전스 피드(IOC) 자동 업데이트 |

### 3. SOAR 동작 흐름

```
보안 이벤트 발생
      ↓
SIEM 경고 발생 (알림)
      ↓
SOAR Playbook 트리거
      ↓
① 자동 조사: IP/도메인/파일 해시 평판 조회 (TIP 연계)
② 맥락 수집: 관련 사용자·시스템·로그 자동 수집
③ 위협 판단: 오탐 필터링 → 위협 수준 분류
      ↓
[고위험 판단 시]
④ 자동 대응: 악성 IP 차단, 감염 단말 네트워크 격리
⑤ 알림: 보안팀 Slack/이메일 통보, ITSM 티켓 생성
⑥ 사고 기록: Case Management에 전 과정 기록
```

### 4. SOAR vs SIEM 비교

| 항목 | SIEM | SOAR |
|------|------|------|
| **주요 기능** | 로그 수집·분석·상관관계 탐지 | 보안 툴 연계·자동화·대응 |
| **출력** | 경고(Alert), 보고서 | 자동 대응 액션, 사고 티켓 |
| **운영 주체** | 보안 분석가 (수동 대응) | Playbook 자동 실행 + 분석가 검토 |
| **연계** | SOAR에 이벤트 공급 | SIEM 경고를 트리거로 활용 |

### 5. SOAR 기대효과

| 효과 | 내용 |
|------|------|
| **MTTR 단축** | 평균 대응시간 80% 이상 단축 |
| **인력 효율화** | 반복적 1차 분석을 자동화, 고급 분석에 집중 |
| **일관성 확보** | Playbook 기반 표준화된 대응 절차 |
| **가시성 향상** | 모든 보안 활동 통합 기록·추적 |


- 최근 SOAR(Security Orchestration, Automation and Response) 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**SOAR 도입 시 Playbook 품질 관리**
- **챌린지**: SOAR 도입 후 잘못 설계된 Playbook이 오탐을 자동 차단하여 정상 업무가 중단되는 "자동화 역효과"가 발생한다.
- **제언**: 초기 6개월은 Playbook을 "권고 모드(Recommend)"로 운영하여 분석가가 최종 승인 후 실행하고, 오탐률 5% 이하로 검증된 Playbook만 "자동 실행 모드"로 전환하는 단계적 적용 전략과 월간 Playbook 효과성 검토 프로세스를 수립해야 한다.
