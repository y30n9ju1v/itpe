## 문제

국가정보원이 발간한 「제로 트러스트 가이드라인 2.0」에 따라 공공기관의 정보보호 체계가 전환되고 있다. 다음에 대해 설명하시오.
가. 제로 트러스트 가이드라인 2.0의 주요 내용
나. 제로 트러스트 성숙도 모델 (3단계)
다. 공공기관 제로 트러스트 전환 시 고려사항

## 해설

### 1. 제로 트러스트 가이드라인 2.0 주요 내용

- **발간**: 국가정보원 (2023년 가이드라인 1.0 → 2024년 2.0 업데이트)
- **배경**: NIST SP 800-207 기반의 국내 공공기관 맞춤형 지침 마련

| 주요 원칙 | 내용 |
|-----------|------|
| **신원 확인 강화** | MFA 의무화, 특권 계정 PAM 도입 |
| **기기 무결성 검증** | 접속 전 단말기 보안 상태(패치·백신) 검증 |
| **최소 권한 접근** | 업무별 최소 권한, Just-in-Time 접근 |
| **지속 모니터링** | 모든 세션 실시간 로깅·이상 탐지 |
| **데이터 중심 보호** | 데이터 분류·암호화·DRM 연계 |

### 2. 제로 트러스트 성숙도 모델 (3단계)

| 단계 | 명칭 | 특징 |
|------|------|------|
| **1단계** | 초기(Traditional) | 경계 기반 보안, 내부망 신뢰 존재 |
| **2단계** | 발전(Advanced) | 부분적 ZT 적용 (ID 기반 접근, MFA 도입) |
| **3단계** | 최적(Optimal) | 완전한 ZTA — 마이크로세그멘테이션 완성, 자동화된 정책 집행, AI 기반 위협 탐지 |

### 3. 제로 트러스트 6대 핵심 기능 요소

| 요소 | 내용 |
|------|------|
| **신원(Identity)** | IdP, MFA, SSO, 권한 관리(PAM) |
| **기기(Device)** | MDM, EDR, 기기 등록·위협 탐지 |
| **네트워크(Network)** | SDP, 마이크로세그멘테이션, DNS 보안 |
| **응용(Application)** | API Gateway, WAF, 앱 단위 접근 통제 |
| **데이터(Data)** | DRM, 데이터 분류, 암호화 |
| **시각화·분석(Visibility)** | SIEM, SOAR, UEBA |

### 4. 공공기관 제로 트러스트 전환 시 고려사항

| 고려사항 | 내용 |
|----------|------|
| **레거시 시스템 호환** | 구형 시스템은 ZTA 직접 연동 어려움 → 프록시 방식 적용 |
| **내부망(망분리) 정책 조화** | 현행 망분리 규정과 ZTA의 충돌 지점 분석·조정 필요 |
| **사용자 변화 관리** | MFA 강제 도입 시 업무 불편 최소화 교육 |
| **예산·로드맵** | 단기(MFA·PAM) → 중기(마이크로세그멘테이션) → 장기(완전 자동화) |
| **운영 역량** | ZTA 운영 전담 인력 확보 또는 MSS(보안관제 서비스) 활용 |


- 최근 제로 트러스트 가이드라인 2.0 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**공공기관 망분리와 제로 트러스트의 병행 운영**
- **챌린지**: 국내 공공기관은 「국가 사이버안보 기본법」상 망분리 의무가 있어 ZTA의 "경계 제거" 원칙과 충돌하는 상황이 발생한다.
- **제언**: 물리적 망분리는 유지하되, 각 망 내부에서 마이크로세그멘테이션과 신원 기반 접근 통제를 적용하는 "망분리 + 제로 트러스트" 하이브리드 모델을 채택하고, 망 연계 구간에는 CC EAL4 이상 인증 SDP 게이트웨이를 배치하여 보안 수준을 단계적으로 높여야 한다.
