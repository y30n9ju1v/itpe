## 해설

### 1. 해싱의 개념 및 구성요소

- **해싱(Hashing)**: 탐색키를 해시 함수에 적용하여 해시 테이블의 주소(버킷)로 변환, O(1)에 데이터를 저장·검색하는 방법

| 구성요소 | 설명 |
|----------|------|
| 해시 함수(Hash Function) | 탐색키 → 해시 주소로 변환하는 함수 |
| 해시 테이블(Hash Table) | 버킷(Bucket)들의 집합 |
| 버킷(Bucket) | 하나 이상의 레코드를 저장하는 단위 |
| 슬롯(Slot) | 버킷 내의 레코드 저장 공간 |
| 충돌(Collision) | 서로 다른 키가 같은 버킷 주소를 가지는 현상 |
| 동거자(Synonyms) | 동일 버킷 주소를 공유하는 키들의 집합 |
| 오버플로우(Overflow) | 버킷의 슬롯이 모두 채워져 더 이상 저장 불가인 상태 |

### 2. 해싱의 유형

| 유형 | 특징 | 적용 |
|------|------|------|
| **정적 해싱(Static Hashing)** | 해시 테이블 크기 고정, 크기 변경 시 전체 재해싱 필요 | 소규모 고정 데이터 |
| **동적 해싱(Dynamic Hashing)** | 데이터 증가에 따라 테이블 크기 동적 확장 | 가변 데이터 환경 |
| **확장 가능 해싱(Extendible Hashing)** | 디렉터리 + 버킷 분할로 확장 | DB 인덱스 |
| **선형 해싱(Linear Hashing)** | 순서대로 버킷 분할, 디렉터리 불필요 | 분산 DB |

### 3. 해싱 오버플로우 해결 기법

| 기법 | 설명 | 특징 |
|------|------|------|
| **선형 조사(Linear Probing)** | 충돌 시 다음 버킷 순서대로 탐색 | 간단하나 클러스터링 발생 |
| **이차 조사(Quadratic Probing)** | 충돌 시 i², 2i²... 간격으로 탐색 | 1차 클러스터링 감소 |
| **이중 해싱(Double Hashing)** | 두 번째 해시 함수로 탐사 간격 결정 | 최소 클러스터링 |
| **체이닝(Chaining)** | 버킷을 연결 리스트로 확장 | 유연하나 포인터 오버헤드 |
| **오버플로우 영역(Overflow Area)** | 별도 오버플로우 구역을 미리 예약 | DB 파일 구조에 활용 |
| **버킷 분할(Bucket Splitting)** | 오버플로우 시 버킷 분할 후 재배치 | 동적 해싱에서 활용 |


- 최근 해싱 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**분산 캐시(Redis) 설계**
- **챌린지**: Redis 클러스터에서 특정 해시 슬롯에 트래픽이 집중되면 핫스팟이 발생한다.
- **제언**: Redis Cluster의 16384개 해시 슬롯을 노드별로 균등 분배하고, 핫 키(Hot Key)에는 키 이름에 랜덤 접미사를 붙여 여러 슬롯으로 분산하는 키 해시태그 전략을 적용해야 한다.
