## 해설

### 출제 배경 및 의도

동형 암호는 암호화된 상태에서 연산이 가능한 차세대 암호 기술로, 클라우드 프라이버시 보호와 AI 데이터 분석 보안의 핵심 기술이다. 개념·종류·한계·활용 사례를 체계적으로 서술해야 한다.

### 1. 동형 암호 개요

정 의  • 암호화된 데이터를 복호화하지 않고 연산(덧셈·곱셈)을 수행하여, 연산 결과를 복호화하면 평문에서 직접 연산한 결과와 동일한 값을 얻는 암호 기법
- **수학적 표현**: Enc(a) OP Enc(b) = Enc(a op b)
- **의의**: "암호화 상태에서 연산" → 데이터 프라이버시와 연산 기능을 동시에 보장

```
[기존 방식]
평문 데이터 ──암호화──▶ 암호문 ──전송──▶ [서버: 복호화 → 연산 → 재암호화] ──▶ 결과
                                                        ↑ 서버가 평문 노출

[동형 암호 방식]
평문 데이터 ──암호화──▶ 암호문 ──전송──▶ [서버: 암호문 상태로 연산] ──▶ 암호화된 결과
                                                        ↑ 서버는 평문 미접근
```

### 2. 동형 암호 종류

| 종류 | 지원 연산 | 특징 | 대표 스킴 |
|------|-----------|------|-----------|
| **부분 동형(PHE)** | 덧셈 또는 곱셈 중 하나만 | 효율적, 제한적 | Paillier(덧셈), ElGamal(곱셈) |
| **다소 동형(SHE/LHE)** | 덧셈+곱셈, 단 회수 제한 | PHE보다 유연 | BGV, BFV |
| **완전 동형(FHE)** | 덧셈+곱셈 무제한 | 임의 연산 가능, 느림 | CKKS, TFHE, CKKS |

### 3. 주요 FHE 스킴 비교

| 스킴 | 특징 | 적합 용도 |
|------|------|-----------|
| **BGV/BFV** | 정수 연산, 정밀도 높음 | 통계, DB 쿼리 |
| **CKKS** | 부동소수점 근사 연산 | ML 추론, 통계 |
| **TFHE** | 논리 게이트 단위, 빠른 부트스트래핑 | 범용 회로 |

### 4. 한계 및 극복 방안

| 한계 | 극복 방안 |
|------|-----------|
| 연산 속도 매우 느림(평문 대비 1000배 이상) | GPU 가속, 하드웨어 가속기(HEaaN) |
| 암호문 크기 증가 | 최적화된 파라미터 선택, 배치 처리 |
| 복잡한 프로그래밍 | 고수준 API(Microsoft SEAL, OpenFHE) |
| 노이즈 누적 → 부트스트래핑 필요 | FHE 부트스트래핑, 파라미터 설계 |


- 동형 암호는 암호화된 상태에서 연산이 가능하여 의료·금융 데이터의 프라이버시를 보존하며 클라우드 분석이 가능한 혁신 기술로, 연산 오버헤드가 크나 CKKS·BFV 등 스킴 최적화와 GPU 가속으로 실용화가 진행 중임.  "끝"

### 실무 제언

**의료 데이터 AI 분석**
- **챌린지**: 의료 데이터는 개인정보보호법으로 외부 분석이 제한되어 AI 학습 데이터 확보가 어렵다.
- **제언**: CKKS 기반 동형 암호를 적용하면 환자 데이터를 암호화한 상태에서 AI 모델 추론을 수행할 수 있어, 데이터 소유자(병원)가 데이터를 복호화하지 않고도 외부 분석 기관에 연산을 위탁할 수 있다.
