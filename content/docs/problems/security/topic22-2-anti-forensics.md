## 해설

### 1. 안티포렌식 개요

정 의  • 디지털 범죄 수사 과정에서 증거 발견을 방해하거나 분석 도구를 무력화하여 수사 효율을 저하시키기 위한 일체의 기술적·비기술적 행위
- **등장 배경**: 디지털 포렌식 기술 발전 → 범죄자들의 대응 기술도 진화

### 2. 안티포렌식 기법 분류

| 분류 | 기법 | 세부 방법 |
|------|------|-----------|
| **데이터 파기** | 보안 삭제 | DoD 5220.22-M 7회 덮어쓰기, Gutmann 35회 |
| | 물리적 파괴 | HDD 천공, 강자기장 소거(디가우징) |
| | SSD 초기화 | ATA Secure Erase, TRIM 명령 |
| **데이터 은닉** | 스테가노그래피 | 이미지·음성·동영상에 데이터 삽입(LSB 치환) |
| | 파일 시스템 조작 | 슬랙 공간 활용, MFT 엔트리 직접 편집 |
| | 암호화 | 전체 디스크 암호화(VeraCrypt, BitLocker) |
| **증거 변조** | 타임스탬프 조작 | touch 명령, Timestomp 도구로 MAC 시간 변경 |
| | 메타데이터 삭제 | ExifTool로 사진·문서 메타데이터 제거 |
| **포렌식 방해** | 도구 탐지 | 포렌식 소프트웨어 프로세스 탐지 후 증거 삭제 |
| | 허위 증거 삽입 | 엉뚱한 용의자를 가리키는 위조 아티팩트 생성 |
| | 가상머신 탐지 | VM 환경에서 악성코드 자동 비활성화 |

### 3. 디지털 포렌식 대응 방안

| 대응 방안 | 내용 |
|-----------|------|
| **라이브 포렌식** | 전원 차단 전 메모리·네트워크 상태 즉시 수집(휘발성 데이터 보존) |
| **다중 해시 검증** | MD5·SHA-256·SHA-3 복수 해시로 무결성 검증 |
| **스테가노그래피 탐지** | StegDetect, Steganalysis 도구로 은닉 데이터 탐지 |
| **타임라인 분석** | 여러 시간 소스(로그, MFT, 레지스트리) 교차 검증으로 조작 탐지 |
| **로그 외부 보관** | SIEM·외부 서버로 실시간 로그 전송하여 로컬 로그 파기 무력화 |
| **포렌식 레디 아키텍처** | 사고 이전부터 증거 수집 가능한 인프라 설계 |

### 4. 법적 이슈

| 이슈 | 내용 |
|------|------|
| **증거 능력** | 안티포렌식으로 변조된 증거의 법적 효력 인정 어려움 |
| **연계 보관성** | Chain of Custody 일지 유지로 증거 신뢰성 입증 |
| **형사법 적용** | 증거인멸죄(형법 제155조), 전자기록위작·변작죄(형법 제232조의2) |
| **국제 협력** | 국경 넘는 디지털 범죄 수사 시 MLAT(사법공조조약) 활용 |


- 안티포렌식은 데이터 삭제·암호화·타임스탬프 조작·스테가노그래피 등으로 디지털 증거를 은닉·파괴하며, 법원에서의 증거 인멸 고의성 입증이 법적 쟁점이 되므로 사전 로그 무결성 확보와 MLAT 기반 국제 공조 수사 체계 구축이 대응의 핵심임.  "끝"

### 실무 제언

**포렌식 레디(Forensic Readiness) 아키텍처 설계**
- **챌린지**: 사고 발생 후 디지털 증거를 수집하려 할 때, 범인이 안티포렌식 기술로 이미 증거를 파기한 경우가 빈번하다.
- **제언**: 사고 이전 단계부터 불변 로그 저장소(AWS S3 Object Lock, Worm Storage)에 시스템 로그를 실시간 복제하고, 네트워크 패킷의 일정 기간 보관 정책(90일 이상)을 수립하여, 로컬 증거 파기 시에도 법적 유효한 증거를 확보할 수 있는 포렌식 레디 아키텍처를 구축해야 한다.
