전자금융사기가 지속적으로 증가하는 상황에서, 이상금융거래를 탐지하는 시스템인 FDS(Fraud Detection System)에 대해서 다음 물음에 답하시오.

## 해설

### 1. FDS(Fraud Detection System) 개요

- **정의**: 전자금융거래의 단말기 정보, 접속 정보, 거래내용 등을 종합적으로 분석하여 이상 금융거래를 탐지·차단하는 시스템
- **법적 근거**: 전자금융거래법 제21조의4(전자적 침해행위 등의 탐지·분석)
- **의무 적용**: 은행·카드·증권·보험사 등 전자금융업자(금융감독원 지도)

### 2. FDS 구성요소

```
┌─────────────────────────────────────────────────┐
│                   FDS 구성도                     │
│                                                  │
│  [데이터 수집 계층]                               │
│  단말기 정보 · 접속 IP/위치 · 거래 패턴 · 행동분석│
│            ↓                                     │
│  [분석 엔진 계층]                                 │
│  규칙 기반 탐지 · ML 모델 · 이상치 탐지          │
│            ↓                                     │
│  [대응 계층]                                      │
│  실시간 차단 · 추가 인증 · 알림 · 사후 분석       │
└─────────────────────────────────────────────────┘
```

| 구성요소 | 설명 |
|----------|------|
| **정보 수집** | 단말기 정보(OS, 브라우저), IP, 위치, 거래금액·시간, 수신계좌 |
| **분석 엔진** | 규칙 기반 + 머신러닝 + 이상치 탐지 하이브리드 |
| **거래 스코어링** | 각 거래에 위험 점수(Risk Score) 부여 |
| **대응 모듈** | 임계점 초과 시 자동 차단 또는 추가 인증(OTP, ARS) 요청 |
| **모니터링 화면** | 실시간 이상거래 현황 대시보드 (보안담당자) |

### 3. FDS 탐지 방법

| 방법 | 설명 | 예시 |
|------|------|------|
| **규칙 기반(Rule-Based)** | 사전 정의된 이상 패턴 매칭 | 1분 내 3회 이상 결제 실패 → 차단 |
| **통계 기반** | 평균·표준편차 기반 이상치 탐지 | 평소 거래금액 대비 10배 초과 시 경보 |
| **ML/AI 기반** | 비지도 학습(Isolation Forest), 지도 학습(Random Forest) | 정상 거래 패턴 학습 후 이탈 시 탐지 |
| **행동분석(BA)** | 사용자 행동 프로파일링 | 마우스 움직임, 키 입력 패턴 이상 탐지 |
| **위치 기반** | GPS/IP 위치 불일치 탐지 | 서울에서 접속 후 5분 내 뉴욕 IP로 결제 |
| **그래프 분석** | 자금 흐름 네트워크 분석 | 빠른 자금 순환 계좌 패턴 탐지 |

### 실무 제언

**AI 기반 FDS 고도화 시 오탐(False Positive) 최소화**
- **챌린지**: 머신러닝 기반 FDS는 고위험 거래를 과다 차단하여 정상 고객 불편을 초래(오탐률 증가)하거나, 반대로 정교한 신종 사기 패턴을 놓치는(미탐) 딜레마에 빠진다.
- **제언**: 규칙 기반(고속 1차 필터) + ML(정밀 2차 분석) + 행동분석(3차 검증)의 3단계 파이프라인을 구성하고, 분기별 모델 재학습(신종 금융사기 패턴 반영) 및 금융보안원 공유 위협 인텔리전스를 연동하여 오탐률을 5% 이하로 관리해야 한다.
