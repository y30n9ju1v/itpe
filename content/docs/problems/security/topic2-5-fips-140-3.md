## 해설

### 출제 배경 및 의도

FIPS 140-3은 암호 모듈의 보안 요구사항을 정의하는 미국 연방 표준이다. 국내 공공기관과 금융 분야에서도 암호 모듈 검증의 준거로 활용되므로, 보안 레벨(Level 1~4)과 검증 항목을 체계적으로 서술할 수 있어야 한다.

### 1. FIPS 140-3 개요

- **정의**: NIST가 발행한 암호화 모듈에 대한 보안 요구사항 표준(ISO/IEC 19790 기반)
- **목적**: 민감한 정보를 처리하는 암호 모듈의 보안성 검증·인증
- **이전 표준**: FIPS 140-2(2001~2021) → FIPS 140-3(2022 발효)
- **국내 연계**: KCMVP(Korea Cryptographic Module Validation Program)의 참조 표준

### 2. FIPS 140-3 보안 레벨

| 레벨 | 요구사항 | 적용 대상 |
|------|----------|-----------|
| **Level 1** | 기본 보안 요구사항 충족, 물리적 보안 없음 | 소프트웨어 암호 모듈 |
| **Level 2** | 변조 증거(Tamper-Evident) 코팅·씰 요구 | 일반 상용 제품 |
| **Level 3** | 변조 탐지·저항(Tamper-Resistant), 신원 기반 인증 | 고보안 환경 |
| **Level 4** | 물리적 완전 봉인, 환경 공격 탐지 | 군사·금융 최고보안 환경 |

### 3. FIPS 140-3 주요 검증 항목

| 영역 | 주요 요구사항 |
|------|---------------|
| 암호 모듈 명세 | 경계 정의, 승인된 알고리즘만 사용 |
| 모듈 포트·인터페이스 | 데이터 입출력·제어·상태 인터페이스 명시 |
| 역할·서비스·인증 | Crypto Officer, User 역할 구분 |
| 소프트웨어/펌웨어 보안 | 자가 테스트, 무결성 검사 |
| 운영 환경 | 신뢰 OS 또는 비수정 OS 사용 |
| 물리적 보안 | 레벨별 물리 보호 요구 |
| 비침습적 보안 | 부채널 공격 방어 |
| 민감 보안 파라미터 관리 | 키 생성·입력·출력·저장·파기 |
| 자가 테스트 | 전원 인가 시 알고리즘·무결성 자동 테스트 |
| 생명주기 보증 | 설계·구현·테스트 문서화 |
| 기타 공격 완화 | 타이밍 공격 등 대응 |

### 4. FIPS 140-2 vs FIPS 140-3 비교

| 구분 | FIPS 140-2 | FIPS 140-3 |
|------|-----------|-----------|
| 기반 표준 | 자체 표준 | ISO/IEC 19790/24759 |
| 발효 시기 | 2001년 | 2022년 |
| 보안 레벨 | Level 1~4 동일 | Level 1~4 동일, 요구사항 강화 |
| 알고리즘 목록 | TDES 포함 | TDES 제외, 양자내성 암호 대비 |
| 인증 | CMVP(NIST+CSEC) | CMVP 동일 체계 |

### 실무 제언

**공공기관 암호 모듈 선정**
- **챌린지**: 국내 공공기관은 정보보호제품 구매 시 FIPS 140-3 또는 KCMVP 인증 여부를 확인해야 하지만, 기준이 복잡하다.
- **제언**: 국가정보원의 CC 인증 및 KCMVP 인증 제품 목록을 우선 참조하고, 해외 제품은 FIPS 140-3 Level 2 이상을 요구하는 조달 정책을 적용해야 한다.
