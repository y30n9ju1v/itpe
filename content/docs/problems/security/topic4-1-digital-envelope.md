## 해설

### 출제 배경 및 의도

디지털봉투는 비대칭키의 키 배포 문제와 대칭키의 속도 장점을 결합한 하이브리드 암호화의 대표 기법이다. 생성(암호화) → 전송 → 개봉(복호화) 3단계 절차와 사용 이유를 그림과 함께 서술해야 한다.

### 1. 디지털봉투 출현 배경

| 방식 | 문제점 |
|------|--------|
| 대칭키 암호화만 사용 | 키 배포 문제: 수신자에게 비밀키를 안전하게 전달하기 어려움 |
| 비대칭키 암호화만 사용 | 속도 문제: 대용량 데이터 암호화에 RSA 등은 너무 느림 |

→ **해결책**: 대칭키로 데이터를 암호화하고, 그 세션키를 수신자의 공개키로 암호화하여 함께 전송

### 2. 디지털봉투 생성 과정 (송신자)

```
[송신자]
평문(M)
  │
  ├─▶ ①랜덤 세션키(K) 생성
  │         │
  │         ├─▶ ②대칭키 암호화(K)──▶ 암호문(C)
  │         │                              │
  │         └─▶ ③수신자 공개키(PUB_B)      │
  │                    로 K 암호화         │
  │                    ──▶ 봉투(E)         │
  │                              │         │
  └─────────────────────────────┼─────────┘
                                 │
                           [전송: C + E]
```

### 3. 디지털봉투 개봉 과정 (수신자)

```
[수신자]
C + E 수신
  │
  ├─ E ──▶ ①수신자 개인키(PRV_B)로 봉투(E) 복호화 ──▶ 세션키(K)
  │
  └─ C ──▶ ②세션키(K)로 암호문(C) 복호화 ──▶ 평문(M)
```

### 4. 디지털봉투의 특징

| 항목 | 내용 |
|------|------|
| 기밀성 | 세션키가 수신자 공개키로 암호화되어 수신자만 복호화 가능 |
| 속도 | 대용량 데이터는 빠른 대칭키로 암호화 |
| 키 배포 해결 | 공개키 기반으로 세션키를 안전하게 전달 |
| 활용 | S/MIME, PGP, TLS 핸드셰이크의 핵심 메커니즘 |


- 디지털봉투는 대칭키 암호화의 속도와 공개키 암호화의 키 관리 편의성을 결합한 하이브리드 암호화 메커니즘으로, S/MIME·PGP·TLS 핸드셰이크 등 실무 통신 보안의 핵심이며, 세션키 보호와 수신자 인증을 동시에 보장하는 효율적 구조임.  "끝"

### 실무 제언

**이메일 보안(S/MIME) 적용**
- **챌린지**: 기업 이메일에 첨부된 민감 문서가 평문으로 전송되어 도청 위험이 있다.
- **제언**: S/MIME(Secure/Multipurpose Internet Mail Extensions)을 적용하면 디지털봉투 방식으로 이메일 본문과 첨부파일을 암호화하고, 디지털서명으로 발신자 인증까지 동시에 제공할 수 있다.
