## 해설

### 출제 배경 및 의도

해시의 기본 개념과 암호학적 해시 함수의 종류, 그리고 해시 충돌 해결 기법(체이닝, 개방 주소법 등)을 모두 다룰 수 있는지를 검증한다. 자료구조 관점의 해시 충돌과 보안 관점의 해시 충돌을 구분하여 서술해야 한다.

### 1. 해싱 개요

- **해싱(Hashing)**: 키를 해시 함수로 변환하여 해시 테이블의 특정 주소에 저장·검색하는 방법
- **해시 함수**: 임의 길이 입력 → 고정 길이 출력(해시값/다이제스트)
- **용도**: 빠른 검색(O(1)), 무결성 검증, 디지털 서명, 패스워드 보호

### 2. 해시함수 종류

#### 가. 자료구조 관점 (해시 테이블용)

| 함수 | 방법 | 특징 |
|------|------|------|
| 나머지 연산법 | h(k) = k mod m | 간단, m을 소수로 설정 시 분산 우수 |
| 제곱 연산법 | k²의 중간 비트 추출 | 키 분산 효과 |
| 폴딩법 | 키를 부분으로 나눠 XOR/덧셈 | 긴 키 처리 적합 |
| 문자열 해싱 | 다항식 기반(Horner's Method) | 문자열 키 처리 |

#### 나. 보안 관점 (암호학적 해시)

| 알고리즘 | 출력 길이 | 현황 |
|----------|-----------|------|
| MD5 | 128bit | 충돌 취약, 폐기 |
| SHA-1 | 160bit | 충돌 취약, 폐기 |
| SHA-256 | 256bit | 현재 표준 |
| SHA-512 | 512bit | 고보안 표준 |
| SHA-3 (Keccak) | 가변 | 차세대 표준 |
| HMAC | 가변 | 메시지 인증 코드 |

### 3. 해시 충돌(Hash Collision) 해결 방법

서로 다른 키가 동일한 해시 버킷을 가리키는 충돌을 해결하는 두 가지 주요 방법:

#### 가. 체이닝 (Chaining / 개방 해싱)

```
버킷 0: [key1] → [key5] → null
버킷 1: [key2] → null
버킷 2: [key3] → [key6] → null
```

- 각 버킷을 연결 리스트로 구성
- 충돌 발생 시 리스트에 추가
- **장점**: 버킷 수보다 많은 데이터 저장 가능
- **단점**: 포인터 오버헤드, 캐시 미스 증가

#### 나. 개방 주소법 (Open Addressing / 폐쇄 해싱)

| 방법 | 설명 |
|------|------|
| 선형 조사(Linear Probing) | 충돌 시 h(k)+1, h(k)+2... 순서로 빈 버킷 탐색 → 클러스터링 문제 |
| 이차 조사(Quadratic Probing) | 충돌 시 h(k)+1², h(k)+2²... → 1차 클러스터링 해소 |
| 이중 해싱(Double Hashing) | 두 번째 해시 함수로 탐사 간격 결정 → 분산 효과 최대 |


- 최근 해싱의 개요, 해시함수 종류 및 해시충돌 해결방법 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**고성능 캐시 시스템 설계**
- **챌린지**: 해시 테이블에서 클러스터링이 심화되면 탐색 성능이 O(1)에서 O(n)으로 저하된다.
- **제언**: 부하율(Load Factor)을 0.7 이하로 유지하고 이중 해싱을 적용하며, 자바의 HashMap처럼 부하율 초과 시 자동 리해싱(rehashing) 메커니즘을 구현하여 평균 O(1) 성능을 유지해야 한다.
