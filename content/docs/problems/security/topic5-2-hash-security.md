## 해설

### 출제 배경 및 의도

해시 함수의 취약점(충돌 공격, 길이 확장 공격, Rainbow Table 등)과 각 취약점에 대한 구체적인 강화 방안을 체계적으로 서술해야 한다. 알고리즘 선택부터 구현 레벨의 강화 기법까지 다층적으로 제시해야 한다.

### 1. 해시 함수 주요 공격 유형

| 공격 유형 | 설명 | 영향 |
|-----------|------|------|
| **충돌 공격** (Collision Attack) | H(x) = H(x')인 두 입력 탐색 | 디지털서명 위조 |
| **역상 공격** (Preimage Attack) | 해시값으로 원본 입력 복원 | 패스워드 탈취 |
| **Rainbow Table** | 사전 계산된 해시 테이블로 역산 | 패스워드 크랙 |
| **길이 확장 공격** (Length Extension) | HMAC 미사용 시 메시지 임의 확장 | MAC 위조 |
| **생일 공격** (Birthday Attack) | 생일 역설로 충돌 탐색 시간 단축 | n비트 → 2^(n/2) 탐색 |

### 2. 해시 함수 안전성 강화 방안

#### 가. 알고리즘 수준 강화

| 방안 | 내용 |
|------|------|
| SHA-2/SHA-3 사용 | MD5·SHA-1 대신 SHA-256/512 이상 적용 |
| 출력 길이 증가 | 128bit → 256bit 이상으로 생일 공격 저항성 2배 강화 |
| SHA-3(Keccak) 전환 | Sponge 구조로 길이 확장 공격 원천 차단 |

#### 나. 구현 수준 강화

| 방안 | 내용 |
|------|------|
| **Salt 적용** | 랜덤 난수를 PW와 결합 후 해시 → Rainbow Table 무력화 |
| **반복 해시(Stretching)** | bcrypt·PBKDF2·Argon2으로 반복 횟수 적용 → 브루트포스 비용 증가 |
| **HMAC 사용** | 비밀키 + 해시 = 메시지 인증 코드(MAC) → 길이 확장 공격 방어 |
| **Pepper 적용** | Salt에 추가로 서버 측 비밀 상수(Pepper) 결합 → DB 유출 시에도 안전 |

```
강화된 패스워드 저장:
H_final = Argon2(Password || Salt, iterations=100000) XOR Pepper
→ Salt: DB 저장, Pepper: 환경변수/HSM 저장
```

#### 다. 운영 수준 강화

| 방안 | 내용 |
|------|------|
| 해시 알고리즘 갱신 주기 관리 | 취약 알고리즘 발견 시 마이그레이션 계획 수립 |
| 양자 저항성 고려 | NIST PQC 기준 SHA-3·SHA-512 이상 권고 |
| HSM 기반 키 보호 | HMAC 키를 HSM에 저장하여 키 유출 방지 |

### 실무 제언

**레거시 시스템 MD5 마이그레이션**
- **챌린지**: 기존 시스템이 MD5로 패스워드를 저장하고 있어 Rainbow Table 공격에 무방비다.
- **제언**: 점진적 마이그레이션 전략으로, 사용자가 다음 로그인 시 새 알고리즘(Argon2)으로 자동 재해시하고, 기존 MD5 해시는 유효 기간 설정 후 강제 패스워드 변경을 요구하는 방식으로 무중단 전환해야 한다.
