## 해설

### 출제 배경 및 의도

MLS(Multi-Level Security)는 서로 다른 보안 등급의 데이터를 단일 시스템에서 동시에 처리하는 개념으로, 군사·정부 시스템에서 중요하다. 보안 등급 체계와 Bell-LaPadula 모델과의 연관성을 서술해야 한다.

### 1. MLS(Multi-Level Security) 개요

정 의  • 기밀·비밀·대외비 등 서로 다른 보안 등급의 정보를 동일한 시스템에서 안전하게 동시 처리하되, 하위 등급 사용자가 상위 등급 정보에 접근할 수 없도록 보장하는 보안 체계
- **기반 모델**: Bell-LaPadula 모델 (No Read Up, No Write Down)
- **적용**: 군사 기밀 시스템, 정보기관, 고보안 정부 시스템

### 2. MLS 보안 등급 체계

1) 미국 정부 기준 (US Government Classification)

| 등급 | 수준 | 접근 권한 |
|------|------|-----------|
| Top Secret(TS) | 최고 기밀 | TS 인가자만 |
| Secret(S) | 비밀 | S 이상 인가자 |
| Confidential(C) | 대외비 | C 이상 인가자 |
| Unclassified(U) | 비분류 | 누구나 |

2) 국내 기준 (국가보안기본법)

| 등급 | 내용 |
|------|------|
| I급 비밀 | 국가 안보에 치명적 손해 |
| II급 비밀 | 국가 안보에 중대한 손해 |
| III급 비밀 | 국가 안보에 상당한 손해 |
| 대외비 | 공개 시 업무 수행에 지장 |

### 3. MLS 도입 5단계

| 단계 | 활동 | 내용 |
|------|------|------|
| **1단계** | 자산 식별 및 분류 | 보호 대상 정보 자산 목록화, 민감도 분류 기준 수립 |
| **2단계** | 보안 등급 정의 | 조직의 보안 정책에 따른 등급 체계 확립(기밀/비밀/일반 등) |
| **3단계** | 접근제어 정책 수립 | 등급별 접근 규칙 정의(MAC 기반, Bell-LaPadula 적용) |
| **4단계** | 기술적 통제 구현 | MLS 지원 OS(SELinux, Trusted Solaris), 라벨링 시스템, 보안 커널 적용 |
| **5단계** | 감사 및 검증 | 등급별 접근 로그 감사, 컴플라이언스 검증, 주기적 보안 등급 재분류 |

### 4. MLS 구현 기술

| 기술 | 설명 |
|------|------|
| **SELinux** | 리눅스 커널 레벨 MLS 접근제어 |
| **보안 레이블링** | 파일·프로세스에 보안 등급 메타데이터 부착 |
| **Type Enforcement** | 프로세스 타입별 접근 정책 정의 |
| **MLS 지원 DBMS** | Oracle Label Security, 보안 레이블 기반 쿼리 필터링 |


- MLS는 기밀·비밀·대외비 등 보안 등급별로 접근을 강제 통제하는 체계로, Bell-LaPadula 모델을 기반으로 하며 국방·정보기관 등 고보증 환경에서 단일 시스템으로 다중 등급 정보를 안전하게 처리하기 위한 필수 아키텍처임.  "끝"

### 실무 제언

**클라우드 환경 MLS 적용**
- **챌린지**: 공공기관의 클라우드 전환 시 기밀·비밀·공개 데이터가 동일 클라우드 인프라에 혼재할 수 있다.
- **제언**: AWS GovCloud·Naver Cloud의 보안 등급별 격리 서비스를 활용하고, 데이터 분류 태그(Classification Label)를 S3 오브젝트·RDS 행 단위로 부착하여 MLS 수준의 접근 제어를 클라우드 환경에서 구현해야 한다.
