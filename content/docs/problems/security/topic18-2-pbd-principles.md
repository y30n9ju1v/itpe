## 문제

PbD(Privacy by Design)는 광범위한 네트워크 환경에서 발생할 수 있는 데이터 처리의 폐해를 방지하기 위해 개나다 온타리오 주의 정보 및 프라이버시 위원(Information and Privacy Commissioner)을 지낸 Ann Cavoukian이 처음 창안해 낸 개념이다. ICT부야의 프라이버시 보호를 위한 주요방법론으로서 다수의 국가에서 이를 정책에 반영하고 있다. 이와 관련하여 다음을 설명하시오.
가. Privacy by Design의 7대 원칙
나. Privacy by Design의 8대 전략
다. Privacy by Design의 8대 전략과 개인정보 보호원칙 간의 개인정보 보호원칙과의 비교

## 해설

### 출제 배경 및 의도

133회 실제 기출로, PBD의 7대 원칙·8대 전략을 심층적으로 서술하고 개인정보보호법 제3조 개인정보 보호원칙과의 연계성을 비교 분석해야 한다. topic18-1의 심화 버전이다.

### 1. Privacy by Design 7대 원칙 (상세)

| 번호 | 원칙 | 상세 내용 |
|------|------|-----------|
| 1 | 사전적·예방적 | 프라이버시 침해 발생 후 해결이 아닌, 설계 단계에서 침해를 예방 |
| 2 | 기본 설정으로서의 프라이버시 | 사용자가 아무 조치 없이도 자동으로 최대 프라이버시 설정이 적용 |
| 3 | 설계에 내재화 | 프라이버시를 시스템의 부가 기능이 아닌 핵심 아키텍처로 통합 |
| 4 | 완전한 기능 — 보안과 프라이버시의 합 | 보안 강화와 프라이버시 보호가 상충하지 않음. "보안 AND 프라이버시" |
| 5 | 종단간·전 수명주기 보호 | 수집 → 처리 → 저장 → 이용 → 파기 전 과정에서 보호 |
| 6 | 가시성·투명성 | 이해관계자 누구나 독립적으로 시스템의 프라이버시를 검증 가능 |
| 7 | 사용자 중심 | 이용자가 자신의 데이터에 대한 통제권을 보유 |

### 2. Privacy by Design 8대 전략

#### 가. 데이터 지향 전략

| 전략 | 설명 | 기술적 구현 |
|------|------|-------------|
| 최소화(Minimise) | 처리되는 개인정보의 양 최소화 | 필드 제한, 선택적 수집 |
| 분리(Separate) | 개인정보를 기능 데이터와 분리 저장 | DB 분리, 암호화 저장 |
| 추상화(Abstract) | 식별 가능성을 낮추어 처리 | 집계, 통계화, 라운딩 |
| 숨기기(Hide) | 개인정보 관련 속성 은폐 | 가명처리, 암호화, 접근제어 |

#### 나. 프로세스 지향 전략

| 전략 | 설명 | 기술적 구현 |
|------|------|-------------|
| 알리기(Inform) | 처리 내용을 정보주체에게 공지 | 투명한 개인정보처리방침, 알림 |
| 통제(Control) | 정보주체에게 접근·수정·삭제 권한 부여 | 마이페이지, 동의 철회 API |
| 집행(Enforce) | 프라이버시 정책의 기술적 강제 | PET, 기술적 통제 |
| 시연(Demonstrate) | 프라이버시 준수 증명 | PIA, 감사, 컴플라이언스 보고 |

### 3. PbD 8대 전략과 개인정보보호법 제3조 비교

| PbD 전략 | 개인정보보호법 제3조 (개인정보 보호원칙) | 연관 조항 |
|----------|------------------------------------------|----------|
| 최소화(Minimise) | 목적에 필요한 최소한의 개인정보 수집 | 제3조 제1항 |
| 분리(Separate) | 개인정보를 안전하게 관리 | 제3조 제4항 |
| 추상화(Abstract) | 가명 또는 익명으로 처리 가능한 경우 익명 처리 | 제3조 제7항 |
| 숨기기(Hide) | 처리목적 달성 시 지체없이 파기 | 제3조 제5항 |
| 알리기(Inform) | 개인정보의 처리 목적 등을 공개·투명하게 처리 | 제3조 제1항, 제5항 |
| 통제(Control) | 정보주체의 권리 보장 | 제3조 제5항 |
| 집행(Enforce) | 개인정보의 처리 방법 및 종류 등에 따른 조치 | 제3조 제4항 |
| 시연(Demonstrate) | 개인정보 보호 책임과 의무 이행 | 제3조 전체 |

### 실무 제언

**개인정보 보호 중심 설계(PbD)의 DevPrivacy 적용**
- **챌린지**: 개발 완료 후 개인정보 보호 기능을 추가하면 설계 변경 비용이 크고, 설계적 취약점이 남는다.
- **제언**: 애자일 개발 방법론에 "Privacy Backlog"를 추가하여 각 스프린트에서 PbD 8대 전략 적용 여부를 체크리스트로 검증하고, PIA(개인정보 영향평가)를 지속적으로 수행하는 'DevPrivacy' 체계를 도입해야 한다.
