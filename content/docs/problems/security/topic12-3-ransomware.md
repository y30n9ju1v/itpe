## 해설

### 1. 랜섬웨어 동작 과정

```
① 감염(Infection)
   스피어 피싱, 악성 웹사이트, RDP 취약점 등으로 초기 침투

② 통신 수립(C&C)
   C&C 서버와 TOR/HTTPS 통신으로 암호화 키 수신

③ 파일 탐색(Discovery)
   문서, 이미지, DB 등 가치 있는 파일 탐색

④ 암호화(Encryption)
   AES로 파일 암호화, AES 키를 RSA 공개키로 암호화

⑤ 몸값 요구(Ransom Demand)
   비트코인 등 암호화폐 주소와 복호화 키 교환 조건 제시

⑥ 키 파기(Key Destruction) — 미지불 시
   일정 시간 내 미지불 시 복호화 키 파기
```

### 2. 랜섬웨어 유형

| 유형 | 설명 | 대표 사례 |
|------|------|-----------|
| **암호화형** | 파일을 암호화 후 복호화 키 대가로 금전 요구 | WannaCry, LockBit, REvil |
| **화면 잠금형** | 운영체제 화면만 잠금, 파일 암호화 없음 | Reveton, WinLock |
| **이중 공갈형** | 암호화 + 데이터 유출 협박 | Maze, DarkSide, Conti |
| **삼중 공갈형** | 이중 공갈 + DDoS 공격까지 추가 | Avaddon |
| **서비스형(RaaS)** | 랜섬웨어를 서비스로 제공 (공격자가 임대) | LockBit, BlackCat(ALPHV) |
| **워이퍼(Wiper)** | 복구 불가 형태로 데이터 삭제(금전 목적 아님) | NotPetya, Shamoon |

### 3. 랜섬웨어 대응방안

#### 가. 예방 (Prevention)

| 방안 | 내용 |
|------|------|
| 패치 관리 | SMB, RDP 등 취약점 즉시 패치(EternalBlue 등) |
| 이메일 보안 | 첨부파일 샌드박스 분석, 매크로 비활성화 |
| MFA 적용 | RDP, VPN 등 원격 접속에 다중 인증 |
| 최소 권한 | 파일 서버 접근 권한 최소화, 네트워크 세분화 |

#### 나. 탐지 (Detection)

| 방안 | 내용 |
|------|------|
| EDR | 파일 대량 암호화 행위 탐지·차단 |
| FIM(파일 무결성 모니터링) | 파일 변경 이상 탐지 |
| 허니팟 파일 | 미끼 파일에 대한 접근 즉시 탐지 |

#### 다. 복구 (Recovery)

| 방안 | 내용 |
|------|------|
| **3-2-1 백업** | 3개 복사본, 2가지 매체, 1개 오프사이트 | 
| 오프라인 백업 | 네트워크 격리된 백업(에어갭) |
| 백업 무결성 검증 | 정기적 복구 훈련(RTO/RPO 검증) |
| 시스템 이미지 | OS 이미지 백업으로 빠른 복원 |


- 최근 랜섬웨어(Ransomware) 관련 보안 위협 증가에 대응하여, 서비스 안전성 확보를 위한 체계적인 통제 및 기술 적용이 증대되는 추세임.  "끝"

### 실무 제언

**랜섬웨어 내성 백업 설계**
- **챌린지**: 네트워크 연결된 백업 서버도 랜섬웨어에 감염되어 백업이 무력화되는 사례가 빈번하다.
- **제언**: 불변 스토리지(AWS S3 Object Lock, Azure Immutable Blob)를 활용한 WORM(Write Once Read Many) 백업을 구성하고, 백업 서버의 네트워크 접근을 단방향(밀어넣기 방식)으로 제한하여 랜섬웨어가 백업을 암호화할 수 없도록 설계해야 한다.
