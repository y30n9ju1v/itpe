## 해설

### 출제 배경 및 의도

양자컴퓨팅의 발전으로 RSA·ECC 등 기존 공개키 암호가 위협받는 상황에서, 양자 컴퓨터에도 안전한 수학적 난제 기반의 PQC 개념·종류·NIST 표준화 현황을 서술할 수 있어야 한다.

### 1. PQC 개요

- **정의**: 양자 컴퓨터의 공격에도 안전한 암호 알고리즘 — 고전 컴퓨터와 양자 컴퓨터 모두에 저항
- **필요성**: Shor's Algorithm으로 RSA·ECC·DH 등 기존 공개키 암호가 다항 시간 내 해독 가능
  - Grover's Algorithm으로 대칭키·해시 보안 강도 절반 감소(128bit → 64bit 등가)
- **NIST PQC 표준화**: 2016년 공모 → 2022년 후보 확정 → **2024년 최종 표준 공표**

### 2. 주요 PQC 기반 수학적 난제

| 기반 | 난제 | 대표 알고리즘 |
|------|------|---------------|
| **격자(Lattice)** | SVP(최단벡터문제), LWE, RLWE | ML-KEM(CRYSTALS-Kyber), ML-DSA(Dilithium) |
| **해시(Hash)** | 해시함수 역산 불가 | SPHINCS+(SLH-DSA) |
| **코드(Code)** | 임의 선형부호 복호화 문제 | BIKE, HQC |
| **다변수 다항식** | MQ(다변수 이차방정식) | Rainbow(취약점 발견, 제외) |
| **등장타원곡선** | SIDH | SIKE(2022 취약점 발견, 제외) |

### 3. NIST PQC 최종 표준 (2024)

| 표준명 | 원래 이름 | 기능 | 기반 |
|--------|-----------|------|------|
| **ML-KEM** (FIPS 203) | CRYSTALS-Kyber | 키 캡슐화(KEM) | 격자(Module-LWE) |
| **ML-DSA** (FIPS 204) | CRYSTALS-Dilithium | 전자서명 | 격자(Module-LWE) |
| **SLH-DSA** (FIPS 205) | SPHINCS+ | 전자서명 | 해시기반 |
| **FN-DSA** (FIPS 206) | FALCON | 전자서명 | 격자(NTRU) |

### 4. PQC 마이그레이션 전략

```
단기(~2026): 크립토 민첩성(Crypto-Agility) 확보
  → 알고리즘 교체가 용이한 아키텍처 설계

중기(~2030): 하이브리드 방식 적용
  → 기존 알고리즘 + PQC 동시 사용
  → TLS: X25519+ML-KEM 하이브리드 키 교환

장기(2030~): PQC 단독 적용
  → 국가·산업별 마이그레이션 일정 준수
```

### 실무 제언

**TLS 1.3 PQC 하이브리드 적용**
- **챌린지**: 현재 TLS 1.3에서 사용 중인 X25519(ECDH)는 양자컴퓨팅에 취약하다.
- **제언**: IETF에서 표준화 중인 X25519+ML-KEM 하이브리드 키 교환을 웹서버·로드밸런서에 적용하고, 하위 호환성을 유지하면서 점진적으로 ML-KEM 단독 방식으로 전환하는 마이그레이션 로드맵을 수립해야 한다.
