## 문제

개인정보보호위원회는 '23년 10 월부터 시범운영하던 사전적 정성 검토제를 '24년 3월 고시 제정하여 본격 운영으로 접어 들었다. 다음에 대해 설명하시오.
가. 사전적정성 검토 개요
나. 사전적정성 검토 절차
다. 사전적정성 검토제 기대효과

## 해설

### 1. 사전적정성 검토 개요

- **정의**: 개인정보처리자가 새로운 서비스나 기술을 도입하기 전에 개인정보 처리의 적법성·안전성을 사전에 검토하여 개인정보보호위원회로부터 의견을 받는 제도
- **법적 근거**: 개인정보보호법 제8조의 2(2023년 개정)
- **시행**: 2023년 10월 시범 운영 → 2024년 3월 고시 제정 → 본격 시행
- **유사 제도**: GDPR의 DPIA(Data Protection Impact Assessment)와 유사

### 2. 사전적정성 검토 절차

```
① 검토 신청
   개인정보처리자 → 개인정보보호위원회 제출
   (서비스 개요, 개인정보 처리 계획, 자체 검토 결과 포함)

② 접수 및 검토 개시
   위원회 접수 → 검토 대상 적합성 확인

③ 검토 수행
   법률·기술 전문가 자문, 정보주체 영향 분석

④ 의견 통보
   적정/부적정/조건부 적정 의견 통보(60일 이내)

⑤ 이행 확인(선택)
   조건부 적정 시 개선 사항 이행 여부 확인
```

### 3. 검토 주요 항목

| 항목 | 검토 내용 |
|------|-----------|
| 수집·이용의 적법성 | 동의, 법적 근거, 목적 명확성 |
| 개인정보 최소화 | 수집 항목 최소화 여부 |
| 안전성 확보 조치 | 암호화, 접근통제, 보안 취약점 |
| 정보주체 권리 | 열람·정정·삭제 요구권 보장 |
| 제3자 제공·위탁 | 적법성, 계약 조건 |

### 4. 사전적정성 검토제 기대효과

| 효과 | 내용 |
|------|------|
| **법적 리스크 감소** | 서비스 출시 전 위법 요소를 사전에 제거 |
| **설계 단계 개선** | PbD(Privacy by Design) 원칙 조기 적용 |
| **신뢰도 향상** | 개인정보보호위원회 검토를 통한 사용자 신뢰 확보 |
| **글로벌 경쟁력** | GDPR DPIA와 유사한 프로세스로 해외 서비스 진출 용이 |
| **사고 예방** | 사후 제재보다 사전 예방으로 비용 절감 |

### 실무 제언

**신규 AI 서비스 사전적정성 검토 활용**
- **챌린지**: AI 기반 신규 서비스(안면인식, 행동 분석 등)는 개인정보 처리 범위가 넓어 출시 후 규제 리스크가 크다.
- **제언**: 서비스 기획 단계에서 사전적정성 검토를 신청하여 위원회 의견을 설계에 반영하고, 이를 투명성 보고서에 공개하여 정보주체의 신뢰를 확보하는 것이 장기적으로 유리하다.
