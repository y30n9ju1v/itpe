## 해설

### 1. 사이버 킬 체인 개요

정 의  • Lockheed Martin이 군사 킬 체인 개념을 사이버 공격에 적용하여, APT 공격을 7단계 연쇄 과정으로 모델화한 프레임워크
       - 공격자가 목표를 달성하기 전에 어느 단계에서든 공격 체인을 끊어 피해를 방지

### 2. 사이버 킬 체인 7단계

```
① 정찰 (Reconnaissance)
   ↓ 공개 정보 수집(OSINT), 소셜 엔지니어링, 취약점 스캔

② 무기화 (Weaponization)
   ↓ 악성코드 + 취약점 익스플로잇 결합 (악성 PDF, DOCX, 페이로드)

③ 전달 (Delivery)
   ↓ 이메일 첨부, 피싱 URL, USB, 웹 드라이브-바이 다운로드

④ 익스플로잇 (Exploitation)
   ↓ 취약점 실행, 악성코드 실행 트리거

⑤ 설치 (Installation)
   ↓ 백도어·RAT·웹쉘 설치, 지속성 확보(레지스트리, 자동시작)

⑥ C2 통신 (Command & Control)
   ↓ 공격자 서버와 통신 채널 개설 (HTTPS, DNS Tunneling)

⑦ 목표 달성 (Actions on Objectives)
   데이터 유출, 랜섬웨어 실행, 시스템 파괴
```

### 3. 단계별 탐지·대응 방안

| 단계 | 탐지 | 대응 |
|------|------|------|
| ① 정찰 | 웹 접근 로그, 포트 스캔 탐지 | IPS, 허니팟 배치 |
| ② 무기화 | 위협 인텔리전스(TIP) IOC 공유 | 취약점 신속 패치 |
| ③ 전달 | 이메일 보안(샌드박스), URL 필터링 | SEG, 웹 프록시 |
| ④ 익스플로잇 | EDR 행위 기반 탐지 | EMET, 취약점 패치 |
| ⑤ 설치 | 무결성 모니터링(FIM), AV/EDR | AppLocker, 화이트리스팅 |
| ⑥ C2 통신 | DNS 이상 탐지, 네트워크 행위 분석 | DNS 싱크홀, C2 차단 |
| ⑦ 목표 달성 | DLP, 데이터 접근 감사 | 백업, 세그멘테이션 |

### 4. MITRE ATT&CK 연계

| 항목 | 킬 체인 | MITRE ATT&CK |
|------|---------|---------------|
| **단계 수** | 7단계 | 14개 전술(Tactics) |
| **세부 기법** | 개략적 | 300개 이상 세부 기법(Techniques) |
| **활용** | 공격 흐름 이해 | 구체적 TTP 매핑·대응 |
| **연계** | 각 킬 체인 단계 ↔ ATT&CK 전술 매핑 | 정밀한 탐지 룰 작성 가능 |

- **킬 체인 + ATT&CK 활용**: 킬 체인으로 공격 단계를 파악하고, ATT&CK의 세부 기법(TTP)을 참조하여 SIEM 탐지 룰을 정밀하게 설계.  "끝"

### 실무 제언

**위협 인텔리전스 연계 킬 체인 대응**
- **챌린지**: 킬 체인 초기 단계(정찰·무기화)는 공격자 인프라에서 이루어져 방어자가 탐지하기 거의 불가능하다.
- **제언**: ISAC(정보공유분석센터), MISP(Malware Information Sharing Platform) 등 위협 인텔리전스 공유 체계를 통해 최신 IOC(침해 지표)를 자동 수집·적용하고, SOAR 플랫폼에서 킬 체인 단계별 Playbook을 실행하여 ③ 전달 단계 이후 공격 체인을 조기 차단하는 전략을 수립해야 한다.
