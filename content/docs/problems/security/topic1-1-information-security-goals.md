## 문제

정보보안의 주요한 세 가지 목적 및 각각의 목적을 위협할 수 있는 공격방법에 대하여 설명하시오.

## 해설

### 출제 배경 및 의도

정보보안의 가장 기본을 묻는 문제로, CIA 트라이어드(기밀성·무결성·가용성)와 각 목적을 위협하는 공격 유형, 그리고 대응방안까지 종합적으로 서술할 수 있는지를 검증한다. 목적별 공격 사례를 풍부하게 작성하고, 관리적·기술적 대응 방안 및 거버넌스 관점의 4단락 구성으로 답안을 완성해야 고득점을 얻을 수 있다.

### 1. 정보보안 세 가지 목적 (CIA 트라이어드)

```
              기밀성(Confidentiality)
                    △
                   / \
                  /   \
                 /     \
    무결성 ——— ◎ ——— 가용성
  (Integrity)   정보   (Availability)
               시스템
```

| 목적 | 정의 | 핵심 개념 |
|------|------|-----------|
| 기밀성(Confidentiality) | 허가받지 않은 사용자에게 정보가 노출되지 않도록 보호 | 암호화, 접근제어, 인증 |
| 무결성(Integrity) | 허가받지 않은 사용자에 의해 정보가 변경되지 않도록 보호 | 해시함수, 디지털서명, MDC |
| 가용성(Availability) | 허가받은 사용자의 정상적인 접근을 보장 | 이중화, BCP, DRS, Anti-DDoS |

- 정보보안이란 인가되지 않은 접근·사용·폭로·붕괴·수정·파괴로부터 정보와 정보시스템을 보호해 기밀성·무결성·가용성을 제공하는 것 (4가지 목적: 책임추적성 포함)

### 2. 세 가지 목적을 위협하는 공격 방법

#### 가. 기밀성을 위협하는 공격

| 공격 방법 | 설명 | 주요 공격 기법 |
|-----------|------|----------------|
| 스누핑(Snooping) | 네트워크 상에 떠도는 중요 정보를 몰래 획득 | 무선랜 해킹, IGMP 스누핑, DHCP 스누핑 |
| 스니핑(Sniffing) | 상대방의 패킷·정보 등을 훔쳐보거나 다른 형태의 정보를 획득 | Switch Jamming, ARP Redirect, ICMP Redirect, 트래픽 분석 |
| 스푸핑(Spoofing) | TCP/IP 구조적 결함을 이용해 임의의 웹사이트로 방문을 유도하여 정보 획득(중간자 공격) | ARP 스푸핑, IP 스푸핑, DNS 스푸핑 |
| 가로채기(Interception) | 접근 권한이 없는 사용자가 정보에 대한 권한을 얻는 행위(정보 탈취) | SQL Injection, Cross Site Scripting |

#### 나. 무결성을 위협하는 공격

| 공격 방법 | 설명 | 주요 공격 기법 |
|-----------|------|----------------|
| 변경(Modification) | 정보를 차단하거나 접근한 후 자신에게 이득이 되는 정보를 변경 | 악성코드, 바이러스, 웜, 트로이목마, 랜섬웨어 |
| 위조(Fabrication) | 주로 발신 근원지를 변경하는 공격 기법 | 파밍 |
| 가장(Masquerading) | 사용자의 PIN을 훔쳐 해당 사용자인 척 하거나, 은행사이트를 가장하는 등의 공격 기법 | 피싱, 스피어 피싱 |
| 재연(Replaying) | 사용자가 보내는 메시지의 사본을 얻어 나중에 다시 전송하는 기법 | 재생공격 |
| 부인(Repudiation) | 송신자나 수신자가 자신의 행동을 부인하면서 발생하는 기법 | Man in the middle 공격(MITM) |

#### 다. 가용성을 위협하는 공격

| 공격 방법 | 설명 | 주요 공격 기법 |
|-----------|------|----------------|
| 서비스거부(DoS) | 서버의 리소스를 독점하여 시스템의 일부가 파괴되거나 사용할 수 없도록 하는 공격 | Ping of Death, Land Attack, Slowloris, Smurf 공격 |
| 분산서비스거부(DDoS) | 여러 대의 공격자를 분산적으로 배치해 동시다발적으로 트래픽을 급증시켜 가용성을 훼손 | Anti-Spoofing, SYN Flooding, UDP Flooding, ICMP Flooding |

### 3. 공격에 대한 대응방안

#### 가. 기밀성 공격 대응방안

| 대응방안 | 설명 | 주요 적용 기술 |
|----------|------|----------------|
| 사용자 인증 | 시스템에 접근하려는 사용자에 대한 인증 | 생체인식, 토큰, 2팩터 인증 |
| 암호화 | 제3자가 정보의 내용을 알 수 없도록 메시지를 변환하여 보내는 과정 | 전치, 치환, 대칭키/공개키 암호, SSH, VPN, PGP, S/MIME |
| 접근제어 | 인가된 사용자만 시스템에 접근할 수 있도록 설정하는 정책 | SAC(서버), NAC(NW), MAC, DAC, RBAC, 방화벽, 망분리 |

#### 나. 무결성 공격 대응방안

| 대응방안 | 설명 | 주요 적용 기술 |
|----------|------|----------------|
| 메시지 무결성 검증 | 원본 메시지 무결성을 보장하는 기법 | 해쉬 함수, MDC |
| 메시지 인증 | 원본 메시지 무결성을 보장하고, 메시지 출원자를 인증하는 기법 | 해쉬 함수, MAC, HMAC |
| 디지털 서명 | 네트워크에서 송신자의 신원을 증명하는 방법 | 부인방지(Non-Repudiation), 공인인증서, PKI |
| 암호 알고리즘 | 인가되지 않은 사용자가 임의의 정보를 변경·삭제·생성하는 것을 방지 | SHA-2, MD5, HMAC, NMAC, 3DES, AES, SEED |

#### 다. 가용성 공격 대응방안

| 대응방안 | 설명 | 주요 적용 기술 |
|----------|------|----------------|
| Anti-DDoS | DDoS 공격을 방어하는 Appliance 인라인 장비 | DDoS 공격 방어 |
| Fault Tolerance | 시스템 구성 부품의 결함/고장이 발생해도 정상 기능 수행 | RAID, Replication, Redundancy, Diversity |
| High Availability | 2대 이상의 시스템을 연결, 한 시스템 장애 시 서비스 제어권을 이관 | Fail-Over, Active-Active/Standby, Clustering, Heartbeat |
| 백업 | 정보 영속성을 위해 데이터를 안전하게 보관 | 전체 백업, 증분 백업, 차등 백업 |
| DRS | 해킹·재해 대비 사후 조치로 시스템을 복구 가능한 상태로 구축 | Mirror, Hot, Warm, Cold 지표(RPO, RTO, RSP, RCO) |
| BCP | 각종 재해·장애·재난으로부터 위기관리를 기반으로 재해복구·업무복구·재개·비상계획 등 비즈니스 연속성 보장 | BIA, MTPD, RTO |

### 4. 정보보안 세 가지 목적 달성을 위한 기업의 정보보안 전략

- IT 거버넌스·정보보호 거버넌스 구축을 기업의 전략으로 수립
- 보안 조직 구성, 컴플라이언스 준수, 통합 보안 관리, 24×365 보안 모니터링 체계
- ISMS-P 획득 등을 마련하여 기업의 기밀성·가용성·무결성 관점의 공격에 체계적으로 대응

### 실무 제언

**융합 보안 거버넌스 적용**
- **챌린지**: CIA 각 목적별 대응이 분절적으로 운영되어 통합 관제가 어렵다.
- **제언**: IT 거버넌스와 정보보호 거버넌스를 연계하여 Defense-in-depth 전략을 수립하고, 공통 IT 자산에 대한 통합 보안 정책을 적용해야 한다.

**Zero Trust 아키텍처 도입**
- **챌린지**: 내부 사용자 인증 후 무제한 접근을 허용하는 경계 기반 보안은 내부자 공격에 취약하다.
- **제언**: "절대 신뢰하지 말고, 항상 검증하라(Never Trust, Always Verify)"는 Zero Trust 원칙을 적용하여 모든 접근 요청에 대해 기밀성·무결성·가용성을 동시에 검증해야 한다.
