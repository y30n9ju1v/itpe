## 해설

### 출제 배경 및 의도

클라우드 서비스, 원격 근무, OTT 스트리밍, 화상회의의 보편화로 네트워크 트래픽은 폭증하고 있으며, 서비스 유형별로 품질 요구사항이 상이하다. 모든 트래픽을 동등하게 처리하는 Best-Effort 인터넷만으로는 음성·영상 서비스의 품질을 보장할 수 없다.

QoS(Quality of Service)는 이러한 다양한 트래픽 요구를 충족하기 위해 네트워크 자원의 우선 배분과 제어를 수행하는 기술 체계다. 기술·모델·측정요소를 가·나·다로 구분하여 종합적 이해를 측정하는 문제로, 각 항목을 체계적 표와 구성도로 답안화하는 것이 고득점 전략이다.

### 1. QoS(Quality of Service), 개요

정 의  • 네트워크에서 특정 트래픽에 대해 지연·지터·대역폭·패킷 손실을 보장하기 위해 자원을 우선 배분하고 제어하는 기술 체계
       - 분류(Classification)→마킹(Marking)→큐잉(Queuing)→혼잡회피→쉐이핑 계층 적용

```
[트래픽 유입]
     |
  [분류·마킹]-- VoIP(EF/46), 영상(AF41/34), 데이터(BE/0)
     |
  [큐잉 스케줄링]-- LLQ(VoIP) / CBWFQ(영상·데이터)
     |
  [혼잡 회피]-- WRED(클래스별 선제적 드롭)
     |
  [쉐이핑·폴리싱]-- 토큰버킷/리키버킷
     |
  [링크 출력]
```

- QoS는 트래픽 입력부터 출력까지 계층적 파이프라인으로 구성되며, 각 단계가 협력하여 서비스별 품질을 보장한다.

### 2. QoS 기술·모델·측정요소

1) 가. QoS 기술

| 구분 | 기술 | 설명 |
|------|------|------|
| ① | 분류·마킹 | DPI·ACL로 트래픽 식별, DSCP 헤더에 우선순위 마킹 |
| ② | 큐잉 | LLQ+CBWFQ로 실시간/보장/일반 트래픽 분리 처리 |
| ③ | 혼잡 회피 | WRED: 클래스별 임계값으로 선제적 패킷 드롭, TCP 혼잡 제어 유도 |
| ④ | 쉐이핑·폴리싱 | 토큰 버킷(폴리싱), 리키 버킷(쉐이핑)으로 전송률 제어 |
| ⑤ | 링크 효율화 | CRTP: VoIP 헤더 40B→2~4B 압축, LFI: 큰 패킷 분할·인터리빙 |

- CRTP + LFI는 저속 WAN 회선에서 VoIP 품질을 효과적으로 개선한다.

2) 나. QoS 모델 비교

| 구분 | Best Effort | IntServ | DiffServ |
|------|-------------|---------|---------|
| QoS 보장 | 없음 | 엄격한 보장 | 상대적 우선순위 |
| 자원 예약 | 없음 | RSVP 흐름별 예약 | 없음(마킹만) |
| 확장성 | 높음 | 낮음(상태 유지) | 높음(Stateless) |
| 복잡도 | 낮음 | 높음 | 중간 |
| 표준 프로토콜 | - | RSVP | DSCP, MPLS |
| 사용 환경 | 일반 인터넷 | 기업 내부 실시간 | ISP·기업 WAN 표준 |

- DiffServ가 확장성과 단순성으로 현재 기업망·ISP의 실질적 표준이다.

3) 다. QoS 측정요소

| 구분 | 측정 요소 | 설명 | VoIP 허용 기준 |
|------|----------|------|--------------|
| 시간 | 지연(Latency) | 발신→수신 패킷 전달 시간 | 150ms 이하 (ITU-T G.114) |
| 변동 | 지터(Jitter) | 패킷 간 지연 편차 | 30ms 이하 |
| 손실 | 패킷 손실률 | 전체 패킷 대비 손실 비율 | 1% 이하 |
| 용량 | 대역폭 | 단위 시간당 전송 가능 데이터 | 서비스별 SLA 계약 |
| 실효 | 처리량(Throughput) | 실제 전달된 유효 데이터 | 대역폭의 70~80% |
| 가동 | 가용성(Availability) | 서비스 정상 운영 시간 비율 | 99.99% (4-nine) |

```
지연 구성 요소:
전체 지연 = 처리 지연(μs) + 큐잉 지연(가변, QoS로 제어)
          + 전송 지연(패킷크기/링크속도) + 전파 지연(물리 거리)
```

- 지터 버퍼(Jitter Buffer)를 수신단에 적용하면 지터를 흡수하여 연속 재생이 가능하나, 버퍼 크기와 지연 간 트레이드오프가 발생한다.  "끝"

### 실무 제언

**VoIP/UC 환경 QoS SLA 수립**
- **챌린지**: 지연·지터·손실 기준 없이 VoIP 도입 시 통화 품질 민원 반복 발생
- **제언**: ITU-T G.114 기준(지연<150ms, 지터<30ms, 손실<1%)을 SLA로 명문화, DSCP EF 마킹+LLQ 스케줄링 전 구간 적용, 주기적 IPSLA 측정으로 SLA 준수 모니터링

**SD-WAN 애플리케이션 인식 QoS**
- **챌린지**: 수백 지점 라우터에서 애플리케이션별 수동 QoS 정책 설정의 운영 부담
- **제언**: SD-WAN 컨트롤러에서 DPI 기반 Zoom·Teams·Salesforce 자동 인식 후 DSCP 마킹 자동 배포, 중앙 정책 관리로 일관성 확보

**5G 네트워크 슬라이싱 연계**
- **챌린지**: 엔터프라이즈 5G 도입 시 산업 자동화(URLLC)와 일반 데이터(eMBB)의 QoS 공존 요구
- **제언**: 5G 네트워크 슬라이싱으로 URLLC(초저지연) 전용 슬라이스 분리, DiffServ DSCP 마킹을 5G QoS 식별자(5QI)와 매핑하여 엔드투엔드 QoS 보장
