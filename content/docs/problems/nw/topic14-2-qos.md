## 해설

> **QoS는 "인터넷이라는 공공 도로에 VIP 차선을 만드는 것"이다 — 구급차(실시간 서비스)가 막히지 않도록 제도를 만드는 기술 체계**

### 1. 출제 배경 및 의도

클라우드, 원격 근무, 스트리밍 서비스 확산으로 네트워크 QoS의 중요성이 더욱 커졌습니다. QoS 기술, 모델, 측정요소를 종합적으로 이해하는 것이 목표입니다.

### 2. 개념의 본질과 작동 메커니즘

#### 가. QoS 기술

QoS를 구현하는 핵심 기술은 크게 4가지로 분류됩니다.

**1) 분류 및 마킹 (Classification & Marking)**
```
트래픽 유형 식별 → 우선순위 마킹
VoIP  → DSCP EF (46)  → 최우선
영상  → DSCP AF41(34) → 고우선
데이터 → DSCP BE (0)  → 기본
```

**2) 큐잉 (Queuing)**
```
우선순위 큐(PQ), 가중 공정 큐(WFQ), CBWFQ, LLQ
(상세 내용: topic14-1 스케줄링 참조)
```

**3) 혼잡 회피 (Congestion Avoidance)**
```
RED (Random Early Detection):
큐가 임계값 초과 → 랜덤으로 패킷 드롭 → TCP 혼잡 제어 유도
큐가 가득 찰 때까지 기다리지 않고 미리 패킷 드롭으로 조절

WRED (Weighted RED):
트래픽 클래스별 다른 드롭 임계값
VoIP: 드롭 없음, 데이터: 적극 드롭
```

**4) 트래픽 쉐이핑/폴리싱 (Traffic Shaping/Policing)**
```
쉐이핑: 트래픽을 버퍼링하여 속도 조절 (버스트 평탄화)
폴리싱: 초과 트래픽 즉시 드롭 (SLA 강제)
→ 리키/토큰 버킷으로 구현
```

**5) 링크 효율화 (Link Efficiency)**
```
헤더 압축 (CRTP): VoIP 헤더 40바이트 → 2~4바이트
링크 단편화/인터리빙 (LFI): 큰 패킷이 작은 패킷을 막지 않도록 분할
```

#### 나. QoS 모델

**1) Best Effort (최선형 서비스)**
```
QoS 보장 없음, 현재 인터넷의 기본 동작
모든 패킷 동등 처리
→ 빠를 수도, 느릴 수도 있음
```

**2) IntServ (통합 서비스)**
```
흐름별 QoS 보장 (RSVP 자원 예약)
엄격한 QoS 보장 가능
확장성 낮음 (대규모 인터넷 불가)
```

**3) DiffServ (차등 서비스)**
```
트래픽 클래스별 처리 (DSCP 마킹)
상대적 우선순위 제공
확장성 높음 → 기업망, ISP에서 표준
```

**세 모델 비교:**

| 항목 | Best Effort | IntServ | DiffServ |
|---|---|---|---|
| **보장** | 없음 | 엄격한 보장 | 상대적 우선순위 |
| **확장성** | 높음 | 낮음 | 높음 |
| **구현 복잡도** | 낮음 | 높음 | 중간 |
| **사용 환경** | 일반 인터넷 | 기업 내부 멀티미디어 | ISP, 기업 WAN |

#### 다. QoS 측정 요소

**핵심 QoS 파라미터:**

| 측정 요소 | 설명 | 허용 기준 (VoIP 예) |
|---|---|---|
| **지연 (Delay/Latency)** | 패킷 전송 시간 (발신→수신) | 150ms 이하 |
| **지터 (Jitter)** | 패킷 간 지연 편차 | 30ms 이하 |
| **패킷 손실 (Packet Loss)** | 손실된 패킷 비율 | 1% 이하 |
| **대역폭 (Bandwidth)** | 단위 시간당 전송 가능한 데이터 양 | 서비스별 상이 |
| **처리량 (Throughput)** | 실제 전송된 데이터 양 (유효 대역폭) | 대역폭의 70~80% |
| **가용성 (Availability)** | 네트워크 서비스 가동 시간 비율 | 99.99% (4-nine) |

**지연 유형:**

```
전체 지연 = 처리 지연 + 큐잉 지연 + 전송 지연 + 전파 지연

처리 지연:  라우터 패킷 처리 시간 (μs 수준)
큐잉 지연:  대기 큐에서 대기 시간 (가변, QoS로 제어)
전송 지연:  패킷을 링크에 싣는 시간 = 패킷크기/링크속도
전파 지연:  신호의 물리적 전파 시간 (빛의 속도 기반)
```

**지터 측정 및 완화:**
```
지터 버퍼: 수신측에서 패킷을 일정 시간 모아 재생
           지터를 흡수하여 연속 재생 보장
           단, 버퍼 크기 ↑ = 지연 ↑ (트레이드오프)
```

### 3. 핵심 키워드 (Must-Have)

- **DSCP:** IP 헤더 DiffServ 마킹 필드 (6비트)
- **EF/AF/BE:** Expedited/Assured/Best-Effort Forwarding
- **지연/지터/손실:** QoS 3대 핵심 지표
- **RED/WRED:** 혼잡 회피를 위한 능동적 큐 관리
- **SLA (Service Level Agreement):** QoS 목표값을 계약으로 명시

### 4. 맥락과 비교

클라우드 시대에는 SD-WAN의 **애플리케이션 인식 QoS**가 발전하고 있습니다. 패킷을 분석하여 Zoom, Teams 등을 자동 인식하고 적절한 QoS 정책을 동적으로 적용합니다. 5G 네트워크 슬라이싱(Network Slicing)은 QoS를 네트워크 인프라 수준에서 보장하는 혁신적 기술입니다.

### 5. 실무 제언

VoIP 서비스 구축 시 QoS 요구사항을 SLA로 명문화하고 (지연<150ms, 지터<30ms, 손실<1%), 이를 달성하기 위한 기술 스택(DSCP 마킹 → LLQ 스케줄링 → WRED 혼잡 회피)을 계층별로 적용합니다. 기술사 답안에서 QoS 기술→모델→측정요소를 각각 표와 다이어그램으로 체계화하고, 실제 VoIP/스트리밍 서비스에서의 적용 사례로 연결하면 실무 역량을 효과적으로 어필할 수 있습니다.
