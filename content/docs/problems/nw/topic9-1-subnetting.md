## 해설

> **서브네팅은 큰 땅(IP 블록)을 필요한 크기의 필지로 나누는 것이고, VLSM은 필지마다 다른 크기를 허용하는 것이다**

### 1. 출제 배경 및 의도

IP 주소 설계는 네트워크 엔지니어의 핵심 실무 능력입니다. VLSM을 통해 IP 낭비를 최소화하는 효율적 서브넷 설계 능력을 평가합니다. 계산 문제이므로 단계별 정확한 계산이 중요합니다.

### 2. 개념의 본질과 작동 메커니즘

#### 가. 수퍼네팅과 서브네팅

**서브네팅 (Subnetting):**
- 하나의 큰 네트워크를 여러 작은 서브넷으로 분할
- 서브넷 마스크의 비트를 늘려 네트워크 영역 확장
- 목적: 브로드캐스트 도메인 축소, IP 낭비 방지

```
192.168.1.0/24 → 서브네팅 →
  192.168.1.0/25  (128개 주소)
  192.168.1.128/25 (128개 주소)
마스크 비트 증가: /24 → /25
```

**수퍼네팅 (Supernetting / CIDR):**
- 여러 개의 연속된 작은 네트워크를 하나의 큰 네트워크로 합산
- 서브넷 마스크의 비트를 줄여 네트워크 영역 축소
- 목적: 라우팅 테이블 축소, 라우팅 효율 향상

```
192.168.0.0/24
192.168.1.0/24  → 수퍼네팅 → 192.168.0.0/23
192.168.2.0/24
192.168.3.0/24  → 수퍼네팅 → 192.168.2.0/23
마스크 비트 감소: /24 → /23
```

| 항목 | 서브네팅 | 수퍼네팅 |
|---|---|---|
| **방향** | 하나 → 여러 개 분할 | 여러 개 → 하나로 합산 |
| **마스크** | 비트 증가 (/24 → /25) | 비트 감소 (/24 → /23) |
| **목적** | IP 효율, 브로드캐스트 분리 | 라우팅 테이블 축소 |
| **기법** | VLSM | CIDR |

#### 나. VLSM 설계: 192.168.0.0/22

**전체 주소 공간:**
- 192.168.0.0/22 = 192.168.0.0 ~ 192.168.3.255 (1024개 주소)

**필요 호스트 수 (큰 순서로 정렬 - VLSM 기본 원칙):**
1. 개발팀: 500명 → 최소 502개 주소 필요 (네트워크 + 브로드캐스트 +2)
2. 영업팀: 250명 → 최소 252개 주소 필요
3. 인사팀: 125명 → 최소 127개 주소 필요
4. 보안팀: 125명 → 최소 127개 주소 필요

**서브넷 크기 계산:**
- 2^n ≥ 호스트수 + 2 를 만족하는 최소 n

**① 개발팀 (500명 + 2 = 502개 필요):**
- 2^9 = 512 ≥ 502 → /23 (512개 주소)
- 서브넷: **192.168.0.0/23**
- 서브넷 마스크: 255.255.254.0
- 네트워크: 192.168.0.0
- 브로드캐스트: 192.168.1.255
- 호스트 범위: **192.168.0.1 ~ 192.168.1.254** (510개)

**② 영업팀 (250명 + 2 = 252개 필요):**
- 2^8 = 256 ≥ 252 → /24 (256개 주소)
- 서브넷: **192.168.2.0/24**
- 서브넷 마스크: 255.255.255.0
- 네트워크: 192.168.2.0
- 브로드캐스트: 192.168.2.255
- 호스트 범위: **192.168.2.1 ~ 192.168.2.254** (254개)

**③ 인사팀 (125명 + 2 = 127개 필요):**
- 2^7 = 128 ≥ 127 → /25 (128개 주소)
- 서브넷: **192.168.3.0/25**
- 서브넷 마스크: 255.255.255.128
- 네트워크: 192.168.3.0
- 브로드캐스트: 192.168.3.127
- 호스트 범위: **192.168.3.1 ~ 192.168.3.126** (126개)

**④ 보안팀 (125명 + 2 = 127개 필요):**
- 2^7 = 128 → /25 (128개 주소)
- 서브넷: **192.168.3.128/25**
- 서브넷 마스크: 255.255.255.128
- 네트워크: 192.168.3.128
- 브로드캐스트: 192.168.3.255
- 호스트 범위: **192.168.3.129 ~ 192.168.3.254** (126개)

**최종 요약표:**

| 팀 | 필요 인원 | 서브넷 | 서브넷 마스크 | 호스트 범위 | 사용 가능 수 |
|---|---|---|---|---|---|
| 개발팀 | 500명 | 192.168.0.0/23 | 255.255.254.0 | .0.1 ~ .1.254 | 510 |
| 영업팀 | 250명 | 192.168.2.0/24 | 255.255.255.0 | .2.1 ~ .2.254 | 254 |
| 인사팀 | 125명 | 192.168.3.0/25 | 255.255.255.128 | .3.1 ~ .3.126 | 126 |
| 보안팀 | 125명 | 192.168.3.128/25 | 255.255.255.128 | .3.129 ~ .3.254 | 126 |

### 3. 핵심 키워드 (Must-Have)

- **VLSM:** Variable Length Subnet Mask, 서브넷마다 다른 마스크 길이
- **CIDR:** Classless Inter-Domain Routing, 수퍼네팅 기반
- **서브넷 공식:** 호스트수 = 2^n - 2 (네트워크/브로드캐스트 제외)
- **VLSM 원칙:** 큰 서브넷부터 할당하여 주소 공간 낭비 최소화

### 4. 맥락과 비교

VLSM은 EIGRP, OSPF 같은 Classless 라우팅 프로토콜이 지원합니다. 반면 RIPv1, IGRP는 Classful 라우팅으로 VLSM을 지원하지 않습니다.

### 5. 실무 제언

실무에서 IP 주소 계획 시 **미래 확장성**을 위해 서브넷 크기에 여유를 두고 설계합니다. 개발팀 500명이 향후 1000명으로 늘 수 있다면 /22(1024개)를 처음부터 할당하는 것이 나중에 IP 재설계 비용을 줄입니다. 기술사 답안에서 VLSM 계산 결과를 표로 정리하고, 할당 순서(큰 것부터)와 주소 공간 연속성을 설명하면 체계적 이해를 보여줄 수 있습니다.
