## 문제

패킷 데이터의 송수신 과정의 전진오류수정 중 전송 데이터가 1011010, 디바이더(divider)가 1101인 경우 CRC(Cyclic Redundancy Check) 값을 구하는 과정을 설명하시오.

## 해설

> **CRC는 나눗셈의 나머지를 오류 체크에 이용하는 방식이다 — 마치 영수증 끝의 체크 숫자처럼**

CRC는 데이터 전송 오류를 검출하기 위해 송신 전 계산한 "나머지 값"을 데이터에 덧붙여 전송하는 방식입니다.

### 1. 출제 배경 및 의도

CRC는 이더넷 프레임의 FCS(Frame Check Sequence), USB, 하드디스크 등 거의 모든 데이터 전송 프로토콜에 사용됩니다. 계산 과정을 직접 수행하는 능력이 요구되며, 이진 나눗셈(XOR 연산)의 원리를 이해해야 합니다.

### 2. 개념의 본질과 작동 메커니즘

#### CRC 개요

- **원리:** 데이터를 생성 다항식(Generator/Divider)으로 모듈로-2 나눗셈을 수행, 나머지(CRC 값)를 데이터에 첨부
- **모듈로-2 나눗셈:** 일반 나눗셈과 동일하지만 뺄셈 대신 **XOR** 사용 (올림/빌림 없음)

```
XOR 규칙:
0 XOR 0 = 0
0 XOR 1 = 1
1 XOR 0 = 1
1 XOR 1 = 0  ← 중요: 빌림 없음
```

#### 단계별 CRC 계산 (전송 데이터: 1011010, 디바이더: 1101)

**Step 1: 데이터 뒤에 0 추가**
- 디바이더 길이 = 4비트 → (4-1) = 3개의 0 추가
- 피제수: `1011010 000` (10비트)

**Step 2: 모듈로-2 나눗셈 수행**

```
     1011010000
÷         1101
─────────────────
1011010000 ÷ 1101

1011         ← 첫 4비트
XOR 1101
─────
 110 0       ← 나머지 앞에 다음 비트 내림
 
 1100
XOR 1101
─────
  001 1      ← 나머지 앞에 다음 비트 내림

  0011
 (0011 < 1101이므로 몫은 0, 다음 비트 내림)
  00110
XOR 1101 × 0 → 그대로
  (앞의 0 처리: 0으로 나누므로 XOR 안 함)

실제 계산 상세:
단계1: 1011 XOR 1101 = 0110  → 내림: 01100
단계2: 1100 XOR 1101 = 0001  → 내림: 00011
단계3: 0011 (< 1101)         → 0으로 나눔: 내림: 00111
단계4: 0111 (< 1101)         → 내림: 01110 (빌 비트 없음이면 끝)
단계5: 1110 XOR 1101 = 0011  → 내림: 0011 (마지막)
단계6: 0011 (< 1101)         → 나머지 = 011
```

정리된 계산:
```
1011010000
÷ 1101
──────────────────────────────
  1011 XOR 1101 = 0110
   1100 XOR 1101 = 0001
    00110 → 0011 XOR 0 → 0011 (자리 이동)
     01110 → 0111 XOR 0 → 0111
      11100 → 1110 XOR 1101 = 0011
       0011  → 나머지 (CRC = 011)
```

**CRC 값 = 011**

**Step 3: 전송 데이터**
```
원본: 1011010
CRC:       011
전송: 1011010 011
```

**Step 4: 수신측 검증**
```
수신된 데이터: 1011010011
÷ 1101
나머지 = 000 → 오류 없음
나머지 ≠ 000 → 오류 발생!
```

#### CRC 동작 흐름

```
[송신]                        [수신]
데이터                        데이터 + CRC
  │                               │
  │ + (n-1)개 0 추가              │ ÷ 생성 다항식
  │ ÷ 생성 다항식                 │
  │                               ▼
  ▼                          나머지 = 0? → 정상
CRC 값                       나머지 ≠ 0? → 오류!
  │
  ▼
데이터 + CRC 전송
```

### 3. 핵심 키워드 (Must-Have)

- **생성 다항식 (Generator Polynomial):** 나눗셈의 제수, 표준 CRC-32, CRC-16 등
- **모듈로-2 나눗셈:** XOR 기반 이진 나눗셈
- **FCS (Frame Check Sequence):** 이더넷 프레임의 CRC 필드
- **CRC-32:** 이더넷에서 사용 (생성다항식 32비트)

### 4. 맥락과 비교

| 오류 검출 방식 | 원리 | 검출 능력 | 오버헤드 |
|---|---|---|---|
| **패리티 비트** | 1의 개수 홀짝 확인 | 단일 비트 오류만 | 1비트 |
| **체크섬** | 합산 후 보수 | 다중 비트 일부 | 가변 |
| **CRC** | 다항식 나눗셈 | 버스트 오류 우수 | 16~32비트 |
| **해밍코드** | 패리티 비트 다중 배치 | 1비트 수정 가능 | 추가 비트 |

### 5. 실무 제언

CRC-32는 이더넷 프레임, ZIP, PNG 파일 등에 사용됩니다. 하드웨어 수준에서는 LFSR(Linear Feedback Shift Register)로 고속 계산이 구현됩니다. 기술사 답안 작성 시 계산 과정을 단계별로 명확히 제시하고, "송신측 첨부 → 수신측 검증 → 나머지 0이면 정상"의 동작 흐름도를 포함하면 높은 점수를 받을 수 있습니다.
