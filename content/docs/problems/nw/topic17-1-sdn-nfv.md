## 문제

차세대 네트워크 기술로 부각되고 있는 SDN(Software Defined Network)과 NFV(Network Function Virtualization)의 구조와 특징을 비교 설명하시오.

## 해설

> **SDN은 "네트워크의 두뇌(제어)와 손발(데이터 전달)을 분리"하는 것이고, NFV는 "전용 하드웨어 대신 범용 서버에서 네트워크 기능을 소프트웨어로 실행"하는 것이다**

### 1. 출제 배경 및 의도

클라우드 데이터센터와 5G 코어 네트워크는 SDN과 NFV 위에 세워집니다. 전통적 하드웨어 기반 네트워크의 한계를 소프트웨어로 극복하는 두 기술의 구조와 차이점을 이해하는 것이 핵심입니다.

### 2. 개념의 본질과 작동 메커니즘

#### SDN (Software Defined Network)

**핵심 개념: 제어 평면과 데이터 평면 분리**

```
전통적 네트워크:
라우터 = [제어 평면(라우팅 결정)] + [데이터 평면(패킷 전달)]
→ 각 장비가 독자적 결정 → 중앙 통제 어려움

SDN:
┌─────────────────────────────────┐
│        SDN 컨트롤러             │ ← 제어 평면 (중앙집중)
│    (전체 네트워크 뷰 + 정책)    │
└──────────┬──────────────────────┘
           │ Southbound API (OpenFlow)
    ┌──────┴───────────────┐
    │  스위치1  스위치2   │ ← 데이터 평면 (단순 전달)
    │  스위치3  스위치4   │   (Flow Table에 따라 전달)
    └─────────────────────┘
```

**SDN 3계층 구조:**

```
┌──────────────────────────────────────────┐
│  응용 계층 (Application Layer)           │
│  트래픽 관리 앱, 보안 앱, 로드밸런서 앱  │
└──────────────┬───────────────────────────┘
               │ Northbound API (REST API)
┌──────────────┴───────────────────────────┐
│  제어 계층 (Control Layer)               │
│  SDN 컨트롤러 (OpenDaylight, ONOS)       │
│  - 전체 토폴로지 뷰                      │
│  - 경로 계산 및 정책 결정                │
└──────────────┬───────────────────────────┘
               │ Southbound API (OpenFlow, NETCONF)
┌──────────────┴───────────────────────────┐
│  데이터 계층 (Data/Infrastructure Layer) │
│  OpenFlow 스위치, 라우터                 │
│  - Flow Table에 따라 패킷 전달           │
└──────────────────────────────────────────┘
```

**OpenFlow 동작:**
```
패킷 도착 → 스위치 Flow Table 조회
매칭 규칙 있음 → 해당 액션 실행 (전달, 드롭, 수정)
매칭 없음 → 컨트롤러에 문의 → 규칙 다운로드 → 실행
```

#### NFV (Network Function Virtualization)

**핵심 개념: 전용 하드웨어 → 범용 서버의 소프트웨어**

```
전통 네트워크:
[전용 방화벽 장비] [전용 로드밸런서] [전용 IDS] [전용 DPI]
→ 고비용, 벤더 종속, 확장 어려움

NFV:
┌───────────────────────────────────────────┐
│         범용 x86 서버 (COTS)              │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐        │
│  │vFW  │ │vLB  │ │vIDS │ │vDPI │ ← VNF  │
│  └─────┘ └─────┘ └─────┘ └─────┘        │
│         하이퍼바이저 (KVM, VMware)         │
└───────────────────────────────────────────┘
→ 소프트웨어로 네트워크 기능 구현
→ 필요시 인스턴스 추가로 확장
```

**ETSI NFV 참조 아키텍처:**

```
┌─────────────────────────────────────────────┐
│        OSS/BSS (운영/비즈니스 지원 시스템)   │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────┴──────────────────────────┐
│              NFV 오케스트레이터              │
│        (VNF 생명주기 관리, 자원 할당)        │
└──────────┬───────────────────┬──────────────┘
           │                   │
┌──────────┴──────┐   ┌────────┴──────────────┐
│   VNF Manager   │   │  가상화 인프라 관리자   │
│   (VNF 관리)    │   │  (NFVI 자원 관리)      │
└─────────────────┘   └──────────┬─────────────┘
                                  │
┌─────────────────────────────────┴────────────┐
│      NFVI (NFV Infrastructure)               │
│   컴퓨팅 | 스토리지 | 네트워크 자원          │
│   하이퍼바이저, 물리 서버, 스위치             │
└──────────────────────────────────────────────┘
```

#### SDN vs NFV 비교

| 비교 항목 | SDN | NFV |
|---|---|---|
| **핵심 아이디어** | 제어-데이터 분리 | 하드웨어 → 소프트웨어 |
| **대상** | 네트워크 제어 방식 | 네트워크 기능 구현 방식 |
| **목적** | 네트워크 프로그래밍 가능성 | 비용 절감, 유연성 |
| **표준** | OpenFlow, ONF | ETSI NFV |
| **컨트롤러** | SDN 컨트롤러 (필수) | 오케스트레이터 |
| **연관성** | 상호 보완 (함께 사용) | 상호 보완 |
| **주요 활용** | 데이터센터, WAN | 통신사 코어, VNF |
| **대표 사례** | Google B4, OpenDaylight | 5G NFc, AT&T Domain 2.0 |

**상호 보완 관계:**
```
SDN + NFV 결합:
SDN 컨트롤러 → NFV로 구현된 가상 스위치/라우터 제어
→ 완전한 소프트웨어 정의 네트워크 실현

예: OpenStack + OpenDaylight + KVM 기반 가상 네트워크
```

### 3. 핵심 키워드 (Must-Have)

- **제어 평면 / 데이터 평면 분리:** SDN의 핵심 원리
- **OpenFlow:** SDN 사우스바운드 API 표준
- **VNF (Virtual Network Function):** 소프트웨어로 구현된 네트워크 기능
- **NFVI (NFV Infrastructure):** VNF 실행 하드웨어/하이퍼바이저
- **오케스트레이터:** VNF 생명주기 자동 관리

### 4. 맥락과 비교

5G 코어 네트워크는 **SBA(Service-Based Architecture)**로 설계되어 AMF, SMF, UPF 등 모든 코어 기능이 VNF(또는 CNF: Cloud-native NF)로 구현됩니다. SDN/NFV + 클라우드 네이티브의 결합이 현대 통신 인프라의 방향입니다.

### 5. 실무 제언

대기업 데이터센터에서 VMware NSX(SDN), Cisco ACI(SDN), OpenStack(NFV)이 광범위하게 사용됩니다. 국내 통신사들도 5G 코어를 NFV 기반으로 구축하여 새로운 서비스 신속 배포와 자원 효율화를 달성하고 있습니다. 기술사 답안에서 SDN 3계층 구조 다이어그램과 NFV ETSI 참조 아키텍처, 두 기술의 명확한 비교표를 제시하면 구조적 이해를 효과적으로 보여줄 수 있습니다.
