## 해설

> **NAT는 한 집에 여러 가족이 살지만 대외적으로 집 번호(공인 IP)는 하나인 구조다**

IPv4 주소 고갈 문제를 해결하는 가장 현실적이고 널리 쓰이는 방법이 NAT입니다. 공유기를 사용하는 가정이나 기업이 모두 NAT를 통해 인터넷에 접속합니다.

### 1. 출제 배경 및 의도

IPv4 주소 공간(약 43억 개)은 인터넷 폭발적 성장으로 이미 고갈되었습니다. IPv6 전환이 더디게 진행되는 현실에서 NAT는 사설 IP 주소를 공인 IP로 변환하여 주소 부족을 해소하는 핵심 기술입니다. 보안 측면(내부망 은폐)과 IP 주소 절약이라는 두 가지 실무적 가치를 함께 묻습니다.

### 2. 개념의 본질과 작동 메커니즘

#### NAT 동작 원리

```
[내부 사설망]                    [공인 인터넷]
PC1: 192.168.1.10 ─┐
PC2: 192.168.1.11 ─┤  NAT 라우터  ─── 목적지 서버
PC3: 192.168.1.12 ─┘ [공인IP: 1.2.3.4]   8.8.8.8

NAT 변환 테이블:
사설IP:포트          공인IP:포트
192.168.1.10:1025 → 1.2.3.4:5001
192.168.1.11:1026 → 1.2.3.4:5002
192.168.1.12:1027 → 1.2.3.4:5003
```

패킷이 내부에서 외부로 나갈 때 소스 IP를 공인 IP로 교체하고, 응답 패킷이 돌아오면 다시 사설 IP로 역변환합니다.

#### NAT의 핵심 기능 분류

| 기능 | 설명 | 종류 |
|---|---|---|
| **주소 변환** | 사설↔공인 IP 변환 | Static NAT, Dynamic NAT |
| **포트 다중화** | 하나의 공인 IP로 다수 호스트 지원 | PAT (NAPT) |
| **보안** | 내부 IP 주소 은폐 | 방화벽 보완 역할 |
| **로드밸런싱** | 여러 서버로 요청 분산 | DNAT 기반 |

#### NAT 유형 상세

**1) Static NAT (정적 NAT)**
- 사설 IP ↔ 공인 IP 1:1 고정 매핑
- 내부 서버를 외부에 공개할 때 사용 (웹서버, 메일서버)

**2) Dynamic NAT (동적 NAT)**
- 공인 IP 풀(Pool)에서 동적으로 할당
- 공인 IP 개수만큼만 동시 접속 가능

**3) PAT / NAPT (Port Address Translation)**
- 하나의 공인 IP + 포트 번호 조합으로 다수 사설 IP 구분
- 가정용 공유기의 핵심 기술
- 최대 65,535개의 포트 × 공인 IP 수만큼 동시 세션 지원

#### NAT 순방향/역방향 변환 흐름

```
[출발] 192.168.1.10:1025 → 8.8.8.8:80
       ↓ NAT 라우터 (소스 변환)
[전송] 1.2.3.4:5001 → 8.8.8.8:80

[응답] 8.8.8.8:80 → 1.2.3.4:5001
       ↓ NAT 라우터 (목적지 역변환)
[도착] 8.8.8.8:80 → 192.168.1.10:1025
```

### 3. 핵심 키워드 (Must-Have)

- **Static NAT:** 1:1 고정 변환, 서버 공개
- **Dynamic NAT:** IP 풀 기반 동적 할당
- **PAT/NAPT:** 포트 기반 다중화, 공유기 핵심 기술
- **NAT 테이블:** 사설IP:포트 ↔ 공인IP:포트 매핑 유지
- **Hairpin NAT (NAT Loopback):** 내부에서 공인 IP로 자사 서버 접근

### 4. 맥락과 비교

| 항목 | NAT | IPv6 전환 |
|---|---|---|
| **주소 부족 해결** | 즉시 적용 가능 | 장기 근본 해결책 |
| **복잡성** | 중간 (NAT 테이블 관리) | 높음 (전체 인프라 교체) |
| **종단간 연결** | 제한됨 (P2P 어려움) | 완전 지원 |
| **보안** | 내부 IP 은폐 | IPSec 기본 내장 |

**NAT의 한계:** P2P 통신, VoIP, IPSec 터널링과 충돌 가능. 이를 해결하기 위해 **STUN, TURN, ICE** 등의 NAT 트래버설 기법이 등장했습니다.

### 5. 실무 제언

기업 네트워크 설계 시 NAT 정책을 명확히 문서화해야 합니다. 특히 **DNAT(Destination NAT)**를 활용한 서버 공개 시 방화벽 ACL과 함께 구성하여 허용된 포트만 외부에 노출되도록 합니다. 클라우드 환경(AWS, Azure)에서도 동일한 개념이 적용되며, AWS의 NAT Gateway가 프라이빗 서브넷 인스턴스의 인터넷 접근을 제공하는 방식이 대표적입니다. 기술사 답안에서는 "IPv4 주소 절약 + 보안 강화 + 클라우드 적용"의 세 축으로 서술하면 고득점이 가능합니다.
