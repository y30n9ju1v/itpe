## 문제

ISO 7498이 필요한 이유와 각 계층별 역할 및 기능, 취급하는 자료 형태와 관련 장비에 대하여 설명하시오.

## 출제 정보

| 항목 | 내용 |
|------|------|
| 토픽(대) | OSI 7 Layer |
| 토픽(소) | OSI 7 Layer |
| 출제 | 120회 |
| 유형 | 응용 |
| 번호 | 4 |

## 해설

> **OSI 7계층은 네트워크 통신을 "레고 블록"처럼 표준화한 설계도다**

서로 다른 제조사의 장비, 다른 OS의 컴퓨터가 통신할 수 있는 이유는 모두가 동의한 공통 규칙(표준)을 따르기 때문입니다. ISO가 1984년에 제정한 OSI 참조 모델(ISO 7498)이 그 기준입니다.

### 1. 출제 배경 및 의도

네트워크 기술의 모든 프로토콜과 장비는 OSI 모델의 어느 계층에 속하는지로 분류됩니다. 이 모델을 이해하지 못하면 TCP/IP, 라우팅, 스위칭, 보안 등 후속 주제를 이해하기 어렵습니다. 출제 의도는 단순 암기가 아닌 "왜 계층이 필요한가"와 각 계층 간 상호작용을 논리적으로 서술하는 능력 평가입니다.

### 2. 개념의 본질과 작동 메커니즘

#### OSI 7계층이 필요한 이유

- **이기종 시스템 호환:** IBM, HP, Cisco 등 다양한 벤더 장비 간 통신 표준
- **복잡성 분리:** 각 계층이 독립적으로 개발/교체 가능 (모듈화)
- **표준화:** 프로토콜 개발사가 공통 인터페이스를 따름

#### 7계층 구조 전체도

```
┌──────────────────────────────────────────────────┐
│ 7. 응용 계층 (Application)    - 데이터 (Data)     │
│    HTTP, FTP, SMTP, DNS                          │
├──────────────────────────────────────────────────┤
│ 6. 표현 계층 (Presentation)   - 데이터 (Data)     │
│    암호화, 압축, 인코딩 (SSL/TLS, JPEG)           │
├──────────────────────────────────────────────────┤
│ 5. 세션 계층 (Session)        - 데이터 (Data)     │
│    세션 설정/유지/종료 (NetBIOS, RPC)             │
├──────────────────────────────────────────────────┤
│ 4. 전송 계층 (Transport)      - 세그먼트          │
│    TCP, UDP (포트번호, 오류제어, 흐름제어)         │
├──────────────────────────────────────────────────┤
│ 3. 네트워크 계층 (Network)    - 패킷              │
│    IP, ICMP, 라우터 (IP주소, 라우팅)              │
├──────────────────────────────────────────────────┤
│ 2. 데이터링크 계층 (Data Link) - 프레임           │
│    Ethernet, MAC, 스위치 (MAC주소, 오류검출)      │
├──────────────────────────────────────────────────┤
│ 1. 물리 계층 (Physical)       - 비트              │
│    전기신호, 광신호, 허브, 케이블                 │
└──────────────────────────────────────────────────┘
```

#### 계층별 상세 정리

| 계층 | 이름 | PDU | 주요 프로토콜 | 관련 장비 | 주요 역할 |
|---|---|---|---|---|---|
| 7 | 응용 | Data | HTTP, FTP, SMTP, DNS | - | 사용자 인터페이스 제공 |
| 6 | 표현 | Data | SSL/TLS, MIME, JPEG | - | 데이터 형식 변환, 암호화 |
| 5 | 세션 | Data | NetBIOS, RPC, SSH | - | 연결 세션 관리 |
| 4 | 전송 | Segment | TCP, UDP, SCTP | L4 스위치 | 종단간 신뢰성, 포트 다중화 |
| 3 | 네트워크 | Packet | IP, ICMP, OSPF | 라우터 | 논리 주소(IP), 라우팅 |
| 2 | 데이터링크 | Frame | Ethernet, PPP, ARP | 스위치, 브리지 | 물리 주소(MAC), 오류검출 |
| 1 | 물리 | Bit | RS-232, IEEE 802.3 | 허브, 케이블 | 비트를 전기/광 신호로 전송 |

#### 데이터 캡슐화/역캡슐화 (Encapsulation)

```
송신측:                              수신측:
[7] 데이터                           데이터 [7]
[6] 데이터                           데이터 [6]
[5] 데이터                           데이터 [5]
[4] TCP헤더 | 데이터    (세그먼트)   TCP헤더 | 데이터 [4]
[3] IP헤더  | 세그먼트  (패킷)       IP헤더  | 세그먼트 [3]
[2] MAC헤더 | 패킷 | FCS (프레임)   MAC헤더 | 패킷 | FCS [2]
[1] 010110101010...    (비트)        010110101010... [1]
```

각 계층은 자신의 헤더를 추가(캡슐화)하며, 수신 측은 역순으로 헤더를 제거(역캡슐화)합니다.

### 3. 핵심 키워드 (Must-Have)

- **PDU(Protocol Data Unit):** 비트→프레임→패킷→세그먼트→데이터
- **캡슐화/역캡슐화:** 각 계층 헤더 추가/제거
- **L2 스위치:** MAC 주소 기반, L3 라우터: IP 주소 기반
- **계층 독립성:** 하위 계층 교체해도 상위 계층 영향 없음

### 4. 맥락과 비교

**OSI 모델 vs TCP/IP 모델**

| OSI 7계층 | TCP/IP 4계층 |
|---|---|
| 7. 응용 / 6. 표현 / 5. 세션 | 응용 계층 |
| 4. 전송 | 전송 계층 |
| 3. 네트워크 | 인터넷 계층 |
| 2. 데이터링크 / 1. 물리 | 네트워크 접근 계층 |

OSI는 이론적 참조 모델이고, 실제 인터넷은 TCP/IP 4계층을 따릅니다. 그러나 문제 분석과 장비 분류에 OSI 모델이 표준 도구로 사용됩니다.

### 5. 실무 제언

네트워크 장애 분석 시 OSI 7계층을 체계적으로 활용합니다. "1계층(케이블 연결?) → 2계층(MAC 테이블?) → 3계층(IP 라우팅?) → 4계층(방화벽 포트?) → 7계층(애플리케이션 설정?)" 순서로 확인하면 체계적 트러블슈팅이 가능합니다. 기술사 답안에서는 각 계층의 역할을 명확히 정의하고, 실제 프로토콜과 장비 예시로 연결하며, 캡슐화 메커니즘을 ASCII 다이어그램으로 표현하면 높은 점수를 받을 수 있습니다.
