## 문제

ISO 7498이 필요한 이유와 각 계층별 역할 및 기능, 취급하는 자료 형태와 관련 장비에 대하여 설명하시오.

## 해설

### 출제 배경 및 의도

서로 다른 제조사의 장비와 다른 운영체제의 컴퓨터가 통신할 수 있는 이유는 모두가 동의한 공통 규칙(표준)을 따르기 때문이다. 1970년대까지 IBM SNA, DEC DECnet 등 각 벤더의 독자적 프로토콜이 혼재하며 이기종 시스템 간 통신이 불가능했다. 이를 해결하기 위해 ISO가 1984년 OSI 참조 모델(ISO 7498)을 제정했다.

OSI 모델은 이후 개발되는 모든 네트워크 프로토콜과 장비 분류의 표준 척도가 되었다. TCP/IP, 이더넷, TLS, HTTP 등 현재 사용되는 기술은 모두 OSI 모델의 특정 계층에 매핑하여 설명된다. 네트워크 장애 분석, 보안 설계, 클라우드 아키텍처 설계 모두 OSI 모델을 기준으로 이루어진다.

이 문제는 단순한 계층 암기가 아니라 "왜 계층이 필요한가"와 각 계층 간 상호작용, 캡슐화 메커니즘을 논리적으로 서술하는 능력을 평가한다.

### 1. OSI 7 Layer(참조 모델), 개요

정 의  • 이기종 시스템 간 통신 표준화를 위해 ISO가 제정한 7개 계층 구조의 네트워크 참조 모델
       - 각 계층은 독립적으로 구현·교체 가능하며, 인접 계층에만 SAP(Service Access Point)를 통해 서비스를 제공

```
┌──────────────────────────────────────────────────────┐
│ 7. 응용 계층 (Application)    PDU: Data              │
│    HTTP, FTP, SMTP, DNS, SNMP                        │
├──────────────────────────────────────────────────────┤
│ 6. 표현 계층 (Presentation)   PDU: Data              │
│    암호화·압축·인코딩 (SSL/TLS, JPEG, ASCII)          │
├──────────────────────────────────────────────────────┤
│ 5. 세션 계층 (Session)        PDU: Data              │
│    세션 설정·유지·종료 (NetBIOS, RPC, SSH)            │
├──────────────────────────────────────────────────────┤
│ 4. 전송 계층 (Transport)      PDU: Segment           │
│    TCP, UDP (포트번호·오류제어·흐름제어·다중화)        │
├──────────────────────────────────────────────────────┤
│ 3. 네트워크 계층 (Network)    PDU: Packet            │
│    IP, ICMP, OSPF  관련장비: 라우터                  │
├──────────────────────────────────────────────────────┤
│ 2. 데이터링크 계층 (Data Link) PDU: Frame            │
│    Ethernet, PPP, ARP  관련장비: L2 스위치·브리지    │
├──────────────────────────────────────────────────────┤
│ 1. 물리 계층 (Physical)       PDU: Bit               │
│    RS-232, IEEE 802.3  관련장비: 허브·케이블·NIC      │
└──────────────────────────────────────────────────────┘
```

- 계층 독립성으로 인해 하위 계층(예: 이더넷 → Wi-Fi 교체)이 변경되어도 상위 계층 프로토콜(TCP/IP)은 영향을 받지 않는다.

### 2. 계층별 역할 및 기능 상세

1) 하위 계층 (1~3계층): 데이터 전달 담당

| 계층 | 이름 | PDU | 주요 프로토콜 | 관련 장비 | 주요 역할 |
|------|------|-----|--------------|----------|----------|
| 1 | 물리 | Bit | RS-232, IEEE 802.3 | 허브, 케이블 | 비트를 전기·광·무선 신호로 전송 |
| 2 | 데이터링크 | Frame | Ethernet, PPP, ARP | L2 스위치, 브리지 | MAC 주소 기반 전달, 오류 검출(CRC) |
| 3 | 네트워크 | Packet | IP, ICMP, OSPF | 라우터, L3 스위치 | 논리 주소(IP) 기반 경로 선택(라우팅) |

- 1~3계층은 패킷을 목적지까지 물리적으로 전달하는 역할을 담당하며, 네트워크 장비(허브·스위치·라우터)가 동작하는 영역이다.

2) 상위 계층 (4~7계층): 애플리케이션 서비스 담당

| 계층 | 이름 | PDU | 주요 프로토콜 | 관련 장비 | 주요 역할 |
|------|------|-----|--------------|----------|----------|
| 4 | 전송 | Segment | TCP, UDP, SCTP | L4 스위치 | 종단 간 신뢰성, 포트 기반 다중화 |
| 5 | 세션 | Data | NetBIOS, RPC, SSH | - | 연결 세션 설정·유지·종료 |
| 6 | 표현 | Data | SSL/TLS, MIME, JPEG | - | 데이터 형식 변환·암호화·압축 |
| 7 | 응용 | Data | HTTP, FTP, SMTP, DNS | - | 사용자 인터페이스 및 애플리케이션 서비스 |

- 4~7계층은 OS와 애플리케이션이 담당하며, 소켓 프로그래밍 API가 4계층과 상위 계층 사이의 인터페이스를 구현한다.

### 3. 캡슐화/역캡슐화 및 TCP/IP 비교

```
[송신측 캡슐화]                      [수신측 역캡슐화]
[7] 데이터                            데이터 [7]
[6] 데이터                            데이터 [6]
[5] 데이터                            데이터 [5]
[4] TCP헤더 | 데이터    (세그먼트)    TCP헤더 | 데이터 [4]
[3] IP헤더  | 세그먼트  (패킷)        IP헤더  | 세그먼트 [3]
[2] MAC헤더 | 패킷 | FCS (프레임)    MAC헤더 | 패킷 | FCS [2]
[1] 0101101010... (비트)              0101101010... [1]
```

| 구분 | OSI 7계층 | TCP/IP 4계층 |
|------|----------|-------------|
| 응용 | 7. 응용 / 6. 표현 / 5. 세션 | 응용 계층 |
| 전송 | 4. 전송 | 전송 계층 |
| 네트워크 | 3. 네트워크 | 인터넷 계층 |
| 하위 | 2. 데이터링크 / 1. 물리 | 네트워크 접근 계층 |

- OSI 모델은 이론적 참조 모델이고, 실제 인터넷은 TCP/IP 4계층을 따르지만 네트워크 분석과 장비 분류에는 OSI 모델이 표준 도구로 활용된다.  "끝"

### 실무 제언

**네트워크 장애 계층별 트러블슈팅**
- **챌린지**: 네트워크 장애 발생 시 원인 계층을 특정하지 못해 복구 시간이 길어짐
- **제언**: 1계층(케이블·링크 상태) → 2계층(MAC 테이블·VLAN) → 3계층(IP·라우팅 테이블) → 4계층(방화벽 포트 허용) → 7계층(애플리케이션 설정) 순서로 체계적 점검

**보안 아키텍처 설계 시 계층 적용**
- **챌린지**: 보안 솔루션을 임의로 배치하여 중복·공백 구간이 발생
- **제언**: L2(포트 보안·802.1X), L3(ACL·방화벽), L4(IPS·WAF), L7(DLP·API 게이트웨이) 계층별 방어 심층화(Defense in Depth) 설계 적용

**클라우드 네트워크 서비스 계층 매핑**
- **챌린지**: AWS·GCP 서비스 선택 시 어느 계층에서 동작하는지 혼동하여 잘못된 서비스 선택
- **제언**: L4(NLB/TCP 로드밸런서) vs L7(ALB/HTTP 로드밸런서)의 계층 차이를 명확히 하고, 요구사항에 맞는 서비스를 선택하는 기준으로 OSI 계층 활용
