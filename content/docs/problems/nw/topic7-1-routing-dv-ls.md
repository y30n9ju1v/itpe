## 문제

네트워크 라우팅 기법 중 거리벡터라우팅(Distance Vector Routing)과 링크스테이트라우팅(Link State Routing) 기법을 설명하고, 두 기법의 차이점을 비교하시오.

## 해설

> **거리벡터는 "이웃에게 소문을 전달받는 방식", 링크스테이트는 "지도를 전체적으로 공유하는 방식"이다**

라우터들이 목적지까지의 최적 경로를 어떻게 알아내는가? 두 가지 철학이 있습니다.

### 1. 출제 배경 및 의도

RIP, OSPF, BGP 등 모든 라우팅 프로토콜의 기반이 되는 알고리즘입니다. 대규모 기업 네트워크 설계 시 어떤 라우팅 프로토콜을 선택할지에 직접 영향을 주는 핵심 개념입니다.

### 2. 개념의 본질과 작동 메커니즘

#### 거리벡터 라우팅 (Distance Vector Routing)

**비유:** 각 마을(라우터)이 이웃 마을에게만 "내가 아는 목적지와 거리"를 알려줍니다. 전체 지도를 알 필요 없이, 이웃의 정보를 믿고 경로를 결정합니다.

**알고리즘:** Bellman-Ford 알고리즘

```
라우터 A의 관점:
A → B: 1홉
A → C: A→B→C = 2홉 (B가 알려준 정보)
A → D: A→B→C→D = 3홉 (B→C가 알려준 정보)

라우터 테이블:
목적지 | 다음홉 | 거리
  B   |   B   |  1
  C   |   B   |  2  ← B를 통해 C까지 2홉
  D   |   B   |  3
```

**동작 과정:**
1. 초기: 직접 연결 이웃 정보만 보유
2. 주기적(RIP: 30초)으로 이웃과 라우팅 테이블 교환
3. Bellman-Ford로 최소 거리 갱신
4. 수렴(Convergence)될 때까지 반복

**문제점: 카운팅 투 인피니티 (Count to Infinity)**
```
링크 단절: A─B 연결 끊김
B: "A까지 ∞" 라고 인식
그러나 C: "나는 B를 통해 A에 2홉으로 갈 수 있어"
B: "그럼 C를 통해 3홉으로 A 갈 수 있네"
C: "그럼 B를 통해 4홉으로..." → 무한 루프!

해결: Split Horizon, Poison Reverse, 최대 홉 제한(RIP=15)
```

#### 링크스테이트 라우팅 (Link State Routing)

**비유:** 모든 라우터가 자신의 직접 연결 링크 상태(가중치, 속도)를 전체 네트워크에 플러딩(Flooding)합니다. 모든 라우터가 완전한 네트워크 지도를 갖고 각자 최단 경로를 계산합니다.

**알고리즘:** Dijkstra's Shortest Path Algorithm

```
각 라우터가 LSA(Link State Advertisement) 생성:
"나 A야. 내 이웃은 B(비용1), C(비용2)야"

모든 LSA를 모아 LSDB(Link State Database) 구성
→ 전체 토폴로지 맵 완성
→ 다익스트라 알고리즘으로 최단 경로 계산

결과: 각 라우터가 동일한 토폴로지 뷰, 일관된 라우팅
```

**대표 프로토콜:** OSPF (Open Shortest Path First)

#### 거리벡터 vs 링크스테이트 비교

| 비교 항목 | 거리벡터 | 링크스테이트 |
|---|---|---|
| **알고리즘** | Bellman-Ford | Dijkstra |
| **정보 공유** | 이웃에게만 라우팅 테이블 | 전체 네트워크에 링크 상태 |
| **네트워크 뷰** | 부분적 (이웃만) | 전체 토폴로지 |
| **수렴 속도** | 느림 | 빠름 |
| **메모리/CPU** | 낮음 | 높음 (LSDB 유지) |
| **루프 위험** | 높음 (카운팅 투 인피니티) | 낮음 |
| **확장성** | 소규모 적합 | 대규모 적합 |
| **대표 프로토콜** | RIP, EIGRP | OSPF, IS-IS |
| **업데이트 방식** | 주기적 (30초) | 변화 시에만 |

### 3. 핵심 키워드 (Must-Have)

- **Bellman-Ford vs Dijkstra:** 두 알고리즘의 특성 차이
- **LSA (Link State Advertisement):** OSPF의 링크 상태 정보 패킷
- **LSDB (Link State Database):** 전체 토폴로지 데이터베이스
- **카운팅 투 인피니티:** DV 라우팅의 대표 문제점
- **수렴 (Convergence):** 모든 라우터 라우팅 테이블이 일치하는 상태

### 4. 맥락과 비교

**프로토콜별 분류:**

| 분류 | 프로토콜 | 알고리즘 | 특징 |
|---|---|---|---|
| **IGP - DV** | RIPv1/v2 | Bellman-Ford | 최대 15홉, 소규모 |
| **IGP - LS** | OSPF, IS-IS | Dijkstra | 대규모 기업망 |
| **IGP - 하이브리드** | EIGRP (Cisco) | DUAL | DV+LS 혼합 |
| **EGP** | BGP | Path Vector | 인터넷 AS 간 라우팅 |

### 5. 실무 제언

대기업 WAN 환경에서는 OSPF를 Area 단위로 분할 설계하여 LSDB 크기를 제한합니다. 인터넷 서비스 사업자(ISP) 간에는 BGP(Border Gateway Protocol)를 사용하며, 이는 경로 벡터(Path Vector) 방식으로 DV의 루프 문제를 AS_PATH 속성으로 해결합니다. 기술사 답안에서 두 알고리즘의 차이를 명확히 하고, OSPF와 RIP의 실제 특성으로 연결 서술하면 실무 이해도를 효과적으로 보여줄 수 있습니다.
