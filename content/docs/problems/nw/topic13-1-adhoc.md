## 해설

### 출제 배경 및 의도

전통적 무선 네트워크는 기지국(AP)이라는 고정 인프라에 의존하므로, 인프라가 파괴되거나 존재하지 않는 환경에서는 통신이 불가능하다. 재난 현장, 전장, 오지 탐험 등의 특수 환경에서 즉각적 통신망 구성 능력의 필요성이 Ad-hoc 네트워킹 연구를 촉발시켰다.

5G 시대의 D2D(Device-to-Device) 통신, 드론 군집(FANET), 자율주행 차량 네트워크(VANET), IoT 메시 네트워크 등 Ad-hoc 기술의 응용 분야가 급격히 확대되면서 기술사 시험의 핵심 토픽으로 부상하였다. 119회 시험에서 라우팅 프로토콜 분류체계와 대표 프로토콜의 동작 원리가 출제 핵심이다.

### 1. Ad-hoc 네트워킹, 개요

정 의  • 고정 인프라(기지국·AP) 없이 단말들이 자율적으로 라우터 역할을 수행하며 멀티홉 네트워크를 구성하는 분산형 무선 네트워크
       - MANET(Mobile Ad-hoc Network)이 대표 형태, IEEE 802.11 기반

```
[Infrastructure 모드]        [Ad-hoc / MANET 모드]
단말A                         단말A
  |                             |    (1홉)
  +--[AP]--인터넷--[AP]--단말B  단말B---(2홉)---단말C
           (고정 인프라 필수)             (단말이 라우터 역할)
                                          (인프라 불필요)
```

- 단말의 이동으로 토폴로지가 동적으로 변하므로, 변화에 적응하는 자율 라우팅 프로토콜이 핵심이다.

### 2. Ad-hoc 라우팅 프로토콜 분류 및 동작

1) 선제적(Proactive) 라우팅

| 구분 | 항목 | 설명 |
|------|------|------|
| 방식 | 테이블 유지 | 항상 최신 라우팅 테이블 유지, 경로 요청 시 즉시 응답 |
| 대표 | OLSR | MPR(Multi-Point Relay) 선택으로 플러딩 오버헤드 감소 |
| 대표 | DSDV | 시퀀스 번호 추가 거리 벡터, 루프 방지 |
| 장점 | 낮은 지연 | 경로 요청 즉시 응답 가능 |
| 단점 | 높은 오버헤드 | 토폴로지 변화마다 전체 업데이트 전파 |

```
OLSR MPR 동작:
A -- B -- C -- D
     |
     E

B가 MPR로 선출 --> B만 전체에 플러딩 전달
A, E는 수신만 --> 제어 트래픽 오버헤드 절감
```

2) 반응형(Reactive) 라우팅

| 구분 | 항목 | 설명 |
|------|------|------|
| 방식 | On-demand | 경로 필요 시에만 탐색, 평상시 테이블 미유지 |
| 대표 | AODV | RREQ/RREP/RERR 3종 메시지로 경로 탐색·유지·복구 |
| 대표 | DSR | 패킷 헤더에 전체 경로 포함(Source Routing) |
| 장점 | 낮은 오버헤드 | 이동이 잦은 대규모 환경에 적합 |
| 단점 | 초기 지연 | 첫 통신 시 경로 탐색(Route Discovery) 지연 발생 |

```
AODV Route Discovery:
① A --> RREQ(브로드캐스트) --> B, C, D ... --> 목적지 Z
② Z --> RREP(유니캐스트) --> ... --> A (최적 역방향 경로)
③ 링크 끊김 --> RERR 발생 --> 출발지에 알림 --> 재탐색
```

3) 라우팅 프로토콜 3대 분류 비교

| 항목 | Proactive | Reactive | Hybrid |
|------|-----------|----------|--------|
| 라우팅 테이블 | 항상 유지 | 요청 시 탐색 | 존 내 유지 / 존 간 탐색 |
| 통신 지연 | 낮음 | 초기 탐색 지연 | 중간 |
| 제어 오버헤드 | 높음 | 낮음 | 중간 |
| 적합 환경 | 소규모·안정 | 대규모·이동 빈번 | 중규모 |
| 대표 프로토콜 | OLSR, DSDV | AODV, DSR | ZRP |

- 환경 특성(노드 수·이동성·밀도)에 따른 최적 프로토콜 선택이 Ad-hoc 네트워크 설계의 핵심이다.

### 3. Ad-hoc 응용 분야 및 발전 트렌드

| 구분 | 응용 | 특징 |
|------|------|------|
| VANET | 차량 간 Ad-hoc (V2V/V2I) | 고속 이동, GPS 위치 활용 지리적 라우팅 |
| WSN | 무선 센서 네트워크 | 에너지 효율 최적화, 계층적 클러스터링 |
| FANET | 드론 군집 Ad-hoc | 3차원 이동, UAV 기반 Flying Relay |
| 군사통신 | 재난·전장 즉석 망 | 보안·생존성 최우선, 기지국 불필요 |
| 5G D2D | ProSe 기반 직접 통신 | 기지국 부하 감소, 근접 서비스 활성화 |

- Ad-hoc은 6G의 분산 인텔리전스 네트워크와 융합하여 인프라리스(Infrastructureless) 자율 통신망으로 진화하고 있다.  "끝"

### 실무 제언

**재난망 Ad-hoc 구축**
- **챌린지**: 지진·화재 등 재난 현장에서 인프라 파괴 시 긴급 통신망 즉시 구성 필요
- **제언**: AODV 기반 메시 네트워크로 소방·경찰 단말 간 직접 통신망 구축, 드론 Flying Relay로 커버리지 확장, 재난안전통신망(PS-LTE)과 Ad-hoc 하이브리드 운용

**VANET 보안 강화**
- **챌린지**: 중앙 인증 없는 Ad-hoc 환경에서 악의적 노드의 경로 조작(Black Hole, Wormhole 공격) 위험
- **제언**: PKI 기반 V2X 인증서(IEEE 1609.2), 신뢰도 기반 라우팅, 이상 행위 탐지 모듈 탑재

**에너지 효율 최적화**
- **챌린지**: 배터리 기반 IoT·WSN 노드의 제어 메시지 오버헤드로 인한 급격한 에너지 소모
- **제언**: OLSR MPR 선택으로 플러딩 오버헤드 최소화, 클러스터 기반 계층 라우팅으로 비클러스터헤드 노드의 전송 부하 분산
