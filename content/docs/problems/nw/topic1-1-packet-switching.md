## 문제

교환방식에 있어서 회선교환(Circuit Switching)방식과 패킷교환(Packet Switching) 방식에 대하여 설명하고, 음성데이터나 영상데이터와 같은 실시간 트래픽 서비스를 패킷교환방식의 네트워크에서 제공하고자 할 때 발생할 수 있는 문제점 및 해결방안에 대하여 설명하시오.

## 출제 정보

| 항목 | 내용 |
|------|------|
| 토픽(대) | 패킷 교환 |
| 토픽(소) | 회선교환, 패킷교환 |
| 출제 | 120회 |
| 유형 | 응용 |
| 번호 | 4 |

## 해설

> **전화망과 인터넷의 차이는 "길을 미리 예약하느냐, 짐을 쪼개서 보내느냐"의 차이다**

두 교환 방식은 현대 통신 인프라의 핵심 철학을 대비합니다. 유선 전화망(PSTN)은 회선교환, 인터넷은 패킷교환 위에 세워져 있습니다.

### 1. 출제 배경 및 의도

120회 응용 문제로, 네트워크 교환 방식의 근본 원리와 실시간 서비스(VoIP, 화상회의)를 IP망에서 제공할 때의 기술적 도전을 묻습니다. QoS, RSVP, DiffServ 등 심화 주제와 연결되며, 실무에서 IP 기반 멀티미디어 서비스 설계 능력을 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

#### 회선교환 (Circuit Switching)

마치 **고속도로 전용 차선**처럼, 통신 전 물리적 경로를 예약하고 통화 내내 독점합니다.

```
발신자 ─── 교환기A ─── 교환기B ─── 교환기C ─── 수신자
           [──────── 전용 회선 설정 (연결 지속) ────────]
  1단계: 회선 설정 (Call Setup)
  2단계: 데이터 전송 (독점 대역폭)
  3단계: 회선 해제 (Call Teardown)
```

#### 패킷교환 (Packet Switching)

마치 **택배 분할 발송**처럼, 데이터를 패킷으로 쪼개 각각 독립적으로 최적 경로로 전송합니다.

```
데이터 → [패킷1][패킷2][패킷3]으로 분할
패킷1: A → B → D (경로1)
패킷2: A → C → D (경로2)  ← 서로 다른 경로 가능
패킷3: A → B → D (경로1)
수신측: 순서 재조합 후 원본 복원
```

| 비교 항목 | 회선교환 | 패킷교환 |
|---|---|---|
| **자원 할당** | 연결 전 전용 회선 예약 | 필요 시 동적 할당 |
| **대역폭** | 고정 (사용 안 해도 점유) | 가변 (공유, 효율적) |
| **지연** | 일정 (낮은 지연) | 가변 (큐잉 지연 발생) |
| **신뢰성** | 회선 끊기면 전체 실패 | 경로 우회 가능 |
| **비용** | 고가 (전용 자원) | 저렴 (자원 공유) |
| **대표 사례** | PSTN(전화망), ISDN | 인터넷(IP), LTE |

#### 패킷교환에서 실시간 트래픽의 문제점

| 문제 | 설명 | 영향 |
|---|---|---|
| **지연(Delay)** | 라우터 큐잉, 전파 지연 누적 | VoIP 통화 끊김 (150ms 초과 시) |
| **지터(Jitter)** | 패킷마다 다른 지연 | 영상 재생 버벅임 |
| **패킷 손실** | 버퍼 오버플로우 시 드롭 | 음질 저하, 화면 깨짐 |
| **순서 역전** | 다른 경로 도착 순서 불일치 | 재조합 오류 |

#### 해결방안

```
┌─────────────────────────────────────────────┐
│              QoS 보장 기법                   │
├─────────────┬───────────────┬───────────────┤
│  IntServ    │   DiffServ    │    RSVP       │
│ (개별 흐름) │ (트래픽 분류) │ (자원 예약)   │
└─────────────┴───────────────┴───────────────┘
```

- **RSVP (Resource Reservation Protocol):** 경로상 라우터에 대역폭 예약 → 회선교환처럼 보장
- **DiffServ:** 패킷에 DSCP 마킹 → 실시간 트래픽 우선 처리
- **버퍼링/지터 버퍼:** 수신측에서 패킷을 일정 시간 모아 순서 정렬 후 재생
- **FEC(Forward Error Correction):** 패킷 손실 대비 여분 데이터 추가 전송

### 3. 핵심 키워드 (Must-Have)

- **회선교환:** 전용 회선, 3단계(설정-전송-해제), 일정 지연, PSTN
- **패킷교환:** 저장-전달(Store-and-Forward), 큐잉 지연, 가상회선/데이터그램
- **실시간 트래픽 문제:** 지연, 지터, 패킷 손실
- **QoS 해결:** RSVP, DiffServ, IntServ, 우선순위 큐잉

### 4. 맥락과 비교

**가상회선 방식 vs 데이터그램 방식 (패킷교환 내 구분)**

| 항목 | 가상회선 | 데이터그램 |
|---|---|---|
| 경로 설정 | 연결 전 논리 경로 설정 | 패킷마다 독립 라우팅 |
| 순서 보장 | 보장 | 미보장 |
| 오버헤드 | 설정/해제 오버헤드 있음 | 없음 |
| 대표 기술 | ATM, Frame Relay | IP |

### 5. 실무 제언

기업 환경에서 화상회의(Zoom, Teams) 도입 시 네트워크 담당자는 반드시 **QoS 정책**을 라우터/스위치에 설정해야 합니다. DSCP EF(Expedited Forwarding) 마킹으로 음성/영상 패킷을 최우선 큐에 배치하고, 대역폭의 20~30%를 실시간 트래픽 전용으로 예약합니다. 또한 WAN 구간에서는 SD-WAN의 **애플리케이션 인식 라우팅**을 활용해 실시간 트래픽을 지연이 낮은 경로로 자동 우회시키는 설계가 현대적 해결책입니다.
