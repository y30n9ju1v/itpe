## 문제

생성형 AI 모델의 성능과 신뢰성을 높이기 위해서는 고품질 데이터 구축이 필수수이다. 이에 생성형 AI 데이터 품질관리 가이드라인 v2.0 이 2025 년 2 월 발간되었다. 다음에 대해 서술 하시오.

## 해설

### 출제 배경 및 의도

"Garbage in, Garbage out" — 생성형 AI 모델의 성능과 안전성은 학습 데이터의 품질에 의해 결정됩니다. 편향된 데이터로 학습된 LLM은 차별적 콘텐츠를 생성하고, 불완전한 데이터는 환각(Hallucination)을 유발합니다.

ChatGPT 이후 생성형 AI가 전 산업으로 확산되면서 AI 학습 데이터 품질 관리가 국가 AI 경쟁력의 핵심 이슈로 부상했습니다. EU AI Act(2024), 국내 AI 기본법(2024) 등 글로벌 규제 강화와 함께 AI 데이터 품질 관리 체계 수립이 기업의 의무가 되고 있습니다.

2025년 2월 과기부·NIA가 발간한 생성형 AI 데이터 품질관리 가이드라인 v2.0이 이러한 문제에 대응하는 국내 최초의 체계적 지침으로, 채점관은 생성형 AI 특수 품질 문제(편향·환각·독성)와 프레임워크·6대 지표를 실무적으로 서술할 수 있는지 평가합니다.

### 1. 생성형 AI 데이터 품질관리, 개요

정 의  • 대규모 학습 데이터로부터 패턴을 학습하여 텍스트·이미지·코드 등 새로운 콘텐츠를
       - 자율 생성하는 AI 모델(LLM·Diffusion Model·GAN)의 데이터 품질을 관리하는 체계

```
[생성형 AI 데이터의 특수한 품질 이슈]

  일반 DB 데이터 품질 이슈      생성형 AI 추가 품질 이슈
  ─────────────────────       ─────────────────────────
  정확성                   +   편향(Bias): 성별·인종·이념 편향
  완전성                   +   독성(Toxicity): 혐오·폭력·성적 콘텐츠
  유효성                   +   환각(Hallucination): 사실과 다른 생성
  일관성                   +   저작권 침해: 학습 데이터 출처 문제
  유일성                   +   개인정보: PII 포함 학습 데이터
  접근성                   +   최신성: 학습 데이터 컷오프 이슈
```

- 생성형 AI 데이터는 수십억~수조 토큰 규모로 전수 검증이 불가하며, 편향·독성·환각 등 AI 특유의 품질 위험이 추가됨

### 2. 핵심 내용 — 프레임워크 프로세스 및 6대 품질지표

1) 나. 생성형 AI 품질관리 프레임워크 프로세스 (가이드라인 v2.0)

| 단계 | 주요 활동 | 품질 관리 포인트 |
|------|----------|----------------|
| 1. 데이터 수집·구축 | 출처 다양화, 라이선스 검토, 동의 확보 | 편향 방지를 위한 다양성 확보 |
| 2. 전처리 | 중복 제거, 노이즈 필터링, PII 마스킹 | 독성·유해 콘텐츠 필터링 |
| 3. 품질 검증 | 6대 품질지표 기준 자동·수동 검증 | Human Annotation 병행 |
| 4. 학습 적용 | 품질 가중치 적용, RLHF 데이터 관리 | 학습 데이터 버전 관리 |
| 5. 모니터링 | 모델 출력 편향·환각 모니터링 | 지속적 피드백 루프 운영 |

2) 다. 생성형 AI 데이터 품질지표 (6대 지표)

| 품질 지표 | 정의 | 측정 방법 | 생성형 AI 특수 고려사항 |
|---------|------|---------|----------------------|
| 정확성(Accuracy) | 데이터가 실제 사실과 일치하는 정도 | 팩트 체크 도구, 전문가 검수 | 환각 방지를 위한 사실 검증 필수 |
| 완전성(Completeness) | 필요한 데이터가 빠짐없이 존재하는 정도 | NULL 비율, 필수 필드 충족률 | 지식 격차(Knowledge Gap) 탐지 |
| 일관성(Consistency) | 동일 개념의 표현이 일관된 정도 | 용어 통일도, 형식 표준화율 | 모달리티 간 일관성 포함 |
| 최신성(Timeliness) | 데이터가 현재 시점을 반영하는 정도 | 데이터 생성 시점, 업데이트 주기 | LLM 지식 컷오프 관리 |
| 다양성(Diversity) | 다양한 관점·주제·출처를 포함하는 정도 | 도메인·언어·성별/연령 분포 | 편향 방지 핵심 지표 |
| 공정성(Fairness) | 특정 집단에 대한 편향이 없는 정도 | 편향 측정 도구, 공정성 벤치마크 | AI 윤리·규제 준수의 핵심 |

### 3. 일반 데이터 품질 vs 생성형 AI 데이터 품질 비교

| 비교 항목 | 일반 데이터 품질 | 생성형 AI 데이터 품질 |
|---------|---------------|---------------------|
| 규모 | 수백만~수억 건 | 수억~수조 토큰 |
| 검증 방식 | 전수·샘플링 검증 | 주로 샘플링·통계 기반 |
| 추가 지표 | 정확성·완전성·일관성 등 | + 다양성·공정성·최신성 |
| 주요 위험 | 무결성 위반, 정합성 오류 | 편향, 독성, 환각, Model Collapse |
| 법적 요구 | DQ 인증 | AI 기본법, EU AI Act |

- 생성형 AI 데이터 품질관리는 기존 6대 지표에 다양성·공정성·최신성이 추가된 확장 체계이며, AI 규제 강화로 공정성 지표 준수가 법적 의무화 추세.  "끝"

### 실무 제언

**편향(Bias) 제거 — 데이터 수집 단계 다양성 설계**
- **챌린지**: 웹 크롤링 기반 학습 데이터는 영어·남성·서구 중심의 구조적 편향이 내재되며, 사후 De-biasing은 모델 성능을 저하시킵니다.
- **제언**: 데이터 수집 계획 단계에서 다양성 쿼터(Diversity Quota)를 언어·지역·성별·연령별로 명시적으로 설정하고, 공정성 지표를 배포 전 필수 통과 게이트로 운영해야 합니다.

**환각(Hallucination) 방지 — RAG 기반 정확성 강화**
- **챌린지**: LLM은 학습 데이터에 없는 정보를 생성할 때 그럴듯한 거짓 정보를 확신에 차게 출력하며, 의료·법률·금융 전문 도메인에서 심각한 피해를 야기합니다.
- **제언**: RAG(Retrieval-Augmented Generation) 아키텍처로 검증된 지식 베이스에서 실시간 검색 후 생성하고, 생성 결과에 출처(Citation)를 의무 제공하는 팩트 체킹 레이어를 설계해야 합니다.

**Model Collapse 방지 — 합성 데이터 비율 관리**
- **챌린지**: AI 생성 합성 데이터를 재학습에 과도하게 활용하면 모델 품질이 점점 저하되는 Model Collapse 현상이 발생합니다.
- **제언**: 학습 데이터 내 합성 데이터 비율을 모니터링하고, 검증된 인간 생성 데이터(Human-generated Data) 비중을 일정 수준 이상 유지하는 데이터 혼합 정책을 수립해야 합니다.
