## 해설

> **국민의 데이터 신뢰, 공공의 품질 책임**  
> 공공데이터 품질관리 매뉴얼 V2.0 — 생명주기별 품질관리와 품질 진단·개선 절차

공공데이터는 행정·교통·기상·의료 등 국민 생활과 직결된 핵심 정보 자원으로, 품질 문제는 민간 AI·빅데이터 서비스의 오류로 직결됩니다. 기초 재료의 품질이 나쁘면 아무리 훌륭한 요리사도 좋은 요리를 만들 수 없는 것처럼, 공공데이터 품질은 다양한 디지털 서비스의 신뢰성을 결정하는 기초 토대입니다. 공공데이터법(2013) 시행 이후 개방·활용이 급증하면서 품질 관리 체계의 표준화 필요성이 높아졌고, 행정안전부가 공공데이터 품질관리 매뉴얼 V2.0을 발간하여 기관별 품질 관리의 지침을 제공하고 있습니다.

### 1. 출제 배경 및 의도

공공데이터 품질 관리는 데이터 기본법(2022), 공공데이터법(2013), 전자정부법 등 복수의 법령이 교차하는 정책 중심 토픽입니다. 국가 데이터 댐 사업, 마이데이터 서비스 확산과 함께 공공데이터 품질이 민간 서비스의 경쟁력을 좌우하는 기반 인프라로 부상했습니다. 채점관은 매뉴얼의 생명주기별 품질 관리 활동과 진단·개선 절차를 구체적이고 체계적으로 서술하는 능력을 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

공공데이터 품질관리 매뉴얼 V2.0은 공공기관이 보유·관리·개방하는 데이터의 품질을 정보 생명주기 전 단계에 걸쳐 체계적으로 관리하기 위한 절차와 기준을 정의한 지침서입니다.

- **Input:** 공공기관의 행정 데이터, 통계 데이터, 개방 데이터
- **Mechanism:** 생산 단계 → 관리 단계 → 활용 단계에 걸친 지속적 품질 관리 활동
- **Output:** 고품질 공공데이터 → 신뢰성 높은 공공 서비스 및 민간 활용 데이터

### 가. 정보 생명주기에 따른 단계별 품질관리 활동

```text
[공공데이터 정보 생명주기 품질관리]

  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
  │  생산 단계   │────→│  관리 단계   │────→│  활용 단계   │
  │ (Production) │     │(Management) │     │  (Usage)    │
  └─────────────┘     └─────────────┘     └─────────────┘
        │                    │                    │
        ▼                    ▼                    ▼
  데이터 설계 시          데이터 운영 중          개방·배포 후
  품질 기준 반영          지속적 품질 관리         품질 피드백 관리
```

#### 생산 단계 품질관리

| 활동 | 세부 내용 | 목적 |
|------|----------|------|
| **데이터 표준 정의** | 데이터 사전, 코드 체계, 명명 규칙 수립 | 입력 단계부터 일관성 확보 |
| **데이터 모델 설계 검토** | 정규화, 필수 항목 정의, 제약 조건 설정 | 구조적 품질 기반 마련 |
| **입력 검증 규칙 설계** | 도메인 제약, 필수 입력, 형식 검증 | 소스 단계 오류 방지 |
| **메타데이터 등록** | 데이터 설명·출처·갱신 주기 등록 | 활용 시 데이터 이해 지원 |

#### 관리 단계 품질관리

| 활동 | 세부 내용 | 목적 |
|------|----------|------|
| **정기 품질 진단** | 품질 6대 지표 기준 주기적 측정 | 품질 수준 모니터링 |
| **오류 데이터 정제** | 발견된 품질 이슈의 원인 분석 및 수정 | 데이터 정확성 복원 |
| **갱신·최신화 관리** | 데이터 갱신 주기 준수, 최신 정보 반영 | 최신성·완전성 유지 |
| **접근 권한 관리** | 역할 기반 접근 제어(RBAC) | 보안·접근성 균형 |

#### 활용 단계 품질관리

| 활동 | 세부 내용 | 목적 |
|------|----------|------|
| **개방 데이터 품질 검증** | 개방 전 최종 품질 검증 게이트 | 외부 배포 품질 보증 |
| **사용자 피드백 수집** | 오류 신고 채널 운영, 이슈 추적 | 현장 품질 이슈 발굴 |
| **활용 실적 모니터링** | 다운로드·API 호출 통계 분석 | 고수요 데이터 우선 관리 |
| **국가데이터맵 연계** | 공공데이터포털 메타데이터 최신화 | 데이터 발견 가능성 향상 |

### 나. 데이터 품질 진단 및 개선의 절차

```text
[공공데이터 품질 진단 및 개선 절차]

  ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
  │ 1. 진단  │──→│ 2. 분석  │──→│ 3. 개선  │──→│ 4. 검증  │──→│ 5. 모니터│
  │  계획    │   │  및 측정 │   │  실행    │   │  및 확인 │   │  링      │
  └──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
      │               │               │               │               │
   진단 범위        품질 지표        오류 유형별       정제 결과        대시보드
   선정             측정             정제 방법         검증             운영
   진단 기준        이상 탐지        근본 원인         품질 목표        지속 관리
   수립             원인 분석        제거              달성 확인
```

| 단계 | 주요 활동 | 산출물 |
|------|----------|--------|
| **1. 진단 계획** | 진단 범위·대상 선정, 품질 기준·목표 수립, 진단 도구 선택 | 품질 진단 계획서 |
| **2. 분석·측정** | 데이터 프로파일링, 6대 품질 지표 측정, 오류 패턴 분석 | 품질 측정 결과 보고서 |
| **3. 개선 실행** | 오류 데이터 정제, 표준 코드 매핑, 입력 프로세스 개선 | 정제 데이터, 개선 이력 |
| **4. 검증·확인** | 정제 결과 재측정, 품질 목표 달성 여부 확인, 책임자 승인 | 품질 개선 완료 보고서 |
| **5. 모니터링** | 품질 대시보드 운영, 주기적 재진단, 이상 알람 설정 | 품질 관리 KPI 보고서 |

#### 공공데이터 품질 6대 지표

| 품질 지표 | 측정 기준 | 공공데이터 특화 예시 |
|---------|---------|-------------------|
| **정확성** | 실제 행정 사실과 데이터의 일치 여부 | 주소 → 도로명주소 DB와 일치 여부 |
| **완전성** | 필수 항목의 NULL·공백 비율 | 사업자 등록번호 누락 건수 |
| **유효성** | 도메인 형식 준수 여부 | 주민등록번호 유효성 검사 |
| **일관성** | 연관 데이터 간 논리 모순 여부 | 기관 코드 ↔ 기관명 불일치 |
| **유일성** | 중복 레코드 존재 여부 | 도로명 주소 중복 등록 |
| **접근성** | 필요 시 데이터 이용 가능 여부 | 공공데이터포털 다운로드 성공률 |

### 3. 핵심 키워드 (Must-Have)

- **근거 법령:** 공공데이터법, 데이터 기본법, 전자정부법
- **생명주기 단계:** 생산(Production), 관리(Management), 활용(Usage)
- **품질 진단 절차:** 진단 계획 → 분석·측정 → 개선 실행 → 검증·확인 → 모니터링
- **품질 6대 지표:** 정확성, 완전성, 유효성, 일관성, 유일성, 접근성
- **핵심 활동:** 데이터 프로파일링, 표준 코드 관리, 메타데이터 등록, 사용자 피드백

### 4. 맥락과 비교

| 비교 항목 | 공공데이터 품질관리 | 민간 데이터 품질관리 |
|---------|----------------|------------------|
| **근거** | 공공데이터법, 매뉴얼 V2.0(의무) | 자율 기준, DQ 인증(선택) |
| **목적** | 국민 서비스 신뢰성, 개방 의무 이행 | 비즈니스 가치 창출 |
| **관리 주체** | 공공기관(책임 관리) | 데이터 오너, 스튜어드 |
| **특징** | 투명성·공개 의무 강조 | 보안·경쟁우위 강조 |

### 5. 실무 제언

**레거시 공공 시스템의 품질 개선 — 소스 시스템 원인 제거 전략**

- **챌린지:** 공공기관의 레거시 시스템은 오래된 입력 검증 로직의 부재로 인해 수년간 오류 데이터가 누적되어 있으며, 정제 후에도 소스 시스템 입력 단계에서 지속적으로 오염이 재발합니다.
- **제언:** 단순 사후 정제보다 소스 시스템의 입력 검증 규칙을 강화하고, 표준 코드 체계와 연계한 자동 검증 로직을 시스템에 내재화해야 합니다. 데이터 품질 책임자(Data Quality Officer)를 기관별로 지정하여 품질 관리를 전담하고, 매뉴얼 V2.0에 따른 연 1회 정기 진단 체계를 제도화하는 것이 핵심입니다.
