## 문제

공공데이터 중요성이 증대 되면서 공공데이터의 품질 관리와 품질 진단 및 개선을 위해 공공데이터품질관리 매뉴얼 V2.0 을 제공하였다. 다음에 대하여 설명하시오.

## 해설

### 출제 배경 및 의도

공공데이터는 행정·교통·기상·의료 등 국민 생활과 직결된 핵심 정보 자원으로, 품질 문제는 민간 AI·빅데이터 서비스의 오류로 직결됩니다. 공공데이터법(2013) 시행 이후 개방·활용이 급증하면서 품질 관리 체계의 표준화 필요성이 높아졌습니다.

공공데이터 품질 관리는 데이터 기본법(2022), 공공데이터법(2013), 전자정부법 등 복수의 법령이 교차하는 정책 중심 토픽입니다. 국가 데이터 댐 사업, 마이데이터 서비스 확산과 함께 공공데이터 품질이 민간 서비스 경쟁력을 좌우하는 기반 인프라로 부상했습니다.

채점관은 매뉴얼의 생명주기별 품질 관리 활동과 진단·개선 절차를 구체적이고 체계적으로 서술하는 능력을 평가합니다.

### 1. 공공데이터 품질관리 매뉴얼 V2.0, 개요

정 의  • 공공기관이 보유·관리·개방하는 데이터의 품질을 정보 생명주기 전 단계에 걸쳐
       - 체계적으로 관리하기 위한 절차와 기준을 정의한 행정안전부 지침서

```
[공공데이터 정보 생명주기 품질관리]

  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
  │  생산 단계   │────→│  관리 단계   │────→│  활용 단계   │
  │ (Production) │     │(Management) │     │  (Usage)    │
  └─────────────┘     └─────────────┘     └─────────────┘
        │                    │                    │
        ▼                    ▼                    ▼
  데이터 설계 시          데이터 운영 중          개방·배포 후
  품질 기준 반영          지속적 품질 관리         품질 피드백 관리
  표준 정의/입력검증      정기진단/정제/갱신       사용자 피드백 수집
```

- 생명주기 3단계 전반에 걸친 품질 관리가 매뉴얼 V2.0의 핵심 원칙

### 2. 핵심 내용 — 생명주기별 품질관리 활동 및 진단·개선 절차

1) 가. 정보 생명주기에 따른 단계별 품질관리 활동

| 단계 | 활동 | 세부 내용 | 목적 |
|------|------|----------|------|
| 생산 | 데이터 표준 정의 | 데이터 사전, 코드 체계, 명명 규칙 수립 | 입력 단계부터 일관성 확보 |
| 생산 | 입력 검증 규칙 설계 | 도메인 제약, 필수 입력, 형식 검증 | 소스 단계 오류 방지 |
| 생산 | 메타데이터 등록 | 데이터 설명·출처·갱신 주기 등록 | 활용 시 데이터 이해 지원 |
| 관리 | 정기 품질 진단 | 품질 6대 지표 기준 주기적 측정 | 품질 수준 모니터링 |
| 관리 | 오류 데이터 정제 | 원인 분석 및 수정 | 데이터 정확성 복원 |
| 관리 | 갱신·최신화 관리 | 데이터 갱신 주기 준수 | 최신성·완전성 유지 |
| 활용 | 개방 데이터 품질 검증 | 개방 전 최종 품질 검증 게이트 | 외부 배포 품질 보증 |
| 활용 | 사용자 피드백 수집 | 오류 신고 채널 운영, 이슈 추적 | 현장 품질 이슈 발굴 |
| 활용 | 국가데이터맵 연계 | 공공데이터포털 메타데이터 최신화 | 데이터 발견 가능성 향상 |

2) 나. 데이터 품질 진단 및 개선 절차

```
[공공데이터 품질 진단 및 개선 절차]

  ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
  │1.진단│──→│2.분석│──→│3.개선│──→│4.검증│──→│5.모니│
  │ 계획 │   │·측정 │   │ 실행 │   │·확인 │   │터링  │
  └──────┘   └──────┘   └──────┘   └──────┘   └──────┘
  진단범위    품질지표    오류유형별   정제결과    대시보드
  선정·기준   측정/탐지   정제·원인    검증·승인   운영·알람
  수립        원인분석    제거
```

| 단계 | 주요 활동 | 산출물 |
|------|----------|--------|
| 1. 진단 계획 | 진단 범위·대상 선정, 품질 기준·목표 수립 | 품질 진단 계획서 |
| 2. 분석·측정 | 데이터 프로파일링, 6대 품질 지표 측정 | 품질 측정 결과 보고서 |
| 3. 개선 실행 | 오류 데이터 정제, 표준 코드 매핑, 입력 프로세스 개선 | 정제 데이터, 개선 이력 |
| 4. 검증·확인 | 정제 결과 재측정, 품질 목표 달성 여부 확인 | 품질 개선 완료 보고서 |
| 5. 모니터링 | 품질 대시보드 운영, 주기적 재진단, 이상 알람 | 품질 관리 KPI 보고서 |

### 3. 공공 vs 민간 데이터 품질관리 비교

| 비교 항목 | 공공데이터 품질관리 | 민간 데이터 품질관리 |
|---------|----------------|------------------|
| 근거 | 공공데이터법, 매뉴얼 V2.0(의무) | 자율 기준, DQ 인증(선택) |
| 목적 | 국민 서비스 신뢰성, 개방 의무 이행 | 비즈니스 가치 창출 |
| 관리 주체 | 공공기관(책임 관리) | 데이터 오너, 스튜어드 |
| 특징 | 투명성·공개 의무 강조 | 보안·경쟁우위 강조 |
| 6대 지표 특화 예시 | 주소→도로명주소 DB 일치, 기관코드↔기관명 일관성 | ERP-CRM 간 고객명 일관성 |

- 공공데이터 품질관리는 의무 법령에 근거하며, 국민 서비스 신뢰성 확보와 민간 AI·빅데이터 생태계 기반 조성이 핵심 목표.  "끝"

### 실무 제언

**레거시 공공 시스템 소스 단계 원인 제거**
- **챌린지**: 공공기관 레거시 시스템은 입력 검증 로직 부재로 수년간 오류 데이터가 누적되어 있으며, 정제 후에도 소스 시스템 입력 단계에서 지속적으로 오염이 재발합니다.
- **제언**: 사후 정제보다 소스 시스템의 입력 검증 규칙을 강화하고, 표준 코드 체계와 연계한 자동 검증 로직을 시스템에 내재화해야 합니다.

**데이터 품질 책임자(DQO) 지정**
- **챌린지**: 품질 관리 전담 조직이 없는 공공기관은 진단 후 개선 조치의 책임 소재가 불명확하여 이행이 지연됩니다.
- **제언**: 기관별 데이터 품질 책임자(Data Quality Officer)를 지정하고, 매뉴얼 V2.0에 따른 연 1회 정기 진단 체계를 제도화해야 합니다.

**사용자 피드백 기반 현장 품질 이슈 발굴**
- **챌린지**: 내부 진단만으로는 민간 활용 시 발생하는 다양한 품질 이슈를 충분히 탐지하기 어렵습니다.
- **제언**: 공공데이터포털에 오류 신고 채널을 운영하고, 수집된 피드백을 품질 진단 계획에 반영하는 피드백 루프를 구축하여 현장 중심의 품질 개선을 실현해야 합니다.
