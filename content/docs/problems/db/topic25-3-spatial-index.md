공간 인덱스 구조에서 MBR(Minimum Bounding Rectangle)을 설명하고, R트리, R+트리, R*트리를 비교하시오.

## 해설

> **공간 탐색의 엔진 — MBR과 R-Tree 계열 인덱스**  
> MBR 개념, R-Tree/R+-Tree/R*-Tree 비교 분석

공간 데이터에는 일반 B-Tree 인덱스를 적용할 수 없습니다. 점·선·면의 다차원 위치 정보는 단일 키로 정렬할 수 없기 때문입니다. R-Tree는 MBR(최소 경계 사각형)을 계층적으로 조직화하여 공간 탐색을 효율화하는 트리 구조로, PostGIS·Oracle Spatial·MySQL의 공간 인덱스 핵심 알고리즘입니다.

### 1. 출제 배경 및 의도

111회 기술사 시험에서 출제된 이 문제는 공간 인덱스의 이론적 이해와 트리 변형들의 차이점 분석을 요구합니다. R-Tree 계열의 세 변형(R-Tree, R+-Tree, R*-Tree)은 각각 노드 분할 전략과 오버랩 처리 방식이 다르며, 이를 명확히 구분하는 것이 채점 포인트입니다. 채점관은 MBR의 정확한 정의와 R-Tree 계열의 트리오프(Trade-off)를 비교표 형태로 설명할 수 있는지 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

R-Tree는 각 공간 객체를 포함하는 최소 경계 사각형(MBR)을 계층적으로 그룹화하여, 범위 질의와 공간 관계 질의를 효율적으로 처리하는 균형 트리(Balanced Tree) 구조입니다.

- **Input:** 다수의 공간 객체(점·선·면)와 공간 질의(범위 검색·관계 판별)
- **Mechanism:** 삽입 시 MBR 면적 최소화 기준으로 노드 선택 → 노드 오버플로 시 분할(Split) → 트리 균형 유지
- **Output:** 공간 질의에 대한 고속 후보 집합 반환 (2단계 필터의 1단계)

### MBR (Minimum Bounding Rectangle) 설명

MBR(최소 경계 사각형, Minimum Bounding Rectangle)은 임의 형상의 공간 객체를 완전히 포함하는 가장 작은 축 정렬 사각형(Axis-Aligned Rectangle)입니다.

```text
[MBR 개념]

  실제 폴리곤            MBR (최소 경계 사각형)
       *                 ┌─────────────┐
      * *                │     *       │
     *   *     →         │    * *      │
    *     *              │   *   *     │
     *   *               │  *     *   │
      * *                │   *   *    │
       *                 │    * *     │
                         │     *      │
                         └─────────────┘
  복잡한 형상을 단순 사각형으로 근사 → 빠른 1차 필터 가능

  MBR(객체 A) = (x_min, y_min, x_max, y_max)
```

**MBR의 특성:**
- 좌표 최솟값·최댓값으로 단순 정의 → 빠른 교차 판별
- 실제 형상보다 큰 영역 포함 → False Positive 발생 가능 (2단계 정밀 판별로 제거)
- 인덱스 노드 크기 최소화에 기여

### R-Tree 기본 구조

```text
[R-Tree 계층 구조]

  루트 노드 (Root)
  ┌─────────────────────────────┐
  │  MBR1(큰 영역)  MBR2(큰 영역) │
  └──────┬──────────────┬───────┘
         │              │
  ┌──────▼──────┐ ┌─────▼───────┐  ← 내부 노드 (Internal Node)
  │MBR3  MBR4   │ │MBR5  MBR6   │     각 노드는 자식의 MBR들
  └──┬────┬─────┘ └──┬────┬─────┘
     │    │          │    │
  [obj] [obj]      [obj] [obj]       ← 리프 노드 (Leaf Node)
                                        실제 공간 객체 포인터

  특성:
  - 균형 트리 (모든 리프 같은 깊이)
  - 각 노드 m ~ M개 엔트리 (m=최소, M=최대)
  - 삽입: 면적 증가 최소 노드 선택
```

### 3. 핵심 키워드 (Must-Have)

**R-Tree / R+-Tree / R*-Tree 비교**

| 비교 항목 | R-Tree (1984, Guttman) | R+-Tree (1987) | R*-Tree (1990) |
|---------|----------------------|----------------|----------------|
| **핵심 철학** | 기본 공간 인덱스 | 오버랩 완전 제거 | 오버랩+면적 동시 최소화 |
| **오버랩(Overlap)** | 허용 (MBR 겹침 가능) | 완전 제거 | 최소화 (완전 제거 아님) |
| **객체 중복 저장** | 없음 | 있음 (경계 걸치면 다중 저장) | 없음 |
| **노드 분할 방식** | Quadratic/Linear Split | 경계 객체 다중 삽입 | Forced Reinsert(재삽입) 우선 |
| **검색 성능** | 중간 | 단순 포함 질의에 유리 | 최적 (대부분의 질의 유형) |
| **삽입 비용** | 낮음 | 높음 (다중 삽입) | 중간 (재삽입 오버헤드) |
| **저장 공간** | 보통 | 많음 (중복 저장) | 보통 |
| **구현 복잡도** | 낮음 | 중간 | 높음 |
| **실무 채택** | 초기 시스템 | 제한적 | PostGIS·Oracle Spatial (표준) |

#### R-Tree 노드 분할 알고리즘

```text
[R-Tree 노드 분할 전략 비교]

Linear Split    :  두 가장 먼 객체를 씨앗으로 선택 후 나머지 분배
                   → 빠르지만 품질 낮음

Quadratic Split :  모든 쌍을 검사하여 낭비 면적 최대 쌍을 씨앗으로 선택
                   → O(n²) 비용, 균형 있는 품질

R*-Tree의 방법  :  분할 전 Forced Reinsert로 재삽입 시도
                   → 오버랩과 면적 동시 최소화 목표 함수 사용
```

#### R+-Tree의 객체 중복 처리

```text
[R+-Tree 경계 객체 처리]

      R-Tree                    R+-Tree
  ┌───────────┐              ┌──────┐  ┌──────┐
  │  Node1    │              │Node1 │  │Node2 │
  │  ┌────────┼──┐           │  Obj │  │  Obj │ ← 경계 객체가
  │  │  Obj   │  │           └──────┘  └──────┘   두 노드에
  └──┼────────┘  │                                 모두 저장
     │  Node2    │
     └───────────┘

  Obj가 두 노드의 경계에 걸침:
  - R-Tree: 한 노드에만 저장, MBR 겹침 발생
  - R+-Tree: 두 노드 모두에 저장, MBR 겹침 없음
```

### 5. 실무 제언

**R*-Tree 선택과 인덱스 설계 최적화**

- **챌린지:** R-Tree 계열 인덱스는 노드 오버랩이 증가하면 검색 시 여러 노드를 중복 방문하는 Dead-End Path 문제가 발생하여 성능이 급격히 저하됩니다. 특히 클러스터 분포가 없는 균일 분포의 공간 데이터에서 이 현상이 심화됩니다.
- **제언:** 실무에서는 R*-Tree를 기본 선택(PostGIS GIST 인덱스의 기반)하되, 데이터 삽입 완료 후 `CLUSTER table USING idx_geom;` 명령으로 물리적 클러스터링을 적용하여 디스크 I/O를 최소화하는 것이 효과적입니다. 대용량 배치 삽입 시에는 STRtree(Sort-Tile-Recursive) 방식의 벌크 로딩을 활용하면 균형 잡힌 초기 R-Tree를 구성할 수 있습니다.
