## 해설

### 출제 배경 및 의도

제5정규형(5NF)은 관계형 데이터베이스 정규화 이론의 실질적 최상위 단계로, ERP·금융·제조 SCM 등에서 3개 이상의 엔티티가 순환적 다대다(M:N) 관계로 얽히는 삼항 관계 구조를 다룹니다. 함수 종속성(FD)이나 다치 종속(MVD)처럼 직관적으로 식별하기 쉬운 제약조건과 달리, 결합성과 대칭성에 의존하는 조인 종속성(JD)을 다루기 때문에 최고 난이도의 아키텍처 역량 검증에 출제됩니다.

채점관은 5NF의 정의와 함께 조인 종속(JD) 개념, 2개 분해 시 가짜 튜플이 발생하는 이유, 그리고 반드시 3개 이상으로 n-분해해야 무손실 복원이 가능하다는 구조적 특성을 사례와 함께 서술하는 답안을 고득점으로 평가합니다.

### 1. 제5정규형(5NF), 개요

정 의  • 제4정규형(4NF)을 만족하면서, 릴레이션에 존재하는 모든 조인 종속성(JD)의 결정자가 후보키(CK)가 되도록 강제하여 투플 생성 이상을 제거하는 최상위 정규화 프로세스
       - 테이블을 분해했다가 다시 Natural Join하는 대칭적 성질 → PJ/NF(Projection-Join Normal Form)라고도 명명

```
[정규화 최상위 계층 흐름]

 3NF/BCNF            4NF                  5NF
(함수 종속성)      (다치 종속성)         (조인 종속성)
     │                 │                    │
결정자=후보키    독립 다치 속성 2개 분해  순환 삼항 관계 3개 이상 분해
     │                 │                    │
FD 제거 완료     MVD 제거 완료        JD 제거 완료 → PJ/NF 달성

조인 종속성(JD): R을 R1,R2,...,Rn으로 분해 후 Natural Join → R과 완전 일치
```

- 5NF는 2개로 분해 시 가짜 튜플이 필연적으로 발생하며, 반드시 3개 이상의 테이블로 대칭 분해해야 무손실 복원이 가능하다.

### 2. 조인 종속성 개념 및 5NF 적용 메커니즘

1) 조인 종속성(JD) 사례: 공급망 삼항 관계

| 비즈니스 제약 | 내용 |
|--------------|------|
| ① 공급 관계 | 공급업자는 특정 부품을 생산·공급 가능 |
| ② 조달 관계 | 기업(소비자)은 프로젝트를 위해 특정 부품 조달 사용 |
| ③ 계약 관계 | 기업은 특정 공급업자와 계약 체결 |
| ④ **핵심 순환 룰** | 공급업자가 부품P 생산 + 기업C가 부품P 사용 + 기업C가 해당 공급업자와 거래 → 공급업자는 반드시 기업C에 부품P 공급 의무 |

```
[원시 릴레이션 <공급망(SPC)>]          [2개 분해 시 가짜 튜플 문제]

공급업자  | 부품명    | 기업명           SP ⋈ PC ≠ SPC  (가짜 튜플 발생!)
──────────────────────────────
김기술   | 반도체    | S전자            [5NF 3원 무손실 분해]
박공학   | 디스플레이| S전자            SP ⋈ PC ⋈ SC = SPC (무손실 복원 ✓)
김기술   | 디스플레이| L디스플레이
김기술   | 디스플레이| S전자  ← 순환룰  <공급부품(SP)>: (공급업자, 부품명)
                              필수 존재  <부품기업(PC)>: (부품명, 기업명)
                                         <공급기업(SC)>: (공급업자, 기업명)
```

- 분해된 3개 테이블을 동시에 Natural Join(SP ⋈ PC ⋈ SC)하면 가짜 데이터 없이 원본 4건으로 완벽하게 무손실 복원된다.

2) 4NF와 5NF 비교

| 비교 항목 | 제4정규형(4NF) | 제5정규형(5NF) |
|-----------|----------------|----------------|
| **핵심 타깃** | 다치 종속성(MVD): X ↠ Y | 조인 종속성(JD) |
| **관계 구조** | 결정자 1개에 독립적 다치 속성 2개 | 3개 이상 속성의 순환 구조 |
| **최소 분해 수** | **2개** 테이블로 분해 가능 | 반드시 **3개 이상** 대칭 분해 필수 |
| **식별 난이도** | 비교적 용이 (속성 간 독립성 확인) | 매우 어려움 (삼항 이상 순환 룰 검증) |
| **동의어** | - | PJ/NF (Projection-Join Normal Form) |

- 5NF에서 2개로만 분해하면 3개 제약 중 하나의 조합만 표현되어 대칭적 순환 규칙의 일부 데이터가 반드시 누락 또는 가짜 튜플로 나타난다.

### 3. 정규화 단계 전체 비교 및 트렌드

| 정규형 | 제거 대상 | 분해 특징 | 달성 상태 |
|--------|-----------|-----------|-----------|
| **3NF/BCNF** | 함수 종속성(FD) | 결정자→후보키, 2개 분해 | 기본키 기반 무결성 확보 |
| **4NF** | 다치 종속성(MVD) | 독립 다치 속성, 2개 이원 분해 | 조합 중복 폭발 제거 |
| **5NF(PJ/NF)** | **조인 종속성(JD)** | 순환 삼항, **3개 이상 n-분해 필수** | 순환 관계 구조 완전화 |

- 실무에서 5NF를 완벽 적용하면 3개 이상 테이블의 동시 조인이 필수가 되어 OLTP 성능이 급감하므로, 5NF 구조로 판명된 영역의 DML 빈도를 분석하여 변경 빈도가 낮은 영역은 PostgreSQL JSONB나 Document Model로 통합 임베딩하는 융합형 아키텍처 전략이 현실적 대안이다.  "끝"

### 실무 제언

**실무 적용의 극악한 난이도와 트레이드오프**
- **챌린지**: 5NF 스키마를 채택하면 조회 시 3개 이상 테이블을 항상 동시에 조인해야 하므로 SQL 작성이 복잡해지고, 클라우드 OLTP 환경에서 분산 노드 간 Shuffle과 디스크 I/O 병목으로 응답 속도(Latency)가 기하급수적으로 저하된다.
- **제언**: 분석 단계에서는 반드시 5NF 수준의 논리 모델을 설계하여 비즈니스 규칙 왜곡과 연결함정을 방지하고, 물리 단계에서는 DML 빈도 분석 후 조회 중심 영역은 JSONB 임베딩으로 무결성과 고성능을 동시에 달성하는 전략을 리드해야 한다.

**논리 모델과 물리 모델의 분리 원칙**
- **챌린지**: 5NF의 이론적 완결성만 추구하다가 물리 구현 시 성능 현실을 무시하면 시스템 운영 불가 수준의 응답 지연이 발생한다.
- **제언**: 논리 모델에서는 5NF 무결성을 확보하되, 물리 모델에서는 트랜잭션 패턴 분석(CRUD 매트릭스)을 기반으로 3개 분해·JSONB 통합·교차 엔티티 패턴 중 최적 전략을 선택적으로 적용하는 이중 레이어 설계 방법론을 채택해야 한다.

**ERP·SCM 도메인에서의 순환 관계 설계**
- **챌린지**: ERP·금융·제조 SCM에서 공급자-부품-소비자 같은 삼항 순환 관계를 2개 테이블로만 설계하면 비즈니스 제약 룰이 DB에 내재화되지 않아 가짜 데이터 유입을 차단할 수 없다.
- **제언**: 도메인 분석 단계에서 삼항 이상의 순환 관계를 식별하여 조인 종속성 여부를 검증하고, 5NF 분해 원칙에 따라 교차 엔티티(Intersection Entity)를 명확히 설계하여 순환 룰을 DB 구조 자체에 내재화해야 한다.
