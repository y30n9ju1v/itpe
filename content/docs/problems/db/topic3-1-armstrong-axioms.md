## 해설

### 출제 배경 및 의도

암스트롱 공리는 관계형 데이터베이스(RDBMS) 설계의 핵심인 정규화(Normalization)와 함수 종속성 판별의 가장 근본적인 수학적 추론 규칙입니다. 최근 클라우드 이관 및 레거시 DB 정비 사업이 늘어나면서 수천 개의 테이블 속 함수 종속성을 알고리즘으로 자동 검증하는 이론적 뼈대로서 그 중요성이 더욱 강조되고 있습니다.

채점관이 암스트롱 공리를 출제하는 의도는 정규화의 수학적 정당성을 이해하는지 검증하기 위함입니다. 단순히 규칙을 암기하는 수준이 아니라, 각 규칙이 어떤 정규화 단계와 연결되는지, 그리고 폐포(Closure) 도출 알고리즘을 실제로 적용할 수 있는지까지 설명할 수 있어야 합니다.

### 1. 암스트롱 공리 (Armstrong's Axioms), 개요

정 의  • 릴레이션 스키마 R과 함수 종속성 집합 F가 주어졌을 때, 논리적으로 유도 가능한 모든 함수 종속성을 찾아내기 위한 수학적 추론 규칙(Inference Rules) 집합
       - 1974년 윌리엄 암스트롱(William W. Armstrong)이 제안, 건전성(Soundness)과 완전성(Completeness)을 동시에 만족

```
[암스트롱 공리의 역할 구조]

암스트롱 공리 (기본 3규칙 + 부가 3규칙)
         │
         ▼ (반복 적용하여 도출)
함수 종속성 폐포(F⁺) 및 애트리뷰트 폐포(X⁺) 계산
         │
         ▼ (후보키 식별, 정규형 판별)
데이터베이스 정규화 (1NF → 2NF → 3NF → BCNF → 4NF → 5NF)
```

- 건전성(Soundness): 공리로 유도된 FD는 모든 릴레이션에서 반드시 참 — 잘못된 종속성 절대 유도 안 됨
- 완전성(Completeness): F에 의해 논리적으로 함축된 모든 FD는 공리만으로 반드시 유도 가능

### 2. 3대 기본 공리 및 부가 규칙

1) 3대 기본 공리

| 공리명 | 수학적 정의 | 직관적 의미 | 사례 |
|--------|-------------|-------------|------|
| **반사율** (Reflexivity) | Y ⊆ X 이면, X → Y | 포함하는 하위 속성은 스스로 결정 가능 | (학번, 주민번호) → 학번 |
| **첨가율** (Augmentation) | X → Y 이면, XZ → YZ | 양변에 동일 속성 추가해도 종속성 유지 | 학번→성명 이면, (학번,학과)→(성명,학과) |
| **이행율** (Transitivity) | X→Y 이고 Y→Z 이면, X→Z | 징검다리를 건너 궁극적 종속성 유도 | 학번→학과코드, 학과코드→학과명 ∴ 학번→학과명 |

- 3대 기본 공리는 건전하고(Soundness) 완전한(Completeness) 추론 규칙으로, 나머지 부가 규칙들의 증명 토대가 된다.

2) 3대 부가 규칙 (기본 공리로부터 증명 가능)

| 규칙명 | 수학적 정의 | 증명 메커니즘 |
|--------|-------------|---------------|
| **합집합** (Union) | X→Y 이고 X→Z 이면, X→YZ | ① X→Y에 Z 첨가율 → XZ→YZ, ② X→Z에 Y 첨가율 → XY→YZ, ③ 이행율 적용 → X→YZ |
| **분해** (Decomposition) | X→YZ 이면, X→Y 이고 X→Z | ① 반사율: YZ→Y, ② X→YZ이고 YZ→Y이므로 이행율 → X→Y |
| **의사이행** (Pseudo-transitivity) | X→Y이고 WY→Z이면, WX→Z | ① X→Y에 첨가율 → WX→WY, ② WX→WY이고 WY→Z이므로 이행율 → WX→Z |

- 부가 규칙은 기본 공리만으로 증명되므로 새로운 공리가 아닌 파생 규칙이며, 폐포 계산 시 연산 효율을 높이는 데 활용된다.

### 3. 애트리뷰트 폐포(X⁺) 도출 알고리즘 및 정규화 연계

```
[X⁺ 도출 알고리즘]            [폐포 도출 사례: R(A,B,C,D,E), F={A→B, B→C, CD→E}]

Input: X, F                   Step 1. 초기화 ──────────> (AD)+ = {A, D}
1. X+ := X                    Step 2. A→B 적용 ────────> B 추가 → {A, D, B}
2. F의 Y→Z에 대해:             Step 3. B→C 적용 ────────> C 추가 → {A, D, B, C}
   Y ⊆ X+ 이면 X+ := X+ ∪ Z  Step 4. CD→E 적용 ───────> E 추가 → {A, D, B, C, E}
3. 추가 없을 때까지 반복       결과: (AD)+ = {A,B,C,D,E} → AD는 슈퍼키(Super Key)
```

| 활용 목적 | 암스트롱 공리 적용 결과 | 연계 정규형 |
|-----------|------------------------|-------------|
| 후보키(CK) 판별 | 폐포가 전체 속성 집합을 포함하면 슈퍼키 → 최소 슈퍼키가 후보키 | 모든 정규화의 기준 |
| 부분 FD 식별 | 복합키의 진부분집합으로 폐포 계산 → 일반 속성 포함 시 부분 종속 | **2NF** |
| 이행 FD 식별 | 일반 속성 X의 폐포에 다른 일반 속성 Y 포함 여부 판단 | **3NF** |
| 결정자 판별 | 후보키가 아닌 속성이 다른 속성을 결정하는지 검증 | **BCNF** |

- 애트리뷰트 폐포 계산은 CASE 도구에서 슈퍼키·후보키를 자동 식별하는 알고리즘의 근간이 되며, 정규화 자동화의 이론적 토대를 제공한다.  "끝"

### 실무 제언

**수학적 이론과 실무 튜닝의 균형**
- **챌린지**: 암스트롱 공리를 엄격하게 적용하여 3NF·BCNF에 도달하면 이상 현상은 완벽히 차단되지만, 테이블 파편화로 인해 과도한 조인(Join) 연산이 발생하여 시스템 성능이 저하되는 트레이드오프가 발생한다.
- **제언**: 암스트롱 공리로 완벽한 논리 스키마를 도출한 후, 실제 대용량 트랜잭션 환경의 부하를 고려하여 의도적이고 통제된 반정규화(Denormalization)나 인덱스 전략을 설계 단계에서 역으로 제시할 수 있는 안목을 갖추어야 한다.

**최소 보존 집합(Minimal Cover)을 통한 기술 부채 최소화**
- **챌린지**: 중복되거나 무의미한 함수 종속성이 존재할 경우 정규화 과정이 비효율적으로 진행되어 불필요한 테이블 분해가 발생한다.
- **제언**: 공리를 통해 릴레이션의 모든 종속성을 표현하는 최소 보존 집합(Minimal Cover)을 도출하고, DA는 이를 기반으로 설계 검증을 자동화하여 시스템 고도화 시 데이터 구조적 리스크와 기술 부채를 설계 초기 단계부터 통제해야 한다.

**레거시 DB 정비 시 공리 기반 종속성 분석**
- **챌린지**: 레거시 시스템의 수천 개 테이블에서 함수 종속성을 수작업으로 분석하는 것은 현실적으로 불가능하며, 누락 시 이상현상 재발 위험이 존재한다.
- **제언**: 암스트롱 공리 기반의 종속성 자동 분석 도구(CASE Tool, ERwin 등)를 활용하여 폐포 계산을 자동화하고, 정규화 위반 테이블을 일괄 스캔·진단하는 데이터 품질 거버넌스 프로세스를 수립해야 한다.
