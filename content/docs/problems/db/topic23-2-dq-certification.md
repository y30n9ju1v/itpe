## 해설

### 출제 배경 및 의도

데이터 경제 시대에 데이터 품질은 AI·분석 결과의 신뢰성을 좌우하는 핵심 자산입니다. DQ 인증은 기업·기관이 보유한 데이터의 품질 수준을 제3자 기관이 공식 검증하는 제도로, 데이터 유통·거래 시장의 신뢰 기반을 형성합니다.

데이터 기본법(2022) 및 데이터산업진흥법에 근거한 정부 정책과 직결되는 실무 토픽입니다. 데이터 거래소·마켓플레이스 활성화로 데이터 품질의 객관적 검증 필요성이 높아졌으며, 공공·민간 기관 모두 DQ 인증 취득이 경쟁력의 지표가 되었습니다.

채점관은 인증 제도의 법적 근거, 심사 항목(관리 체계 + 값 품질), 인증 절차를 체계적으로 서술할 수 있는지 평가합니다.

### 1. DQ 인증(Data Quality Certification), 개요

정 의  • 조직이 관리하는 데이터 품질 수준이 일정 기준에 부합함을
       - K-DATA(한국데이터산업진흥원)가 공식 확인·인증서를 부여하는 제도

```
[DQ 인증 프로세스]

신청기관              K-DATA 심사팀           인증위원회
   │                      │                      │
   ├─ 인증신청서 제출 ──→  │                      │
   │  (신청서+자체점검표)   │                      │
   │                      ├─ 서류심사 (2주)        │
   │                      │  · 관련 문서 검토      │
   │  ←── 보완요청 ─────── │                      │
   │                      │                      │
   ├─ 현장심사 수감 ───────→│                      │
   │                      ├─ 현장심사 (2~3일)     │
   │                      │  · 데이터 관리 체계    │
   │                      │  · 데이터 값 품질      │
   │                      │                      │
   │                      ├─ 심사결과 보고 ───────→│
   │                      │                      ├─ 인증위원회 심의
   │  ←──────── 인증서 발급 (유효기간 2년, 갱신 가능) ──┘
```

- DQ 인증은 데이터 관리 체계(8대 영역)와 데이터 값 품질(6대 지표) 두 축으로 심사됨

### 2. 심사 항목 상세

1) 데이터 값 품질 6대 지표

| 품질 지표 | 정의 | 측정 방법 |
|-----------|------|---------|
| 정확성(Accuracy) | 실제 사실과 데이터가 일치하는 정도 | 오류 건수 / 전체 건수 |
| 완전성(Completeness) | 필수 항목에 값이 빠짐없이 채워진 정도 | NULL·공백 건수 / 전체 건수 |
| 유효성(Validity) | 형식·도메인·업무 규칙에 부합하는 정도 | 유효하지 않은 값 건수 / 전체 건수 |
| 일관성(Consistency) | 연관 데이터 간 논리적 모순이 없는 정도 | 불일치 건수 / 전체 비교 건수 |
| 유일성(Uniqueness) | 중복 없이 단 하나의 레코드로 존재 | 중복 건수 / 전체 건수 |
| 접근성(Accessibility) | 필요한 시점에 데이터를 이용 가능한 정도 | 서비스 가용률, 응답시간 |

- 6대 지표의 측정값이 인증 기준점(Pass/Fail 임계값) 이상이어야 인증 획득

2) 데이터 관리 체계 심사 8대 영역

```
┌─────────────────────────────────────────────────┐
│  1. 데이터 관리 조직       2. 데이터 관리 계획      │
│  3. 데이터 표준 관리       4. 데이터 모델 관리      │
│  5. 데이터 흐름 관리       6. 데이터 보안 관리      │
│  7. 데이터 품질 관리       8. 요구사항 관리         │
└─────────────────────────────────────────────────┘
```

- 8대 영역은 데이터를 다루는 조직의 프로세스·규정·도구 성숙도를 종합 평가

### 3. 유사 인증 제도 비교 및 트렌드

| 비교 항목 | DQ 인증 | ISMS-P 인증 | ISO 9001 |
|-----------|---------|-------------|----------|
| 대상 | 데이터 품질 | 정보보호·개인정보 | 품질경영시스템 |
| 인증 기관 | K-DATA | KISA | KAB 등 인증기관 |
| 법적 근거 | 데이터산업진흥법 | 정보통신망법·개인정보보호법 | ISO 국제표준 |
| 유효기간 | 2년 | 3년(갱신) | 3년(매년 사후심사) |
| 주요 심사 | 관리체계·값 품질 | 보안 관리체계·기술 통제 | 품질경영 프로세스 |
| 적용 분야 | 데이터 거래·유통 | 정보시스템 운영 기관 | 제조·서비스 전반 |

- 데이터 기본법 시행 이후 DQ 인증이 데이터 담보 금융·데이터 거래소 연계 요건으로 확대 적용 중.  "끝"

### 실무 제언

**DQ 인증 준비의 선순환 구조 구축**
- **챌린지**: 심사 전 단기 대량 정제는 근본 원인을 해결하지 않아, 인증 이후 품질이 재저하되는 '페인트 칠하기' 현상이 발생합니다. 레거시 시스템 배치 적재 과정에서 유효성·일관성 규칙 미적용으로 오염이 지속됩니다.
- **제언**: ETL/ELT 파이프라인에 DQ Rule을 임베드하여 원천 데이터 유입 시점부터 자동 검증하고, DQ 대시보드로 6대 지표를 상시 모니터링해야 합니다.

**데이터 관리 체계의 현실적 조직화**
- **챌린지**: 중소·중견 기업은 DA팀이 없어 '데이터 관리 조직' 심사 영역에서 감점이 빈번합니다.
- **제언**: 부서별 데이터 스튜어드(Data Steward) 역할 지정 및 데이터 관리 위원회(DGC) 구성으로 의사결정 구조를 문서화하는 것이 현실적 대안입니다.

**지속적 관리 문화 내재화**
- **챌린지**: DQ 인증을 취득 목적으로만 접근하면 인증 후 품질 관리 활동이 형식화되고, 실질적 데이터 품질 개선 효과가 사라집니다.
- **제언**: DQ 인증은 '결과'가 아닌 '지속적 관리 문화'의 인증임을 조직에 내재화하고, 품질 KPI를 부서 성과지표에 연계하여 데이터 품질 거버넌스를 제도화해야 합니다.
