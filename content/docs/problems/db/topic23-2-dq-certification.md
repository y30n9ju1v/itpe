## 해설

> **데이터 신뢰성의 공식 보증 제도**  
> 한국데이터산업진흥원(K-DATA)이 운영하는 데이터 품질 인증 체계

데이터 경제 시대에 데이터의 품질은 AI/분석 결과의 신뢰성을 좌우하는 핵심 자산입니다. DQ 인증은 기업·기관이 보유한 데이터의 품질 수준을 제3자 기관이 공식 검증하여 인증하는 제도로, 데이터 유통·거래 시장에서의 신뢰 기반을 형성합니다. 2022년 데이터 기본법 시행 이후 데이터 가치평가·거래 활성화와 함께 DQ 인증 수요가 급증하고 있습니다.

### 1. 출제 배경 및 의도

데이터 품질 인증(DQ 인증)은 데이터 기본법(2022) 및 데이터산업진흥법에 근거한 정부 정책과 직결되는 실무 토픽입니다. 데이터 거래소·데이터 마켓플레이스가 활성화되면서 데이터 품질의 객관적 검증 필요성이 높아졌으며, 공공 및 민간 기관 모두 DQ 인증 취득이 경쟁력의 지표가 되었습니다. 채점관은 인증 제도의 법적 근거, 심사 항목, 인증 절차를 체계적으로 서술할 수 있는지 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

DQ 인증(Data Quality Certification)은 조직이 관리하는 데이터의 품질 수준이 일정 기준에 부합함을 한국데이터산업진흥원(K-DATA)이 공식 확인하고 인증서를 부여하는 제도입니다.

- **Input:** 인증 신청 기관의 데이터 관리 체계(프로세스·조직·규정)와 데이터 값(실제 저장 데이터)
- **Mechanism:** 서류 심사 → 현장 심사(데이터 관리 체계 + 데이터 값 품질) → 인증위원회 심의 → 인증 부여
- **Output:** DQ 인증서(유효기간 2년) 및 품질 등급(우수·보통)

```text
[DQ 인증 프로세스]

신청기관              K-DATA 심사팀           인증위원회
   │                      │                      │
   ├─ 인증신청서 제출 ──→  │                      │
   │  (신청서+자체점검표)   │                      │
   │                      ├─ 서류심사(2주)         │
   │                      │  ∙ 관련 문서 검토      │
   │  ←── 보완요청 ────── │                      │
   │                      │                      │
   ├─ 현장심사 수감 ──────→│                      │
   │                      ├─ 현장심사(2~3일)      │
   │                      │  ∙ 데이터 관리 체계    │
   │                      │  ∙ 데이터 값 품질      │
   │                      │                      │
   │                      ├─ 심사결과 보고 ──────→│
   │                      │                      ├─ 인증위원회 심의
   │                      │                      │
   │  ←────── 인증서 발급 ──────────────────────── │
   │  (유효기간 2년, 갱신 가능)
```

### 3. 핵심 키워드 (Must-Have)

| 구분 | 핵심 키워드 | 설명 |
|------|------------|------|
| **법적 근거** | 데이터산업진흥법, 데이터 기본법 | 2022년 시행, DQ 인증의 제도적 기반 |
| **인증 기관** | 한국데이터산업진흥원(K-DATA) | 과학기술정보통신부 산하 전문 기관 |
| **심사 항목 1** | 데이터 관리 체계(Management) | 조직·프로세스·규정·도구 등 관리 인프라 |
| **심사 항목 2** | 데이터 값 품질(Value Quality) | 정확성·완전성·유효성·일관성·유일성·접근성 |
| **인증 유효기간** | 2년 (갱신 심사 가능) | 갱신 시 약식 심사 적용 |
| **품질 6대 지표** | 정확성/완전성/유효성/일관성/유일성/접근성 | DQ 인증 심사의 정량 기준 |

#### 데이터 값 품질 6대 지표 상세

| 품질 지표 | 정의 | 측정 방법 예시 |
|-----------|------|---------------|
| **정확성(Accuracy)** | 실제 세계의 사실과 데이터가 일치하는 정도 | 오류 데이터 건수 / 전체 데이터 건수 |
| **완전성(Completeness)** | 필수 항목에 값이 빠짐없이 채워진 정도 | NULL·공백 건수 / 전체 건수 |
| **유효성(Validity)** | 정해진 형식·도메인·업무 규칙에 부합하는 정도 | 유효하지 않은 값 건수 / 전체 건수 |
| **일관성(Consistency)** | 연관 데이터 간 논리적 모순이 없는 정도 | 불일치 건수 / 전체 비교 건수 |
| **유일성(Uniqueness)** | 중복 없이 단 하나의 레코드로 존재하는 정도 | 중복 건수 / 전체 건수 |
| **접근성(Accessibility)** | 필요한 시점에 데이터를 이용할 수 있는 정도 | 서비스 가용률, 응답시간 |

### 5. 실무 제언

**DQ 인증 준비 시 데이터 값 품질 개선의 선순환 구조 구축**

- **챌린지:** 인증 심사 전 단기간에 데이터 값을 대량 정제하는 방식은 근본 원인을 해결하지 않아, 인증 이후 품질이 다시 저하되는 '페인트 칠하기' 현상이 발생합니다. 특히 레거시 시스템의 배치 적재 과정에서 유효성·일관성 규칙 미적용으로 오염이 지속됩니다.
- **제언:** ETL/ELT 파이프라인에 데이터 품질 룰(DQ Rule)을 임베드하여 원천 데이터 유입 시점부터 자동 검증하고, DQ 대시보드를 통해 6대 지표를 상시 모니터링하는 체계를 구축해야 합니다. DQ 인증은 '결과'가 아닌 '지속적 관리 문화'의 인증임을 조직에 내재화하는 것이 핵심입니다.

**데이터 관리 체계의 조직화**

- **챌린지:** 중소·중견 기업의 경우 데이터 관리 전담 조직(DA팀)이 없어 심사 영역 중 '데이터 관리 조직' 항목에서 감점 요인이 발생하는 경우가 많습니다.
- **제언:** 데이터 스튜어드(Data Steward) 역할을 업무 부서별로 지정하고, 데이터 관리 위원회(Data Governance Committee)를 형식적으로라도 구성하여 의사결정 구조를 문서화하는 것이 현실적인 대안입니다.
