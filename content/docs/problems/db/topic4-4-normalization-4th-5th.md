## 해설

### 출제 배경 및 의도

BCNF까지의 정규화는 함수 종속성(FD) 기반으로 결정자가 후보키가 되도록 스키마를 분해하는 과정이지만, 현실의 복잡한 비즈니스 규칙은 이를 초월하는 고차 종속성과 관계 설계 오류를 내포합니다. 정보시스템 감리나 전사 데이터 거버넌스 정비 시, 테이블 간 관계 설정 오류로 인해 엉뚱한 데이터가 유도되는 연결함정과 다치·조인 종속성을 다루는 4NF·5NF의 메커니즘은 고급 데이터 아키텍처 역량을 검증하는 핵심 주제입니다.

채점관은 연결함정의 두 가지 유형(무부착 함정, 균열 함정)과 고차 정규화(4NF, 5NF)의 대상 종속성 및 분해 메커니즘을 명확히 구분하여 설명할 수 있는지를 평가합니다. 특히 4NF와 5NF의 분해 테이블 수(각각 2개, 3개 이상)의 차이가 핵심 변별 키워드입니다.

### 1. 연결함정과 고차 정규화, 개요

정 의  • BCNF를 만족하면서도 잔존하는 다치 종속(MVD)·조인 종속(JD) 및 관계 설계 오류(연결함정)를 제거하여 데이터의 논리적 완결성을 확보하는 고급 데이터 무결성 엔지니어링 활동
       - 연결함정은 조인 시 가짜 튜플 유도 또는 경로 단절, 4NF·5NF는 다차원 종속성 분해로 중복 제거

```
[고차 정규화 및 연결함정 관계 흐름]

 BCNF 달성 릴레이션
       │
       ├──[연결함정 검사]──> Fan Trap(가짜 튜플)   → 관계 수직 재배치
       │                     Chasm Trap(경로 단절) → 직접 관계 추가
       │
       ├──[4NF 적용]─────> 다치 종속(MVD) X ↠ Y   → 2개 테이블로 이원 분해
       │
       └──[5NF 적용]─────> 조인 종속(JD)          → 3개 이상 n-분해
                                                     (2개 분해 시 가짜 튜플 발생)
```

- 연결함정은 ERD 관계 설계 오류이고, 4NF·5NF는 함수 종속성 이상의 다차원 종속성 분해로서 접근 계층이 다르다.

### 2. 연결함정 2대 유형 및 고차 정규화 메커니즘

1) 연결함정(Connection Trap) 2대 유형

| 구분 | 항목 | 설명 |
|------|------|------|
| **무부착 함정** (Fan Trap) | 발생 구조 | 자식A (M) ← (1) 부모 (1) → (M) 자식B |
| | 현상 | 두 자식 간 직접 연결 소실 → 조인 시 가짜 튜플(Spurious Tuple) 대량 양산 |
| | 사례 | 교수가 여러 학생 지도 + 여러 학과 소속 → 특정 학생의 학과 추론 불가 |
| | 해결 | 학과(1)→교수(M)→학생(M) 인과 관계 순서로 수직 재배치 |
| **균열 함정** (Chasm Trap) | 발생 구조 | 엔티티A (1)→(M) 엔티티B (0..1)→(M) 엔티티C |
| | 현상 | 선택적 관계로 경로 단절 → 특정 데이터 간 연결 경로 유실(Lost Path) |
| | 사례 | 학과→교수(필수), 교수→학생(선택) 구조에서 미배정 교수의 학생 학과 추론 불가 |
| | 해결 | 학과와 학생 간 직접 관계(Direct Relationship) 신설 |

- 연결함정은 SQL 구문 오류 없이 정상 실행되므로 개발 단계 발견이 어렵고, 통계·정산 시스템에서 치명적 데이터 왜곡을 일으킨다.

2) 제4정규형(4NF)과 제5정규형(5NF) 메커니즘

```
[4NF: 다치 종속(MVD) 분해]          [5NF: 조인 종속(JD) 분해]

원시 릴레이션 (BCNF이나 중복 폭발)  원시 릴레이션 <공급망(SPC)>
과목명 | 교재명 | 취미               공급업자 | 부품명 | 기업명
데이터 | DB개론  | 등산               김기술  | 반도체  | S전자
데이터 | DB개론  | 독서               박공학  | 디스플레이| S전자
데이터 | SQL완성 | 등산  ← 중복       김기술  | 디스플레이| L디스플레이
데이터 | SQL완성 | 독서  폭발         김기술  | 디스플레이| S전자 ← 대칭 룰

과목명↠교재명, 과목명↠취미 (독립)   2개 분해 시 가짜 튜플 발생

  ▼ 2개 이원 분해(4NF)               ▼ 3개 n-분해(5NF)
<과목교재>(과목명, 교재명)           <공급부품(SP)>(공급업자, 부품명)
<과목취미>(과목명, 취미)             <부품기업(PC)>(부품명, 기업명)
                                     <공급기업(SC)>(공급업자, 기업명)
```

- 4NF는 독립적 다치 속성 2개를 이원 분해하고, 5NF는 순환 삼항 관계를 반드시 3개 이상 테이블로 n-분해해야 가짜 튜플 없는 무손실 복원이 가능하다.

### 3. 고차 정규화 단계 비교 및 트렌드

| 구분 | 대상 종속성 | 분해 테이블 수 | 핵심 현상 | 달성 목적 |
|------|-------------|----------------|-----------|-----------|
| **BCNF** | 함수 종속성 (FD) | 2개 | 후보키 아닌 결정자 | 기본키 패러다임 완성 |
| **4NF** | 다치 종속 (MVD) X↠Y | 2개 | 독립 다치 속성 조합 중복 폭발 | 무의미한 조합 중복 제거 |
| **5NF** | 조인 종속 (JD) | 3개 이상 (필수) | 2개 분해 시 가짜 튜플 발생 | 순환 관계 구조 무결성 확보 |
| **연결함정** | 관계 설계 오류 | ERD 재설계 | Fan Trap / Chasm Trap | 비즈니스 관계 왜곡 방지 |

- 실무 대규모 MSA·OLTP 환경에서 4~5NF를 기계적으로 적용하면 다중 조인 부하가 급증하므로, 논리 모델에서 무결성을 확보한 후 다치 속성 변경 빈도를 분석하여 JSON/Document 타입 통합 저장이나 교차 엔티티(Intersection Table) 설계 패턴으로 유연하게 대응해야 한다.  "끝"

### 실무 제언

**이론적 엄격성과 실무 효용성의 괴리**
- **챌린지**: 실무 프로젝트에서 4·5차 정규화까지 도달하는 경우는 드물며, 과도한 테이블 분해는 SQL 작성 난이도 급상승과 다중 조인 성능 저하를 유발한다.
- **제언**: ERD 설계 단계에서 연결함정을 조기 식별하여 비즈니스 관계 왜곡을 막고, 4~5NF는 교차 엔티티(Intersection Table) 패턴을 정착시키고 데이터 표준 가이드라인 도구로 가치 있게 활용해야 한다.

**다치 속성의 물리 설계 전략**
- **챌린지**: 다치 종속이 있는 영역을 4NF로 분해하면 변경이 적고 조회가 많은 경우 불필요한 조인 부하가 발생한다.
- **제언**: 다치 속성의 DML 빈도를 분석하여 변경이 거의 없는 영역은 PostgreSQL JSONB나 Document Model로 통합 임베딩하여 무결성과 고성능 조회를 동시에 달성하는 융합형 아키텍처를 적용해야 한다.

**MSA 환경에서의 연결함정 리스크 확대**
- **챌린지**: MSA 환경에서 DB-per-Service 분리로 관련 테이블이 파편화되면 연결함정이 서비스 간 API 다중 호출 시 데이터 정합성 파탄과 트래픽 병목을 유발한다.
- **제언**: DDD 적용 시 애그리거트(Aggregate) 경계를 명확히 하고, 삼항 이상의 복잡한 순환 관계는 교차 엔티티 패턴을 의무 적용하여 명확한 1:M 구조로 단순화해야 한다.
