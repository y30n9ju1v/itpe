## 해설

### 출제 배경 및 의도

데이터 기본법(2022) 시행으로 데이터가 자산으로 공식 인정되면서, "이 데이터의 가치는 얼마인가?"라는 질문에 객관적으로 답하는 가치평가 제도가 본격화되었습니다. 마치 부동산 감정평가처럼 데이터도 이제 담보 대출·M&A·거래 가격 산정에 활용할 수 있는 공인된 평가 방법론이 필요합니다.

데이터 거래소·마켓플레이스 활성화, 데이터 담보 금융상품 출시 등으로 데이터의 화폐가치 산정 필요성이 급증했습니다. 무형 자산인 데이터의 특수성(비소모성·비경합성·낮은 재생산 비용)으로 인해 전통적 유형자산 평가와는 다른 특화된 방법론이 필요합니다.

채점관은 가치평가 제도의 법적 근거와 3대 접근법(비용·시장·수익)을 구체적으로 서술하고, 각 접근법의 특성과 절차를 논리적으로 설명하는 능력을 평가합니다.

### 1. 데이터 가치평가(Data Value Assessment), 개요

정 의  • 데이터 자산이 보유한 경제적 가치를 화폐 단위로 측정·산정하는 체계적 절차로
       - 비용·시장·수익 3대 접근법을 활용하여 데이터의 화폐가치를 공식 산정하는 제도

```
[국내 데이터 가치평가 제도 체계]

  데이터 기본법(2022.4 시행)
       │
       ├── 과학기술정보통신부
       │    └── 데이터 가치평가 가이드라인 제정 (3대 접근법 표준화)
       │
       └── 한국데이터산업진흥원(K-DATA)
            ├── 데이터 가치평가 서비스 운영
            ├── 가치평가 전문기관 지정·관리
            └── 한국데이터거래소(KDX) 연계 평가 지원
```

- 데이터 기본법 제20조에 근거하며, 데이터 자산 인식 3요건(식별 가능성·통제·미래 경제적 효익)을 충족해야 가치평가 적용 가능

### 2. 데이터 가치평가 기법 및 절차

1) 가. 데이터 가치평가 제도 — 법적 근거 및 자산 인식 요건

| 법적 근거 | 내용 |
|---------|------|
| 데이터 기본법 제20조 | 데이터 가치평가 지원 근거 마련 |
| 데이터산업진흥법 | 데이터 거래·유통 활성화를 위한 평가 체계 |
| 기업 회계기준(K-IFRS) | 무형자산 인식 요건: 식별 가능성·통제·미래 경제적 효익 |

| 자산 인식 요건 | 내용 |
|------------|------|
| 식별 가능성 | 다른 자산과 분리 가능하고 계약·법적 권리에서 발생 |
| 통제 | 데이터의 사용을 통제하고 제3자의 접근 제한 가능 |
| 미래 경제적 효익 | 수익 창출·비용 절감 등의 경제적 효과 기대 가능 |

2) 나. 데이터 가치평가 3대 접근법

```
[데이터 가치평가 3대 접근법]

  ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
  │  비용 접근법    │   │  시장 접근법    │   │  수익 접근법    │
  │ (Cost Approach)│   │(Market Approach)│   │(Income Approach)│
  │                │   │                │   │                │
  │ "만드는 데     │   │ "유사 데이터가  │   │ "이 데이터로   │
  │  얼마가 들었나?"│   │  얼마에 거래?" │   │  얼마 벌 수?" │
  └────────────────┘   └────────────────┘   └────────────────┘
   데이터 희소·고유 시   거래 시장 활성화 시   미래 수익 예측 시
```

| 접근법 | 방식 | 장점 | 단점 | 적합 데이터 |
|-------|------|------|------|------------|
| 비용 접근법 | 재현/대체 비용 = 수집+정제+저장+관리 비용 | 객관성·명확성 | 미래 가치 과소평가 | 고유 내부 데이터 |
| 시장 접근법 | 유사 데이터 거래 가격 비교 + 조정 계수 | 시장 반영 | 비교 사례 부족 | 상용 데이터 상품 |
| 수익 접근법 | DCF / 로열티 면제법 / 초과 수익법 | 경제적 효익 직접 반영 | 불확실성·예측 오류 | 수익화 데이터 서비스 |

**수익 접근법 DCF 산정 공식:**

```
             n    FCFt
가치(V) =   Σ  ────────
            t=1  (1+r)^t

FCFt: t기의 데이터 활용 순현금흐름
r   : 할인율 (데이터 리스크 반영)
n   : 평가 기간 (데이터 수명)
```

3) 나. 데이터 가치평가 수행 절차

| 단계 | 주요 활동 | 산출물 |
|------|----------|--------|
| 1. 목적·범위 정의 | 거래·담보·회계 목적 명확화 | 평가 계획서 |
| 2. 대상 특성 분석 | 볼륨·품질·희소성·유효 기간 파악 | 데이터 특성 보고서 |
| 3. 접근법 선택 | 비용/시장/수익 중 택일 또는 복합 적용 | 평가 모델 설계서 |
| 4. 가치 산정 | 정량적 계산, 보정 계수 적용 | 가치 산정 결과 |
| 5. 검증·보고 | 전문가 검토, 최종 가치 확정 | 가치평가 보고서 |

### 3. 데이터 가치평가 비교 및 트렌드

| 비교 항목 | 비용 접근법 | 시장 접근법 | 수익 접근법 |
|---------|-----------|-----------|-----------|
| 산정 기준 | 구축 비용 | 시장 비교 가격 | 미래 창출 수익 |
| 주요 장점 | 객관성·명확성 | 시장 반영 | 경제적 효익 직접 반영 |
| 주요 단점 | 가치 과소평가 가능 | 비교 사례 부족 | 불확실성·예측 오류 |
| 데이터 요건 | 구축 비용 기록 | 유사 거래 사례 | 수익 예측 모델 |

- 데이터 담보 금융·DQ 인증 연계·AI 학습 데이터 거래 확대로 데이터 가치평가가 기업 재무 전략의 핵심 요소로 제도화 중.  "끝"

### 실무 제언

**복합 접근법으로 가치 산정 신뢰성 확보**
- **챌린지**: 단일 접근법으로는 데이터의 다면적 가치를 온전히 반영하기 어렵습니다. 비용 접근법은 미래 가치를 과소평가하고, 시장 접근법은 국내 데이터 거래 시장 미성숙으로 비교 사례가 부족합니다.
- **제언**: 평가 목적(거래·담보·회계)에 따라 주 접근법을 선정하고, 나머지 접근법으로 합리성을 교차 검증하는 복합 접근법(Ensemble) 방식을 적용해야 합니다.

**데이터 자산 회계 처리 준비**
- **챌린지**: 현행 K-IFRS에서 내부 창출 무형자산은 식별 가능성·통제·미래 효익 요건을 모두 충족해야 하며, 데이터는 이 요건 입증이 어려운 경우가 많습니다.
- **제언**: 데이터 자산 회계 인식을 위해 데이터 라이선스·소유권 계약을 명확히 하고, 수익화 트랙 레코드를 축적하며 데이터 자산 목록(Data Asset Register)을 체계적으로 관리해야 합니다.

**데이터 자산 목록 및 품질 연계 관리**
- **챌린지**: 가치평가 대상 데이터의 품질·최신성·접근성이 보장되지 않으면 평가 후 실제 활용 가치가 급감하여 담보·거래 과정에서 분쟁이 발생합니다.
- **제언**: DQ 인증 기준(6대 품질 지표)과 데이터 가치평가를 연계하여, 품질이 보증된 데이터만 가치평가 대상으로 등록하고 정기적인 품질 재평가 체계를 구축해야 합니다.
