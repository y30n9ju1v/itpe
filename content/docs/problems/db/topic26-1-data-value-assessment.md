## 해설

> **데이터를 돈으로 환산하는 공식 제도**  
> 데이터 가치평가 3대 접근법(비용·시장·수익)과 국내 제도(과기부·K-DATA)

데이터 기본법(2022) 시행으로 데이터가 자산으로 공식 인정되면서, "이 데이터의 가치는 얼마인가?"라는 질문에 객관적으로 답하는 가치평가 제도가 본격화되었습니다. 데이터 담보 대출, M&A 시 데이터 자산 평가, 데이터 거래 가격 산정 — 모두 표준화된 가치평가 방법론이 필요합니다.

### 1. 출제 배경 및 의도

데이터 가치평가는 데이터 기본법(2022)과 과학기술정보통신부의 데이터 가치평가 가이드라인에 근거한 정책 중심 토픽입니다. 데이터 거래소·데이터 마켓플레이스의 활성화, 데이터 담보 금융상품 출시 등으로 데이터의 화폐가치 산정 필요성이 증가하고 있습니다. 채점관은 가치평가 제도의 법적 근거와 3대 접근법(비용·시장·수익)을 구체적으로 서술하고, 각 접근법의 특성과 절차를 논리적으로 설명하는 능력을 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

데이터 가치평가(Data Value Assessment)는 데이터 자산이 보유한 경제적 가치를 화폐 단위로 측정·산정하는 체계적 절차입니다. 무형 자산인 데이터의 특수성(비소모성·비경합성·재생산 비용 낮음)으로 인해 전통적 유형자산 평가와는 다른 특화된 방법론이 필요합니다.

- **Input:** 평가 대상 데이터(볼륨·품질·희소성·활용 가능성)
- **Mechanism:** 평가 목적 정의 → 접근법 선택 → 데이터 특성 분석 → 가치 산정 → 검증
- **Output:** 화폐 단위의 데이터 자산 가치 (원화/달러)

       └── 한국데이터산업진흥원(K-DATA)

            ├── 가치평가 전문기관 지정·관리

### 나. 데이터 가치평가 기법 및 절차

#### 데이터 가치평가 3대 접근법

```text
[데이터 가치평가 3대 접근법]

  ┌────────────────┐   ┌────────────────┐   ┌────────────────┐
  │  비용 접근법    │   │  시장 접근법    │   │  수익 접근법    │
  │ (Cost Approach)│   │(Market Approach)│   │(Income Approach)│
  │                │   │                │   │                │
  │ "이 데이터를   │   │ "유사 데이터가  │   │ "이 데이터로   │
  │  만드는 데     │   │  시장에서       │   │  얼마의 수익을  │
  │  얼마가 들었나?"│   │  얼마에 거래    │   │  낼 수 있나?"  │
  │                │   │  되었나?"       │   │                │
  └────────────────┘   └────────────────┘   └────────────────┘
         ↑                    ↑                    ↑
    데이터 희소 시          거래 시장 활성 시       미래 수익 예측 가능 시
    활용에 적합             활용에 적합             활용에 적합
```

#### 1. 비용 접근법 (Cost Approach)

데이터를 구축하거나 대체하는 데 소요된 비용을 기준으로 가치를 산정합니다.

| 방식 | 내용 |
|------|------|
| **재현 비용(Reproduction Cost)** | 현재 시점에 동일한 데이터를 다시 구축하는 데 드는 비용 |
| **대체 비용(Replacement Cost)** | 유사한 기능의 데이터로 대체하는 데 드는 비용 |
| **비용 구성 요소** | 수집 비용 + 정제 비용 + 저장 비용 + 관리 비용 + 기회비용 |

**적합 시나리오:** 데이터 구축 비용이 명확하고, 시장 거래 사례가 없는 고유한 내부 데이터

#### 2. 시장 접근법 (Market Approach)

유사한 데이터의 시장 거래 가격을 참조하여 비교 평가합니다.

| 방식 | 내용 |
|------|------|
| **비교 거래법** | 유사 데이터의 실제 거래 가격 비교 |
| **조정 요소** | 데이터 볼륨, 품질, 희소성, 독점성, 갱신 주기 등 |
| **데이터 거래소** | 한국데이터거래소(KDX), 민간 데이터 마켓플레이스 거래 사례 활용 |

**적합 시나리오:** 데이터 거래 시장이 활성화되어 비교 가능한 거래 사례가 존재하는 경우

#### 3. 수익 접근법 (Income Approach)

데이터가 미래에 창출할 것으로 예상되는 경제적 효익을 현재 가치로 환산합니다.

| 방식 | 내용 |
|------|------|
| **DCF(현금흐름할인법)** | 미래 예상 현금흐름 × 할인율로 현재 가치 산정 |
| **로열티 면제법** | 유사 데이터의 시장 로열티율 × 예상 매출로 산정 |
| **초과 수익법** | 데이터 활용으로 발생하는 초과 수익의 현재 가치 |

**산정 공식 (DCF 기준):**

```text
                  n    FCFt
데이터 가치(V) =  Σ  ────────
                 t=1  (1+r)^t

FCFt: t기의 데이터 활용 순현금흐름
r   : 할인율 (데이터 리스크 반영)
n   : 평가 기간 (데이터 수명)
```

**적합 시나리오:** 데이터 비즈니스 모델이 명확하고 수익 예측이 가능한 경우

| **3대 접근법** | 비용 접근법, 시장 접근법, 수익 접근법(DCF/로열티면제법) |

| **핵심 공식** | DCF(현금흐름할인법), 로열티 면제법, 재현/대체 비용 |

### 4. 맥락과 비교

| 비교 항목 | 비용 접근법 | 시장 접근법 | 수익 접근법 |
|---------|-----------|-----------|-----------|
| **산정 기준** | 구축 비용 | 시장 비교 가격 | 미래 창출 수익 |
| **주요 장점** | 객관성·명확성 | 시장 반영 | 경제적 효익 직접 반영 |
| **주요 단점** | 가치 과소평가 가능 | 비교 사례 부족 | 불확실성·예측 오류 |
| **데이터 요건** | 구축 비용 기록 | 유사 거래 사례 | 수익 예측 모델 |
| **적합 데이터** | 내부 고유 데이터 | 상용 데이터 상품 | 수익화 데이터 서비스 |

### 5. 실무 제언

**데이터 가치평가의 현실적 한계와 복합 접근법 활용**

- **챌린지:** 단일 접근법으로는 데이터의 다면적 가치를 온전히 반영하기 어렵습니다. 비용 접근법은 미래 가치를 과소평가하고, 시장 접근법은 국내 데이터 거래 시장 미성숙으로 비교 사례가 부족하며, 수익 접근법은 불확실한 미래 예측에 의존합니다.
- **제언:** 실무에서는 두 가지 이상의 접근법을 병행(복합 접근법)하여 결과를 교차 검증하는 방식이 권장됩니다. 평가 목적(거래·담보·회계)에 따라 주 접근법을 선정하고, 나머지 접근법으로 합리성을 검증하는 앙상블(Ensemble) 방식을 적용하면 가치 산정의 신뢰성을 높일 수 있습니다.

**데이터 자산의 회계 처리 준비**

- **챌린지:** 현행 회계기준(K-IFRS)에서 내부 창출 무형자산은 엄격한 인식 요건(식별 가능성·통제·미래 효익)을 충족해야 하며, 데이터는 이 요건 입증이 어려운 경우가 많습니다.
- **제언:** 데이터 자산의 회계 인식을 위해 데이터 라이선스·소유권 계약을 명확히 하고, 데이터 활용을 통한 수익 창출 증거(수익화 트랙 레코드)를 축적해야 합니다. 중장기적으로는 데이터 자산 전용 회계 기준 개발(IASB 논의 중)에 대비하여 데이터 자산 목록(Data Asset Register)을 체계적으로 관리하는 것이 핵심입니다.
