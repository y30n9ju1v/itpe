## 문제

데이터 통합 및 마이그레이션 프로젝트에서 데이터 무결성 목표를 달성하기 위해서는 데이터들의 정합성을 확보하고 신뢰도를 높이는 일이 매우 중요하다. 다음의 내용을 설명하시오.

가. 데이터 무결성(Integrity)과 정합성(Consistency)의 차이  
나. 데이터 값(Value) 진단 프로파일링(Data Profiling)의 중점 분석 관점  
다. 데이터 마이그레이션 검증 테스트 방법

## 해설

> **데이터 이전의 신뢰성 확보 3대 축**  
> 무결성(Integrity) vs 정합성(Consistency), 값 진단 프로파일링, 마이그레이션 검증 테스트

클라우드 전환, 시스템 통합, 레거시 현대화 프로젝트에서 데이터 마이그레이션은 가장 리스크가 높은 단계입니다. 마이그레이션 후 데이터 오류는 운영 중단·업무 오류로 직결되며, 사후 복구 비용은 사전 검증 비용의 10배 이상에 달합니다. 따라서 무결성·정합성의 개념적 구분과 체계적인 검증 방법론이 필수입니다.

### 1. 출제 배경 및 의도

128회 정보처리기술사 시험에서 출제된 이 문제는 데이터 품질 관리와 마이그레이션 엔지니어링이 결합된 실무 중심 토픽입니다. 클라우드 DB 마이그레이션(Oracle → Aurora, MS-SQL → PostgreSQL), 빅데이터 플랫폼 구축 시 데이터 레이크 적재 등이 활발히 진행되면서, 마이그레이션 품질 검증 역량이 핵심 기술사 역량으로 부각되었습니다. 채점관은 무결성과 정합성의 개념 차이를 명확히 구분하고, 프로파일링 관점과 검증 방법을 구체적으로 서술하는 능력을 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

데이터 마이그레이션 품질 확보는 원천 데이터 진단(프로파일링) → 마이그레이션 실행 → 목적지 검증(테스트)의 3단계 사이클로 구성됩니다.

```text
[데이터 마이그레이션 품질 확보 흐름]

원천 시스템                 마이그레이션                목적지 시스템
    │                          │                            │
    ├─ 데이터 프로파일링 ──────→│                            │
    │  (값 진단, 이상 탐지)      │                            │
    │                          │                            │
    │  ←── 품질 규칙 정의 ────── │                            │
    │                          │                            │
    ├─ 추출(Extract) ──────────→│                            │
    │                          ├─ 변환(Transform) ──────────│
    │                          │  (매핑, 정제, 변환)         │
    │                          ├─ 적재(Load) ───────────────→│
    │                          │                            │
    │                          │          ├─ 검증 테스트 ───│
    │                          │          │  (Count/Hash/   │
    │                          │          │   Sampling)     │
    │  ←──────────── 결과 비교 ─────────────────────────────│
```

### 가. 데이터 무결성(Integrity)과 정합성(Consistency)의 차이

| 항목 | 데이터 무결성(Integrity) | 데이터 정합성(Consistency) |
|------|-------------------------|--------------------------|
| **정의** | 데이터가 정해진 규칙과 제약 조건을 위반하지 않는 상태 | 연관된 데이터들이 서로 논리적으로 모순되지 않는 상태 |
| **관점** | 개별 데이터 항목의 규칙 준수 여부 | 복수 데이터 간의 관계·일관성 |
| **위반 사례** | PK 중복, NOT NULL 위반, FK 불일치, 도메인 위반 | 주문 금액 = 단가 × 수량 불일치, 주소 코드 ≠ 실제 주소 |
| **DB 관점** | 엔티티 무결성, 참조 무결성, 도메인 무결성 | 트랜잭션 ACID의 Consistency, 교차 테이블 일관성 |
| **마이그레이션 시** | 원천 DB 제약 위반 데이터 탐지 | 원천-목적지 간 집계 값, 연관 데이터 일치 여부 |

```text
[무결성 vs 정합성 개념 비교]

무결성(Integrity)              정합성(Consistency)
    │                              │
    ├─ 엔티티 무결성                ├─ 수평 정합성
    │  (PK 유일성·NOT NULL)        │  (동일 테이블 내 논리 규칙)
    │                              │
    ├─ 참조 무결성                  ├─ 수직 정합성
    │  (FK → PK 참조 보장)         │  (상세-요약 테이블 간 합계 일치)
    │                              │
    └─ 도메인 무결성                └─ 시스템 간 정합성
       (값 범위·형식 준수)             (원천 ↔ 목적지 데이터 일치)
```

### 나. 데이터 값(Value) 진단 프로파일링의 중점 분석 관점

데이터 프로파일링(Data Profiling)은 데이터 마이그레이션 전 원천 데이터의 실제 상태를 통계적으로 분석하여 품질 이슈를 사전 탐지하는 활동입니다.

| 분석 관점 | 세부 내용 | 발견 이슈 유형 |
|---------|----------|--------------|
| **컬럼 통계 분석** | 최솟값·최댓값·평균·분포·NULL 비율 측정 | 이상값(Outlier), 과도한 NULL, 편향 분포 |
| **패턴 분석** | 전화번호·이메일·날짜 등 형식 패턴 검사 | 형식 불일치, 다중 형식 혼재 |
| **중복 분석** | 동일·유사 레코드 탐지(퍼지 매칭 포함) | 중복 고객, 중복 제품 코드 |
| **참조 무결성 분석** | FK 컬럼이 PK 테이블에 존재하는지 검사 | 고아 레코드(Orphan Record) |
| **도메인 분석** | 허용 가능한 값 목록(코드 도메인) 준수 여부 | 미정의 코드값, 폐기된 코드 사용 |
| **관계 분석** | 테이블 간 업무 규칙(예: 주문 → 고객 필수) 준수 여부 | 논리적 정합성 위반 |

### 다. 데이터 마이그레이션 검증 테스트 방법

```text
[마이그레이션 검증 3단계 방법론]

 1단계: 전수 카운트 검증         2단계: 해시 비교 검증         3단계: 샘플링 검증
 ─────────────────────         ──────────────────────        ─────────────────
 원천 건수 == 목적지 건수?        컬럼별 체크섬 비교            랜덤/계층 샘플 추출
 테이블별 ROW COUNT 비교         (MD5/SHA256 해시)             육안 및 자동화 비교
 집계 합계 비교(SUM, COUNT)      대량 데이터 고속 검증 가능     업무 규칙 검증 포함
```

| 검증 방법 | 설명 | 장점 | 단점 |
|---------|------|------|------|
| **레코드 수 검증(Count Test)** | 원천·목적지 테이블별 ROW COUNT 비교 | 전수 검증, 누락 탐지 | 값 오류는 탐지 불가 |
| **해시 비교(Hash Comparison)** | 행·컬럼 단위 MD5/SHA256 체크섬 비교 | 대량 데이터 고속 검증 | 원본 데이터 변환 시 해시 불일치 허용 필요 |
| **샘플링 검증(Sampling Test)** | 통계적 샘플링으로 실제 값 비교 | 상세 값 검증, 업무 규칙 확인 | 전수 검증 아님, 오류 누락 가능성 |
| **집계 검증(Aggregation Test)** | SUM·AVG·MAX·MIN 비교 | 수치 정합성 검증에 효과적 | 집계에 사용되지 않는 필드 검증 불가 |
| **참조 무결성 검증** | FK 제약 위반 건수 확인 | 관계 정합성 확인 | 목적지 DB 제약 설정 필요 |

### 3. 핵심 키워드 (Must-Have)

- **무결성 유형:** 엔티티 무결성, 참조 무결성, 도메인 무결성, 사용자 정의 무결성
- **정합성 유형:** 수평 정합성, 수직 정합성, 시스템 간 정합성
- **프로파일링 관점:** 컬럼 통계, 패턴 분석, 중복 분석, 참조 무결성, 도메인 분석, 관계 분석
- **검증 방법:** 레코드 수 검증, 해시 비교, 샘플링 검증, 집계 검증, 참조 무결성 검증
- **핵심 개념:** 고아 레코드(Orphan Record), 골든 레코드, 데이터 프로파일링, ETL

### 4. 맥락과 비교

| 비교 항목 | 데이터 마이그레이션 | 데이터 통합(Integration) | 데이터 동기화(Synchronization) |
|---------|----------------|------------------------|-------------------------------|
| **시점** | 일회성 이전 | 지속적 또는 일회성 | 지속적·실시간 |
| **방향** | 단방향 | 단방향/양방향 | 양방향 |
| **주요 관심사** | 완전성·정확성 보장 | 변환 규칙·통합 로직 | 지연시간·충돌 해결 |
| **검증 방법** | Count/Hash/Sampling | 데이터 품질 모니터링 | CDC 기반 변경 추적 |

### 5. 실무 제언

**마이그레이션 전 데이터 정제 vs 마이그레이션 후 정제 전략**

- **챌린지:** 원천 데이터에 품질 이슈가 많을 경우, 마이그레이션 전 정제를 완료하면 일정이 지연되고, 마이그레이션 후 정제를 선택하면 오염된 데이터가 신규 시스템에 그대로 유입되는 딜레마가 발생합니다.
- **제언:** '비즈니스 크리티컬' 데이터(고객 식별자, 계약 정보 등)는 마이그레이션 전 필수 정제하고, '비크리티컬' 데이터는 마이그레이션 후 점진적 정제 로드맵을 수립하는 이원화 전략이 효과적입니다. 검증 자동화 스크립트를 CI/CD 파이프라인에 통합하여 전체 마이그레이션 단계마다 검증 게이트(Quality Gate)를 운영해야 합니다.
