## 해설

> **데이터 품질의 국제 공통 언어**  
> ISO 8000 시리즈 — 마스터 데이터 품질관리와 프로세스 참조 모델

데이터 글로벌화가 가속되면서 서로 다른 국가·조직 간의 데이터 교환 시 품질 기준의 불일치로 인한 오류가 심각한 비즈니스 리스크로 부각되었습니다. ISO 8000은 이러한 문제를 해결하기 위한 데이터 품질 국제 표준으로, 특히 마스터 데이터(Master Data) 관리와 데이터 품질관리 프로세스 참조 모델을 핵심으로 합니다.

### 1. 출제 배경 및 의도

ISO 8000은 국내 DQ 인증의 이론적 기반이자 글로벌 데이터 품질 관리의 지침서로 활용됩니다. 데이터 기반 의사결정(DDDM)과 AI 학습 데이터 품질 문제가 부각되면서 국제 표준 기반의 품질관리 체계 수립이 요구되고 있습니다. 채점관은 ISO 8000 시리즈의 구조를 알고, 프로세스 참조 모델과 마스터 데이터 품질관리 프레임워크를 논리적으로 설명할 수 있는지 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

ISO 8000은 데이터 및 정보 품질에 관한 국제 표준 시리즈로, 데이터의 교환·포터블리티(Portability)·마스터 데이터 품질관리를 체계적으로 정의합니다. 국제표준화기구(ISO)가 제정하였으며, 한국은 KS X ISO 8000으로 채택하였습니다.

- **Input:** 조직이 생성·수집한 원시 데이터(Raw Data)와 데이터 관리 프로세스
- **Mechanism:** 데이터 품질 요구사항 정의 → 데이터 품질 측정 → 데이터 품질 개선 → 지속적 모니터링(PDCA 사이클)
- **Output:** 표준화된 고품질 데이터(마스터 데이터 포함)와 성숙도 수준 달성

#### ISO 8000 시리즈 구조

```text
[ISO 8000 시리즈 전체 구조]

ISO 8000
 ├── Part 1~10    : 개요 및 기본 개념
 │    ├── Part 2  : 용어(Vocabulary)
 │    └── Part 8  : 데이터 품질 개념(핵심)
 │
 ├── Part 60~69   : 데이터 품질 관리(Management)
 │    └── Part 61 : 데이터 품질관리 프로세스 참조 모델
 │
 ├── Part 100~119 : 마스터 데이터(Master Data)
 │    └── Part 110: 마스터 데이터 교환
 │
 └── Part 150~    : 데이터 품질 성숙도 모델(DQMM)
      └── Part 150: 조직 데이터 품질 성숙도
```

### 가. 데이터 품질관리 프로세스 참조 모델 (ISO 8000-61)

ISO 8000-61은 조직의 데이터 품질관리 활동을 표준화된 프로세스 모델로 정의합니다.

```text
[ISO 8000-61 프로세스 참조 모델]

  ┌─────────────────────────────────────────────────────────┐
  │               데이터 품질관리 프로세스                    │
  │                                                         │
  │  1. 데이터 품질    2. 데이터 품질    3. 데이터 품질      │
  │     요구사항 정의     측정·분석        개선               │
  │     (Plan)           (Check)         (Do/Act)           │
  │                                                         │
  │  4. 데이터 품질 모니터링 및 통제 (Control)               │
  │                                                         │
  │  지원 프로세스: 품질 기록 관리, 역량 관리, 도구 관리      │
  └─────────────────────────────────────────────────────────┘
```

| 프로세스 | 활동 내용 | 주요 산출물 |
|---------|----------|------------|
| **1. 요구사항 정의** | 비즈니스 목적에 맞는 데이터 품질 요구사항 수집·정의 | 품질 요구사항 명세서 |
| **2. 측정·분석** | 데이터 프로파일링, 품질 지표 측정, 원인 분석 | 품질 측정 결과 보고서 |
| **3. 개선** | 오류 데이터 정제, 근본 원인 제거, 예방 조치 | 정제 데이터, 개선 계획 |
| **4. 모니터링·통제** | 지속적 품질 수준 감시, 임계치 초과 시 알람 | 품질 대시보드, KPI 보고서 |

### 나. 마스터 데이터 품질관리 프레임워크 (ISO 8000-150)

마스터 데이터(Master Data)란 조직의 핵심 비즈니스 엔티티(고객·제품·공급업체·직원 등)를 표현하는 공유 참조 데이터입니다.

```text
[마스터 데이터 품질관리 프레임워크]

  비즈니스 컨텍스트
       │
       ▼
  ┌─────────────────────────────────────┐
  │     마스터 데이터 요구사항 정의       │
  │  (식별자 체계, 속성, 관계 정의)       │
  └──────────────┬──────────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────────┐
  │     마스터 데이터 품질 측정          │
  │  (6대 품질 지표 기준 측정)           │
  └──────────────┬──────────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────────┐
  │     마스터 데이터 정제·통합          │
  │  (중복 제거, 골든 레코드 생성)        │
  └──────────────┬──────────────────────┘
                 │
                 ▼
  ┌─────────────────────────────────────┐
  │     마스터 데이터 배포·동기화        │
  │  (MDM 허브를 통한 전사 배포)         │
  └─────────────────────────────────────┘
```

#### 마스터 데이터 품질 핵심 개념

| 개념 | 설명 |
|------|------|
| **골든 레코드(Golden Record)** | 중복 마스터 데이터를 통합한 단일 신뢰 원본(Single Source of Truth) |
| **MDM(Master Data Management)** | 마스터 데이터의 생성·관리·배포를 통합 관리하는 시스템·프로세스 |
| **데이터 스튜어드십(Stewardship)** | 마스터 데이터의 품질 책임자를 지정하여 관리하는 거버넌스 체계 |
| **데이터 포터빌리티(Portability)** | ISO 8000의 핵심 가치 — 시스템 간 데이터 교환 시 품질 손실 방지 |

### 3. 핵심 키워드 (Must-Have)

| 구분 | 키워드 |
|------|--------|
| **표준 시리즈** | ISO 8000-2(용어), ISO 8000-8(개념), ISO 8000-61(프로세스 참조 모델), ISO 8000-110(마스터 데이터), ISO 8000-150(성숙도) |
| **마스터 데이터** | 골든 레코드, MDM, 마스터 데이터 통합, 데이터 스튜어드 |
| **품질 프로세스** | 요구사항 정의, 측정·분석, 개선, 모니터링·통제 |
| **성숙도(DQMM)** | 인식(Aware) → 반응(Reactive) → 능동(Proactive) → 관리(Managed) → 최적화(Optimized) |
| **핵심 원칙** | 데이터 포터빌리티, 구문 적합성(Syntactic Conformance), 의미 적합성(Semantic Conformance) |

### 4. 맥락과 비교

| 비교 항목 | ISO 8000 | DAMA-DMBOK | CMMI-Data |
|-----------|----------|------------|-----------|
| **성격** | 국제 표준(규범) | 업계 가이드(지식체계) | 성숙도 모델 |
| **초점** | 데이터 품질·마스터 데이터 | 데이터 관리 전 영역 | 프로세스 역량 개선 |
| **인증** | 있음(DQ 인증 기반) | 없음(CDMP 자격증) | 있음(CMMI 레벨) |
| **국내 적용** | DQ 인증과 연계 | DA 자격시험 참조 | 일부 SI 기업 |

### 5. 실무 제언

**마스터 데이터 품질 확보 — 골든 레코드 전략**

- **챌린지:** 조직 내 여러 시스템(ERP, CRM, SCM)에 동일 고객·제품 정보가 각기 다른 형식으로 분산 저장되어 있어, 단일 신뢰 원본(SSOT) 확보가 어렵습니다. 특히 기업 인수합병(M&A) 후 이기종 시스템 통합 시 마스터 데이터 충돌이 심각해집니다.
- **제언:** MDM 허브 아키텍처를 도입하여 소스 시스템의 마스터 데이터를 중앙에서 통합하고, 매칭·병합 규칙(Matching/Merge Rule)을 ISO 8000-110 기준으로 정의해야 합니다. 데이터 스튜어드를 지정하여 골든 레코드의 생성·검증·배포 프로세스를 거버넌스화하는 것이 핵심입니다.
