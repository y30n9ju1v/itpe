## 해설

### 출제 배경 및 의도

데이터 글로벌화가 가속되면서 국가·조직 간 데이터 교환 시 품질 기준의 불일치로 인한 오류가 심각한 비즈니스 리스크로 부각되었습니다. ISO 8000은 이러한 문제를 해결하기 위한 데이터 품질 국제 표준으로, 마스터 데이터 관리와 품질관리 프로세스 참조 모델을 핵심으로 합니다.

ISO 8000은 국내 DQ 인증의 이론적 기반이자 글로벌 데이터 품질 관리의 지침서로 활용됩니다. 데이터 기반 의사결정(DDDM)과 AI 학습 데이터 품질 문제가 부각되면서 국제 표준 기반의 품질관리 체계 수립이 요구되고 있습니다.

채점관은 ISO 8000 시리즈 구조를 알고, 프로세스 참조 모델(ISO 8000-61)과 마스터 데이터 품질관리 프레임워크(ISO 8000-150)를 논리적으로 설명할 수 있는지 평가합니다.

### 1. ISO 8000, 개요

정 의  • 데이터·정보 품질에 관한 국제 표준 시리즈로 데이터 교환·포터빌리티·
       - 마스터 데이터 품질관리를 체계적으로 정의하는 ISO 제정 표준(KS X ISO 8000 국내 채택)

```
[ISO 8000 시리즈 전체 구조]

ISO 8000
 ├── Part 1~10    : 개요 및 기본 개념
 │    ├── Part 2  : 용어(Vocabulary)
 │    └── Part 8  : 데이터 품질 개념(핵심)
 │
 ├── Part 60~69   : 데이터 품질 관리(Management)
 │    └── Part 61 : 데이터 품질관리 프로세스 참조 모델
 │
 ├── Part 100~119 : 마스터 데이터(Master Data)
 │    └── Part 110: 마스터 데이터 교환
 │
 └── Part 150~    : 데이터 품질 성숙도 모델(DQMM)
      └── Part 150: 조직 데이터 품질 성숙도
```

- ISO 8000은 구문 적합성(Syntactic Conformance)과 의미 적합성(Semantic Conformance)을 데이터 품질의 핵심 원칙으로 정의

### 2. 핵심 내용 — 프로세스 참조 모델 및 마스터 데이터 프레임워크

1) 가. 데이터 품질관리 프로세스 참조 모델 (ISO 8000-61)

| 프로세스 | 활동 내용 | 주요 산출물 |
|---------|----------|------------|
| 1. 요구사항 정의(Plan) | 비즈니스 목적에 맞는 품질 요구사항 수집·정의 | 품질 요구사항 명세서 |
| 2. 측정·분석(Check) | 데이터 프로파일링, 품질 지표 측정, 원인 분석 | 품질 측정 결과 보고서 |
| 3. 개선(Do/Act) | 오류 데이터 정제, 근본 원인 제거, 예방 조치 | 정제 데이터, 개선 계획 |
| 4. 모니터링·통제(Control) | 지속적 품질 감시, 임계치 초과 시 알람 | 품질 대시보드, KPI 보고서 |

- ISO 8000-61은 PDCA 사이클 기반으로 데이터 품질관리를 지속적 개선 프로세스로 정의

2) 나. 마스터 데이터 품질관리 프레임워크 (ISO 8000-150)

```
[마스터 데이터 품질관리 프레임워크]

  비즈니스 컨텍스트
       │
       ▼
  ┌────────────────────────────────────┐
  │  마스터 데이터 요구사항 정의         │
  │  (식별자 체계, 속성, 관계 정의)      │
  └────────────────┬───────────────────┘
                   │
                   ▼
  ┌────────────────────────────────────┐
  │  마스터 데이터 품질 측정            │
  │  (6대 품질 지표 기준 측정)          │
  └────────────────┬───────────────────┘
                   │
                   ▼
  ┌────────────────────────────────────┐
  │  마스터 데이터 정제·통합            │
  │  (중복 제거, 골든 레코드 생성)       │
  └────────────────┬───────────────────┘
                   │
                   ▼
  ┌────────────────────────────────────┐
  │  마스터 데이터 배포·동기화          │
  │  (MDM 허브를 통한 전사 배포)        │
  └────────────────────────────────────┘
```

| 핵심 개념 | 설명 |
|---------|------|
| 골든 레코드(Golden Record) | 중복 마스터 데이터를 통합한 단일 신뢰 원본(SSOT) |
| MDM(Master Data Management) | 마스터 데이터의 생성·관리·배포를 통합 관리하는 시스템·프로세스 |
| 데이터 스튜어드십(Stewardship) | 마스터 데이터 품질 책임자를 지정하여 관리하는 거버넌스 체계 |
| 데이터 포터빌리티(Portability) | ISO 8000 핵심 가치 — 시스템 간 데이터 교환 시 품질 손실 방지 |

### 3. 유사 표준 비교 및 트렌드

| 비교 항목 | ISO 8000 | DAMA-DMBOK | CMMI-Data |
|-----------|----------|------------|-----------|
| 성격 | 국제 표준(규범) | 업계 가이드(지식체계) | 성숙도 모델 |
| 초점 | 데이터 품질·마스터 데이터 | 데이터 관리 전 영역 | 프로세스 역량 개선 |
| 인증 | 있음(DQ 인증 기반) | 없음(CDMP 자격증) | 있음(CMMI 레벨) |
| 국내 적용 | DQ 인증과 연계 | DA 자격시험 참조 | 일부 SI 기업 |

- ISO 8000은 국내 DQ 인증 심사 기준의 이론적 토대이며, AI 학습 데이터 품질 표준화 논의에서도 참조 표준으로 채택 중.  "끝"

### 실무 제언

**마스터 데이터 골든 레코드 전략**
- **챌린지**: 조직 내 ERP·CRM·SCM에 동일 고객·제품 정보가 각기 다른 형식으로 분산 저장되어 단일 신뢰 원본(SSOT) 확보가 어렵습니다. 특히 M&A 후 이기종 시스템 통합 시 마스터 데이터 충돌이 심각해집니다.
- **제언**: MDM 허브 아키텍처를 도입하여 소스 시스템의 마스터 데이터를 중앙에서 통합하고, 매칭·병합 규칙을 ISO 8000-110 기준으로 정의해야 합니다.

**ISO 8000 기반 데이터 품질 성숙도 향상**
- **챌린지**: 데이터 품질관리 성숙도가 낮은 조직은 품질 문제가 발생해야 반응하는 사후(Reactive) 수준에 머물러 예방적 품질 관리가 이루어지지 않습니다.
- **제언**: ISO 8000-150의 성숙도 모델(인식→반응→능동→관리→최적화) 기준으로 현재 수준을 진단하고, 단계별 개선 로드맵을 수립하여 능동(Proactive) 수준 이상으로 향상시켜야 합니다.

**데이터 포터빌리티 확보**
- **챌린지**: 시스템 간 데이터 교환 시 코드 체계·날짜 형식·단위 불일치로 품질 손실이 발생하며, 글로벌 협업 시 국가별 표준 차이로 데이터 오류가 증가합니다.
- **제언**: 데이터 교환 인터페이스 설계 시 ISO 8000-110 기반의 구문·의미 적합성 검증 규칙을 반드시 포함하고, 전사 데이터 표준(코드·형식·단위)을 ISO 8000 원칙에 맞게 정비해야 합니다.
