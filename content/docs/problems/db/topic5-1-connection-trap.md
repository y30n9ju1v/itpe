## 해설

> **데이터 모델링의 숨겨진 논리적 결함**  
> 연결함정(Connection Trap): 관계 왜곡이 만드는 가짜 정보와 경로 단절

데이터 모델링 단계에서 엔티티 간의 관계를 잘못 파악하거나 생략하면, 조인(Join) 시 원래 존재하지 않는 가짜 정보(Spurious Tuple)가 나타나거나 관계 경로가 단절되는 구조적 모순이 발생합니다. 이를 연결함정(Connection Trap)이라 하며, SQL 구문 오류 없이 쿼리가 정상 실행되므로 개발 단계에서 발견하기 매우 어렵습니다. 컴파일 에러 없이 통계·정산 시스템에서 치명적인 데이터 왜곡을 일으킨다는 점에서 가장 위험한 설계 오류 중 하나입니다.

### 1. 출제 배경 및 의도

연결함정은 전사 데이터 표준화 사업이나 차세대 시스템 고도화 시, 관계의 모호성으로 인해 엉뚱한 데이터가 조인되어 가짜 정보가 추출되는 현상이 빈번하게 발생한다는 점에서 중요합니다. 채점관은 수험생이 단순히 ERD 가독성만 보지 않고, 관계의 카디널리티와 선택성(Optionality)을 공학적으로 검증하여 데이터 모델의 정합성과 정보 보존 능력을 진단하는 역량을 갖추었는지 확인하고자 합니다.

### 2. 개념의 본질과 작동 메커니즘

연결함정은 주로 3개 이상의 엔티티가 다차원적인 비즈니스 규칙을 형성할 때 나타납니다. 다차원 업무 도메인을 단순한 1:M 선형 구조로 엮을 때 발생하며, 공통 부모를 가진 자식 간의 관계 왜곡(무부착 함정, Fan Trap)이나 선택적 관계선 배치 오류에 따른 경로 미아 현상(균열 함정, Chasm Trap)으로 구체화됩니다.

**무부착 함정(Fan Trap):** 하나의 부모 엔티티가 두 개의 자식 엔티티와 각각 1:M 관계를 가질 때, 두 자식 엔티티 간의 직접 연결 고리가 소실되어 조인 시 존재하지 않는 가짜 튜플(Spurious Tuple)이 대량 양산됩니다.

```text
[Fan Trap 발생 구조]
 ┌───────────┐ 1       M ┌───────────┐ M       1 ┌───────────┐
 │   소 속   ├───────────┤   교 수   ├───────────┤   학 생   │
 └───────────┘           └───────────┘           └───────────┘

[Fan Trap 해결: 인과 관계에 따라 수직 재배치]
 ┌───────────┐ 1       M ┌───────────┐ 1       M ┌───────────┐
 │   소 속   ├───────────┤   교 수   ├───────────┤   학 생   │
 └───────────┘           └───────────┘           └───────────┘
```

**균열 함정(Chasm Trap):** 엔티티 간 경로상에 선택적(Optional, 0..1) 관계가 존재하여 특정 데이터 간의 연결 경로가 완전히 끊어지는 현상입니다.

```text
[Chasm Trap 발생 구조]
 ┌───────────┐ 1       M ┌───────────┐ 0..1    M ┌───────────┐
 │   학 과   ├───────────┤   교 수   ├──  ──  ──┤   학 생   │
 └───────────┘           └───────────┘ (선택적)  └───────────┘

[Chasm Trap 해결: 직접 관계선 추가]
     ┌──────────────────────────────────────────┐
     │                (직접 관계 추가)          │
     ▼ 1                                      M ▲
 ┌───────────┐ 1       M ┌───────────┐ 0..1    M ┌───────────┐
 │   학 과   ├───────────┤   교 수   ├──  ──  ──┤   학 생   │
 └───────────┘           └───────────┘           └───────────┘
```

### 3. 핵심 키워드 (Must-Have)

- **연결함정 종류:** 무부착 함정(Fan Trap), 균열 함정(Chasm Trap)
- **유발 현상:** 가짜 튜플(Spurious Tuple), 미아 경로(Lost Path), 추론 오류, 관계 단절
- **대응 기법:** 관계 재배치(Re-ordering), 직접 관계 추가(Direct Relationship), 필수(Mandatory) 제약 조건, 교차 엔티티(Intersection Entity)

### 4. 맥락과 비교

연결함정은 고차 정규화 및 관계 대수와 직접 연계되며, 두 유형은 상반된 조치 전술을 가집니다.

| 비교 항목 | 무부착 함정 (Fan Trap) | 균열 함정 (Chasm Trap) |
| --- | --- | --- |
| **발생 원인** | 한 부모 엔티티에 2개의 자식 엔티티가 1:M 관계로 결합 | 엔티티 간 경로상에 선택적(Optional) 관계가 존재하여 단절 |
| **핵심 현상** | 조인 시 가짜 데이터가 대량 양산됨 (과도한 관계 성립) | 특정 데이터 간의 연결 경로가 상실됨 (미아 경로 발생) |
| **구조적 형상** | `자식 A (M) <- (1) 부모 (1) -> (M) 자식 B` | `엔티티 A (1) -> (M) 엔티티 B (0..1) -> (M) 엔티티 C` |
| **해결 전략** | 관계의 순서를 인과 관계에 따라 수직 재배치 | 단절을 유발하는 두 엔티티 간 직접 관계선 추가 |

### 5. 실무 제언

**눈에 보이지 않는 위험성 — 연결함정 진단 체계 확립**

- **챌린지:** 연결함정은 컴파일 에러나 SQL 구문 오류를 발생시키지 않습니다. 쿼리는 정상 작동하여 결과 집합을 반환하기 때문에 개발자가 초기에 인지하기 어렵고, 오픈 후 통계·정산 시스템에서 치명적인 데이터 왜곡을 일으킵니다.
- **제언:** 기술사는 ERD 가독성만 봐서는 안 되며, 'CRUD 매트릭스'와 '비즈니스 시나리오 시뮬레이션'을 정기적으로 가동해야 합니다. 특히 다대다(M:N) 관계를 단순 1:M으로 치환할 때 연결함정이 숨어들기 쉬우므로, 개념 설계 단계부터 교차 엔티티를 명확히 설계하는 거버넌스가 정착되어야 합니다.

**MSA 환경에서의 연결함정 리스크 확대**

- **챌린지:** 마이크로서비스 아키텍처(MSA) 환경에서는 서비스별로 DB가 분리(Database-per-Service)되므로 관련 테이블들이 서로 다른 네트워크 노드에 파편화됩니다. 이 상태에서 연결함정이 내포되면 서비스 간 API 다중 호출(Chaining) 시 데이터 정합성 파탄과 대규모 트래픽 병목을 유발합니다.
- **제언:** 프로젝트 초기 도메인 주도 설계(DDD) 적용 시 애그리거트(Aggregate) 경계를 명확히 하고, 삼항 이상의 복잡한 순환 관계는 교차 엔티티(Intersection Entity) 패턴을 의무 적용하여 명확한 1:M 구조로 단순화해야 합니다.
