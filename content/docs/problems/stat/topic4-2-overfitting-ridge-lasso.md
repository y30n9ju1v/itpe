## 해설

### 1. 다중회귀 분석 시 발생할 수 있는 문제점의 개요

```
[다중회귀분석 문제점]    →   [해결방안]    →  [Regularization]
  ① 다중공선성              정규화  ──L2──→  Ridge Regression
  ② 오버피팅               데이터            L1──→  Lasso Regression
                           추가확보
                           드롭아웃
```

- 다중회귀 분석 진행 시, 독립변수의 수가 많을수록 High variance로 모델이 학습되어 Train data에 Overfitting 문제 발생

### 2. 다중회귀 분석에서의 오버피팅(Overfitting) 문제 설명

1) 오버피팅 개념

```
훈련데이터에만 과하게 학습        → 훈련 데이터에서는 높은 정확도
                                    테스트 데이터나 실제 적용시 성능 저하 문제
편향 분산 딜레마:
  High Variance / Low Bias
  특성: High Feature

가장 학습이 잘된 모델은 에러가 가장 적은 모델
에러가 가장 적은 모델은 편향과 분산이 모두 낮은 모델
```

2) 오버피팅 문제 판별 및 해결 방안

| 구분 | 해결방안 | 설명 |
|------|---------|------|
| 판별 | 학습 시 | 에러가 기대하는 에러율보다 낮게 나타나는 경우 |
| 판별 | 테스트 시 | 에러가 기대하는 에러율보다 높게 나타나는 경우 |
| 해결방안 | 학습 데이터 | 충분한 데이터 학습 |
| 해결방안 | 조기 종료(Early Stopping) | 검증단계에서 검증 데이터를 통해 검증 정확도에 큰 변화가 없을 때 반복 학습 종료 |
| 해결방안 | 드롭아웃 | 학습 시 히든레이어의 몇 개의 노드를 랜덤하게 사용안함 |
| 해결방안 | 정규화 | Ridge Regression, Lasso Regression |

- 특성에 대한 파라미터 값(가중치)의 값을 줄이는 정규화를 통해 오버피팅 문제 해결

### 3. Ridge Regression과 Lasso Regression 설명

1) Ridge Regression (L2 정규화)

정 의  • 회귀 계수 β의 제곱합에 패널티를 부여하여 계수를 0에 가깝지만 0이 되지는 않도록 축소

수식: 비용함수 = RSS + λΣβⱼ²  (L2 Norm: 가중치의 제곱합)

| 항목 | 설명 |
|------|------|
| 특성 | Non sparse model: 모든 특성 변수를 유지하되 계수만 작게 만듦 |
| 장점 | 다중공선성이 있는 경우 계수 안정화에 효과적 |
| 단점 | 변수 선택 불가(계수가 0이 되지 않음) |
| 적용 | 모든 특성이 유의미할 것으로 예상될 때 |

2) Lasso Regression (L1 정규화)

정 의  • 회귀 계수 β의 절댓값의 합에 패널티를 부여하여 일부 계수를 정확히 0으로 만들어 변수 선택 가능

수식: 비용함수 = RSS + λΣ|βⱼ|  (L1 Norm: 가중치의 절댓값 합)

| 항목 | 설명 |
|------|------|
| 특성 | Sparse model: 일부 특성 변수의 계수를 정확히 0으로 만들어 변수 선택 |
| 장점 | 자동 변수 선택 가능, 해석이 단순해짐 |
| 단점 | 상관관계 높은 변수 중 임의로 하나만 선택 |
| 적용 | 불필요한 특성이 많을 것으로 예상될 때 |

3) Ridge vs Lasso 비교

| 구분 | Ridge (L2) | Lasso (L1) |
|------|-----------|-----------|
| 패널티 항 | λΣβⱼ² | λΣ\|βⱼ\| |
| 계수 축소 방식 | 0에 가깝게 축소 (0이 되지 않음) | 일부 계수를 정확히 0으로 만듦 |
| 모델 유형 | Non-sparse model | Sparse model |
| 변수 선택 | 불가 | 가능 |
| 적합한 상황 | 다중공선성 존재 시 | 불필요한 변수 많을 때 |

- 최근 AI/ML 모델의 일반화(Generalization) 성능 향상을 위해 Ridge·Lasso 정규화 기법 활용 증대.  "끝"

### 실무 제언

**λ(람다) 값 튜닝의 중요성**

- **챌린지**: Ridge와 Lasso 모두 패널티 강도를 조절하는 λ 값이 핵심입니다. λ=0이면 일반 회귀분석과 동일하고, λ가 너무 크면 모든 계수가 0에 수렴하는 언더피팅(Underfitting)이 발생합니다.
- **제언**: 교차검증(Cross-Validation)을 통해 최적 λ를 탐색하는 것이 실무 핵심입니다. 그리드 서치(Grid Search) 또는 랜덤 서치(Random Search)와 K-Fold CV를 결합하여 최적 하이퍼파라미터를 선정해야 합니다.
