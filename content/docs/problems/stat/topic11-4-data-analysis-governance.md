## 문제

단순히 대용량 데이터를 수집·축적하는 것보다는 어떤 목적으로 어떤 데이터를 어떻게 분석해 활용할 것인가가 더욱 중요하기 때문에 데이터가 강조될수록 데이터 분석과 활용에 대한 체계적인 관리를 위한 분석 거버넌스의 중요성 또한 올라간다. 데이터 분석 거버넌스에 대하여 다음을 설명하시오.

가. 데이터 분석 거버넌스 체계
나. 데이터 분석 준비도와 성숙도 모델

## 해설

### 출제 배경 및 의도

데이터 경제 시대에 단순 데이터 수집을 넘어 체계적 분석·활용 관리의 중요성이 높아짐에 따라, 분석 거버넌스 체계의 구성 요소(CoA 중심)와 조직의 분석 역량을 진단하는 준비도·성숙도 모델의 이해 여부를 확인하기 위해 출제

### 1. 전사 데이터 분석 관리 체계, 데이터 거버넌스 개요

정 의  • 전사 차원의 모든 데이터에 대한 정책·지침·표준화·전략을 수립하고, 데이터를 관리하는 조직과 프로세스를 구축함으로써 고품질 데이터를 활용하여 기업의 가치 창출을 지원하는 체계
       - 데이터 품질 보장, 프라이버시 보호, 데이터 수명관리, 소유권 및 관리권 명확화, 데이터 적시 공급 등을 핵심 요구사항으로 함

```
[데이터 거버넌스 기대효과]

  가용성(Availability)   유용성(Usability)   통합성(Integration)
         ↘                    ↓                  ↙
              ┌─────────────────────────────┐
              │     데이터 거버넌스 체계      │
              └─────────────────────────────┘
         ↗                    ↑                  ↖
  보완성(Security)       안정성(Stability)
```

### 2. 데이터 분석 거버넌스 체계 (CoA 중심)

```
                    ┌──────────────────────────┐
                    │   CoA (Center of         │
                    │      Analytics)          │
                    └──────────┬───────────────┘
          ┌───────────┬────────┴────────┬───────────────┐
          ▼           ▼                 ▼               ▼
    ┌───────────┐ ┌─────────┐   ┌──────────────┐ ┌──────────┐
    │ 분석      │ │ 분석    │   │ 분석수준     │ │ 분석     │
    │ 프로세스  │ │ 전문인력│   │ 진단         │ │ 교육     │
    └───────────┘ └─────────┘   └──────────────┘ └──────────┘
          ▲
    ┌───────────┐
    │ 분석 조직 │
    └───────────┘
```

| 구성요소 | 역할 | 내용 |
|----------|------|------|
| 분석 프로세스 | 분석 업무의 체계적 수행 | 요건정의 → 모델링 → 검증 및 테스트 → 적용 (EDA/CDA 방법론 기반) |
| 분석 전문 인력 | 분석 실행 주체 | 기초통계학 및 분석 방법에 대한 지식과 경험을 갖춘 데이터 사이언티스트 |
| 분석 조직 | 분석 방향 설정 및 총괄 | 데이터 분석 가치 발견·과제 정의, 인사이트 실행 (데이터 분석 컨트롤 타워) |
| 분석수준 진단 | 현재 역량 평가 | 분석 준비도 및 분석 성숙도 점검 |
| 분석 교육 | 역량 내재화 | 데이터 훈련 프로그램 운영, 비즈니스 전체 분야에 분석 문화 확산 |

### 3. 데이터 분석 준비도와 성숙도 모델

#### 가. 분석 준비도(Readiness) — 6가지 기준

| 기준 | 점검 항목 |
|------|----------|
| ① 분석 업무 파악 | 예측 분석 및 시뮬레이션 분석 수행 여부 |
| ② 인력 및 조직 | 전사 분석 업무 총괄 조직 구성 여부 |
| ③ 분석 기법 | 업무별 적합 분석 기법 사용, 기법 평가 및 개선 활동 |
| ④ 분석 데이터 | 충분성·신뢰성·적시성 확보 여부 |
| ⑤ 분석 문화 | 데이터 공유 및 협업 문화 형성 수준 |
| ⑥ IT 인프라 | EAI·ETL 등 데이터 유통 체계 구비 여부 |

#### 나. 분석 성숙도(Maturity) — 4단계

| 단계 | 명칭 | 내용 |
|------|------|------|
| ① 도입단계 | 분석 시작 | 분석을 시작해 환경과 시스템을 구축하는 초기 단계 |
| ② 활용단계 | 결과 적용 | 분석 결과를 실제 업무에 적용하여 가치를 창출하는 단계 |
| ③ 확산단계 | 전사 확산 | 전사 차원에서 분석을 관리하고 공유하는 단계 |
| ④ 최적화단계 | 혁신·성과 향상 | 분석을 통한 지속적 혁신 및 성과 향상을 추구하는 단계 |

### 4. 준비도·성숙도 2×2 매트릭스 — 4가지 유형 진단

```
성숙도
  높음│ 2) 정착형           │ 4) 확산형
      │ 준비도 낮으나 조직/ │ 빅데이터 환경 갖추고
      │ 인력/기법 제한적    │ 전사 차원 분석 관리·
      │ 사용. 1차 정착 필요 │ 공유 가능
      ├────────────────────┼────────────────────
  낮음│ 1) 준비형           │ 3) 도입형
      │ 데이터/인력/기법    │ 분석업무·기법 부족하나
      │ 미적용 상태         │ 적용 조직 준비도 높아
      │ 사전 준비 필요      │ 바로 도입 가능
      └────────────────────┴────────────────────
             낮음                    높음
                       준비도
```

| 유형 | 특징 | 목표 방향 |
|------|------|----------|
| 준비형 | 데이터·인력·기법 미적용 | 분석 기반 환경·인력 확보 우선 |
| 정착형 | 준비도 낮으나 조직·기법 제한적 활용 | 분석 1차 정착 후 준비도 향상 |
| 도입형 | 준비도 높으나 분석 업무·기법 부족 | 분석 기법·업무 모델 즉시 도입 |
| 확산형 | 준비도·성숙도 모두 높음 | 전사 확산 및 지속적 최적화 추구 |

끝

### 실무 제언

분석 거버넌스 체계에서 CoA는 단순 분석 조직이 아닌 비즈니스 전략과 데이터 분석을 연결하는 컨트롤 타워 역할을 수행한다. 준비도·성숙도 진단을 통해 현재 위치(4개 유형 중 하나)를 파악하고, 단계별 목표 방향(인프라 구축 → 업무 적용 → 전사 확산 → 혁신)을 수립함으로써 데이터 분석 역량의 체계적 고도화가 가능하다.
