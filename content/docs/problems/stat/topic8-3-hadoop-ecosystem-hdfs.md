### 3. 하둡 에코시스템과 HDFS NameNode/DataNode 쓰기·읽기 과정

**가. 하둡 에코시스템 구성**

| 구성요소 | 설명 |
|----------|------|
| Hcatalog | 메타데이터와 테이블 관리 지원 |
| Pig | 하둡 대용량 데이터 처리를 위한 고급언어이자 처리엔진 |
| Hive | 하둡 동작 데이터 웨어하우스 인프라 구조 |
| Zookeeper | 분산 시스템 환경 프로젝트 |
| Ambari | 하둡 클러스터 프로비저닝, 관리 및 모니터링 |
| Sqoop | 대용량 벌크 데이터 전송 |
| Avro | 데이터의 재구성 |

**나. HDFS 쓰기 과정**

```
[Client] ──1.Write Request──→ [Name Node]
[Client] ←──2.Response──────  [Name Node]
[Client] ──3.Write Data──→ [Data Node 1]
               [Data Node 1] ──4.Write Replica──→ [Data Node 2]
                              [Data Node 2] ──4.Write Replica──→ [Data Node 3]
[Client] ←──5.Write Done──── [Data Node 1]
```

| 단계 | 설명 |
|------|------|
| 1 | NameNode에 파일 쓰기 요청 |
| 2 | NameNode는 메모리에 파일 경로 생성, 저장 DataNode 목록 전송 |
| 3 | 첫 번째 DataNode 쓰기 수행 |
| 4 | 첫 번째부터 세 번째 DataNode까지 순차 쓰기 |
| 5 | 복제 완료 시 Client 작업 완료 안내 |

**다. HDFS 읽기 과정**

| 단계 | 설명 |
|------|------|
| 1 | NameNode에 파일 읽기 요청 |
| 2 | NameNode는 Client와 가장 가까운 DataNode 위치 파악 후 목록 전송 |
| 3 | DataNode 파일 블록 정보 요청 |
| 4 | DataNode는 요청 받은 블록 전송 |
| 5 | Client 조회 |

### 4. 빅데이터 시스템 구축 시 실무측면의 고려사항

| 측면 | 고려사항 |
|------|----------|
| 프로젝트 관리 측면 | 비즈니스 사용자들과 업임 선도 기술 연구 전행, 프로젝트 외부 Usecase 개발, 연계와의 도입 이후 Gap 등을 도입 제품 테스트 환경 구축 |
| 적용 기술 측면 | 빅데이터 플랫폼 무부 파악, 빅데이터 플랫폼 선정, 분석 기능과 기술관리, 처리시간과 실시간 관리, DW와의 통합 여부 |
| 분석 측면 | 인 메모리 분석의 고려, 하둡에 데이터 처리 방식, 데이터 전처리 방식, 빅데이터 탐색과 인사이트 도출, 온/오프라인의 모델 적용 |
| 수행 조직 및 예산 측면 | 전문 아키텍트·전사조직 운영, 빅데이터 전문가 개발자 적절한 예산, 총운영비용(CAPEX, OPEX) 등 |
