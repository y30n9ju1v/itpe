## 해설

### 출제 배경 및 의도

빅데이터·AI 시대에 데이터 분석은 단순 집계를 넘어 예측과 처방으로 진화하고 있습니다. 채점관은 CDA/EDA의 차이를 명확히 이해하고, 가트너의 분석 성숙도 모델을 통해 분석 고도화 방향을 설명할 수 있는 능력을 평가합니다. 또한 데이터 분석의 표준적 5단계 절차를 체계적으로 서술하는 능력을 요구합니다.

### 1. 가설 기반 vs. 데이터 기반, CDA와 EDA 비교

1) 개념 정의

| 구분 | 개념 |
|------|------|
| 확증적 분석(CDA, Confirmatory Data Analysis) | 가설을 설정한 후 수집 데이터로 가설 평가·추정하는 목적 지향적 분석 방법 |
| 탐색적 분석(EDA, Exploratory Data Analysis) | 데이터에 대한 그래프·시각화 분석을 통해 근본 원인을 규명하는 귀납적 분석 방법 |

2) CDA와 EDA 상세 비교

| 구분 | 확증적 분석 (CDA) | 탐색적 분석 (EDA) |
|------|-----------------|-----------------|
| 절차 | 가설검정→데이터수집→통계분석→가설검증 | 데이터수집→시각화탐색→패턴도출→인사이트 발견 |
| 방향성 | 연역적 (Deductive) | 귀납적 (Inductive) |
| 투입자료 | 상관관계 행렬 | 공분산 행렬 |
| 요인구조 | 특정한 요인구조의 확증 | 새로운 요인구조의 발견 |
| 모형식별 | 모형의 식별문제 발생 없음 | 모형 식별을 위한 요인 분산 재제약 필요 |
| 요인간 상관관계 | 직각 회전의 경우 요인간 관계없음 | 요인들 간의 상관관계 가정 |
| 주요 도구 | 통계 검정(Z, t, X², F) | 히스토그램, 박스플롯, 산점도 |
| 활용 사례 | 임상시험, A/B 테스트, 품질 검정 | 신규 데이터셋 탐색, 이상치 탐지 |

- CDA는 가설 기반 연역적 접근, EDA는 데이터 기반 귀납적 접근으로 상호보완적 관계.

### 2. 가트너 데이터 분석 성숙도 모델 (4단계)

```
가치↑
  │                                      [처방적분석]
  │                         [예측적분석]  ← 어떻게 할 것인가? (최적화)
  │              [진단적분석]  ← 어떻게 될 것인가?
  │   [설명적분석]  ← 원인은 무엇인가?
  │ ← 무엇이 발생하였는가?
  └──────────────────────────────────────────────────→ 난이도↑
     (낮음)   (낮음)    (보통)    (높음)    (매우높음)
```

1) 4가지 분석 유형 상세

| 유형 | 핵심 질문 | 설명 | 특징 |
|------|----------|------|------|
| 설명적 분석 (Descriptive Analytics) | 무엇이 발생하였는가? | 과거/현재 발생사실을 다양한 기준으로 서술 | 확보된 데이터에 대한 다양한 관점 분석 |
| 진단적 분석 (Diagnostic Analytics) | 원인은 무엇인가? | 인과관계·상관관계를 파악하여 원인 규명 | 문제의 근본 원인(Root Cause) 분석 |
| 예측적 분석 (Predictive Analytics) | 어떻게 될 것인가? | 미래의 불확실한 사실을 사전 예측 | 패턴 인식, 확률 모형, ML 활용 |
| 처방적 분석 (Prescriptive Analytics) | 무엇을 할 것인가? | 분석을 넘어 대안 도출과 의사결정 실행 | To-Do 제시, 최적화·시뮬레이션 |

- 가트너 모델에서 설명적→처방적으로 갈수록 분석 난이도와 비즈니스 가치 모두 증가.

### 3. 데이터 분석 처리를 위한 5단계 절차

```
[문제 정의] → [데이터 수집] → [데이터 전처리] → [데이터 모델링] → [탐색 및 시각화]
  (What)        (Collect)       (Cleanse)          (Model)            (Insight)
```

1) 5단계 상세

| 절차 | 설명 | 주요 활동 | 특징 |
|------|------|----------|------|
| 1. 문제 정의 | 분석하고자 하는 분야 이해, 해결할 문제를 객관적으로 정의 | 분석 목적 설정, KPI 정의 | 분석·처리를 위한 핵심 요소 |
| 2. 데이터 수집 | 분석에 필요한 데이터 요건 정의 후 데이터 확보 | 내부 DB, 외부 API, 크롤링 | 일관성, 명료성, 정확성 등 필요 |
| 3. 데이터 전처리 | 수집한 데이터에 존재하는 결측값·오류를 수정·보완 | 결측치 처리, 이상치 탐지, 정규화 | 분석·처리 목적에 맞게 처리 |
| 4. 데이터 모델링 | 다수의 테이블을 사용하는 분석에서 효율적 데이터 분석 | 회귀, 분류, 클러스터링, 시계열 | 데이터 나누기, 분석 데이터 결정 |
| 5. 탐색 및 시각화 | 도구를 이용하여 시각화하고, 탐색을 통해 문제를 해결 | R, Python, SPSS, Tableau 활용 | 인사이트 도출, 의사결정 지원 |

2) 전처리 주요 기법

| 기법 | 설 명 |
|------|-------|
| 결측치 처리 | 평균·중앙값 대체, 행 삭제, 예측 모델로 대체 |
| 이상치 탐지 | IQR(사분위수 범위), Z-score 기반 이상치 제거 |
| 정규화·표준화 | Min-Max 정규화, Z-score 표준화로 스케일 통일 |
| 차원 축소 | PCA(주성분분석)로 고차원 데이터 압축 |

- 최근 데이터 기반 AI 의사결정에서 설명적→처방적 분석으로 고도화 증대. "끝"

### 실무 제언

**데이터 전처리의 중요성 (Garbage In, Garbage Out)**

- **챌린지**: 데이터 분석 프로젝트 실패의 80% 이상은 데이터 품질 문제에서 비롯됩니다. 아무리 정교한 모델을 사용해도 입력 데이터에 결측치·오류·편향이 있으면 분석 결과를 신뢰할 수 없습니다.
- **제언**: 전처리 단계에 전체 분석 시간의 60~70%를 투자하는 것이 현업의 관행입니다. 특히 EDA를 통해 데이터 분포, 이상치, 상관관계를 충분히 탐색한 후 CDA로 가설을 검증하는 순서를 지키는 것이 분석 품질을 높이는 핵심입니다.
