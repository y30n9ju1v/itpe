### 1. 두 제품 또는 사건 사이의 연관성 발견을 위한 연관규칙 개요

**가. 연관규칙(Association Rules) 개념**

| 구분 | 내용 |
|------|------|
| 정 의 | 특정 아이템 집합에서 의미있는 연관성이 존재하며, 해당 연관성을 일반화하고 정량적으로 표현하는 작업 |
| 목 적 | 대량의 데이터로부터 품목간의 어떠한 종속 관계가 존재하는지를 찾아내는 작업 |

**나. 연관규칙 장점 및 단점**

| 구분 | 항목 | 설명 |
|------|------|------|
| 장점 | 탐색적 기법 | 조건 반응(if-then)으로 표현되는 연관 분석의 결과를 이해하기 쉬움 |
| | 강력한 비목적성 분석 기법 | 분석 방향이나 목적이 특별하게 없는 경우 목적 변수가 없으므로 유용 |
| | 사용이 편리한 분석 데이터의 형태 | 거래 내용에 대한 데이터를 변환 없이 그 자체로 이용할 수 있는 간단한 자료구조를 갖는 분석방법 |
| | 계산의 용이성 | 분석을 위한 계산이 상당히 간단함 |
| 단점 | 상당한 수의 계산과정 | 품목수가 증가하면 분석에 필요한 계산은 기하급수적으로 늘어남 |
| | 적절한 품목의 결정 | 너무 세분화된 품목을 가지고 연관규칙을 찾으려고 하면 의미 없는 분석 결과가 나올 수 있음 |
| | 품목의 비율차이 | 상대적으로 거래량이 적은 품목은 당연히 포함된 거래수가 적을 것이고, 규칙발견시 제외되기 쉬움 |

### 2. 연관규칙의 평가척도, 지지도·신뢰도·향상도

**가. 지지도(Support)**

| 구분 | 내용 |
|------|------|
| 개념 | 전체 거래 중에서 품목 X, Y가 동시에 포함되는 거래의 비율 |
| 수식 | S = (X와 Y가 동시에 포함된 거래수 / 전체거래수) = P(X∩Y) |
| 사례 | 전체 거래 4건 중 기저귀(X)와 맥주(Y)를 동시에 구입한 거래는 2건, S = P(X∩Y) = 2/4 = 0.5 |

**나. 신뢰도(Confidence)**

| 구분 | 내용 |
|------|------|
| 개념 | 항목 X를 포함하는 거래 중에서 항목 Y가 포함될 확률, 조건부 확률로 계산 |
| 수식 | C = (X와 Y가 동시에 포함된 거래수 / X를 포함하는 거래 수) = P(X∩Y)/P(X) |
| 사례 | 기저귀(X)만을 포함하는 거래는 3건이면 P(X) = 3/4 = 0.75 이고, C = P(X∩Y)/P(X) = 0.5/0.75 = 0.67 |

**다. 향상도(Lift)**

| 구분 | 내용 |
|------|------|
| 개념 | 항목 X를 구매한 경우 그 거래가 항목 Y를 포함하는 경우와 항목 Y가 임의로 구매되는 경우의 비율 |
| 수식 | L = P(Y\|X)/P(Y) = P(X∩Y)/P(X)P(Y) |
| 분석 | 향상도(L) = 1: 독립 사건 / 향상도(L) > 1: 양의 연관관계(연관성 높음) / 향상도(L) < 1: 음의 연관관계(연관성 낮음) |
| 사례 | 맥주(Y)만을 포함하는 거래가 2건이라면 P(Y) = 2/4 = 0.5이고, L = P(X∩Y)/P(X)P(Y) = 0.5/(0.75 × 0.5) = 1.33 → 양의 상관관계를 가짐으로 맥주와 기저귀 구매 연관성 높음 |

### 3. 연관분석을 위한 Apriori 알고리즘

**가. Apriori 알고리즘 개념**

- 데이터베이스에서 후보 항목집합(Candidate Itemset)을 생성하고, 이를 데이터베이스 트랜잭션과 비교하여 후보 항목 집합들의 발생 빈도를 계산하고, 사용자가 정의한 최소지지도를 기준으로 빈발 항목집합(Large Itemset)을 결정하는 알고리즘

**나. 선험적 규칙**

| 구분 | 내용 | 예시 |
|------|------|------|
| 규칙1 | 한 항목집합이 빈발 하다면, 이 항목집합의 모든 부분집합은 빈발항목 집합 | 모든 항목집합 {a,b,c,d}, 빈발항목집합 {b,c,d}라면, 이 집합의 부분집합 {b,c},{b,d},{c,d},{b},{c},{d}는 빈발항목 집합 |
| 규칙2 | 한 항목집합이 비 빈발 하다면, 이 항목집합을 포함하는 모든 집합은 비 빈발항목 집합 | — |
| Join | 빈발항목 집합을 찾기 위해 후보집합을 생성하는 연산 | — |
| Prune | 비 빈발항목에 따라서 후보집합에서의 후보를 제외하는 연산 | — |

**다. Apriori 알고리즘 연관규칙 탐색 과정**

| 단계 | 과정 | 설명 |
|------|------|------|
| ① | 빈발 항목 집합 탐색 | 대용량 데이터베이스 내의 단위 트랜잭션에서 빈번하게 발생하는 사건의 유형을 발견 |
| ② | 최소 지지도 확인 | 트랜잭션을 대상으로 최소지지도 이상을 만족하는 빈발항목 집합을 발견하는 과정 (최소지지도 2 이상이면, item set의 {D}는 support가 1이므로 제외) |
| ③ | 후보 집합 생성 | 빈발항목으로 집합 생성 ({A},{B},{C},{E}로부터 2가지 품목 집합 생성) |
| ④ | 반복 | 새로운 빈발항목이 생성되지 않을 때까지 반복 수행 (최소지지도가 넘는 3가지 품목 집합 {B,C,E} 찾음) |

- 연관규칙은 Apriori 알고리즘을 통해 빈발항목집합을 효율적으로 탐색하고, 지지도/신뢰도/향상도로 유의미한 규칙을 선별하는 과정
