## 문제

데이터 웨어하우스(Data Warehouse)와 데이터 레이크(Data Lake)의 특징을 비교하고, 빅데이터의 요소기술과 기술영역별 표준화 대상항목을 설명하시오.

## 해설

### 출제 배경 및 의도

빅데이터 시대 도래로 정형 데이터 중심의 DW 한계가 부각되고 Data Lake·Data Fabric 등 신규 아키텍처가 확산됨에 따라, 각 개념의 특징 비교 및 빅데이터 표준화 현황을 논술할 수 있는지 확인하기 위해 출제

### 1. 데이터 웨어하우스와 데이터 레이크 개요

정 의  • DW: 정형화된 데이터를 목적에 맞게 변환하여 저장하는 중앙 저장소 (OLTP → ETL → DW)
       • Data Lake: 스키마 없이(schemaless) 원천 데이터(raw data)를 그대로 저장하는 대용량 저장소

```
[OLTP/ERP/CRM] --ETL--> [데이터 웨어하우스] <--> [데이터 레이크] <-- [Devices/Web/Sensors/Social]
      (정형화 변환)         (BI 활용)           (Schema on Read)    (빅데이터 AI/ML)
```

### 2. DW와 데이터 레이크 비교

| 구분 | 데이터 웨어하우스 | 데이터 레이크 |
|------|-----------------|--------------|
| 데이터 구조 | 수집 데이터를 목적에 맞게 가공하여 정형화된 형태로 저장 | 스키마 없이(schemaless) 모든 데이터를 원천 데이터(raw data) 그대로 저장 |
| 데이터 처리 | 다양한 기업형 애플리케이션 데이터 흡수, 사전 정의된 스키마에 맞춰 데이터 변환 | 조직의 스키마에 맞추기 위한 과정 불필요, 필요한 정보는 수집된 데이터에서 탐색/검색, 데이터 처리 오케스트레이션 기능 필요 |
| Schema | Schema on Write (데이터 저장할 때 부여) | Schema on Read (데이터를 활용·분석하고자 할 때 부여) |
| 데이터 품질 | 고품질 정제된 데이터 | 원시 데이터 포함 (데이터 스웜프 위험) |
| 분석 속도 | 구조화로 빠른 쿼리 | 유연하지만 처리 시간 필요 |
| 사용 목적 | 비즈니스 인텔리전스(BI), 정형 리포팅 | AI/ML, 빅데이터 처리, 고급 분석 |

### 3. 빅데이터 요소기술 및 기술영역별 표준화 대상항목

| 기술영역 | 주요 기술 | 표준화 대상 |
|---------|----------|------------|
| 수집 | Flume, Kafka, Sqoop | 데이터 수집 인터페이스, 메시지 포맷 |
| 저장 | HDFS, HBase, NoSQL | 파일 시스템 포맷, 스토리지 인터페이스 |
| 처리 | MapReduce, Spark, Flink | 분산 처리 API, 배치/스트림 처리 프레임워크 |
| 분석 | Hive, Pig, Presto | 데이터 마이닝 알고리즘, ML 인터페이스 |
| 시각화 | Tableau, Kibana | 대시보드 표준, 차트 포맷 |
| 보안 | Ranger, Knox | 인증/인가 프레임워크, 암호화 표준 |

### 4. 데이터 패브릭(Data Fabric) 개념

정 의  • 온프레미스, 클라우드, 하이브리드 환경에서 데이터를 통합적으로 관리하고 분석할 수 있도록 지원하는 데이터 관리 아키텍처
       - 자동화된 데이터 발견(Data Discovery), 메타데이터 기반 통합, 실시간 데이터 접근을 핵심 특징으로 함

| 구분 | DW | Data Lake | Data Fabric |
|------|-----|-----------|-------------|
| 초점 | 정형 데이터 분석 최적화 | 비정형 원시 데이터 보관 | 이기종 환경 통합 데이터 관리 |
| 스키마 | Schema on Write | Schema on Read | 메타데이터 자동화 |
| 환경 | 단일 저장소 | 단일 저장소(대용량) | 멀티클라우드/하이브리드 |

### 5. 결론

빅데이터 시대에 DW의 정형성과 Data Lake의 유연성을 결합한 Data Fabric이 차세대 데이터 관리 아키텍처로 부상하고 있으며, 기술영역별 표준화를 통해 이기종 환경 간 상호운용성 확보가 핵심 과제이다. **끝**

### 실무 제언

- **Data Swamp 방지**: Data Lake 도입 시 데이터 카탈로그와 메타데이터 관리 체계를 선행 구축하여 데이터 품질 저하 방지
- **단계적 아키텍처 전환**: DW → Data Lakehouse → Data Fabric 순으로 점진적 전환 전략 권장
- **표준화 우선 적용**: 수집·저장 계층의 인터페이스 표준화를 우선 적용하여 벤더 종속성 최소화
