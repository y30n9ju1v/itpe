## 해설

### 출제 배경 및 의도

멀티클라우드·하이브리드 환경의 확산으로 데이터가 여러 플랫폼에 분산되어 데이터 사일로(Data Silo) 문제가 심화되었다. 데이터 패브릭은 이를 해결하기 위해 메타데이터와 AI 기반으로 이기종 데이터를 통합 관리하는 아키텍처로, Gartner가 2022년 핵심 데이터 트렌드로 선정하였다. 이 문제는 개념·특징·구성요소와 기존 DW·데이터 레이크와의 차별점을 논리적으로 서술할 것을 요구한다.

---

### 1. 데이터 패브릭 개요

가. 정의

정 의  • 온프레미스, 클라우드, 하이브리드 등 이기종 환경에서 모든 데이터를 통합적으로 관리하고 분석할 수 있도록 지원하는 데이터 관리 아키텍처
       - AI/ML 기반 능동적 메타데이터를 핵심으로 데이터 발견·통합·거버넌스를 자동화

나. 등장 배경

- 멀티클라우드 환경 확산으로 데이터 위치가 다변화
- 데이터 사일로(Data Silo) 문제로 인해 전사 데이터 통합 분석이 어려움
- 실시간 데이터 접근 및 자동화된 거버넌스 필요성 증대

다. 데이터 패브릭 구성도

```
[온프레미스 DB] [클라우드 DW] [데이터 레이크] [SaaS 애플리케이션]
       ↓              ↓              ↓               ↓
┌──────────────────────────────────────────────────────┐
│              데이터 패브릭 레이어                      │
│  메타데이터 관리 | 데이터 카탈로그 | 거버넌스 | 보안    │
└──────────────────────────────────────────────────────┘
       ↓              ↓              ↓
[데이터 사이언티스트] [비즈니스 분석가] [AI/ML 플랫폼]
```

---

### 2. 데이터 패브릭 주요 특징

| 특징 | 설명 |
|------|------|
| 자동화된 데이터 발견 | AI/ML 기반으로 데이터 소스를 자동 탐색하고 카탈로그화 |
| 메타데이터 기반 통합 | 기술적·비즈니스·운영 메타데이터를 통합하여 데이터 연계성 확보 |
| 능동적 메타데이터 활용 | 정적 메타데이터에서 벗어나 실시간으로 업데이트되는 능동적 메타데이터 활용 |
| 데이터 가상화 | 데이터를 물리적으로 이동하지 않고 논리적 통합 뷰 제공 |
| 실시간 데이터 접근 | 온프레미스/클라우드 어디서든 실시간 데이터 접근 가능 |
| 내장된 거버넌스 | 보안, 개인정보보호, 컴플라이언스를 자동화하여 통합 관리 |

---

### 3. 데이터 패브릭 구성요소

| 구성요소 | 설명 |
|----------|------|
| 데이터 카탈로그(Data Catalog) | 데이터 자산의 메타데이터를 수집·관리·검색하는 중앙 저장소 |
| 데이터 통합(Data Integration) | ETL/ELT, 데이터 파이프라인을 통해 다양한 소스 데이터를 통합 |
| 데이터 가상화(Data Virtualization) | 물리적 이동 없이 분산된 데이터에 통합 뷰 제공 |
| 데이터 거버넌스(Data Governance) | 데이터 품질, 보안, 개인정보보호 정책을 중앙화하여 관리 |
| 지식 그래프(Knowledge Graph) | 데이터 간 관계를 그래프로 표현하여 데이터 연관성 시각화 |
| AI/ML 엔진 | 메타데이터 분석, 이상 탐지, 추천 등 자동화된 인사이트 제공 |

---

### 4. DW, 데이터 레이크, 데이터 패브릭 비교

| 구분 | 데이터 웨어하우스 | 데이터 레이크 | 데이터 패브릭 |
|------|-----------------|--------------|--------------|
| 데이터 형태 | 정형 | 정형 + 비정형 원시 | 모든 형태 이기종 |
| 처리 방식 | Schema on Write | Schema on Read | 가상화 + 통합 |
| 거버넌스 | 중앙화 정형 | 느슨한 관리 | 자동화된 통합 거버넌스 |
| 유연성 | 낮음 | 높음 | 매우 높음(멀티클라우드) |
| 주요 기술 | SQL, BI | Hadoop, Spark | 메타데이터, AI, 가상화 |

---

### 5. 결론

데이터 패브릭은 멀티클라우드·분산 환경에서 데이터 사일로 문제를 해결하고 실시간 인사이트를 제공하는 차세대 데이터 관리 아키텍처이다. 능동적 메타데이터와 AI 기반 자동화가 핵심 경쟁력으로, 기존 DW·데이터 레이크의 한계를 보완하여 전사 데이터 통합 분석 환경을 실현한다. **끝**

### 실무 제언

- 데이터 패브릭 도입 시 데이터 카탈로그 구축이 선행 과제로, 조직 내 메타데이터 표준화가 필수
- 데이터 가상화는 물리적 데이터 이동 비용을 줄이나, 실시간 쿼리 성능 보장을 위한 최적화 설계 필요
- 지식 그래프와 AI/ML 엔진의 결합으로 데이터 계보(Data Lineage) 추적 및 이상 탐지 자동화 가능
