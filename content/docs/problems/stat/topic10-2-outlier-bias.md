## 해설

### 출제 배경 및 의도

AI 기반 의사결정 시스템에서 이상치와 편향은 분석 결과의 신뢰성과 공정성을 동시에 저해한다. 단순 개념 암기를 넘어, 두 개념의 차이를 구분하고 각각의 영향과 기술적·제도적 처리방안을 종합적으로 제시하는 능력을 평가한다.

### 1. 이상치와 편향 개요

**이상치(Outlier)**

정 의  • 관측된 데이터 범위에서 많이 벗어나서 데이터 품질 저하에
         영향을 미치기 때문에 검출 및 처리가 필요한 극단적 관측치 값

발생원인:
- 입력 오류 (Human Error)
- 측정 오류 (하드웨어/소프트웨어 결함)
- 실험 오류 (초기조건 통제 실패)
- 고의적 이상값 (Data Poisoning)
- 표본추출 에러 (Sampling Bias)

**편향(Bias)**

정 의  • 데이터 수집, 처리, 알고리즘 설계 과정에서 특정 방향으로
         치우침이 발생하여 분석 결과의 객관성과 공정성을 저해하는 현상

편향 유형:

| 유형 | 설명 |
|------|------|
| 인간의 편향 | 수집자의 주관적 판단이 데이터에 반영 |
| 숨겨진 편향 | 표면적으로 드러나지 않는 구조적 편향 |
| 데이터 표본 편향 | 특정 집단이 과대/과소 대표된 표본 |
| 롱테일 편향 | 소수 극단값이 분포를 왜곡 |
| 고의적 편향 | 특정 결과 유도를 위한 의도적 데이터 조작 |

### 2. 이상치가 데이터 분석 결과에 미치는 영향

| 관점 | 영향 항목 | 상세 설명 |
|------|-----------|-----------|
| 통계적 관점 | 평균 및 분산의 왜곡 | 극단값이 평균을 이탈시키고 분산 비대칭을 증폭 |
| 통계적 관점 | 정규성 훼손 | 왜도/첨도 변화로 정규분포 가정 위반 |
| 통계적 관점 | 검정 오류 증가 | 1종 오류(False Positive) 및 2종 오류(False Negative) 증가 |
| 예측모델 관점 | 회귀 계수 및 설명력 왜곡 | 최소제곱법(SSE 최소화)이 이상치에 과도하게 견인되어 회귀선 왜곡 |
| 예측모델 관점 | 거리 기반 알고리즘 척도 왜곡 | K-Means, KNN, SVM 등 유클리디안 거리 알고리즘에서 공간적 척도 왜곡 |
| 예측모델 관점 | 과적합 및 일반화 상실 | 이상치에 민감하게 피팅되어 일반화 성능 저하 |

### 3. 편향이 데이터 분석 결과에 미치는 영향

| 영향 항목 | 상세 설명 |
|-----------|-----------|
| 모집단 대표성 상실 | 편향된 표본으로 인해 분석 결과가 전체 모집단을 반영하지 못함 |
| 인과관계 착시 및 허위 상관관계 도출 | 편향된 데이터로 인해 실제로는 관계없는 변수 간 상관관계 도출 |
| 왜곡 데이터 피드백 루프 지속 | AI 모델이 편향된 결과를 생성하고, 이를 재학습하는 악순환 발생 |
| 사회적·윤리적 차별성 고착화 | 성별, 인종, 연령 등 특정 집단 차별 알고리즘 고착 |
| 소수 클래스 미고려 모델 생성 | 불균형 데이터로 인한 다수 클래스 편향 모델 생성 (Class Imbalance) |

### 4. 이상치 처리방안

| 단계 | 기법 | 설명 |
|------|------|------|
| 검출 | 개별 데이터 관찰 (ESD) | Extreme Studentized Deviate 검정으로 단일 이상치 탐지 |
| 검출 | 시각화 | 박스플롯(Box-Plot/IQR), 산점도로 분포 이상 확인 |
| 검출 | 머신러닝 기법 | K-Means 클러스터링으로 군집에서 멀리 떨어진 점 탐지 |
| 검출 | 마할라노비스 거리 | 다변량 이상치 탐지 — 변수 간 상관관계 고려 |
| 검출 | LoF (Local Outlier Factor) | 주변 밀도 대비 밀도가 낮은 점을 이상치로 판단 |
| 검출 | Isolation Forest | 이상치는 분리가 쉽다는 특성을 이용한 트리 기반 탐지 |
| 처리 | 삭제 | 이상치 행 제거 (데이터 손실 주의) |
| 처리 | 대체법 | 평균/역측적/회귀/Hot-Deck 대체 |
| 처리 | 변환 (Winsorizing) | 극단값을 유효범위 백분위수 값으로 대체하여 설명력 유지 |
| 처리 | Box-Plot / IQR | Q1-1.5×IQR 이하, Q3+1.5×IQR 이상을 이상치 기준으로 판단 후 처리 |
| 처리 | 분류 | 이상치를 별도 클래스로 분리하여 이상탐지 모델 적용 |

```
IQR 기반 이상치 기준:

  이상치         │◀── IQR ──▶│         이상치
  ●●●           │            │           ●●●
──────────────────────────────────────────────
  Q1-1.5×IQR   Q1   Median  Q3    Q3+1.5×IQR
```

### 5. 편향 처리방안

| 측면 | 처리방안 | 상세 내용 |
|------|----------|-----------|
| 정부 측면 | 설명을 요구할 권리 규정 | AI 의사결정에 대한 설명 요구권 법제화 |
| 정부 측면 | 자동화된 의사결정 제한 권리 | 알고리즘 기반 자동 결정에 이의 제기 및 인간 검토 요구권 |
| 기업 측면 | 기업 AI 윤리 수립 | AI 윤리 원칙 및 책임 거버넌스 체계 구축 |
| 기업 측면 | 정보 품질 진단과 감리 | 데이터 품질 주기적 감사 및 편향 모니터링 |
| 기업 측면 | 설명가능 AI(XAI) 개발 | 모델 의사결정 근거를 투명하게 제시하는 XAI 도입 |
| 기술적 측면 | 편향 인식 알고리즘 개발 | Fairness-aware ML 알고리즘 적용 |
| 기술적 측면 | 재샘플링 | SMOTE(과소표집 클래스 합성 증강), Tomek Links(경계 샘플 제거) |
| 기술적 측면 | 재가중치 부여 | 소수 클래스에 높은 가중치 부여하여 불균형 보정 |
| 기술적 측면 | 정규화 | L1(Lasso) — 불필요 특성 제거, L2(Ridge) — 편향 특성 가중치 축소 |
| 기술적 측면 | 적대적 편향 제거 (GAN) | GAN을 활용한 편향 없는 데이터 생성 및 모델 학습 |
| 기술적 측면 | Cross Validation (K-Fold) | 데이터 분할을 달리하며 검증하여 편향 영향 최소화 |

결 론  • 이상치는 통계적 기법(IQR, Z-score, Isolation Forest)으로 검출·처리
       • 편향은 기술적 기법과 제도적 규제의 병행 적용이 필수
       • AI 공정성 확보를 위해 설명가능 AI(XAI)와 Fairness-aware 알고리즘 도입 필요  "끝"

### 실무 제언

- 신용평가, 채용 AI에서 편향 방치 시 사회적 차별 고착화 — SMOTE 및 재가중치 기법으로 클래스 불균형 해소 필요
- Isolation Forest는 고차원 빅데이터 환경에서 이상치 탐지 효율이 높아 금융 FDS에 활용 증가
- 정부 측면에서 EU AI Act, 한국 인공지능기본법 등 법적 규제가 강화되고 있어 설명가능 AI 도입이 기업 리스크 관리의 핵심 과제
