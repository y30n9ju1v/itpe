## 해설

### 출제 배경 및 의도

AI 학습 및 데이터 마이닝에서 데이터 품질은 분석 결과의 신뢰성을 좌우한다. 아웃라이어는 모델 편향을 초래하는 핵심 원인으로, 개념·유형·노이즈와의 구분 및 대응 방안까지 체계적으로 서술하는 능력을 평가한다.

### 1. 아웃라이어 제거를 통한 데이터 적절성 확보 — 데이터 전처리 개요

정 의  • AI 학습, 데이터 마이닝 결과의 정확성·신뢰성 향상을 위해
         데이터 분석 및 처리에 적합하도록 데이터를 가공하는 작업

```
[원시 데이터]
    │
    ▼
① 데이터 정제 ── 결측치 처리 / 아웃라이어 처리 / 노이즈 처리
    │
    ▼
② 데이터 통합 ── 중복 처리 / ETL 수행
    │
    ▼
③ 데이터 변환 ── 형식 변환 / 속성 변환
    │
    ▼
④ 데이터 정리 ── 차원 축소 / 샘플링
    │
    ▼
[분석 가능 데이터]
```

### 2. 아웃라이어와 노이즈(Noise) 비교

| 구분 | 아웃라이어(Outlier) | 노이즈(Noise) |
|------|-------------------|--------------|
| 개념 | 데이터 중앙값에서 벗어난 극단적 값 | 원본값을 왜곡하거나 잘못 라벨링된 데이터 |
| 데이터 특성 | 과대 또는 과소 | 무작위적 변화 |
| 발생 원인 | 측정 오류, 자연적 요인, 데이터 변환 시 오류 | 데이터 측정 시 오류, 환경적 요인, 간섭 |
| 구분 관점 | 데이터 분포 측면 Garbage Data | 모델 관점 Garbage Data |
| 모델 영향 | 모델 편향 초래 | 모델 성능 저하 |
| 검출 방법 | 왜도/첨도, 박스플롯(IQR), z-score 기반 | 산점도, 히스토그램, 관찰 수행 |
| 대응 방안 | 삭제, 원저라이징(Winsorizing) | 수정, 리샘플링(Resampling) |

### 3. 아웃라이어의 3가지 유형

| 유형 | 개념 | 설명 | 검출 방법 |
|------|------|------|-----------|
| 전역(Global) 아웃라이어 | 데이터셋 전체 분포에서 과도하게 크거나 작게 벗어난 아웃라이어 | 데이터 전체 기준으로 명백히 비정상적인 단일 값 (예: 체온 데이터에서 60°C) | Z-score, Mahalanobis 거리 |
| 맥락(Contextual) 아웃라이어 | 특정 맥락이나 서브 그룹 내에서 예상 범위를 벗어난 아웃라이어 | 전체 기준으로는 정상이나 특정 조건 내에서 비정상 (예: 6월의 이상 고온 30°C→한국 기준 정상, 북극 기준 이상) | Contextual Clustering, Contextual Anomaly Detection |
| 군집(Collective) 아웃라이어 | 데이터셋 전체 분포에서 과도하게 벗어나 군집을 이루는 아웃라이어 | 개별적으로는 정상이나 함께 나타날 때 비정상으로 간주되는 데이터 집합 (예: 특정 시간대에 집중된 비정상 거래) | Clustering 알고리즘 (K-Means, DBSCAN) |

```
[전역 아웃라이어]          [맥락 아웃라이어]         [군집 아웃라이어]
 ●  ●  ●  ●  ●            ●  ●  ●  ●                ●  ●  ●  ●  ●
 ●  ●  ●  ●  ●            ●  ●  ●  ●                ●  ●  ●  ●  ●
              ★ ←이상치   ●★ ←맥락 내 이상          ●●●●←군집 이상
 (전체 기준 극단값)         (서브그룹 내 이상)         (군집 전체가 이상)
```

### 4. Winsorizing(원저라이징) 개념 및 절차

정 의  • 일부의 극단값을 유효범위 내 데이터 값으로 대체하여
         데이터 설명력을 향상 및 유지하는 통계적 기법

```
[데이터 정렬]
    │
    ▼
[백분위수 결정] ── 예: 하위 5%, 상위 95% 기준 설정
    │
    ▼
[극단값 대체] ── 하위 극단값 → 5th Percentile 값으로 대체
              └─ 상위 극단값 → 95th Percentile 값으로 대체
    │
    ▼
[전처리 완료]
```

| 구분 | 삭제 방법 | Winsorizing |
|------|-----------|-------------|
| 데이터 손실 | 극단값 행 제거 → 설명력 저하 | 극단값을 유효범위 값으로 대체 → 설명력 유지 |
| 적용 기준 | 극단값이 오류임이 명확할 때 | 극단값이 의미 있으나 영향을 줄여야 할 때 |

결 론  • 아웃라이어를 단순 삭제 시 모델 설명력 저하 발생
       • Winsorizing을 통해 데이터 설명력 유지 가능
       • 유형(전역·맥락·군집)에 따라 차별화된 검출·처리 전략 적용 필요  "끝"

### 실무 제언

- 금융 이상거래 탐지(FDS)는 군집 아웃라이어 검출이 핵심 — 개별 거래는 정상이나 패턴이 이상한 경우
- IoT 센서 데이터에서는 맥락 아웃라이어 비중이 높음 — 계절·시간대별 정상 범위 기준 설정 필요
- Winsorizing은 금융·의료 분야에서 극단값 보존 및 분포 왜곡 완화에 널리 사용
