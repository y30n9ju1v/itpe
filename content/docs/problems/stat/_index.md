## Part 1. 가설검정

### "내가 옳은지 데이터로 증명하기"

#### 통계적 가설검정이란 무엇인가?

요즘 회사에서 흔히 이런 상황이 생깁니다. "스마트 팩토리를 도입했더니 생산 시간이 줄었다는 보고서가 나왔는데, 이게 진짜로 줄어든 건지, 아니면 그냥 운이 좋아서 표본이 우연히 짧게 나온 건지 어떻게 알 수 있죠?" 이 질문에 답하는 도구가 바로 **통계적 가설검정(Statistical Hypothesis Test)**입니다.

핵심 아이디어는 간단합니다. "일단 **현 상태에는 아무 변화가 없다**고 가정하자. 그런데 우리 데이터가 이 가정 아래에서 나올 확률이 극히 희박하다면, 가정 자체가 틀렸다고 결론 내리자." 이것이 **귀무가설(H₀)**과 **대립가설(H₁)**의 논리적 관계입니다.

```
[귀무가설 H₀] "스마트 팩토리 전후 생산 시간은 차이 없다 (μ ≥ μ₀)"
       ↓ 표본 데이터 수집 및 검정통계량 계산
[검정통계량이 기각역에 들면?]
       ↓
[대립가설 H₁ 채택] "스마트 팩토리가 생산 시간을 단축시켰다 (μ < μ₀)"
```

검정 절차는 6단계로 정리됩니다: ① 가설 설정 → ② 유의수준(α) 결정 → ③ 검정통계량 선택 → ④ 기각역 설정 → ⑤ 검정통계량 계산 → ⑥ 결과 해석(H₀ 기각 or 채택).

---

#### CDA vs EDA — 목적에 따라 달라지는 분석 방식

데이터 분석에는 두 가지 큰 접근법이 있습니다.

* **확증적 분석(CDA, Confirmatory Data Analysis):** 먼저 가설을 세운 뒤 데이터를 수집해 가설을 검증하는 전통적인 방식입니다. 연구자가 "이 약이 효과가 있을 것이다"는 가설을 품고 임상 실험을 설계하는 것이 좋은 예입니다.
* **탐색적 분석(EDA, Exploratory Data Analysis):** 가설 없이 데이터를 먼저 쌓아두고, 시각화와 통계 지표를 활용해 패턴과 인사이트를 자유롭게 발견하는 방식입니다. 마치 보물지도 없이 방대한 숲을 헤치며 보물을 탐험하는 것에 비유할 수 있습니다.

가트너(Gartner)는 데이터 분석의 성숙도를 4단계로 정의했습니다: 설명적 분석(무엇이 일어났는가?) → 진단적 분석(왜 일어났는가?) → 예측적 분석(어떻게 될 것인가?) → 처방적 분석(무엇을 해야 하는가?). 단계가 올라갈수록 난도와 비즈니스 가치가 모두 높아집니다.

---

#### 1종 오류와 2종 오류 — "억울하게 유죄 판결받는 비극"

가설검정에는 두 가지 실수가 존재합니다.

| 의사결정 | H₀가 참(실제로 무죄) | H₀가 거짓(실제로 유죄) |
|---------|---------------------|----------------------|
| H₀ 채택(무죄 판결) | 옳은 결정 ✓ | **2종 오류(β)** — 유죄인데 무죄 방면 |
| H₀ 기각(유죄 판결) | **1종 오류(α)** — 무죄인데 유죄 판결 | 옳은 결정 ✓ |

* **1종 오류(Type I Error, α):** "사실 스마트 팩토리 효과가 없는데, 우연히 표본이 좋게 나와서 효과 있다고 잘못 결론" → 억울한 유죄 판결 같은 상황. 실무에서 제품 불량률 기준보다 더 엄격하게 잡으면 됩니다.
* **2종 오류(Type II Error, β):** "사실 스마트 팩토리 효과가 있는데, 표본이 나쁘게 나와서 효과 없다고 잘못 결론" → 진짜 범인을 놓아주는 상황.

1종 오류를 줄이면 2종 오류가 늘어나는 **상충 관계(Trade-off)**가 있습니다. 유의수준 α = 0.05(5%)는 1종 오류를 5%까지 허용한다는 의미입니다.

## Part 2. 검정방법

### "표본이 몇 개냐에 따라 쓰는 검정법이 다릅니다"

#### t-검정 vs Z-검정 vs ANOVA

같은 '평균 차이를 검정하는 도구'라도 상황마다 다른 검정법을 씁니다. 마치 짧은 거리는 버스, 먼 거리는 기차, 더 먼 거리는 비행기를 타듯이 말이죠.

```
표본 30개 미만, 모분산 모름     → t-검정 (소표본 전용)
표본 30개 이상, 모분산 알거나 대표본 → Z-검정 (대표본)
비교 집단이 3개 이상            → ANOVA (분산분석)
```

* **t-검정:** 소규모 실험처럼 표본이 적을 때 사용합니다. 표본 평균이 모평균에 얼마나 가까운지 t-분포를 기반으로 검정합니다. t = (표본평균 - 모평균) / (표본표준편차 / √n)
* **Z-검정:** 중심극한정리 덕분에 표본이 충분히 크면(n ≥ 30) 표본 평균이 정규분포에 수렴합니다. Z = (x̄ - μ) / (σ / √n)
* **ANOVA(분산분석):** "A 공장, B 공장, C 공장의 불량률이 모두 같은가?"처럼 3개 이상 집단을 한꺼번에 비교할 때 씁니다. 집단 **간(Between)** 분산과 집단 **내(Within)** 분산의 비율(F 통계량)로 유의성을 판단합니다.

---

#### 중심극한정리(CLT) — 통계학의 가장 위대한 정리

**중심극한정리(Central Limit Theorem)**는 "**모집단의 분포가 어떤 모양이든, 표본 크기 n이 충분히 크면(30 이상) 표본 평균들의 분포는 정규분포에 가까워진다**"는 법칙입니다. 이 정리 덕분에 우리는 복잡한 비정규 분포를 가진 현실 데이터에서도 정규분포 기반 검정법을 안심하고 쓸 수 있습니다.

**대수의 법칙(Law of Large Numbers)**도 같이 기억하세요. "동전을 던질수록 앞면 비율이 0.5에 수렴한다"처럼 **표본이 많아질수록 표본 통계량이 모수(진짜 모집단 값)에 점점 가까워진다**는 법칙입니다. 중심극한정리와 대수의 법칙은 표본 조사로 전수 조사를 대신할 수 있는 통계학의 이론적 근거입니다.

## Part 3. 기술통계와 추론통계

### "데이터를 숫자로 요약하고, 표본으로 전체를 추측한다"

#### 기술통계(Descriptive Statistics) — 데이터의 자기소개서

기술통계는 수집한 데이터 자체를 요약해서 보여주는 도구입니다. 중간고사 점수 200개를 받았을 때, "평균이 65점이고 70점대가 가장 많으며 몇 명은 100점과 0점대 극단값을 보였다"처럼 데이터의 특징을 요약하는 것입니다.

핵심 지표들입니다:

* **왜도(Skewness):** 분포가 좌우 어느 쪽으로 꼬리를 길게 뻗었는지 나타냅니다. 양수면 오른쪽 꼬리(저소득층이 많고 소수 고소득자가 있는 소득 분포), 음수면 왼쪽 꼬리입니다.
* **첨도(Kurtosis):** 분포가 정규분포보다 뾰족한지(양수), 납작한지(음수)를 나타냅니다.
* **IQR(사분위 범위):** 중앙 50% 데이터 구간(Q3 - Q1). 아웃라이어 탐지에 씁니다. `Q1 - 1.5×IQR` 이하나 `Q3 + 1.5×IQR` 이상이면 아웃라이어로 판단합니다.

---

#### 추론통계 — 표본으로 모집단을 추측하기

**추론통계(Inferential Statistics)**는 전체 모집단을 다 조사하기 어려울 때 일부 표본(Sample)을 뽑아 통계 공식으로 모집단의 특성을 **추론**하는 방법입니다.

추정의 두 종류를 구분해야 합니다:

* **점추정(Point Estimation):** 모평균을 딱 하나의 숫자로 추정. (예: "이 공장 제품 평균 무게는 100g이다") 간편하지만 불확실성을 표현하지 못합니다.
* **구간추정(Interval Estimation):** 신뢰구간으로 범위를 제시. (예: "95% 확률로 98~102g 사이") 더 정직한 추정입니다.

**불편 추정량(Unbiased Estimator)**은 평균적으로 편향되지 않는 추정량입니다. 표본 분산을 구할 때 n이 아닌 **(n-1)로 나누는 이유**가 바로 이 때문입니다. n으로 나누면 계통적으로 과소추정되기 때문입니다.

---

#### 정규분포와 확률분포

**정규분포(Normal Distribution)**는 자연계에서 가장 많이 나타나는 분포입니다. 종 모양(Bell Curve)의 좌우 대칭 형태이며, 평균 μ과 표준편차 σ로 완전히 정의됩니다.

```
        μ-3σ  μ-2σ  μ-σ   μ   μ+σ  μ+2σ  μ+3σ
         |     |     |     |     |    |     |
         └──68.27%──┘
         └──────95.45%──────┘
         └──────────99.73%──────────┘
```

실무에서는 표준화(Z-score = (x - μ) / σ)를 통해 모든 정규분포를 표준정규분포(μ=0, σ=1)로 변환해 비교합니다.

**베르누이 분포**는 성공/실패 두 결과만 있는 시행의 분포(동전 던지기), **기하 분포**는 첫 성공이 나올 때까지 필요한 시행 횟수의 분포입니다.

## Part 4. 회귀분석

### "변수들 사이의 관계를 수식으로 모델링하기"

#### 다중회귀 vs 로지스틱 회귀

**회귀분석(Regression Analysis)**은 독립변수(원인)와 종속변수(결과) 사이의 관계를 수식으로 모델링하는 기법입니다.

* **선형회귀(Linear Regression):** "아파트 면적(독립변수)이 클수록 가격(종속변수)이 높다"처럼 연속형 결과를 예측합니다. y = β₀ + β₁x₁ + β₂x₂ + ... + ε
* **로지스틱 회귀(Logistic Regression):** 종속변수가 "합격/불합격", "스팸/정상"처럼 **범주형**일 때 씁니다. 결과를 시그모이드 함수를 통해 0~1 사이 확률로 변환합니다.

통계학의 4가지 척도도 알아야 합니다: 명목척도(이름), 서열척도(순위), 등간척도(간격이 동일, 온도), 비율척도(절대 0이 존재, 몸무게).

---

#### Ridge와 Lasso — 과적합을 막는 정규화 기법

**과적합(Overfitting)**은 학습 데이터에 너무 딱 맞춰져서 새 데이터에는 엉터리 예측을 하는 현상입니다. 반대는 **과소적합(Underfitting)**입니다.

```
과소적합       적정 모델       과적합
(단순한 직선)  (적절한 곡선)  (너무 구불구불)
  ∽∽∽           ~              ~∿∿∿~
데이터를 못 따라감  딱 맞음   훈련 데이터에만 딱 맞음
```

해결책이 바로 **정규화(Regularization)**입니다:

* **Ridge(L2 정규화):** 손실함수에 모든 계수의 제곱합(λΣβ²)을 더해 계수가 지나치게 커지지 못하게 압박합니다. 계수를 0에 가깝게 줄이되 완전히 0으로 만들지는 않아, 모든 변수를 살려둡니다.
* **Lasso(L1 정규화):** 계수의 절댓값 합(λΣ|β|)을 더합니다. 불필요한 변수의 계수를 **완전히 0으로** 만들어 자동으로 변수 선택을 해줍니다. 중요하지 않은 변수를 솎아낼 때 유용합니다.

λ(람다)가 클수록 정규화가 강해져 단순한 모델이 됩니다.

---

#### 다중공선성 — 설명변수들끼리 너무 닮아버린 문제

**다중공선성(Multicollinearity)**은 독립변수들 사이에 강한 상관관계가 있어서, 어떤 변수가 종속변수에 미치는 영향인지 분리하기 어려워지는 문제입니다.

"키와 몸무게 둘 다 독립변수로 넣어서 건강 점수를 예측하려 했더니, 키와 몸무게가 서로 너무 비슷하게 움직여서 각각의 영향력을 따로 계산하지 못하는" 상황입니다.

진단 도구: **VIF(분산 팽창 인수)** = 1/(1 - R²ᵢ). VIF > 10이면 다중공선성 의심.
해결책: 상관 높은 변수 제거, 변수 변환, Ridge 회귀, 주성분 회귀(PCR) 적용.

## Part 5. 차원축소

### "수백 개의 변수를 소수의 핵심으로 압축하기"

고차원 데이터는 시각화가 안 되고, 계산이 느리며, '차원의 저주(Curse of Dimensionality)' 문제가 생깁니다. 이를 해결하는 두 가지 대표 기법을 비교합니다.

| 구분 | PCA (주성분 분석) | LDA (선형 판별 분석) |
|------|-----------------|---------------------|
| 학습 방식 | 비지도학습 | 지도학습 |
| 목적 | 분산 최대화 (데이터 퍼짐을 최대한 보존) | 클래스 간 분리 최대화 |
| 라벨 필요 여부 | 불필요 | 필요 |
| 활용 | 데이터 압축, 노이즈 제거, 시각화 | 분류(Classification) 사전 차원 축소 |

**PCA**는 교실에서 학생들 사진을 찍을 때 모든 학생이 최대한 잘 보이는 각도를 찾는 것과 같습니다. 레이블(누가 누군지)과 관계없이 최대한 데이터의 정보를 보존합니다.

**LDA**는 남학생과 여학생을 최대한 명확히 구분해서 두 그룹이 한눈에 분리되는 촬영 각도를 찾는 것에 비유할 수 있습니다.

## Part 6. 상관분석

### "함께 움직이는가? 그리고 그 이유가 있는가?"

#### 상관분석의 종류

**상관분석(Correlation Analysis)**은 두 변수 사이의 선형적 관계 강도를 측정합니다. 상관계수는 -1 ~ +1 사이 값을 가집니다.

| 기법 | 대상 데이터 | 특징 |
|------|-----------|------|
| 피어슨(Pearson) | 등간·비율 척도, 정규분포 | 선형 관계 측정 |
| 스피어만(Spearman) | 서열 척도, 비정규 | 순위 기반, 이상치에 강함 |
| 켄달(Kendall) | 서열 척도 | 일치·불일치 쌍 비율, 소표본에 강함 |

#### 상관관계 ≠ 인과관계

통계 분야의 가장 유명한 함정입니다. "아이스크림 판매량과 익사 사고 수가 강한 양의 상관관계를 보인다. 그러므로 아이스크림이 익사를 유발한다?" — 완전히 틀린 결론입니다.

둘 다 **여름(계절)**이라는 제3의 변수(혼란 변수)에 의해 함께 증가하는 것입니다. **상관관계는 두 변수가 함께 변한다는 사실만 알려주며, 왜 그런지(인과)는 알려주지 않습니다.** 인과관계를 증명하려면 무작위 대조 실험(RCT)이나 인과 추론 방법론이 필요합니다.

## Part 7. 베이즈 정리

### "새로운 증거가 생기면 믿음을 업데이트하라"

**베이즈 정리(Bayes' Theorem)**는 **사전 확률(Prior)**에 새로운 증거(Data)를 결합해 **사후 확률(Posterior)**을 구하는 공식입니다.

$$P(H|E) = \frac{P(E|H) \cdot P(H)}{P(E)}$$

* P(H): 사전 확률 — 증거를 보기 전 내 믿음
* P(E|H): 가능도(Likelihood) — 가설이 맞다면 이 증거가 관찰될 확률
* P(H|E): 사후 확률 — 증거를 본 후 업데이트된 믿음

**실전 예시:** 의장 선출 후보 A, B, C의 당선 확률이 0.3, 0.5, 0.2이고, 이자가 오를 확률이 A 당선 시 24%, B 당선 시 40%, C 당선 시 45%라 하자. 이자가 실제로 올랐을 때, A가 선출되었을 확률은?

```
P(이자 오름) = 0.3×0.24 + 0.5×0.40 + 0.2×0.45 = 0.37
P(A | 이자 오름) = (0.3×0.24) / 0.37 = 0.072 / 0.37 ≈ 19.5%
```

스팸 필터, 의학 진단, 추천 시스템 모두 베이즈 정리를 핵심 엔진으로 씁니다.

## Part 8. 빅데이터 플랫폼

### "대용량 데이터를 빠르게 처리하는 기술 생태계"

#### 람다(Lambda) vs 카파(Kappa) 아키텍처

실시간 스트리밍과 대용량 배치 처리를 동시에 다뤄야 하는 빅데이터 시스템에서 어떤 아키텍처를 선택할지가 핵심입니다.

```
람다 아키텍처 (Lambda Architecture)
 원시 데이터 → [배치 레이어] → 배치 뷰 ──┐
             → [스피드 레이어] → 실시간 뷰 ─┤→ [서빙 레이어] → 쿼리
                                            ┘
카파 아키텍처 (Kappa Architecture)
 원시 데이터 → [스피드 레이어] → 실시간 뷰 → [서빙 레이어] → 쿼리
 (배치 레이어 제거, 단순화)
```

* **람다 아키텍처:** 배치(정확도)와 스피드(실시간)를 모두 갖추되, 두 레이어를 병렬로 유지해 복잡도가 높습니다. 금융 이상 탐지처럼 정확도와 실시간이 둘 다 중요할 때 씁니다.
* **카파 아키텍처:** 카프카 같은 스트리밍 레이어만 남겨 단순화한 구조. 배치 처리도 스트리밍으로 재처리합니다.

---

#### 하둡(Hadoop) 에코시스템

하둡은 구글의 MapReduce 논문을 오픈소스로 구현한 분산 처리 시스템입니다. 핵심은 **HDFS(Hadoop Distributed File System)**입니다.

```
HDFS 쓰기 과정:
클라이언트 → NameNode(경로 확인) → DataNode1(블록 저장) → DataNode2(복제) → DataNode3(복제)

HDFS 읽기 과정:
클라이언트 → NameNode(블록 위치 조회) → 가까운 DataNode에서 블록 직접 읽기
```

하둡 에코시스템 핵심 컴포넌트: HDFS(저장), YARN(자원관리), MapReduce(배치처리), Hive(SQL), Spark(인메모리 고속처리), HBase(NoSQL), Sqoop(RDBMS 연동).

---

#### 카프카(Kafka) — 데이터의 고속도로

**카프카(Apache Kafka)**는 대용량 실시간 메시지 스트리밍 플랫폼입니다. Pub/Sub 모델로 동작합니다.

* **Producer:** 메시지를 생산해서 Topic에 던지는 역할
* **Broker:** 메시지를 저장·관리하는 카프카 서버
* **Topic:** 메시지 분류 단위 (채널 개념)
* **Consumer:** Topic에서 메시지를 가져다 쓰는 소비자
* **Consumer Group:** 부하를 나눠 병렬로 처리하는 소비자 집합

카프카는 메시지를 디스크에 **로그(순서대로) 저장**하기 때문에, 과거 데이터를 재처리하거나 여러 소비자가 독립적으로 읽을 수 있다는 강점이 있습니다.

## Part 9. 데이터 마이닝

### "숨겨진 패턴과 지식을 데이터에서 캐내기"

#### CRISP-DM vs SEMMA — 데이터 마이닝 방법론

데이터 마이닝 프로젝트를 체계적으로 진행하는 방법론 두 가지입니다.

* **CRISP-DM:** 비즈니스 이해 → 데이터 이해 → 데이터 준비 → 모형 개발 → 평가 → 배포. 산업 표준으로 비즈니스 목적과 데이터 준비를 강조합니다.
* **SEMMA (SAS 제안):** **S**ample(표본 추출) → **E**xplore(탐색) → **M**odify(전처리) → **M**odeling(모델링) → **A**ssessment(평가). 데이터 준비와 모델 구축 과정에 집중합니다.

---

#### 연관규칙 — "맥주 사는 사람은 기저귀도 산다"

슈퍼마켓 계산대 데이터를 분석했더니 "맥주를 산 손님 중 65%가 기저귀도 샀다"는 유명한 발견 사례입니다. 이처럼 데이터에서 'A를 사면 B도 산다'는 규칙을 찾는 것이 **연관규칙 마이닝**입니다.

세 가지 측도로 연관 규칙의 품질을 평가합니다:

| 측도 | 수식 | 의미 |
|------|------|------|
| 지지도(Support) | P(X∩Y) / 전체 | 전체 거래 중 X와 Y가 함께 나타나는 비율 |
| 신뢰도(Confidence) | P(X∩Y) / P(X) | X를 산 사람 중 Y도 산 비율 |
| 향상도(Lift) | P(X∩Y) / P(X)·P(Y) | 1보다 크면 X와 Y는 서로 도움 되는 관계 |

**Apriori 알고리즘**은 '자주 나오는 항목 집합(빈발 항목집합)'을 단계적으로 찾아 연산량을 줄이는 방법입니다. Join(후보 생성) → Prune(가지치기) 반복.

---

#### 의사결정나무 — "스무고개처럼 데이터 분류하기"

**의사결정나무(Decision Tree)**는 데이터를 스무고개처럼 Yes/No 질문으로 나누어 최종 결론에 도달하는 알고리즘입니다.

```
[나이 ≥ 30?]
   ├─ Yes → [소득 > 5000만원?]
   │            ├─ Yes → 대출 승인 ✓
   │            └─ No  → 대출 거절 ✗
   └─ No  → 대출 거절 ✗
```

나무의 분기점(Split)을 결정할 때 사용하는 불순도 측도:

* **지니 지수(Gini):** G(t) = 1 - Σpₖ². 0에 가까울수록 순수한 노드.
* **엔트로피(Entropy):** E(t) = -Σpₖ·log₂(pₖ). 0에 가까울수록 순수한 노드.

**랜덤 포레스트**는 의사결정나무를 수백 개 만들어 다수결로 결정하는 앙상블 기법입니다. 단일 의사결정나무의 과적합 문제를 크게 개선합니다.

---

#### SNA (사회연결망 분석)

**SNA(Social Network Analysis)**는 개인, 집단, 조직 간의 관계를 그래프 이론으로 분석하는 방법입니다. 트위터나 카카오톡의 친구 관계망 분석이 대표적 예입니다.

중심성(Centrality) 지표가 핵심입니다:
* **연결 중심성(Degree Centrality):** 직접 연결된 이웃의 수 (팔로워 수)
* **근접 중심성(Closeness Centrality):** 모든 다른 노드까지의 평균 거리의 역수
* **매개 중심성(Betweenness Centrality):** 다른 두 노드 사이의 최단 경로에 얼마나 자주 놓이는가 (정보 흐름의 허브)

---

#### 시계열 분석 — 시간과 씨름하는 데이터

**시계열 분석**은 연도별 매출, 월별 기온처럼 시간 순서로 배열된 데이터에서 패턴을 파악하고 미래를 예측하는 기법입니다.

* **AR(자기회귀) 모형:** 과거 자신의 값으로 현재를 예측. (전날 주가로 오늘 주가 예측)
* **MA(이동평균) 모형:** 과거 오차(백색 잡음)의 가중합으로 예측.
* **ARIMA(p, d, q):** AR + 차분(I, 비정상성 제거) + MA를 합친 강력한 모형.

시계열 분해 4요인: **추세(Trend)** + **계절(Seasonal)** + **순환(Cyclical)** + **불규칙(Irregular)**

---

#### 텍스트 마이닝과 오피니언 마이닝

**텍스트 마이닝**은 NLP(자연어 처리) 기반으로 비정형 텍스트에서 패턴, 지식을 추출하는 기법입니다. 6단계 절차: 문서 수집 → 전처리 → 의미정보 변환 → 정보 추출 → 패턴 분석 → 정보표현.

**오피니언 마이닝(감성분석)**은 텍스트 마이닝의 특수 형태로, 리뷰나 SNS에서 긍정/부정/중립 감성을 판별합니다. ML 기반(학습 데이터로 분류), Lexicon 기반(감성 사전 활용), 규칙 기반 방식이 있습니다.

---

## Part 10. 데이터 전처리

### "쓰레기를 넣으면 쓰레기가 나온다 (GIGO)"

AI 모델이 아무리 뛰어나도 입력 데이터가 더러우면 쓸모없는 결과가 나옵니다. 데이터 정제는 전체 분석 작업 시간의 60~80%를 차지한다고도 알려져 있습니다.

```
데이터 전처리 4단계:
[데이터 정제] → [데이터 통합] → [데이터 변환] → [데이터 정리]
 결측치/아웃라이어  중복처리/ETL   형식변환/정규화   차원축소/샘플링
```

#### 아웃라이어 vs 노이즈

같은 이상 데이터라도 성격이 다릅니다:

* **아웃라이어(Outlier):** 데이터 **분포** 관점의 문제. "다른 데이터에 비해 극단적으로 크거나 작은 관측값." 키가 230cm인 중학생처럼, 분포에서 튀는 진짜 값일 수도 있고 측정 오류일 수도 있습니다.
* **노이즈(Noise):** **모델** 관점의 문제. 원본 값을 왜곡하거나 잘못 라벨링된 오류 데이터. 강아지 사진을 고양이로 잘못 레이블링한 데이터가 대표적입니다.

아웃라이어 3가지 유형:
* **전역(Global):** 전체 분포에서 과도하게 벗어난 값. Z-score나 Mahalanobis 거리로 탐지.
* **맥락(Contextual):** 특정 맥락 내에서만 이상한 값. (6월의 영하 기온) Contextual Clustering으로 탐지.
* **군집(Collective):** 개별로는 정상이지만 집단으로 나타날 때 이상. Clustering 알고리즘으로 탐지.

처리 방안: 삭제, 대체법(평균/회귀 대체), **Winsorizing(극단값을 상·하한값으로 대체)**, Box-Plot IQR 활용.

---

#### 이상치(Outlier)와 편향(Bias)이 모델에 미치는 영향

이상치는 회귀계수를 왜곡하고, 군집 척도를 무너뜨리며, 분류 경계를 흐립니다. 편향은 더 넓게 시스템 전체를 오염시킵니다.

편향 유형: 인간의 편향, 숨겨진 편향, 데이터 표본 편향, 롱테일 편향, 고의적 편향.

편향 해결책(기술적 측면): SMOTE(오버샘플링), Tomek Links(언더샘플링), L1/L2 정규화, 적대적 편향 제거(GAN), Stratified K-Fold Cross Validation.

## Part 11. 데이터 분석 플로우

### "하나의 모델보다 여럿이 낫다, 그리고 데이터를 잘 다스리는 법"

#### 앙상블 기법 — "다수결이 단수결보다 낫다"

**앙상블(Ensemble)**은 여러 개의 모델을 조합해 더 나은 예측을 만드는 기법입니다. 여러 전문가의 의견을 수렴하는 것처럼, 단일 모델의 약점을 집단 지성으로 보완합니다.

| 기법 | 원리 | 특징 |
|------|------|------|
| **배깅(Bagging)** | 부트스트랩 표본으로 여러 모델 학습 → 다수결 | 과적합 감소, 병렬 학습 가능 |
| **부스팅(Boosting)** | 이전 모델의 오분류에 가중치 부여 → 순차 개선 | 정확도 높음, 이상치에 민감 |
| **랜덤 포레스트** | 배깅 + 무작위 변수 선택 → 의사결정나무 다수 | 해석력과 성능의 균형 |

---

#### 랜덤 샘플링과 필터링

**랜덤 샘플링**의 유형: 단순(Simple), 계통(Systematic), 2단계, 층별(Stratified), 집락(Cluster) 샘플링.

**필터링** 유형:
* 학습 기반: 베이지안, 내용기반, 칼만 필터
* 비학습 기반: 해시, 질의(Query), 블룸(Bloom) 필터

---

#### 데이터 분석 Flow와 모델 평가

데이터 분석 전체 프로세스: **데이터 EDA** → **데이터 전처리** → **분석 모델링** → **분석모델 평가**. 이 과정은 단방향이 아니라 여러 번 반복하는 순환 과정입니다.

분류 모델 평가 핵심 지표:

| 지표 | 수식 | 의미 |
|------|------|------|
| 정확도(Accuracy) | (TP+TN) / (TP+TN+FN+FP) | 전체 예측 중 맞은 비율 |
| 정밀도(Precision) | TP / (TP+FP) | 양성 예측 중 실제 양성 비율 |
| 재현율(Recall) | TP / (TP+FN) | 실제 양성 중 찾아낸 비율 |
| F1 Score | 2 × (Precision × Recall) / (Precision + Recall) | 정밀도와 재현율의 조화평균 |

**ROC Curve**는 임계값 변화에 따른 TPR vs FPR을 그린 곡선. **AUC**가 1에 가까울수록 좋은 모델입니다.

군집 모델은 **실루엣 계수(Silhouette Coefficient)**: Sᵢ = (bᵢ - aᵢ) / max(aᵢ, bᵢ) 로 평가합니다. (aᵢ: 같은 군집 내 평균 거리, bᵢ: 가장 가까운 다른 군집까지의 평균 거리)

---

#### 데이터 분석 거버넌스

**거버넌스(Governance)**는 전사 차원의 데이터 정책, 관리 조직, 프로세스를 수립하는 체계입니다.

중심에 **CoA(Center of Analytics)**를 두고 분석 프로세스, 전문 인력, 분석 조직, 수준 진단, 교육 5가지 요소가 유기적으로 연결됩니다.

분석 준비도(Readiness) × 성숙도(Maturity) 매트릭스:
- **준비형(↓성숙도, ↓준비도):** 전면 사전 준비 필요
- **정착형(↑성숙도, ↓준비도):** 조직·인력 등 1차 정착 우선
- **도입형(↓성숙도, ↑준비도):** 즉시 도입 가능
- **확산형(↑성숙도, ↑준비도):** 전사 확산·최적화 단계

---

## Part 12. LDW

### "데이터웨어하우스와 빅데이터의 결혼"

**LDW(Logical Data Warehouse)**는 기존 EDW(Enterprise Data Warehouse)와 Hadoop의 강점을 통합한 아키텍처입니다. 정형 데이터는 DW에서, 비정형 대용량 데이터는 하둡에서 처리하되 사용자에게는 하나의 통합된 논리적 뷰를 제공합니다.

```
원천 데이터
(소셜/스트리밍/센싱)
        ↓ Sqoop/ETL/CDC
┌─────────────────────────────────────┐
│         LDW 통합 레이어              │
│ Hadoop(HDFS/Hive/Spark)            │
│ + DW(EDW/ODS/In-memory/Cloud DW)  │
│ + 메타데이터/Taxonomy/Ontology       │
└─────────────────────────────────────┘
        ↓
실시간 의사결정 | 리포팅 | 텍스트분석 | 데이터마이닝
```

LDW의 4가지 활용 방안: IoT 플랫폼, Hybrid DW, Data Analytics, Data Virtualization.

---

#### DW 다차원 모델링

DW를 설계할 때 분석 목적에 맞게 **Fact Table**(측정값)과 **Dimension Table**(분류 기준)을 배치하는 방법입니다.

* **Star Schema:** 하나의 Fact Table 주변에 Dimension Table이 별처럼 배치. Join이 적어 조회 속도 빠름.
* **Snowflake Schema:** Star Schema의 Dimension Table을 3차 정규화. 데이터 중복 감소, 조회 속도 약간 저하.

OLTP(정규화 ERD) vs OLAP(다차원 모델): OLTP는 트랜잭션 정확성, OLAP은 분석 편의성에 최적화됩니다.

---

## Part 13. OLAP / OLTP

### "운영 DB와 분석 DB는 왜 반드시 분리되어야 하는가?"

#### OLAP vs OLTP 핵심 비교

| 구분 | OLAP | OLTP |
|------|------|------|
| 목적 | 분석(조회·집계) | 트랜잭션(삽입·수정) |
| 기능 원리 | Drill Down/Up, Slice, Dice, Pivot | ACID 트랜잭션 |
| 데이터 모델 | Star/Snowflake Schema | ERD(정규화) |
| 주요 작업 | 대용량 집계 쿼리 | 건건이 소규모 데이터 처리 |
| 데이터 보호 | Read Only | 락킹, 병행처리 |

#### 왜 분리 운영이 필요한가?

"하나의 DB에서 거래 처리와 보고서 분석을 동시에 하면 어떻게 될까?" — 보고서 쿼리가 대용량 테이블을 풀스캔하는 동안 운영 서비스가 락(Lock)에 걸려 멈춰버리는 참사가 벌어집니다.

분리 이유 세 가지: **서비스 제공 차별화**(배치 부하와 운영 부하 분리), **DB 성능 설정**(파라미터가 서로 반대 방향으로 최적화됨), **데이터 관리**(운영DB는 짧은 주기 백업/고무결성, 분석DB는 긴 주기 백업/대용량 집계 뷰).

## Part 14. 데이터 레이크 / 데이터 패브릭

### "데이터를 어디에, 어떻게 담을 것인가?"

#### 데이터 웨어하우스 vs 데이터 레이크

전통적 DW가 정형화된 금고라면, 데이터 레이크는 모든 것을 원상태로 쏟아붓는 거대한 저수지입니다.

| 구분 | 데이터 웨어하우스 | 데이터 레이크 |
|------|----------------|------------|
| 데이터 구조 | 정형화(Schema on Write) | 원시 데이터 그대로(Schema on Read) |
| 스키마 적용 시점 | 저장 전 정제 후 적재 | 분석할 때 스키마 부여 |
| 데이터 품질 | 고품질 정제 데이터 | 원시 데이터 포함 (데이터 스웜프 위험) |
| 주 활용 | BI, 정형 리포팅 | AI/ML, 빅데이터 고급 분석 |

기존 DW의 비정형 데이터 분석 한계 때문에 데이터 레이크 구축 수요가 급증했습니다.

---

#### 데이터 패브릭 — 이기종 환경을 하나로

**데이터 패브릭(Data Fabric)**은 온프레미스, 퍼블릭 클라우드, 프라이빗 클라우드 등 어디에 데이터가 있든 하나의 통합된 레이어처럼 관리하고 접근할 수 있게 해주는 차세대 데이터 관리 아키텍처입니다.

```
[온프레미스 DB] [AWS S3] [Azure DW] [SaaS 앱]
        ↓           ↓         ↓          ↓
┌──────────────────────────────────────────┐
│          데이터 패브릭 레이어             │
│  데이터 카탈로그 | AI 기반 메타데이터     │
│  데이터 가상화 | 거버넌스 자동화          │
└──────────────────────────────────────────┘
        ↓           ↓         ↓
[데이터 사이언티스트] [분석가] [AI/ML 플랫폼]
```

데이터 패브릭의 핵심 역량: 자동화된 데이터 발견, 능동적 메타데이터 관리, 데이터 가상화, 내장된 거버넌스·보안.

DW → 데이터 레이크 → 데이터 패브릭으로의 진화는 "단일 구조 → 유연한 저장 → 어디서나 통합 관리"로의 여정입니다.
