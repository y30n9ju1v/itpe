## 문제

동등분할 테스트와 경곗값 분석 테스트에 대해서 설명하고, 아래 명세서를 보고 **동등분할 테스트 케이스와 경곗값 분석 테스트 케이스**를 설계하시오.

> (단, 경곗값 테스트의 경우 2-value 기준)
>
> **[명세]** 공기업 채용 기준에 따라 19세에서 60세까지만 고용할 수 있다.
> (단, 19세~20세는 인턴으로 고용해야 하고, 21세~60세는 정규 직원으로 고용해야 한다.)

## 해설

> **경계에서 결함이 태어난다**  
> "18세는 안 되고, 19세부터 된다"는 그 경계값 하나가 가장 많은 버그를 낳는다.

프로그래머들이 가장 흔하게 범하는 실수 중 하나는 경계 조건(Boundary Condition)의 오류입니다. `if (age >= 19)` 대신 `if (age > 19)`로 코딩하면 19세가 처리에서 제외됩니다. 이처럼 딱 1만큼 어긋나는 실수를 Off-by-One 오류라고 합니다. 동등 분할(Equivalence Partitioning, 동일하게 처리될 입력을 그룹으로 묶는 기법)과 경계값 분석(Boundary Value Analysis, 경계 지점의 값을 집중 테스트하는 기법)은 이런 실수를 체계적으로 찾아내기 위한 블랙박스 테스트의 핵심 기법입니다.

### 1. 출제 배경 및 의도

이 문제는 이론 설명과 실제 테스트 케이스 설계를 동시에 요구하는 **응용형** 문제입니다. 채점관은 두 가지를 평가합니다.

1. 동등 분할과 경계값 분석의 **정의와 원리**를 정확히 이해하고 있는가
2. 주어진 명세서를 보고 **실제 테스트 케이스 표**를 올바르게 설계할 수 있는가

특히 경계값 분석에서 **2-value BVA**와 **3-value BVA**의 차이를 명확히 알고 문제 조건(2-value 기준)에 맞게 적용하는 것이 고득점 핵심입니다.

### 2. 개념의 본질과 작동 메커니즘

#### 가. 동등 분할(Equivalence Partitioning)

**원리**: 입력 도메인을 동일한 방식으로 처리될 것으로 예상되는 **동등 클래스(파티션)**로 나누고, 각 클래스에서 대표값 하나만 테스트해도 같은 효과를 기대할 수 있다는 가정에 기반합니다.

```text
[동등 분할 개념도]

나이 입력 범위: 0 ~ 무한

  ← 18세 이하 │ 19~20세 │ 21~60세 │ 61세 이상 →
  (무효 파티션) │(유효:인턴)│(유효:정규)│(무효 파티션)

  각 파티션에서 대표값 하나씩만 선택하면 충분
  예: 10, 19, 30, 65
```

**핵심 개념**:
- **유효 동등 클래스(Valid Equivalence Class)**: 시스템이 정상 처리해야 하는 입력 범위
- **무효 동등 클래스(Invalid Equivalence Class)**: 시스템이 오류로 처리해야 하는 입력 범위
- 각 클래스에서 하나의 대표값을 선택 → 테스트 케이스 수를 최소화

#### 나. 경계값 분석(Boundary Value Analysis, BVA)

**원리**: 동등 클래스의 **경계(Boundary)에서 결함이 가장 자주 발생**한다는 경험칙. 경계값과 그 인접 값들을 집중적으로 테스트합니다.

```text
[경계값 분석 유형 비교]

2-value BVA: 경계 당 2개 값 테스트
  경계-1 (경계 바로 아래) + 경계 (경계 자체)
  또는 경계 + 경계+1 (방향에 따라)
  예: 나이 19세 경계 → {18, 19} 테스트

3-value BVA: 경계 당 3개 값 테스트
  경계-1 + 경계 + 경계+1
  예: 나이 19세 경계 → {18, 19, 20} 테스트
  → 더 철저하지만 테스트 케이스 수가 더 많음
```

| BVA 유형 | 테스트 값 | 특징 |
|---------|---------|------|
| 2-value BVA | 경계-1, 경계 | 경계의 바로 아래와 경계 자체 |
| 3-value BVA | 경계-1, 경계, 경계+1 | 경계 전후를 모두 포함, 더 완전 |

### 3. 명세서 적용: 테스트 케이스 설계

**명세**: 공기업 채용 기준에 따라 19세에서 60세까지만 고용할 수 있다.
- 19세~20세: 인턴으로 고용
- 21세~60세: 정규 직원으로 고용
- 19세 미만, 60세 초과: 고용 불가

#### 가. 동등 분할 테스트 케이스

```text
[파티션 식별]

파티션 1 (무효): 나이 < 19  → 고용 불가
파티션 2 (유효): 19 ≤ 나이 ≤ 20  → 인턴 고용
파티션 3 (유효): 21 ≤ 나이 ≤ 60  → 정규 직원 고용
파티션 4 (무효): 나이 > 60  → 고용 불가
```

| TC | 파티션 | 입력값(나이) | 예상 출력 | 유효/무효 |
|----|-------|-----------|---------|---------|
| TC-EP-1 | 파티션 1 | 10 | 고용 불가 | 무효 |
| TC-EP-2 | 파티션 2 | 20 | 인턴 고용 | 유효 |
| TC-EP-3 | 파티션 3 | 35 | 정규 직원 고용 | 유효 |
| TC-EP-4 | 파티션 4 | 65 | 고용 불가 | 무효 |

#### 나. 경계값 분석 테스트 케이스 (2-value BVA 기준)

경계점 식별:
- 경계 B1: 18/19 (19세 미만/인턴 경계)
- 경계 B2: 20/21 (인턴/정규 직원 경계)
- 경계 B3: 60/61 (정규 직원/고용 불가 경계)

```text
[2-value BVA: 각 경계에서 경계-1 과 경계 값 테스트]

경계 B1 (18|19): → {18, 19}
경계 B2 (20|21): → {20, 21}
경계 B3 (60|61): → {60, 61}
```

| TC | 경계점 | 입력값(나이) | 예상 출력 | 2-value 기준 |
|----|------|-----------|---------|------------|
| TC-BVA-1 | B1 하한 | 18 | 고용 불가 | 경계-1 |
| TC-BVA-2 | B1 하한 | 19 | 인턴 고용 | 경계 |
| TC-BVA-3 | B2 전환 | 20 | 인턴 고용 | 경계-1 |
| TC-BVA-4 | B2 전환 | 21 | 정규 직원 고용 | 경계 |
| TC-BVA-5 | B3 상한 | 60 | 정규 직원 고용 | 경계-1 |
| TC-BVA-6 | B3 상한 | 61 | 고용 불가 | 경계 |

> **2-value BVA의 핵심**: 각 경계에서 "경계 바로 아래(경계-1)"와 "경계 자체"를 테스트하여 Off-by-One 오류를 포착

### 4. 핵심 키워드 (Must-Have)

| 키워드 | 설명 |
|--------|------|
| 동등 클래스 (Equivalence Class/Partition) | 동일하게 처리될 것으로 예상되는 입력값의 집합 |
| 유효 파티션 | 시스템이 정상 처리해야 하는 입력 범위 |
| 무효 파티션 | 시스템이 오류로 처리해야 하는 입력 범위 |
| 경계값 (Boundary Value) | 파티션과 파티션의 경계가 되는 값 |
| 2-value BVA | 경계-1 + 경계 (2개 값으로 경계 테스트) |
| 3-value BVA | 경계-1 + 경계 + 경계+1 (3개 값으로 더 철저히) |
| Off-by-One 오류 | >= 대신 >와 같이 경계 조건을 1만큼 잘못 지정하는 오류 |

### 5. 실무 제언

**도메인 분할이 명확하지 않을 때**

실무에서는 요구사항이 "19세에서 60세"처럼 명확하게 주어지지 않을 때가 많습니다. 요구사항 분석가와 협력하여 동등 클래스 다이어그램을 그리면서 경계를 명확히 하는 작업이 테스트 설계 전에 반드시 선행되어야 합니다. 이 과정에서 모호한 요구사항이 조기에 발견됩니다.

**다차원 동등 분할**

실제 시스템은 단일 입력이 아닌 여러 입력 변수가 조합됩니다. 예를 들어 나이와 학력을 동시에 고려하는 채용 기준이라면, 두 변수의 파티션을 조합하는 **다차원 동등 분할**이 필요하고, 이때는 결정 테이블(Decision Table) 기법과 연계하여 테스트 케이스 수를 효율적으로 관리해야 합니다.
