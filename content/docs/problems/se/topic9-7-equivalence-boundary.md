## 문제

동등분할 테스트와 경곗값 분석 테스트에 대해서 설명하고, 아래 명세서를 보고 **동등분할 테스트 케이스와 경곗값 분석 테스트 케이스**를 설계하시오.

> (단, 경곗값 테스트의 경우 2-value 기준)
>
> **[명세]** 공기업 채용 기준에 따라 19세에서 60세까지만 고용할 수 있다.
> (단, 19세~20세는 인턴으로 고용해야 하고, 21세~60세는 정규 직원으로 고용해야 한다.)

## 해설

### 출제 배경 및 의도

소프트웨어 결함의 상당 부분은 경계 조건 처리 오류에서 발생한다. `if (age >= 19)` 대신 `if (age > 19)`로 코딩하면 19세가 처리에서 제외되는 Off-by-One 오류가 대표적 사례이다. 기술사 시험에서 이 주제는 블랙박스 테스트의 이론과 실전 케이스 설계를 동시에 검증하는 응용형 문제로 자주 출제된다.

채점관은 동등 분할(Equivalence Partitioning)과 경계값 분석(Boundary Value Analysis)의 원리를 정확히 이해하고, 주어진 명세로부터 2-value BVA 기준의 실제 테스트 케이스 표를 올바르게 설계할 수 있는지를 평가한다. 단순 암기를 넘어 실무 적용 능력을 검증하는 문제이므로, 테스트 케이스 설계 표를 명확히 작성하는 것이 고득점의 핵심이다.

### 1. 동등 분할 및 경계값 분석 개요

정 의  • 동등 분할(EP)을 동일하게 처리될 입력을 파티션으로 분류하여 대표값으로 테스트하는 블랙박스 기법이라 하고, 경계값 분석(BVA)을 파티션 경계에서 결함이 집중 발생한다는 경험칙에 기반해 경계값을 집중 검증하는 기법이라 한다.
       - 두 기법은 상호 보완적으로 활용되며 최소 테스트 케이스로 최대 결함 검출을 목표로 한다.

```
[동등 분할 및 경계값 분석 구조도]

나이 입력 도메인: 0 ~ 무한
┌──────────┬────────────┬──────────────┬──────────┐
│  무효 EP  │  유효 EP   │   유효 EP    │  무효 EP  │
│ (나이<19) │ (19~20세)  │  (21~60세)   │ (나이>60) │
│ 고용불가  │  인턴 고용  │ 정규직 고용  │  고용불가 │
└──────────┴────────────┴──────────────┴──────────┘
            ↑           ↑             ↑
          B1(18|19)  B2(20|21)     B3(60|61)
           2-value BVA 경계점 (경계-1, 경계 테스트)
```

- 동등 분할은 테스트 케이스 수를 최소화하고, 경계값 분석은 Off-by-One 오류를 정밀하게 검출한다.

### 2. 핵심 개념 및 기법

1) 동등 분할(Equivalence Partitioning)

| 구분 | 항목 | 설명 |
|------|------|------|
| 유효 EP | Valid Equivalence Class | 시스템이 정상 처리해야 하는 입력 범위 |
| 무효 EP | Invalid Equivalence Class | 시스템이 오류 처리해야 하는 입력 범위 |
| 선택 원칙 | 대표값 1개 선택 | 동일 클래스 내 임의 값은 동일 결과를 기대 |
| 목표 | 테스트 최소화 | 각 클래스에서 1개만 테스트해도 동등 효과 |

- 명세: 공기업 채용 기준(19~60세) → 4개 파티션(무효/인턴/정규/무효) 식별

2) 경계값 분석 BVA 유형 비교

| 구분 | 2-value BVA | 3-value BVA |
|------|-------------|-------------|
| 테스트 값 | 경계-1, 경계 | 경계-1, 경계, 경계+1 |
| 케이스 수 | 경계당 2개 | 경계당 3개 |
| 적용 기준 | 문제 조건(본 문제) | 더 철저한 검증 필요 시 |
| Off-by-One 검출 | 가능 | 더 완전한 검출 |

- 2-value BVA는 경계 바로 아래(경계-1)와 경계 자체를 테스트하여 ">=" vs ">" 오류를 포착한다.

### 3. 명세 적용 — 테스트 케이스 설계

1) 동등 분할 테스트 케이스

```
[파티션 식별]
파티션 1 (무효): 나이 < 19  → 고용 불가
파티션 2 (유효): 19 ≤ 나이 ≤ 20  → 인턴 고용
파티션 3 (유효): 21 ≤ 나이 ≤ 60  → 정규 직원 고용
파티션 4 (무효): 나이 > 60  → 고용 불가
```

| TC | 파티션 | 입력값(나이) | 예상 출력 | 유효/무효 |
|----|-------|------------|---------|---------|
| TC-EP-1 | 파티션 1 | 10 | 고용 불가 | 무효 |
| TC-EP-2 | 파티션 2 | 20 | 인턴 고용 | 유효 |
| TC-EP-3 | 파티션 3 | 35 | 정규 직원 고용 | 유효 |
| TC-EP-4 | 파티션 4 | 65 | 고용 불가 | 무효 |

- 4개 파티션에서 대표값 1개씩 선택하여 최소 테스트 케이스로 전체 입력 도메인을 커버한다.

2) 경계값 분석 테스트 케이스 (2-value BVA 기준)

```
[2-value BVA: 각 경계에서 경계-1과 경계값 테스트]
B1 (18|19): → {18, 19}   B2 (20|21): → {20, 21}   B3 (60|61): → {60, 61}
```

| TC | 경계점 | 입력값(나이) | 예상 출력 | 2-value 기준 |
|----|-------|------------|---------|------------|
| TC-BVA-1 | B1 하한 | 18 | 고용 불가 | 경계-1 |
| TC-BVA-2 | B1 하한 | 19 | 인턴 고용 | 경계 |
| TC-BVA-3 | B2 전환 | 20 | 인턴 고용 | 경계-1 |
| TC-BVA-4 | B2 전환 | 21 | 정규 직원 고용 | 경계 |
| TC-BVA-5 | B3 상한 | 60 | 정규 직원 고용 | 경계-1 |
| TC-BVA-6 | B3 상한 | 61 | 고용 불가 | 경계 |

- 3개 경계점 × 2-value = 6개 테스트 케이스로 Off-by-One 오류를 체계적으로 검출한다.  "끝"

### 실무 제언

**도메인 분할의 모호성 문제**
- **챌린지**: 실무 요구사항은 경계가 명확하지 않아 파티션 식별이 어렵다.
- **제언**: 테스트 설계 전 요구사항 분석가와 동등 클래스 다이어그램을 함께 작성하여 경계를 합의하면 모호한 요구사항이 조기에 발견된다.

**다차원 입력 조합 문제**
- **챌린지**: 실제 시스템은 나이·학력·경력 등 복수 변수가 조합되어 파티션 수가 폭발적으로 증가한다.
- **제언**: 결정 테이블(Decision Table) 기법과 연계하여 조합 폭발을 통제하고, 카테고리 분할(Category-Partition) 기법으로 독립 변수별 파티션을 체계적으로 설계한다.

**회귀 테스트에서의 경계값 활용**
- **챌린지**: 비즈니스 규칙 변경 시 기존 경계값 테스트 케이스가 무효화된다.
- **제언**: 경계값 테스트 케이스를 자동화 테스트(JUnit, pytest)로 관리하면 규칙 변경 시 즉시 검출이 가능하고 회귀 테스트 유지보수 비용을 최소화한다.
