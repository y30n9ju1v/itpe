## 해설

### 출제 배경 및 의도

소프트웨어는 물리적 결함이 눈에 보이지 않아 구매자가 품질을 직접 검증하기 어렵다. 이 정보 비대칭 문제를 해결하기 위해 국가 공인 시험기관이 표준화된 기준으로 평가하고 마크를 부여하는 제도가 SW 품질인증이다. 공공기관 SW 조달 시 GS 인증 제품에 대한 법적 우대가 보장되어 있어 실무 중요도가 매우 높다.

관리 유형 출제로, 채점관은 GS 인증의 법적 근거·절차·등급 체계와 함께 인증 기준이 되는 ISO/IEC 25010 품질 모델(8대 특성), 그리고 SW 프로세스 인증(SP/CMMI)과의 차이를 종합적으로 이해하는지 평가한다. "제3자 검증을 통한 정보 비대칭 해소"라는 제도 본질을 답안 서두에 명시하면 차별화 포인트가 된다.

### 1. 소프트웨어 품질인증 개요

정 의  • **SW 품질인증**을 독립 공인 시험기관이 표준화된 품질 기준으로 SW 제품을 평가하여 일정 수준 이상에 인증 마크를 부여하는 제3자 검증 제도
       - 국내 대표: TTA 주관 GS(Good Software) 인증
       - 법적 근거: 소프트웨어산업 진흥법 제22조

```
[국제 표준 → 국내 인증 연계 구조]

ISO/IEC 25010 (SW 제품 품질 모델)
        ↓
ISO/IEC 25023 (품질 측정 방법)
        ↓
TTA 시험 기준 (국내 적용)
        ↓
GS 인증 마크 부여 (유효기간 2년)
        ↓
조달청 우대 / 공공기관 우선 구매·수의계약 가능

[GS 인증 절차]

신청(개발사 → TTA) → 예비시험(시험 범위 협의)
  → 시험(4~8주, 품질 특성별 테스트)
  → 결과보고(시험성적서)
  → 인증위원회 심의
  → 인증서 발급(1등급/2등급)
  → 사후관리(변경 시 재시험)
```

- GS 인증은 제3자 검증으로 정보 비대칭을 해소하며, 공공조달 우선 구매·수의계약의 법적 근거가 된다.

### 2. 핵심 내용 상세

1) GS(Good Software) 인증 제도 개요

| 구분 | 항목 | 설명 |
|------|------|------|
| 주관기관 | TTA(한국정보통신기술협회) | 행정안전부 위탁 |
| 법적 근거 | SW산업 진흥법 제22조 | 공공기관 우선 구매 의무 |
| 인증 대상 | 패키지·임베디드·모바일 앱 SW | 버전별 인증 |
| 유효기간 | 2년 | 버전 변경 시 재인증 필요 |
| 인증 등급 | 1등급(최우수), 2등급(우수) | 품질 수준에 따라 차등 부여 |

2) 인증 기준: ISO/IEC 25010 품질 특성 (8대 특성)

| 품질 특성 | 설명 | 세부 특성 예시 |
|-----------|------|---------------|
| 기능 적합성 (Functional Suitability) | 명시된 기능을 올바르게 수행하는가 | 기능 완전성·정확성 |
| 성능 효율성 (Performance Efficiency) | 자원(시간·메모리) 사용이 효율적인가 | 시간 반응성·자원 활용성 |
| 호환성 (Compatibility) | 다른 환경·시스템과 잘 작동하는가 | 공존성·상호운용성 |
| 사용성 (Usability) | 사용자가 쉽고 효율적으로 사용 가능한가 | 학습성·운용성·오류 방지성 |
| 신뢰성 (Reliability) | 장애 없이 안정적으로 동작하는가 | 성숙성·가용성·결함 허용성 |
| 보안성 (Security) | 비인가 접근으로부터 보호되는가 | 기밀성·무결성·부인방지성 |
| 유지보수성 (Maintainability) | 수정·개선이 쉬운가 | 모듈성·재사용성·분석성 |
| 이식성 (Portability) | 다른 환경으로 쉽게 이전되는가 | 적응성·설치성 |

- 공공조달 우선 구매·수의계약: 국가기관·지방자치단체 GS 인증 제품 우선 구매 권고, 일정 금액 이하 수의계약 허용, 나라장터 등록 가능.

### 3. SW 품질인증 vs 프로세스 인증 비교 및 트렌드

1) SW 품질인증(GS) vs 프로세스 인증(SP/CMMI) 비교

| 구분 | SW 품질인증 (GS) | SW 프로세스 인증 (CMMI·SP) |
|------|----------------|--------------------------|
| 인증 대상 | 소프트웨어 제품 | 소프트웨어 개발 프로세스 |
| 주요 관심사 | 결과물(Product) 품질 | 과정(Process) 성숙도 |
| 평가 기준 | ISO/IEC 25010 | CMMI 5단계·ISO/IEC 33000 |
| 주관기관 | TTA | KOSA(한국SW산업협회) |
| 조달 효과 | 우선 구매·수의계약 | 기술 평가 가점 |
| 비유 | 완성된 음식의 맛 평가 | 주방 시스템·레시피 평가 |

2) 국내외 SW 품질 인증 비교

| 인증 | 주관 | 특징 |
|------|------|------|
| GS 인증 (국내) | TTA | 공공조달 우대, 2년 유효 |
| Common Criteria (국제) | 국가정보원(CC인증) | 보안 제품 특화 |
| UL Certification (미국) | UL | 안전성 중심 |

- GS 인증과 CMMI/SP 인증은 상호 보완적이며, 공공 입찰에서 두 인증을 모두 보유한 기업이 가장 유리한 경쟁 지위를 확보한다.  "끝"

### 실무 제언

**중소 SW 기업의 GS 인증 취득 전략**
- **챌린지**: GS 인증 시험은 4~8주 소요 및 상당한 비용이 필요하여 스타트업·중소 SW 기업에게 실질적 부담이 된다.
- **제언**: 공공 조달 시장 타깃 제품이라면 GS 인증은 사실상 필수이다. TTA 사전 상담 제도를 활용하고, 인증 범위를 핵심 기능 모듈에 한정하는 모듈형 인증 전략으로 비용을 최소화할 수 있다.

**품질 특성 측정의 객관성 확보**
- **챌린지**: 사용성·유지보수성 등 일부 품질 특성은 측정자 주관이 개입되어 시험 결과의 일관성이 저하될 수 있다.
- **제언**: ISO/IEC 25023(측정법) 기반의 정량적 측정 지표를 사전에 개발사와 협의하고, 동일 시험 환경(OS·브라우저·버전) 고정 후 자동화 테스트 도구를 적용하면 측정 신뢰성을 높일 수 있다.

**AI·클라우드 서비스의 GS 인증 확장**
- **챌린지**: 전통적 패키지 SW 중심으로 설계된 GS 인증 체계가 SaaS·AI 서비스의 동적 특성(지속적 업데이트·확률적 출력)을 평가하기에 한계가 있다.
- **제언**: 클라우드 서비스 품질 특성(가용성 SLA·데이터 이식성·멀티테넌시 격리)과 AI 품질 특성(정확도·공정성·설명 가능성)을 ISO/IEC 25010 확장 체계에 포함하는 인증 기준 현대화가 필요하다.
