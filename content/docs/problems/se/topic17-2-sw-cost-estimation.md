## 해설

### 출제 배경 및 의도

"이 프로젝트에 얼마가 들 것인가?"는 SW 개발에서 가장 어렵고 중요한 질문이다. 잘못된 비용 산정은 프로젝트 실패·계약 분쟁·품질 저하로 직결된다. 공공 SW 사업의 예산 편성·입찰 원가 산정·변경 관리 모든 단계에서 비용 산정 역량이 필수이다.

채점관은 하향식·상향식·알고리즘 3분류 체계를 명확히 이해하고, 각 방법의 적용 시점과 장단점을 "불확실성 콘(Cone of Uncertainty)"의 맥락에서 설명할 수 있는지 평가한다. 단순 암기보다 복합 활용 전략과 실무 판단 논리 서술이 고득점의 핵심이다.

### 1. SW 비용 산정방법 개요

정 의  • **SW 비용 산정**을 프로젝트 개발·운영에 소요되는 비용을 체계적으로 예측하는 방법론
       - 본질: 불확실성 관리 — 초기 불확실성이 크고 산정 오차도 크며, 진행될수록 정확도 향상

```
[불확실성 콘과 산정 방법 적합성]

프로젝트 초기 (불확실성 高)           프로젝트 후기 (불확실성 低)
←──────────────────────────────────────────────────→
│   개념화    │  요구사항   │   설계   │   구현   │
│             │             │          │          │
│ 하향식      │ 하향식/FP   │  FP      │ 상향식   │
│ 전문가·델파이│ COCOMO II  │ 상세 FP  │ WBS 기반 │
│             │             │          │          │
│ 오차 ±4배  │ 오차 ±2배  │ ±1.5배  │ ±1.25배 │

[SW 비용 산정 방법 분류 체계]

SW 비용 산정 방법
├── 하향식 (Top-down)
│   ├── 전문가 판단 (Expert Judgment)
│   └── 델파이 기법 (Delphi Technique)
├── 상향식 (Bottom-up)
│   └── WBS 기반 세부 산정
└── 알고리즘 방식 (Algorithmic/Parametric)
    ├── LOC 기반: COCOMO, COCOMO II
    └── FP 기반: 기능점수 × 단가
```

- 실무에서는 단일 방법이 아닌 복합 활용(상향식 WBS + FP 기반 교차 검증)으로 신뢰성을 확보한다.

### 2. 핵심 기법별 상세 내용

1) 하향식 방법 (Top-down)

| 구분 | 전문가 판단 | 델파이 기법 |
|------|------------|------------|
| 방법 | 경험 많은 PM·아키텍트의 직관적 산정 | 복수 전문가 익명 설문 → 수렴 |
| 장점 | 빠른 산정, 암묵지 활용 | 집단사고 방지, 객관성 향상 |
| 단점 | 주관적·낙관적 편향 | 시간 소요, 전문가 확보 어려움 |
| 적합 | 초기 개념 단계·예비 타당성 조사 | 요구사항 불명확 단계·대형 프로젝트 |

```
[델파이 기법 절차]

전문가 A ──┐
전문가 B ──┼──→ 익명 응답 ──→ 조정자 취합 ──→ 결과 피드백
전문가 C ──┘         ↑                               │
                      └───────────────────────────────┘
                                (수렴할 때까지 반복)
```

2) 상향식 방법 (Bottom-up)

| 구분 | 항목 | 설명 |
|------|------|------|
| 방법 | WBS 기반 세부 공수 산정 | 최소 작업 단위까지 분해 후 합산 |
| 장점 | 높은 정확도·작업별 원가 추적 가능 | 산정 근거 명확 |
| 단점 | 시간 소요·상세 요구사항 필요 | WBS 작성 역량 필요 |
| 적합 | 요구사항 분석 완료 후·상세 설계 단계 | - |

```
[상향식 산정 구조 예시]

프로젝트 전체 (100 MM)
├── 요구분석 (10 MM): 인터뷰 3 + 문서화 4 + 검토 3
├── 설계 (30 MM): 아키텍처 설계 10 + 상세 설계 20
└── 구현 (60 MM): 모듈 A 20 + 모듈 B 40
```

3) 알고리즘 방식 — COCOMO II

| 구분 | COCOMO (1981) | COCOMO II (1995~) |
|------|--------------|------------------|
| 기반 | LOC 중심 | FP + LOC 모두 지원 |
| 개발 방법론 | 폭포수 모델 가정 | 나선형·프로토타입 지원 |
| 재사용 처리 | 없음 | 재사용 코드 보정 계수 포함 |

COCOMO II 3가지 세부 모델:

| 모델 | 적용 단계 | 산정 입력값 | 정확도 |
|------|----------|-----------|--------|
| Application Composition | 요구사항 초기 | Object Points(화면 수 등) | ±4배 |
| Early Design | 개념·아키텍처 설계 | UFP(미조정 FP) | ±2배 |
| Post-Architecture | 상세 설계 이후 | LOC, FP | ±1.5배 |

```
[COCOMO II Post-Architecture 기본 공식]

공수(MM) = A × Size^E × Π(EMi)

  A: 상수(보통 2.94)
  Size: KSLOC(천 줄 소스 코드) 또는 FP
  E: 규모 지수(1.01~1.26)
  EMi: 비용 동인(Cost Driver) 보정 계수 17가지
       (신뢰성 요구·DB 크기·개발자 역량 등)
```

### 3. 종합 비교 및 복합 활용 전략

1) 3가지 방법 종합 비교

| 구분 | 하향식(전문가) | 하향식(델파이) | 상향식(WBS) | 알고리즘(COCOMO/FP) |
|------|--------------|--------------|------------|---------------------|
| 적용 시기 | 초기 | 초기~중기 | 중기~후기 | 중기~후기 |
| 정확도 | 낮음 | 중간 | 높음 | 중간~높음 |
| 소요 시간 | 짧음 | 중간 | 길음 | 중간 |
| 객관성 | 낮음 | 중간 | 높음 | 높음 |
| 주요 단점 | 주관적 편향 | 전문가 확보 어려움 | WBS 상세화 필요 | 파라미터 보정 어려움 |

2) 복합 활용 전략

```
[실무 복합 활용 전략]

초기 단계:    델파이 기법 → 개략 비용 범위 설정
중기 단계:    FP 기반 COCOMO II → 알고리즘 산정
후기 단계:    상향식 WBS 기반 상세 산정
교차 검증:    두 방법 결과 차이 15% 이내 → 신뢰
             차이 15% 초과 → 원인 분석·재산정
```

- PERT 3점 산정(공수 = (O+4M+P)/6)을 WBS와 결합하면 불확실성을 정량화하여 리스크 버퍼를 합리적으로 설정할 수 있다.  "끝"

### 실무 제언

**복합 산정과 교차 검증**
- **챌린지**: 단일 방법으로는 신뢰성 있는 산정이 어렵고, 상향식 WBS만 믿었다가 누락된 작업이 나중에 발견되는 "누락 오류"가 빈번하다.
- **제언**: 상향식 WBS 산정 + FP 기반 알고리즘 교차 검증을 병행하고, 두 결과 차이가 15% 이내면 신뢰, 초과 시 원인 분석 후 재산정하는 프로세스가 효과적이다.

**애자일 환경에서의 비용 산정**
- **챌린지**: 스프린트 기반 애자일 프로젝트에서는 초기 전체 비용 산정이 어렵고, 요구사항이 변경될 때마다 산정값이 달라져 공공 사업 계약 구조와 충돌이 발생한다.
- **제언**: 스토리 포인트(Story Point) 기반 벨로시티(Velocity) 추적으로 릴리스 계획을 수립하고, FP 환산(스토리 포인트 × FP 전환 계수)으로 공공 계약 대가 산정 기준과 연계하는 하이브리드 접근이 현실적이다.

**COCOMO II 파라미터 보정의 어려움**
- **챌린지**: COCOMO II의 17가지 비용 동인(EMi) 값은 조직 내 과거 프로젝트 데이터가 없으면 기본값(Nominal)을 적용할 수밖에 없어 모델 정확도가 낮아진다.
- **제언**: 프로젝트 완료 후 실적 데이터(실제 공수·FP)를 체계적으로 수집하고, 3~5개 프로젝트 이상 데이터가 축적되면 조직 특화 COCOMO II 캘리브레이션(Calibration)을 수행하여 산정 정확도를 ±1.25배 수준으로 향상시킬 수 있다.
