## 해설

> **개발 없이 구매로 조달하는 상용 소프트웨어, 공공기관이 직접 살 수 있는 제도**  
> 대형 IT 통합 사업자를 거치지 않고 수요기관이 상용 SW를 직접 구매하는 공공조달 혁신

기존에는 공공기관이 Microsoft Office나 Oracle DB 같은 상용 SW를 구매할 때도 시스템 통합(SI) 사업자가 납품하는 구조여서, 중간 마진이 발생하고 납품 가격이 불투명했습니다. 상용 SW 직접구매 제도는 수요기관이 조달청 나라장터·디지털서비스몰을 통해 SW 제조사(또는 공식 대리점)와 직접 계약하는 방식으로, 가격 투명성을 높이고 SW 기업의 수익성을 개선하는 공공조달 혁신 제도입니다.

### 1. 출제 배경 및 의도

국내 SW 생태계의 고질적 문제인 "SI 종속", "불공정 하도급 구조"를 해결하기 위한 정책 제도입니다. 채점관은 직접구매 제도의 개념, 기존 구매 방식과의 차이, 조달 절차, 라이선스 유형 등을 종합적으로 설명할 수 있는지 평가합니다.

### 2. 개념의 본질과 작동 메커니즘

상용 SW 직접구매 제도의 본질은 **"유통 단계 단축을 통한 가격 투명성 확보"**입니다.

```text
[기존 구매 방식 vs 직접구매 방식]

기존 방식 (간접 구매):
  SW 제조사 → SI/대리점 → 수요기관
                   ↑
              중간 마진 발생
              가격 불투명
              SW 기업 수익 감소

직접구매 방식:
  SW 제조사 ──────────────→ 수요기관
  (또는 공식 유통사)          ↑
                         나라장터/
                         디지털서비스몰 통해
                         직접 계약

효과:
  → 중간 마진 제거 → 가격 인하
  → 가격 공개 → 투명성 확보
  → SW 기업 직접 수익 확보 → 생태계 건전화
```

### 3. 핵심 키워드 (Must-Have)

#### 1) 상용 SW 직접구매 제도 개요

**정의**: 수요기관이 상용 소프트웨어를 구매할 때, 하드웨어 또는 SI(시스템 통합) 사업에 포함하지 않고 소프트웨어만을 분리하여 나라장터 또는 디지털서비스몰을 통해 구매하는 제도.

**법적 근거**: 「소프트웨어산업 진흥법」, 「국가를 당사자로 하는 계약에 관한 법률」, 조달청 고시

**주관기관**: 과학기술정보통신부, 조달청

| 항목 | 내용 |
|------|------|
| 적용 대상 | 모든 국가기관·지방자치단체·공공기관 |
| 구매 채널 | 나라장터(G2B), 디지털서비스몰 |
| 의무 적용 범위 | SW 구매금액이 일정 금액 이상인 경우 직접구매 권장/의무 |
| 제도 목적 | 가격 투명성, SW 기업 생태계 건전화, 발주자 선택권 강화 |

#### 2) 직접구매 vs 일반(간접) 구매 차이

| 구분 | 일반 구매 (간접) | 직접구매 |
|------|--------------|---------|
| **거래 구조** | 수요기관 → SI/대리점 → SW사 | 수요기관 → SW사(또는 공식 유통사) |
| **가격 결정** | SI가 협상 (불투명) | 등록 가격 기준 (투명) |
| **계약 유형** | SW가 HW·SI에 포함 | SW 단독 계약 |
| **수요기관 선택권** | 제한적 (SI 종속) | 확대 (제품 직접 선택) |
| **SW 기업 수익** | SI 마진 제외 후 수취 | 직접 수취 |
| **AS/지원** | SI가 중간 처리 | SW사 직접 지원 |

#### 3) 공공 SW 조달 절차 (직접구매)

```text
[직접구매 표준 조달 절차]

① 수요기관 구매 계획 수립
   → 필요한 SW 제품, 수량, 예산 확정

② 제품 검색 및 선정
   → 나라장터(G2B) 또는 디지털서비스몰에서 등록 제품 검색
   → GS 인증 여부, 가격, 사양 비교

③ 구매 요청
   → 수요기관 내부 결재 (예산 집행 결의)

④ 계약 체결
   → 조달청 계약 또는 기관 자체 계약
   → SW사(또는 공식 대리점)와 직접 계약

⑤ 납품·검수
   → 라이선스 키 발급 또는 설치 매체 납품
   → 수요기관 검수 (정품 확인, 라이선스 수량 확인)

⑥ 사후 관리
   → 유지보수 계약 (선택적, 별도 계약)
   → 라이선스 갱신 관리
```

#### 4) 라이선스 유형별 특성

공공기관이 상용 SW를 구매할 때 선택하는 라이선스 유형에 따라 비용 구조와 조달 방식이 달라집니다.

| 라이선스 유형 | 특징 | 공공 조달 관점 |
|-------------|------|-------------|
| **영구 라이선스 (Perpetual)** | 한 번 구매 → 영구 사용권 | CAPEX 방식, 예산 1회 집행 |
| **구독 라이선스 (Subscription)** | 연간/월간 구독료 납부 | OPEX 방식, 연도별 예산 편성 필요 |
| **SaaS (Software as a Service)** | 클라우드 기반, 인터넷으로 사용 | 디지털서비스몰 통해 구매, 탄력적 확장 |
| **동시 사용자 (Concurrent User)** | 동시에 접속하는 사용자 수 기준 | 사용 패턴 분석 후 적정 수량 산정 필요 |
| **Named User** | 지정된 개인 사용자 수 기준 | 사용자 목록 관리 필요 |

```text
[라이선스 유형별 비용 흐름 비교]

영구 라이선스:
  Year 1: 구매비 (高) + 유지보수비
  Year 2~: 유지보수비만 (低)
  → 장기 사용 시 총 비용 有利

구독 라이선스:
  Year 1~N: 구독료 (균등)
  → 항상 최신 버전, 단기적으로 저렴

SaaS:
  사용량 기반 과금 → 오토스케일 가능
  → 클라우드 정부 전환 정책과 부합
```

#### 5) 디지털서비스몰 (신설 채널)

기존 나라장터가 HW 중심이었다면, **디지털서비스몰**은 클라우드·SaaS·AI 서비스 등 디지털 서비스 특화 조달 채널입니다.

| 채널 | 나라장터(G2B) | 디지털서비스몰 |
|------|------------|--------------|
| **대상** | 패키지 SW, HW | 클라우드, SaaS, AI 서비스 |
| **계약 방식** | 물품 구매 계약 | 서비스 이용 계약 |
| **가격 방식** | 단가 고정 | 종량제/구독 지원 |
| **기술 검증** | GS 인증 | 보안 인증, 클라우드 인증 |

### 4. 맥락과 비교

#### SW 직접구매 제도와 관련 제도의 연계

```text
[공공 SW 구매 제도 전체 구조]

상용 SW 직접구매 ─── 가격 투명성, 유통 단순화
       ↓
SW 분리발주 ────────── 기능별 최적 사업자 선택
       ↓
단계별(분석/구현) 발주 ─ 요구사항 명확화
       ↓
PMO(사업관리) 제도 ──── 발주자 역량 보완
       ↓
GS 인증 우선 구매 ──── 품질 검증된 제품 구매

→ 네 가지 제도가 상호 보완적으로 공공 SW 발주 생태계 개선
```

#### 클라우드 전환 정책과의 연계 (Cloud-First)

정부의 디지털 전환 정책("디지털 정부 실현")에 따라 공공기관은 SaaS·클라우드 서비스 우선 검토 의무가 강화되고 있습니다. 디지털서비스몰이 이 흐름의 핵심 채널 역할을 합니다.

### 5. 실무 제언

**구독형 SaaS 도입 시 예산 구조 변화 대응**

- **챌린지**: 전통적인 공공 예산은 연간 단위로 편성되며, 특히 대규모 CAPEX(자본 지출)에 익숙합니다. SaaS 구독 모델은 OPEX(운영 지출) 방식이어서 매년 예산 편성이 필요하고, 중간에 예산이 삭감되면 서비스 중단 위기가 발생합니다.
- **제언**: SaaS 도입 시 3~5년 장기 계약으로 단가를 낮추고, 예산 지속성을 법적으로 보장하는 다년도 계약(Multi-year Contract) 활용을 검토해야 합니다. 행정안전부의 클라우드 이용 지침에도 다년도 계약 근거가 마련되어 있습니다.

**기술사 시험 답안 작성 팁**

- 직접구매의 핵심 가치인 "유통 단계 단축 → 가격 투명성"을 다이어그램으로 표현하면 시각적 설득력이 높아집니다.
- 라이선스 유형 3가지(영구/구독/SaaS)를 비교표로 정리하면 체계적인 답안이 됩니다.
- "디지털서비스몰"의 등장과 클라우드 퍼스트 정책과의 연계를 언급하면 최신 트렌드를 반영한 고득점 답안이 됩니다.
