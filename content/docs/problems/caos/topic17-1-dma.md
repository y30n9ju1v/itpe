## 해설

> **CPU 없이 메모리와 I/O 장치가 직접 데이터를 주고받는 고속 전송 방식**  
> DMA 제어기 구조, 버스 사이클 스틸, 버스트 모드, 투명 모드

I/O 장치와 메모리 간 대량의 데이터를 전송할 때, CPU가 직접 바이트 단위로 복사하면 CPU 시간의 대부분을 I/O에 낭비합니다. **DMA(Direct Memory Access)**는 CPU 개입 없이 I/O 장치가 직접 메모리에 데이터를 쓰고 읽을 수 있게 하는 하드웨어 메커니즘입니다.

### 1. 출제 배경 및 의도

DMA는 고성능 I/O의 핵심이며, 네트워크 패킷 처리, 디스크 I/O, 오디오 스트리밍 등 실시간 데이터 전송의 기반입니다. 채점관은 **I/O 제어 방식의 진화(폴링→인터럽트→DMA)**, **DMA 제어기 내부 구조**, 그리고 **버스 사이클 스틸, 버스트, 투명 모드의 차이**를 이해하는지를 봅니다.

### 2. 개념의 본질과 작동 메커니즘

#### I/O 제어 방식의 진화

| 방식 | 동작 | CPU 부담 | 성능 |
|---|---|---|---|
| **폴링(Polling)** | CPU가 반복 확인 | 매우 높음 | 낮음 |
| **인터럽트 방식** | I/O 완료 시 인터럽트 | 중간 | 중간 |
| **DMA 방식** | DMA 컨트롤러가 독립 전송 | 낮음 | 높음 |
| **채널 I/O** | 독립 I/O 프로세서 | 매우 낮음 | 최고 |

#### 가. DMA 제어기 구조와 버스 연결 방식

**DMA 제어기 내부 구조:**

```
┌─────────────────────────────────────────────────────────┐
│                    DMA 제어기 (DMAC)                    │
│                                                          │
│  ┌──────────────────┐   ┌──────────────────────────┐   │
│  │  주소 레지스터   │   │  카운트 레지스터         │   │
│  │ (전송 시작 주소) │   │ (남은 전송 바이트 수)    │   │
│  └──────────────────┘   └──────────────────────────┘   │
│                                                          │
│  ┌──────────────────┐   ┌──────────────────────────┐   │
│  │  제어 레지스터   │   │  상태 레지스터           │   │
│  │ (방향, 모드 등)  │   │ (완료, 오류 상태)        │   │
│  └──────────────────┘   └──────────────────────────┘   │
│                                                          │
│  ┌──────────────────────────────────────────────────┐  │
│  │          버스 인터페이스 로직                    │  │
│  │  (버스 요청 BRQ, 버스 허가 BGA, 주소/데이터 버스)│  │
│  └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

**버스 연결 방식:**

```
┌──────────┐      버스 중재기      ┌──────────┐
│   CPU    │◀─── BRQ/BGA ──────▶│  DMAC    │
│          │                      │          │
└──────┬───┘                      └──────┬───┘
       │                                  │
       └──────────────┬───────────────────┘
                      │
              ┌───────▼────────┐
              │  시스템 버스   │
              │ (주소+데이터+제어)│
              └───────┬────────┘
                      │
               ┌──────▼──────┐
               │  주기억장치  │
               └─────────────┘
```

**BRQ(Bus Request):** DMAC가 CPU에게 버스 제어권 요청  
**BGA(Bus Grant Acknowledge):** CPU가 버스 제어권 양도 승인

#### 나. DMA 동작 모드와 처리 절차

**DMA 처리 절차:**

```
1. CPU → DMAC: DMA 설정
   - 주소 레지스터 ← 메모리 시작 주소
   - 카운트 레지스터 ← 전송 바이트 수
   - 제어 레지스터 ← 방향(읽기/쓰기), 장치 ID

2. CPU → I/O 장치: 데이터 전송 명령 (DMA 시작)

3. DMAC → CPU: BRQ (버스 사용 요청)
4. CPU → DMAC: BGA (버스 제어권 양도)
   - CPU는 버스 사용 중단, 내부 연산 계속 (또는 대기)

5. DMAC: 메모리 ↔ I/O 장치 직접 데이터 전송
   (CPU 개입 없이 독립적으로 수행)
   - 주소 레지스터 증가, 카운트 레지스터 감소
   - 반복...

6. 전송 완료 시 DMAC → CPU: 인터럽트 발생
7. CPU: DMA 완료 인터럽트 처리 (상태 확인 등)
8. CPU: 버스 제어권 복귀
```

**DMA 동작 모드:**

**① 사이클 스틸(Cycle Stealing) 모드**

```
CPU 버스 사용 사이클:  [C][C][C][C][C][C][C][C]
DMA 스틸 사이클:            [D]      [D]      [D]
결합 사용:             [C][C][D][C][C][D][C][D]
```

- DMAC가 버스 사이클을 하나씩 빼앗아(steal) 데이터를 전송
- CPU는 버스를 잠시 양보하지만 완전 정지는 아님
- 소량의 데이터를 간간이 전송할 때 적합

**② 버스트(Burst) 모드**

```
CPU:  [동작] ─────▶ [대기] ─────────────────▶ [동작 재개]
DMA:                  [연속 버스트 전송 ..........]
```

- DMAC가 버스를 독점하여 전체 데이터 블록을 한꺼번에 전송
- CPU는 DMA 완료까지 버스 사용 불가 (완전 정지)
- 대용량 고속 전송에 적합 (디스크 읽기, 비디오 스트림)
- CPU 반응성 저하 가능

**③ 투명(Transparent) 모드**

```
CPU:  [C][C][C][C][C][C][C] ← CPU가 버스를 사용하지 않는 사이클
DMA:     [D]   [D]   [D]   ← CPU 유휴 사이클에만 DMA 전송
```

- DMAC가 CPU의 버스 유휴(Idle) 사이클을 감지하여 그때만 전송
- CPU 성능에 전혀 영향을 주지 않음
- 하드웨어 복잡도 높음, 전송 속도 느릴 수 있음

**DMA 동작 모드 비교:**

| 모드 | CPU 영향 | 전송 속도 | 하드웨어 복잡도 |
|---|---|---|---|
| **사이클 스틸** | 약간 감소 | 중간 | 낮음 |
| **버스트** | 정지 | 빠름 | 낮음 |
| **투명** | 없음 | 느림 | 높음 |

### 3. 핵심 키워드 (Must-Have)

- **DMA(Direct Memory Access):** CPU 개입 없는 메모리-I/O 직접 전송
- **DMAC:** DMA 컨트롤러 (주소/카운트/제어 레지스터)
- **BRQ / BGA:** 버스 요청 / 버스 허가
- **사이클 스틸:** 버스 사이클 하나씩 빼앗아 전송
- **버스트 모드:** 버스 독점 후 블록 전체 전송
- **투명 모드:** CPU 유휴 사이클에만 전송
- **인터럽트:** DMA 완료 후 CPU에 알림
- **채널 I/O:** DMA의 진화형, 독립 I/O 프로세서

### 4. 맥락과 비교

**현대 DMA 기술의 진화:**

| 기술 | 설명 |
|---|---|
| **IOMMU** | I/O 장치의 DMA 주소 변환 및 보안 격리 |
| **DMA-BUF** | Linux의 DMA 버퍼 공유 프레임워크 |
| **RDMA (Remote DMA)** | 네트워크를 통한 원격 메모리 직접 접근 (InfiniBand, RoCE) |
| **PCIe P2P DMA** | CPU 경유 없이 GPU ↔ NVMe 직접 전송 |

### 5. 실무 제언

**AI 학습에서 RDMA와 GPUDirect의 활용**

- **챌린지:** 분산 AI 학습에서 GPU 간 데이터(그래디언트) 공유는 통신 병목의 핵심입니다. 기존 방식(GPU → CPU 메모리 → 네트워크 카드 → 네트워크 → 원격 CPU → 원격 GPU)은 데이터가 CPU를 두 번 거쳐 지연이 큽니다.
- **제언:** **NVIDIA GPUDirect RDMA**를 사용하면 GPU 메모리의 데이터가 CPU를 우회하여 직접 NIC를 통해 원격 GPU 메모리로 전송됩니다. 이는 DMA 원리의 최신 진화형으로, 지연을 수십 마이크로초에서 수 마이크로초로 줄이고 CPU를 AI 연산에만 집중시킵니다. 1000GPU 규모 AI 클러스터에서 이 기술의 활용 여부가 학습 효율을 2~5배 차이 나게 합니다.
