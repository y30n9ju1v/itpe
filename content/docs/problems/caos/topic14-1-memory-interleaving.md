## 해설

### 출제 배경 및 의도

메모리 인터리빙(Memory Interleaving)은 CPU와 메모리 간 속도 격차(Memory Gap)를 해소하기 위해 1960년대부터 활용된 고전적 기법입니다. 단일 뱅크 접근 시간을 개선하는 것이 아니라, 여러 뱅크를 병렬로 접근하여 전체 메모리 대역폭을 향상시키는 접근 방식으로, 현대 DDR 다채널 메모리·HBM·RAID 스트라이핑에 이르기까지 동일 원리가 적용됩니다.

채점관은 **저위/고위 인터리빙의 주소 배치 방식과 용도 차이**, **동시 접근에 의한 대역폭 향상 원리**, 그리고 **파이프라인·캐시와의 시너지 효과**를 이해하는지를 평가합니다.

### 1. 메모리 인터리빙(Memory Interleaving), 개요

정 의  • 메모리 인터리빙을 주기억장치를 k개의 독립 뱅크(Bank)로 분할하여 연속 주소를 각 뱅크에 분산 배치함으로써 동시 접근을 가능하게 하는 메모리 대역폭 향상 기법으로 정의함
       - 단일 뱅크 접근 시간(t)이 동일하더라도 k-way 인터리빙 시 이론적으로 k배 대역폭 향상 가능

```
4-Way 저위 인터리빙 구조도

CPU/캐시 컨트롤러
       │ 캐시 라인 적재 요청(주소 0~3)
       ├──────┬──────┬──────┐
       ▼      ▼      ▼      ▼
  ┌──────┐┌──────┐┌──────┐┌──────┐
  │Bank 0││Bank 1││Bank 2││Bank 3│
  │주소 0││주소 1││주소 2││주소 3│
  │주소 4││주소 5││주소 6││주소 7│
  │ ...  ││ ...  ││ ...  ││ ...  │
  └──────┘└──────┘└──────┘└──────┘
  t=0 동시 읽기 시작
  t=40ns 동시 완료 → 4워드를 단일 뱅크의 1/4 시간에 획득
```

- 저위 인터리빙은 캐시 라인 채우기(Cache Line Fill)에 최적화되어 공간 지역성을 최대한 활용함

### 2. 인터리빙 유형·접근 방식

1) 저위 인터리빙 (Low-Order Interleaving)

| 구분 | 항목 | 설명 |
|------|------|------|
| 주소 매핑 | Bank 번호 = 주소 mod k | 하위 비트로 뱅크 결정 |
| 연속 주소 배치 | 다른 뱅크에 분산 | 주소 0→Bank0, 1→Bank1, 2→Bank2, 3→Bank3 |
| 연속 접근 성능 | 우수(병렬 접근 가능) | 캐시 라인 4워드를 동시에 읽음 |
| 주요 용도 | 캐시 라인 채우기, 파이프라인 메모리 단계 | CPU 데이터 패치 최적화 |
| 뱅크 충돌 | stride=k 패턴에서 발생 | 동일 뱅크에 연속 접근 시 직렬화 |

- 4-Bank, 뱅크 접근 시간 40ns: 단일 뱅크 방식 160ns → 인터리빙 40ns (4배 향상)

2) 고위 인터리빙 (High-Order Interleaving)

| 구분 | 항목 | 설명 |
|------|------|------|
| 주소 매핑 | Bank 번호 = 주소 / (뱅크 크기) | 상위 비트로 뱅크 결정 |
| 연속 주소 배치 | 같은 뱅크에 집중 | 주소 0~3→Bank0, 4~7→Bank1 |
| 연속 접근 성능 | 불량(동일 뱅크 직렬 접근) | 순차 패치 병렬화 불가 |
| 주요 용도 | 멀티프로세서 독립 뱅크 | 각 프로세서가 전용 뱅크 사용 |
| 뱅크 충돌 | 낮음(독립 접근) | 프로세서 간 경쟁 최소화 |

- 멀티프로세서 시스템에서 각 CPU가 독립된 뱅크를 전담하면 뱅크 충돌을 최소화할 수 있음

3) 파이프라인 연계 접근(Pipeline Interleaving) 효과

| 구분 | 항목 | 설명 |
|------|------|------|
| 동작 방식 | 10ns 간격 뱅크 순차 개시 | Bank0 시작 후 10ns 뒤 Bank1 시작 |
| 뱅크 접근 시간 | 40ns | 겹쳐서 매 10ns마다 워드 공급 |
| CPU 수신 패턴 | 매 10ns마다 1워드 | 단일 뱅크 대비 4배 공급 속도 |
| 적합 시나리오 | 캐시 미스 시 캐시 라인 채우기 | 공간 지역성이 높은 순차 접근 |

- 파이프라인 인터리빙은 뱅크 접근 시간이 사이클 시간보다 클 때 특히 효과적임

### 3. 저위 vs 고위 인터리빙 비교 및 현대 적용 트렌드

| 구분 | 저위 인터리빙 | 고위 인터리빙 |
|------|--------------|--------------|
| 주소 결정 | 하위 비트(주소 mod k) | 상위 비트 |
| 연속 주소 배치 | 다른 뱅크에 분산 | 같은 뱅크에 집중 |
| 연속 접근 성능 | 우수(병렬 접근) | 불량 |
| 독립 접근 성능 | 충돌 가능(stride=k) | 우수(뱅크 독립) |
| 현대 적용 | DDR 채널 인터리빙, RAID 0 | NUMA 메모리 노드 분리 |
| AI 적용 | HBM 내부 채널 인터리빙 | GPU VRAM 독립 파티션 |

- 현대 DDR4/DDR5 2채널·4채널 구성이 저위 인터리빙 원리를 직접 적용하며, AI 학습 서버에서 채널 인터리빙 활성화만으로 메모리 대역폭이 2~4배 향상됨.  "끝"

### 실무 제언

**AI 학습 서버 메모리 채널 인터리빙 최적화**
- **챌린지**: Intel Xeon 서버에서 DIMM 슬롯을 불균등하게 채우면 채널 인터리빙이 비활성화되어 메모리 대역폭이 절반 이하로 떨어지며, AI 학습 처리량도 비례하여 감소함
- **제언**: 채널별로 DIMM을 균등하게 장착하고 `dmidecode -t memory` 명령으로 활성 채널 수를 확인하여 8채널 DDR5 구성을 완전히 활성화함으로써 AI 워크로드 성능을 20~50% 향상시킴

**GPU VRAM 뱅크 충돌 최소화**
- **챌린지**: CUDA 커널에서 stride 접근 패턴이 특정 메모리 뱅크에 집중되면 뱅크 충돌(Bank Conflict)이 발생하여 공유 메모리 성능이 급격히 저하됨
- **제언**: 패딩(Padding)을 추가하거나 접근 패턴을 stride=1로 재설계하여 뱅크 충돌을 제거하고, NVIDIA Nsight 프로파일러로 Bank Conflict Rate를 측정하여 최적화 효과를 검증함

**SSD NAND 다이 인터리빙 활용**
- **챌린지**: 단일 NAND Die 접근 지연(70~100μs)이 NVMe SSD 성능의 병목이 되어 기업용 스토리지에서 순차 읽기 속도가 이론값에 미치지 못하는 경우가 발생함
- **제언**: 컨트롤러가 여러 NAND Die를 인터리빙하여 동시에 읽기/쓰기를 수행하도록 설계하면 전체 처리량이 Die 수에 비례하여 향상되며, 메모리 인터리빙 원리가 스토리지 영역에도 동일하게 적용됨
