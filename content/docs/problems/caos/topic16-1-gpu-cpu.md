## 해설

> **소수의 강력한 코어 vs 수천 개의 단순 코어: CPU와 GPU의 설계 철학 차이**  
> 직렬 최적화와 병렬 최적화, 그리고 AI 시대의 GPGPU

AI와 딥러닝의 폭발적 성장과 함께 GPU(Graphics Processing Unit)는 단순한 그래픽 처리 장치를 넘어 현대 컴퓨팅의 핵심 연산 엔진으로 자리 잡았습니다. ChatGPT, 자율주행, 이미지 생성 AI 모두 수천~수만 개의 GPU 코어에 의존합니다.

### 1. 출제 배경 및 의도

AI 시대에 GPU의 중요성이 폭발적으로 높아진 만큼, 기술사 시험에서도 자주 출제됩니다. 채점관은 **GPU와 CPU의 설계 철학 차이(직렬 vs 병렬)**, **GPGPU의 개념과 등장 배경**, 그리고 **GPU가 딥러닝에 적합한 이유**를 논리적으로 연결하기를 기대합니다.

### 2. 개념의 본질과 작동 메커니즘

#### CPU와 GPU의 설계 철학 비교

```
CPU (직렬 처리 최적화)          GPU (병렬 처리 최적화)
┌────────────────────────┐     ┌──────────────────────────────────┐
│  코어 4~64개           │     │  코어 수천~수만 개               │
│                        │     │                                  │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐  │     │ ┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐│
│  │ 강한 ALU │         │     │ │단순 ALU가 수천 개...           ││
│  └──┘ └──┘ └──┘ └──┘  │     │ └─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘│
│                        │     │                                  │
│  ┌──────────────────┐  │     │  ┌────────────────────────────┐ │
│  │  L1/L2/L3 캐시  │  │     │  │  작은 캐시, 큰 메모리      │ │
│  │  (수십 MB)       │  │     │  │  대역폭 (HBM: 수 TB/s)    │ │
│  └──────────────────┘  │     │  └────────────────────────────┘ │
│                        │     │                                  │
│  ┌──────────────────┐  │     │  ┌────────────────────────────┐ │
│  │  복잡한 분기     │  │     │  │  단순/규칙적 연산 최적화   │ │
│  │  예측 로직       │  │     │  │  (행렬 곱셈 등)            │ │
│  └──────────────────┘  │     │  └────────────────────────────┘ │
└────────────────────────┘     └──────────────────────────────────┘
```

#### GPU (Graphics Processing Unit)

**원래 목적:** 3D 그래픽 렌더링에서 **수백만 개의 픽셀을 동시에 처리**하기 위해 설계

**그래픽 파이프라인 병렬 처리:**
```
3D 장면 → [버텍스 처리] → [래스터라이제이션] → [픽셀 처리] → [프레임 버퍼]
            수만 개 버텍스   수백만 개 픽셀 동시 처리
```

**GPU 아키텍처 핵심 요소 (NVIDIA 기준):**
- **CUDA Core (쿠다 코어):** 부동소수점/정수 연산 단순 코어 (수천~수만 개)
- **SM (Streaming Multiprocessor):** 수십 개의 CUDA 코어를 묶은 실행 단위
- **SIMT (Single Instruction Multiple Thread):** 하나의 명령어를 다수의 스레드가 동시 실행
- **Warp:** 32개 스레드의 실행 단위 (분기 발생 시 성능 저하: Warp Divergence)
- **VRAM:** 고대역폭 전용 메모리 (GDDR6: 수백 GB/s, HBM3: 수 TB/s)

#### GPGPU (General Purpose GPU)

**개념:** GPU를 그래픽이 아닌 **범용 병렬 컴퓨팅**에 활용하는 기술

**등장 배경:**
- 2006년 NVIDIA CUDA 출시 → GPU의 병렬 처리 능력을 일반 프로그래밍에 개방
- 딥러닝의 핵심 연산인 행렬 곱셈이 GPU의 SIMT 실행 방식과 완벽하게 부합

**GPGPU 활용 영역:**

| 분야 | 설명 |
|---|---|
| **딥러닝 학습** | 수십억 파라미터의 행렬 곱셈/역전파 계산 |
| **과학 계산** | 기상 시뮬레이션, 분자 동역학, CFD |
| **암호화폐 채굴** | SHA-256 해시 계산 (반복적 단순 연산) |
| **컴퓨터 비전** | 이미지 처리, 특징 추출 |
| **금융 모델링** | 몬테카를로 시뮬레이션 |

**GPGPU 프로그래밍 프레임워크:**
- **CUDA (NVIDIA):** GPU 병렬 프로그래밍의 사실상 표준
- **OpenCL:** 벤더 중립적 오픈 표준
- **ROCm (AMD):** AMD GPU용 오픈소스 플랫폼
- **SYCL / HIP:** 이식성 높은 이기종 컴퓨팅 프레임워크

#### CPU vs GPU 상세 비교

| 비교 항목 | CPU | GPU |
|---|---|---|
| **코어 수** | 4~64개 (고성능) | 수천~수만 개 |
| **코어 설계** | 복잡 (분기 예측, OoO 실행) | 단순 (SIMT 실행) |
| **클럭 속도** | 3~6GHz | 1~3GHz |
| **캐시** | 크고 지능적 (수십 MB L3) | 작고 단순 (공유 L2) |
| **메모리 대역폭** | ~100 GB/s | ~1 TB/s (HBM3) |
| **적합 워크로드** | 순차적, 복잡한 분기, 저지연 | 대규모 병렬, 규칙적 데이터 |
| **프로그래밍** | 범용 (C, Java, Python) | CUDA, OpenCL 필요 |
| **전력 효율** | 높음 (낮은 TDP) | 낮음 (수백 W) |

### 3. 핵심 키워드 (Must-Have)

- **GPU:** 수천 개의 단순 코어, 병렬 그래픽/연산 처리
- **GPGPU:** GPU를 범용 병렬 계산에 사용
- **CUDA:** NVIDIA의 GPGPU 프로그래밍 플랫폼
- **SIMT(Single Instruction Multiple Thread):** GPU의 실행 모델
- **Warp / Warp Divergence:** GPU 스레드 그룹, 분기 시 성능 저하
- **SM(Streaming Multiprocessor):** GPU의 실행 단위 (CUDA 코어 집합)
- **HBM(High Bandwidth Memory):** GPU 전용 고대역폭 메모리
- **텐서 코어(Tensor Core):** 행렬 연산 전용 하드웨어 (NVIDIA Volta 이후)
- **CPU-GPU 협력:** CPU가 제어(직렬 로직), GPU가 대규모 병렬 연산

### 5. 실무 제언

**LLM 추론 최적화에서 CPU-GPU 협력 구조**

- **챌린지:** GPT-4급 초거대 언어 모델(LLM)의 추론 시 모델 가중치가 수백 GB에 달해 단일 GPU VRAM을 초과합니다. 또한 LLM의 Attention 연산은 토큰 수에 O(n²)으로 늘어나므로 GPU 메모리와 대역폭 모두 병목이 됩니다.
- **제언:** **LLM 추론 최적화 기법:**
  1. **모델 양자화(Quantization):** FP16 → INT8/INT4로 메모리 절반~1/4 절감
  2. **텐서 병렬(Tensor Parallelism):** 모델 레이어를 여러 GPU에 분산
  3. **KV 캐시 최적화(PagedAttention):** vLLM 방식으로 Attention KV 캐시를 동적 관리
  4. **CPU 오프로드:** GPU에 적재 불가한 레이어를 CPU RAM에 유지하고 필요 시 GPU로 이동
  GPU-CPU 협력의 원리를 이해해야 이러한 현대적 AI 인프라 설계가 가능합니다.
