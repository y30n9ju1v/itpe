## 해설

### 출제 배경 및 의도

GPU(Graphics Processing Unit)는 3D 게임 렌더링을 위해 개발되었으나, 2006년 NVIDIA CUDA 출시를 기점으로 범용 병렬 컴퓨팅(GPGPU)의 핵심 엔진으로 진화하였습니다. ChatGPT, 자율주행, 의료 AI 등 현대 AI 시스템이 모두 GPU에 의존하면서, 기술사 시험에서도 GPU-CPU 차이와 GPGPU 개념이 고빈도 출제 주제가 되었습니다.

채점관은 **CPU와 GPU의 설계 철학 차이(직렬 vs 병렬 최적화)**, **GPGPU의 개념과 등장 배경**, 그리고 **GPU가 딥러닝에 적합한 이유(행렬 곱셈과 SIMT의 부합성)**를 논리적으로 연결하기를 기대합니다.

### 1. GPU·GPGPU, 개요

정 의  • GPU를 수천~수만 개의 단순 코어로 대규모 병렬 데이터 처리에 특화된 프로세서로 정의함
       - GPGPU(General Purpose GPU)를 GPU의 병렬 연산 능력을 그래픽 이외의 범용 컴퓨팅에 활용하는 기술로 정의함

```
CPU vs GPU 설계 철학 비교 구성도

CPU (직렬·저지연 최적화)          GPU (병렬·고처리량 최적화)
┌──────────────────────┐         ┌──────────────────────────────┐
│ ┌─────┐ ┌─────┐      │         │ SM(Streaming Multiprocessor) │
│ │강력한│ │강력한│      │         │ ┌──┬──┬──┬──┬──┬──┬──┬──┐  │
│ │코어 1│ │코어 2│      │         │ │C │C │C │C │C │C │C │C │  │
│ └─────┘ └─────┘      │         │ └──┴──┴──┴──┴──┴──┴──┴──┘  │
│                      │         │ (1 SM = 수십~수백 CUDA 코어) │
│ ┌──────────────────┐ │         │ × 수십 개 SM = 수천 코어     │
│ │대형 L1/L2/L3 캐시│ │         │                              │
│ │복잡한 분기 예측   │ │         │ ┌──────────────────────────┐│
│ │비순서 실행 로직   │ │         │ │HBM3: 수 TB/s 메모리 대역폭││
│ └──────────────────┘ │         │ └──────────────────────────┘│
│ 코어 수: 4~64개      │         │ 코어 수: 수천~수만 개        │
│ 클럭: 3~6GHz        │         │ 클럭: 1~3GHz                │
└──────────────────────┘         └──────────────────────────────┘
```

- CPU는 소수의 강력한 코어로 복잡한 순차 작업을, GPU는 수천 개의 단순 코어로 규칙적 병렬 연산을 최적화함

### 2. GPU 구조·GPGPU 개념·CPU-GPU 비교

1) GPU 아키텍처 핵심 요소 (NVIDIA 기준)

| 구분 | 요소 | 설명 |
|------|------|------|
| 실행 단위 | CUDA Core | 부동소수점/정수 연산 단순 코어(수천~수만 개) |
| 실행 묶음 | SM(Streaming Multiprocessor) | 수십~수백 CUDA 코어의 실행 단위 |
| 실행 모델 | SIMT(Single Instruction Multiple Thread) | 하나의 명령어를 32개 스레드(Warp)가 동시 실행 |
| 스레드 그룹 | Warp | 32개 스레드 단위; 분기 시 Warp Divergence로 성능 저하 |
| 전용 메모리 | VRAM(HBM3) | 수 TB/s 대역폭, CPU 대비 10배 이상 |
| AI 전용 | Tensor Core | 행렬 곱셈 전용 하드웨어(NVIDIA Volta 이후) |

- SIMT 실행 모델은 딥러닝의 핵심 연산인 행렬 곱셈(GEMM)과 완벽하게 부합하여 GPU가 AI 가속기로 채택된 근본 이유가 됨

2) GPGPU 활용 영역 및 프레임워크

| 구분 | 분야 | 설명 |
|------|------|------|
| AI/ML | 딥러닝 학습·추론 | 수십억 파라미터 행렬 곱셈·역전파 계산 |
| 과학 계산 | 기상·분자 동역학·CFD | 대규모 편미분방정식 병렬 풀이 |
| 암호화폐 | SHA-256 해시 | 반복적 단순 연산의 대량 병렬 처리 |
| 금융 | 몬테카를로 시뮬레이션 | 수백만 시나리오 동시 계산 |
| 컴퓨터 비전 | 이미지 처리·특징 추출 | 컨볼루션 연산의 병렬 처리 |

- GPGPU 프로그래밍 프레임워크: CUDA(NVIDIA 사실상 표준), OpenCL(벤더 중립), ROCm(AMD 오픈소스), SYCL/HIP(이식성 높은 이기종 컴퓨팅)

3) CPU vs GPU 상세 비교

| 비교 항목 | CPU | GPU |
|-----------|-----|-----|
| 코어 수 | 4~64개(고성능) | 수천~수만 개 |
| 코어 설계 | 복잡(분기 예측, OoO 실행) | 단순(SIMT 실행) |
| 클럭 속도 | 3~6GHz | 1~3GHz |
| 캐시 | 크고 지능적(수십 MB L3) | 작고 단순(공유 L2) |
| 메모리 대역폭 | ~100 GB/s | ~4 TB/s(HBM3) |
| 적합 워크로드 | 순차적·복잡 분기·저지연 | 대규모 병렬·규칙적 데이터 |
| 프로그래밍 | 범용(C, Java, Python) | CUDA, OpenCL 필요 |
| 전력 소비 | 낮음(수십 W) | 높음(수백 W) |

- CPU는 제어 흐름이 복잡한 직렬 로직, GPU는 동일 연산을 대량 데이터에 적용하는 병렬 처리에 각각 최적화되어 있으며, 현대 AI 시스템은 두 프로세서의 협력 구조로 동작함

### 3. AI 가속기 생태계 비교 및 트렌드

| 가속기 | 설계 방향 | 특징 |
|--------|-----------|------|
| NVIDIA H100/H200 | GPGPU + Tensor Core | 범용 딥러닝, 생태계 최강 |
| Google TPU v5 | 행렬 곱셈 전용 | TensorFlow/JAX 최적화 |
| Apple Neural Engine | 추론 전용(저전력) | 온디바이스 AI, 모바일 최적화 |
| AMD Instinct MI300X | CPU+GPU 통합(HBM) | LLM 추론 고성능 |
| Cerebras WSE-3 | 웨이퍼 스케일 칩 | 초거대 모델 단일 칩 학습 |

- GPU-CPU 이분법에서 CPU+GPU 통합(AMD MI300, Intel Gaudi) 방향으로 진화하며, LLM 시대에 메모리 대역폭이 연산 능력 못지않게 중요해지고 있음.  "끝"

### 실무 제언

**LLM 추론 서비스에서의 CPU-GPU 협력 최적화**
- **챌린지**: GPT-4급 LLM의 가중치가 수백 GB에 달해 단일 GPU VRAM을 초과하며, Attention 연산이 토큰 수에 O(n²)으로 증가하여 GPU 메모리와 대역폭 모두 병목이 됨
- **제언**: 모델 양자화(FP16→INT8/INT4), 텐서 병렬(Tensor Parallelism), PagedAttention(vLLM)으로 KV 캐시를 동적 관리하고, GPU 미적재 레이어는 CPU RAM에 오프로드하는 계층적 협력 구조를 설계함

**Warp Divergence 방지로 GPU 효율 극대화**
- **챌린지**: CUDA 커널 내에서 if-else 분기가 Warp 내 스레드 간 다른 경로를 실행하게 하면 Warp Divergence가 발생하여 성능이 최대 32배까지 저하됨
- **제언**: 분기를 제거하는 브랜치리스(Branchless) 코드 패턴을 적용하거나, 동일 분기 방향의 스레드끼리 묶이도록 데이터를 사전 정렬하며, NVIDIA Nsight Compute로 Warp Efficiency 지표를 측정하여 최적화 효과를 검증함

**온디바이스 AI를 위한 이기종 컴퓨팅 설계**
- **챌린지**: 스마트폰·엣지 디바이스에서 LLM 추론 시 GPU의 높은 전력 소비가 배터리 수명을 급격히 단축하여 사용자 경험을 저해함
- **제언**: Apple Neural Engine, Qualcomm Hexagon DSP처럼 특정 연산(행렬 곱셈, 컨볼루션)에 특화된 저전력 NPU와 범용 CPU·GPU를 조합하는 이기종 컴퓨팅(Heterogeneous Computing) 아키텍처를 채택하여 전력 효율과 성능을 동시에 달성함
