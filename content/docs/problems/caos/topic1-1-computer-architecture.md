## 문제

컴퓨터시스템의 다섯 가지 구성 요소를 제시하고, 폰 노이만 컴퓨터의 기초가 되는 Stored-program computer의 개념에 대하여 설명하시오.

## 해설

> **모든 디지털 문명의 출발점, 두 가지 혁명적 아이디어**  
> 컴퓨터의 다섯 가지 구성 요소와 프로그램을 메모리에 저장하는 Stored-program 원리

컴퓨터는 단순한 계산기가 아닙니다. 현대의 스마트폰, 데이터센터, AI 가속기 칩까지 1940년대 폰 노이만(John von Neumann)이 정립한 단 하나의 혁명적 아이디어—**"명령어(프로그램)를 메모리에 저장하고 순서대로 꺼내 실행한다"**—위에 세워진 구조물입니다. 정보처리기술사 시험에서 이 주제는 컴퓨터구조 도메인의 가장 기초적인 뼈대로, 이후 파이프라인, 캐시메모리, 가상메모리 등 모든 심화 주제의 출발점이 됩니다.

### 1. 출제 배경 및 의도

101회 응용 문제로 출제된 이 주제는 컴퓨터구조의 원리를 근본에서부터 이해하는지 검증합니다. 채점관은 단순 암기가 아닌 **각 구성 요소 간의 유기적 관계**와 **Stored-program이 왜 혁명적인지**를 논리적으로 서술할 수 있는지를 봅니다. 특히 "프로그램 내장 방식이 나오기 전에는 어떤 방식이었는가(배선 방식, ENIAC)", "이 원리가 가져온 소프트웨어 산업의 탄생"이라는 맥락까지 짚어주면 고득점을 狙할 수 있습니다.

### 2. 개념의 본질과 작동 메커니즘

#### 컴퓨터시스템의 다섯 가지 구성 요소

```
┌─────────────────────────────────────────────┐
│                  CPU                        │
│  ┌──────────────┐   ┌──────────────────┐   │
│  │  제어장치(CU) │   │  연산장치(ALU)   │   │
│  └──────┬───────┘   └────────┬─────────┘   │
└─────────┼────────────────────┼─────────────┘
          │   내부 버스         │
    ┌─────▼─────────────────────▼─────┐
    │        기억장치(Memory)          │
    │  주기억장치(RAM) + 보조기억장치   │
    └─────────────────────────────────┘
          │                    │
   ┌──────▼──────┐      ┌──────▼──────┐
   │  입력장치   │      │  출력장치   │
   └─────────────┘      └─────────────┘
```

| 구성 요소 | 역할 | 실제 부품 예시 |
|---|---|---|
| **입력장치 (Input Unit)** | 외부 데이터를 컴퓨터 내부로 변환하여 전달 | 키보드, 마우스, 스캐너, 마이크 |
| **출력장치 (Output Unit)** | 처리된 결과를 사람이 인식할 수 있는 형태로 변환 | 모니터, 프린터, 스피커 |
| **기억장치 (Memory Unit)** | 명령어와 데이터를 저장 (주기억: RAM, 보조기억: HDD/SSD) | RAM, ROM, SSD, HDD |
| **연산장치 (ALU, Arithmetic Logic Unit)** | 산술 연산(+, -, ×, ÷)과 논리 연산(AND, OR, NOT) 수행 | CPU 내부 ALU 회로 |
| **제어장치 (CU, Control Unit)** | 명령어를 해독하고 각 장치에 제어 신호를 전송하여 동작 조율 | CPU 내부 CU 회로 |

> **핵심:** 연산장치와 제어장치를 합쳐 **CPU(Central Processing Unit)**라 부르며, 이 둘이 프로세서의 핵심을 이룹니다.

#### Stored-program Computer (저장 프로그램 내장 방식)

**등장 배경:** Stored-program 이전의 컴퓨터(ENIAC, 1945)는 전선과 스위치를 물리적으로 재배선해야 다른 계산을 수행할 수 있었습니다. 프로그램 하나 바꾸는 데 수일이 소요되는 심각한 비효율이 있었습니다.

**혁명적 아이디어:** 1945년 폰 노이만은 논문 "First Draft of a Report on the EDVAC"에서 다음 원칙을 제시했습니다:

1. **프로그램(명령어)과 데이터를 동일한 메모리에 2진수(Binary)로 저장한다.**
2. **메모리에서 명령어를 순차적으로 읽어(Fetch) 해독(Decode)하고 실행(Execute)한다.**
3. **프로그램이 스스로 자신의 명령어 순서(분기)를 제어할 수 있다.**

**명령어 처리 사이클 (Instruction Cycle):**

```
Fetch(인출) → Decode(해독) → Execute(실행) → (반복)
  ↑                                              │
  └──────────────────────────────────────────────┘
    PC(Program Counter)가 다음 명령어 주소를 가리킴
```

- **Fetch:** PC(프로그램 카운터)가 가리키는 주소의 명령어를 주기억장치에서 읽어 IR(명령어 레지스터)에 저장
- **Decode:** CU(제어장치)가 IR의 명령어를 해독하여 필요한 제어 신호 생성
- **Execute:** ALU가 연산을 수행하거나, 메모리에 데이터를 읽고 씀

### 3. 핵심 키워드 (Must-Have)

- **5대 구성 요소:** 입력장치, 출력장치, 기억장치, 연산장치(ALU), 제어장치(CU)
- **CPU:** 연산장치 + 제어장치
- **Stored-program:** 명령어와 데이터를 동일 메모리에 2진수로 저장
- **명령어 사이클:** Fetch → Decode → Execute
- **PC(Program Counter):** 다음에 실행할 명령어의 주소를 보관하는 레지스터
- **IR(Instruction Register):** 현재 실행 중인 명령어를 임시 저장
- **폰 노이만 병목(von Neumann Bottleneck):** CPU와 메모리 간 단일 버스가 성능 병목을 일으키는 구조적 한계

### 4. 맥락과 비교

| 비교 항목 | ENIAC 방식 (배선 방식) | 폰 노이만 방식 (Stored-program) |
|---|---|---|
| **프로그램 변경** | 물리적 재배선 필요 (수일 소요) | 메모리의 내용만 교체 (즉시 가능) |
| **유연성** | 특정 계산에 고정됨 | 범용 연산 가능 |
| **명령어 저장** | 하드웨어에 내장 | 메모리에 데이터와 함께 저장 |
| **소프트웨어 산업** | 불가능 | 소프트웨어가 독립 산업으로 탄생 |
| **단점** | 유연성 부재 | 폰 노이만 병목(단일 버스) |

**폰 노이만 병목(von Neumann Bottleneck):**  
명령어와 데이터가 같은 버스(Bus)를 공유하므로 CPU가 메모리를 기다리는 지연(Latency)이 발생합니다. 이를 극복하기 위해 **캐시메모리, 파이프라인, 하버드 아키텍처(명령어 버스/데이터 버스 분리)** 등의 기법이 발전했습니다.

### 5. 실무 제언

**임베디드 시스템에서의 적용**

- **챌린지:** IoT 엣지 디바이스나 마이크로컨트롤러는 ROM에 프로그램을 굽는 방식(Harvard Architecture 변형)을 사용합니다. 순수한 폰 노이만 방식에서는 명령어와 데이터가 같은 주소 공간을 공유하므로, 외부 공격자가 데이터 영역에 악성 코드를 주입하고 실행시키는 **코드 인젝션 공격**이 가능합니다.
- **제언:** 현대 OS는 **NX 비트(No-eXecute bit)** 또는 **DEP(Data Execution Prevention)** 기능을 통해 데이터 영역의 코드 실행을 하드웨어 수준에서 차단합니다. 기술사 답안 작성 시 이론적 원리뿐 아니라 보안 취약점과 현대적 해결책까지 연결하면 심화 역량을 어필할 수 있습니다.

**현대 서버 아키텍처로의 발전**

- 폰 노이만 원리는 여전히 유효하지만, 현대 CPU는 Out-of-Order Execution(비순차 실행), 슈퍼스칼라, VLIW, SIMD 등 성능 최적화 기법을 겹겹이 쌓아 폰 노이만 병목을 최소화합니다. 정보처리기술사는 "원리를 알고 현대적 진화를 연결하는 능력"을 갖추어야 합니다.
