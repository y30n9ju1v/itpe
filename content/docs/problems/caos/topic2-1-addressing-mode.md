## 해설

> **명령어가 데이터를 찾아가는 방법의 모든 것**  
> 즉시 주소부터 변위 주소까지, 유연성과 메모리 접근 횟수의 트레이드오프

CPU는 명령어를 실행할 때마다 "이 데이터가 어디에 있지?"를 알아야 합니다. 주소지정방식(Addressing Mode)은 명령어 내 피연산자(Operand) 필드가 실제 데이터(유효 주소, Effective Address)를 가리키는 방법을 정의한 규칙입니다. 방식에 따라 명령어 길이, 메모리 접근 횟수, 표현 가능한 주소 범위가 달라지므로 CPU 설계와 컴파일러 최적화의 핵심 요소입니다.

### 1. 출제 배경 및 의도

합숙 공통 문제로 출제된 만큼 모든 수험생이 기본적으로 알아야 하는 영역입니다. 채점관은 각 방식의 **동작 원리와 유효 주소 계산 공식**, 그리고 **장단점 비교**를 체계적으로 서술할 수 있는지를 봅니다. 단순 나열이 아니라 "왜 이 방식이 필요했는가"라는 설계 의도까지 연결하면 고득점 답안이 됩니다.

### 2. 개념의 본질과 작동 메커니즘

명령어 구조는 크게 연산코드(Opcode)와 피연산자(Operand)로 구성됩니다:

```
┌─────────────────┬───────────────────────┐
│   Opcode (연산) │   Operand (피연산자)   │
│   ADD, MOV 등   │   주소 또는 데이터     │
└─────────────────┴───────────────────────┘
```

피연산자 필드를 해석하는 방식이 주소지정방식입니다.

#### 주요 주소지정방식 상세 비교

| 방식 | 유효 주소(EA) | 메모리 접근 | 특징 |
|---|---|---|---|
| **즉시 주소 (Immediate)** | 피연산자 자체가 데이터 | 0회 | 빠름. 상수값 처리. 표현 범위 제한 |
| **직접 주소 (Direct)** | EA = Operand | 1회 | 단순. 제한된 주소 범위 |
| **간접 주소 (Indirect)** | EA = M[Operand] | 2회 이상 | 넓은 주소 범위. 느림 |
| **레지스터 주소 (Register)** | EA = Register | 0회(레지스터) | 가장 빠름. 레지스터 수 제한 |
| **레지스터 간접 (Register Indirect)** | EA = M[Register] | 1회 | 포인터 구현에 활용 |
| **변위 주소 (Displacement)** | EA = Register + Offset | 1회 | 배열, 스택, 구조체 접근에 활용 |
| **상대 주소 (PC Relative)** | EA = PC + Offset | 1회 | 분기 명령어, 위치 독립 코드 |
| **베이스 레지스터 (Base Register)** | EA = Base + Offset | 1회 | 세그멘테이션, 다중 프로세스 |
| **인덱스 주소 (Indexed)** | EA = Index + Offset | 1회 | 배열 순차 접근에 최적 |

#### 각 방식 동작 원리 상세 설명

**① 즉시 주소 (Immediate Addressing)**
```
명령어: ADD R1, #5
의미: R1 = R1 + 5  (5는 상수로 명령어에 직접 포함)
유효 주소: 없음 (피연산자 자체가 데이터)
```
- 메모리 접근 불필요 → 가장 빠름
- 피연산자 필드의 비트 수에 따라 표현 가능한 상수 범위 제한

**② 직접 주소 (Direct Addressing)**
```
명령어: LOAD R1, 1000
의미: R1 = M[1000]  (주소 1000의 내용을 읽어옴)
유효 주소 EA = 1000 (피연산자 필드의 값)
```
- 단순하지만 명령어 비트 수만큼의 주소 공간만 접근 가능

**③ 간접 주소 (Indirect Addressing)**
```
명령어: LOAD R1, (1000)
의미: R1 = M[M[1000]]  (주소 1000의 내용이 실제 주소)
유효 주소 EA = M[1000]
```
- 포인터 변수 구현의 기초
- 2회 메모리 접근으로 속도 느림

**④ 레지스터 간접 (Register Indirect)**
```
명령어: LOAD R1, (R2)
의미: R1 = M[R2]  (R2가 가리키는 주소의 내용)
유효 주소 EA = R2의 값
```
- C언어의 포인터 역참조(`*ptr`)가 이 방식으로 구현

**⑤ 변위 주소 - 상대 주소 (PC Relative)**
```
명령어: JMP +50
의미: PC = PC + 50  (현재 위치에서 50 앞으로 분기)
유효 주소 EA = PC + 50
```
- 위치 독립 코드(PIC, Position Independent Code) 생성에 핵심
- 동적 라이브러리(SO, DLL) 로딩의 기반

**⑥ 인덱스 주소 (Indexed Addressing)**
```
명령어: LOAD R1, 1000(R2)
의미: R1 = M[1000 + R2]  (배열 base + index)
유효 주소 EA = 1000 + R2의 값
```
- 배열 원소 순차 접근: R2를 증가시키며 반복

### 3. 핵심 키워드 (Must-Have)

- **유효 주소(EA, Effective Address):** 실제 피연산자가 저장된 메모리 주소
- **즉시 주소:** 피연산자 = 데이터 자체, 메모리 접근 0회
- **직접 주소:** EA = Operand, 1회 접근
- **간접 주소:** EA = M[Operand], 2회 이상 접근
- **레지스터 주소:** EA = Register 값, 최고 속도
- **변위 주소:** EA = Register + Displacement, 배열/스택/세그멘트 접근
- **PC 상대 주소:** EA = PC + Offset, 분기 명령어 구현
- **인덱스 주소:** EA = Base + Index Register

### 4. 맥락과 비교

**RISC vs CISC 설계 철학과 주소지정방식:**

| 구분 | RISC (ARM, RISC-V) | CISC (x86) |
|---|---|---|
| **주소지정방식 수** | 소수(간소화) | 다양(복잡) |
| **메모리 접근** | Load/Store 방식만 허용 | 연산 명령어에서도 직접 메모리 접근 |
| **레지스터 활용** | 많은 레지스터, 레지스터 간 연산 | 적은 레지스터, 메모리-레지스터 혼용 |
| **명령어 길이** | 고정 길이 | 가변 길이 |
| **컴파일러 의존도** | 높음 | 낮음 |

**컴파일러와의 연관성:** 고급 언어의 배열은 인덱스 주소지정방식으로, 포인터는 레지스터 간접 방식으로, 함수 분기는 PC 상대 주소 방식으로 번역됩니다.

### 5. 실무 제언

**보안 관점의 주소지정방식**

- **챌린지:** 간접 주소지정방식의 포인터 체계는 **버퍼 오버플로우(Buffer Overflow)** 공격의 근본 원인이 됩니다. 공격자가 스택의 리턴 주소를 덮어쓰면 PC가 악성 코드 주소를 가리키게 됩니다.
- **제언:** 현대 CPU와 OS는 **ASLR(Address Space Layout Randomization)**로 주소 공간을 무작위화하고, **스택 카나리(Stack Canary)**로 리턴 주소 변조를 탐지하며, **NX/DEP**로 데이터 영역 실행을 차단합니다. 주소지정방식 이해는 단순 아키텍처 지식을 넘어 시스템 보안의 기초이기도 합니다.

**임베디드 시스템 최적화:**

마이크로컨트롤러에서 레지스터 주소지정방식 활용률을 높이면 메모리 접근 횟수를 줄여 전력 소모와 실행 시간을 동시에 줄일 수 있습니다. 컴파일러의 레지스터 할당(Register Allocation) 최적화 옵션(-O2, -O3)이 이를 자동화합니다.
