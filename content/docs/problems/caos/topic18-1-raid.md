## 해설

### 출제 배경 및 의도

RAID(Redundant Array of Independent Disks)는 서버·스토리지 인프라의 근간을 이루는 기술로, 디지털 전환 가속화에 따라 대용량·고가용성 데이터 저장 요구가 급증하면서 중요성이 더욱 부각되고 있습니다. 특히 클라우드 환경에서도 온프레미스 스토리지 설계나 프라이빗 클라우드 구축 시 RAID 레벨 선택은 TCO와 가용성에 직결되는 핵심 결정 사항입니다.

기술사 시험에서 RAID 문제는 단순 개념 열거가 아니라 **디스크 수, 공간 효율, 고장 허용 범위**를 정량적으로 제시하고 레벨 간 트레이드오프를 체계적으로 비교하는 능력을 평가합니다. 채점관은 스트라이핑·미러링·패리티의 원리를 구성도로 명확히 표현하고, 각 레벨의 수치를 정확히 산출하는지를 봅니다.

### 1. RAID(Redundant Array of Independent Disks), 개요

정 의  • 다수의 물리 디스크를 논리적으로 묶어 성능·용량·고장 내성을 동시에 달성하는 저장 기술
       - 스트라이핑(Striping), 미러링(Mirroring), 패리티(Parity) 세 원리의 조합으로 구현

```
┌─────────────────────────────────────────────────────────┐
│               RAID 핵심 원리                             │
│                                                         │
│  스트라이핑  →  데이터를 여러 디스크에 분산 저장        │
│               읽기/쓰기 병렬화 → 성능 향상              │
│                                                         │
│  미러링      →  동일 데이터를 2개 이상 디스크에 복사    │
│               디스크 장애 시 즉시 대체 → 가용성 향상    │
│                                                         │
│  패리티      →  XOR 연산으로 오류 정정 정보 생성        │
│               최소 공간으로 고장 내성 확보              │
└─────────────────────────────────────────────────────────┘
```

- RAID는 성능·공간 효율·고장 내성 간 트레이드오프를 레벨별로 다르게 설정함

### 2. RAID 레벨별 구성 및 특성

1) RAID 0·5·6 (단일 어레이 계열)

| 구분 | 항목 | 설명 |
|------|------|------|
| RAID 0 | 최소 디스크 | 2개 |
| | 공간 효율 | 100% (패리티 없음) |
| | 고장 허용 | 0개 (1개 장애 시 전체 데이터 손실) |
| | 특징 | 스트라이핑만 적용, 성능 n배 향상 |
| | 용도 | 임시 데이터, 영상 편집 작업 파일 |
| RAID 5 | 최소 디스크 | 3개 |
| | 공간 효율 | (n-1)/n (3개: 66.7%, 6개: 83.3%) |
| | 고장 허용 | 1개 |
| | 특징 | 분산 패리티로 패리티 병목 제거, Write Penalty 존재 |
| | 용도 | 일반 파일 서버, NAS (중간 규모 표준) |
| RAID 6 | 최소 디스크 | 4개 |
| | 공간 효율 | (n-2)/n (4개: 50%, 8개: 75%) |
| | 고장 허용 | 2개 (동시 2개 장애 허용) |
| | 특징 | 이중 분산 패리티, 쓰기 성능 RAID 5보다 낮음 |
| | 용도 | 대용량 고가용성 스토리지, 엔터프라이즈 NAS |

```
RAID 5 분산 패리티 구조 (4 디스크):

디스크 0: [A1][B1][C1][P(D)]
디스크 1: [A2][B2][P(C)][D1]
디스크 2: [A3][P(B)][C2][D2]
디스크 3: [P(A)][B3][C3][D3]
           ↑ 패리티(P)가 순환 배치 → 병목 제거
```

- RAID 5/6의 Write Penalty는 배터리 백업 캐시(BBWC) Write Back 모드로 최소화 가능

2) RAID 1+0·5+0 (중첩 어레이 계열)

| 구분 | 항목 | 설명 |
|------|------|------|
| RAID 1+0 | 최소 디스크 | 4개 (짝수) |
| | 공간 효율 | 50% |
| | 고장 허용 | 각 미러쌍에서 1개씩, 최대 n/2개 |
| | 특징 | 미러링(RAID 1) 후 스트라이핑(RAID 0), 고성능+고가용성 |
| | 용도 | DB 서버, 트랜잭션 시스템 |
| RAID 5+0 | 최소 디스크 | 6개 (RAID 5 그룹 2개 × 3개) |
| | 공간 효율 | 각 그룹의 (n-1)/n 유지 |
| | 고장 허용 | 각 RAID 5 그룹에서 1개씩 |
| | 특징 | RAID 5 그룹을 RAID 0으로 스트라이핑 |
| | 용도 | 대용량 고성능 스토리지, 빅데이터 플랫폼 |

```
RAID 1+0 구조:

[미러쌍1: D0+D1] ─── RAID 0 스트라이프 ─── [미러쌍2: D2+D3]
  D0: [A1][A3]                                D2: [A2][A4]
  D1: [A1][A3] (미러)                         D3: [A2][A4] (미러)
```

- RAID 1+0은 재구성(Rebuild) 시 미러 쌍만 읽으므로 RAID 5+0 대비 재구성 속도가 빠름

### 3. RAID 레벨 종합 비교 및 선택 기준

| RAID | 최소 디스크 | 공간 효율 | 고장 허용 | 읽기 성능 | 쓰기 성능 | 주요 용도 |
|------|------------|----------|----------|----------|----------|----------|
| RAID 0 | 2 | 100% | 0개 | 최고 | 최고 | 임시/비임계 고속 |
| RAID 5 | 3 | (n-1)/n | 1개 | 높음 | 중간 | 일반 파일 서버 |
| RAID 6 | 4 | (n-2)/n | 2개 | 높음 | 낮음 | 대용량 고가용성 |
| RAID 1+0 | 4 | 50% | n/2개 | 최고 | 높음 | DB/트랜잭션 |
| RAID 5+0 | 6 | ~(n-그룹수)/n | 그룹당 1개 | 최고 | 높음 | 대용량 고성능 |

현대 16TB+ HDD는 URE(Unrecoverable Read Error) 확률이 1/10^14 비트로, 대용량 어레이 재구성 중 데이터 손실 위험이 있어 대용량 환경에서는 RAID 6 또는 ZFS RAIDZ2 채택이 권장됨.  "끝"

### 실무 제언

**클라우드 환경에서의 RAID 역할 재정립**
- **챌린지**: AWS EBS, Azure Managed Disk 등 클라우드 블록 스토리지는 내부적으로 복제 메커니즘을 내장하여 VM 레벨 RAID 구성 시 이중 오버헤드만 발생
- **제언**: 클라우드 환경에서는 스토리지 서비스의 내장 복제와 스냅샷 정책으로 RAID를 대체하고, 온프레미스에서는 BBWC(Battery Backed Write Cache)를 활성화하여 Write Penalty를 최소화

**RAID 레벨 선택 기준 가이드**
- **챌린지**: 워크로드 특성(IOPS 집약 vs 순차 대용량 vs 고가용성)에 따라 최적 RAID 레벨이 달라지나 선택 기준이 모호한 경우 발생
- **제언**: DB/트랜잭션 → RAID 1+0(성능+가용성 최우선), 일반 파일 서버 → RAID 5(비용 효율), 대용량 아카이브 → RAID 6(이중 고장 허용), 분석/임시 데이터 → RAID 0(성능 최우선)

**대용량 스토리지 재구성 위험 관리**
- **챌린지**: 재구성(Rebuild) 중 추가 디스크 장애 발생 시 RAID 5에서 전체 데이터 손실 위험
- **제언**: 16TB 이상 대용량 어레이에서는 RAID 6 또는 ZFS RAIDZ2 채택, 핫 스페어(Hot Spare) 디스크 상시 준비, 정기적 배열 검증(Scrubbing) 수행으로 잠재 URE 사전 탐지
