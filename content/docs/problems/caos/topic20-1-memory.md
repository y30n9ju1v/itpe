DRAM(Dynamic RAM)과 SRAM(Static RAM)

## 해설

### 출제 배경 및 의도

반도체 메모리는 컴퓨터 성능의 근본을 결정하는 핵심 부품입니다. CPU 처리 속도가 수 나노초인 반면 DRAM 접근 시간은 수십~수백 나노초로 상당한 격차(Memory Wall)가 존재하며, 이를 해소하기 위한 메모리 계층 구조 설계가 현대 컴퓨터 아키텍처의 핵심 과제입니다. AI 가속기 시대에 HBM(High Bandwidth Memory)이 부각되면서 DRAM/SRAM의 기술적 이해는 더욱 중요해졌습니다.

125회 응용 1번 문제로, 채점관은 DRAM·SRAM의 **저장 셀 구조 차이(1T1C vs 6T)**, **리프레시 필요성**, **속도·비용·집적도·전력 비교**, 그리고 **메모리 계층에서의 역할 분담**을 명확히 이해하는지 평가합니다.

### 1. 반도체 RAM(Random Access Memory), 개요

정 의  • 임의의 주소에 동일한 속도로 읽기/쓰기가 가능한 휘발성 반도체 메모리
       - 저장 셀 구조에 따라 커패시터 기반 DRAM과 플립플롭 기반 SRAM으로 분류

```
┌─────────────────────────────────────────────────────────┐
│               메모리 계층 구조 (위로 갈수록 빠름·비쌈)  │
│                                                         │
│  CPU 레지스터 ← SRAM 기반, 수 사이클,  수 KB           │
│       │                                                 │
│  L1 캐시     ← SRAM 기반, ~1ns,        32~64KB         │
│       │                                                 │
│  L2/L3 캐시  ← SRAM 기반, ~5~20ns,    256KB~64MB      │
│       │                                                 │
│  주기억장치  ← DRAM 기반, ~100ns,      16~512GB        │
│       │                                                 │
│  SSD/HDD     ← NAND Flash/자기,        ms 단위         │
└─────────────────────────────────────────────────────────┘
```

- SRAM은 속도가 빠르나 비싸서 캐시에, DRAM은 저렴하고 고집적이어서 주기억장치에 사용됨

### 2. DRAM과 SRAM 셀 구조 및 특성 비교

1) DRAM(Dynamic RAM) 저장 원리와 특성

| 구분 | 항목 | 설명 |
|------|------|------|
| 셀 구조 | 구성 | 1T1C (트랜지스터 1개 + 커패시터 1개) |
| | 저장 원리 | 커패시터 전하로 데이터 저장 (충전=1, 방전=0) |
| | 리프레시 | 필수 (전하 자연 방전 → 수십 ms마다 재충전) |
| 특성 | 접근 시간 | 40~100ns (느림) |
| | 집적도 | 높음 (소형 셀 → 대용량 가능) |
| | 비용 | 저렴 |
| | 전력 | 리프레시로 인한 주기적 전력 소비 |
| 용도 | 주사용처 | 주기억장치(Main Memory) |
| | 최신 규격 | DDR5(51.2GB/s), HBM3(~1.2TB/s/스택), LPDDR5(모바일) |

```
DRAM 단일 셀 구조 (1T1C):

워드선(WL) ──┤트랜지스터├── 커패시터(C)
              │
           비트선(BL)
전하 충전=1, 방전=0 / 수십 ms마다 리프레시 필수
```

- 리프레시 동안 메모리 접근 불가 → 성능 영향, ECC(Error Correcting Code)로 오류 정정 가능

2) SRAM(Static RAM) 저장 원리와 특성

| 구분 | 항목 | 설명 |
|------|------|------|
| 셀 구조 | 구성 | 6T (트랜지스터 6개, 교차 결합 인버터 쌍) |
| | 저장 원리 | 쌍안정 래치(플립플롭)의 안정 상태로 저장 |
| | 리프레시 | 불필요 (전원 공급 시 상태 유지) |
| 특성 | 접근 시간 | 1~10ns (매우 빠름) |
| | 집적도 | 낮음 (대형 셀 → 소용량) |
| | 비용 | 비쌈 |
| | 전력 | 대기 전력 낮음 (정적 상태 유지) |
| 용도 | 주사용처 | CPU L1/L2/L3 캐시, TLB, 레지스터 파일 |
| | 적용 예 | 임베디드 MCU 내부 SRAM, FPGA 블록 RAM |

```
SRAM 6T 셀 구조 (쌍안정 래치):

VDD
 │
P1 ─┬─ P2
    │   │
    ├◀──┤  (교차 결합 인버터)
    │   │
N1 ─┴─ N2
 │
GND
전원 공급 시 데이터 영구 유지, 리프레시 불필요
```

- 6T 구조로 셀 면적이 DRAM 1T1C 대비 약 6배 이상 크나, 리프레시 없이 최고 속도 제공

### 3. DRAM vs SRAM 종합 비교 및 진화 트렌드

| 비교 항목 | DRAM | SRAM |
|------|------|------|
| 저장 원리 | 커패시터 전하 | 쌍안정 래치(플립플롭) |
| 셀 구성 | 1T1C (소형) | 6T (대형) |
| 리프레시 | 필요 (수십 ms 주기) | 불필요 |
| 접근 시간 | 40~100ns (느림) | 1~10ns (빠름) |
| 집적도 | 높음 (대용량) | 낮음 (소용량) |
| 비용 | 저렴 | 비쌈 |
| 전력 소비 | 중간 (리프레시 전력) | 낮음 (정적 상태) |
| 주요 용도 | 주기억장치 | CPU 캐시, 레지스터 |

| DRAM 세대 | 규격 | 최대 대역폭 |
|------|------|------|
| DDR4 | DDR4-3200 | 25.6 GB/s |
| DDR5 | DDR5-6400 | 51.2 GB/s |
| LPDDR5 | LPDDR5X-8533 | 68.3 GB/s |
| HBM3 | HBM3e | ~1.2 TB/s (스택당) |

AI·HPC 워크로드에서 메모리 대역폭이 성능 병목이 되면서, HBM(High Bandwidth Memory)이 여러 DRAM 다이를 TSV(Through-Silicon Via)로 수직 스택하여 기존 DRAM 대비 10~20배 대역폭을 달성하는 방향으로 진화 중.  "끝"

### 실무 제언

**AI 가속기 인프라 선택 시 메모리 아키텍처 고려**
- **챌린지**: LLM 추론에서 수백 GB 모델 가중치를 빠르게 로드하려면 메모리 대역폭이 핵심 병목으로, GDDR6 기반 GPU는 대형 모델 추론 효율이 낮음
- **제언**: NVIDIA H100의 HBM3은 3.35 TB/s 대역폭으로 A100 대비 추론 처리량 2~3배 향상. AI 인프라 GPU 선택 시 VRAM 용량뿐만 아니라 HBM 세대(HBM2e/HBM3)와 대역폭을 핵심 지표로 평가

**서버 메모리 안정성 설계**
- **챌린지**: 대용량 DRAM 서버에서 메모리 오류가 시스템 크래시나 데이터 손상으로 이어지는 사례 발생
- **제언**: 서버용 DRAM은 ECC(Error Correcting Code) 메모리를 의무 적용하여 단일 비트 오류를 자동 정정하고 다중 비트 오류를 탐지. 메모리 채널 이중화(Mirrored Channel) 또는 스페어 랭크(Rank Sparing)로 DIMM 장애 시 무중단 운영

**캐시 최적화를 통한 성능 개선**
- **챌린지**: 데이터 접근 패턴이 캐시 특성과 맞지 않아 캐시 미스율이 높아지면 CPU가 DRAM 접근 대기로 처리량 급감
- **제언**: L1/L2(SRAM) 캐시 지역성(Locality)을 극대화하는 데이터 구조 설계(캐시 라인 정렬, AoS→SoA 변환)와 프리페치(Prefetch) 전략으로 메모리 지연 은폐
