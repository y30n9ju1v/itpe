## 문제

DRAM(Dynamic RAM)과 SRAM(Static RAM)

## 해설

> **메모리 계층 구조의 두 기둥: 빠르고 비싼 SRAM vs 느리고 저렴한 DRAM**  
> 저장 셀 구조의 차이, 리프레시, 접근 시간, 그리고 현대 메모리 계층에서의 역할

주기억장치(Main Memory)를 구성하는 반도체 메모리는 크게 동적(Dynamic) RAM과 정적(Static) RAM으로 나뉩니다. 이 두 종류는 데이터를 저장하는 회로 구조가 근본적으로 달라, 속도, 전력, 비용, 집적도 모든 면에서 서로 다른 특성을 보입니다.

### 1. 출제 배경 및 의도

125회 응용 1번 문제로 출제되었습니다. 채점관은 **DRAM과 SRAM의 저장 셀 구조 차이**, **DRAM의 리프레시(Refresh) 필요성**, **속도/비용/집적도/전력 측면의 비교**, 그리고 **메모리 계층 구조에서의 역할 분담**을 명확히 이해하는지를 봅니다.

### 2. 개념의 본질과 작동 메커니즘

#### DRAM (Dynamic Random Access Memory)

**저장 원리:**
```
단일 셀 구조:
┌─────────────────────────────┐
│  ┌──┐         ┌──────────┐ │
│  │트│ 비트선  │ 커패시터 │ │
│  │랜│─────────│ (콘덴서) │ │
│  │지│         │ C        │ │
│  │스│ 워드선  └──────────┘ │
│  │터│ ─────── 접근         │
│  └──┘                       │
└─────────────────────────────┘
1 트랜지스터 + 1 커패시터 (1T1C)
```

- 데이터를 **커패시터(Capacitor)의 전하**로 저장 (충전=1, 방전=0)
- **리프레시(Refresh) 필수:** 커패시터 전하는 시간이 지남에 따라 자연 방전 → 수십 밀리초마다 재충전 필요
- 리프레시 동안 메모리 접근 불가 → 성능에 영향

**특징:**
- **셀 크기:** 매우 작음 (1T1C) → 높은 집적도, 대용량 가능
- **접근 시간:** 40~100ns (느림)
- **전력 소비:** 리프레시로 인한 주기적 전력 소비
- **비용:** 저렴 (높은 집적도로 가격 하락)
- **휘발성:** 전원 차단 시 데이터 소실

**현대 DRAM 종류:**

| 종류 | 특징 |
|---|---|
| **DDR5 SDRAM** | 현재 주력 PC 메모리, 최대 6400 MT/s |
| **LPDDR5** | 모바일/임베디드용 저전력 버전 |
| **HBM3 (High Bandwidth Memory)** | 3D 스택 DRAM, GPU/AI 가속기용, 수 TB/s 대역폭 |
| **GDDR6X** | GPU 전용 고대역폭 DRAM |

#### SRAM (Static Random Access Memory)

**저장 원리:**
```
쌍안정 래치 셀 구조:
     VDD
      │
   ┌──┴──┐     ┌──┴──┐
   │ P1  │     │ P2  │
   └──┬──┘     └──┬──┘
      ├────▶───────┤  (교차 연결된 인버터 쌍)
      ├───◀────────┤
   ┌──┴──┐     ┌──┴──┐
   │ N1  │     │ N2  │
   └──┬──┘     └──┬──┘
     GND          GND
6 트랜지스터(6T) 구조
```

- 데이터를 **교차 결합된 인버터(플립플롭)** 회로의 안정적 상태로 저장
- 전원이 공급되는 한 데이터가 유지 (리프레시 불필요)

**특징:**
- **셀 크기:** 큼 (6T) → 낮은 집적도, 소용량
- **접근 시간:** 1~10ns (매우 빠름)
- **전력 소비:** 리프레시 없음, 대기 전력 낮음 (다만 누설 전류 존재)
- **비용:** 비쌈 (낮은 집적도)
- **휘발성:** 전원 차단 시 데이터 소실

**현대 SRAM 활용:**
- CPU L1/L2/L3 캐시 (모든 현대 프로세서)
- TLB 캐시
- 레지스터 파일 일부
- 임베디드 MCU 내부 SRAM

#### DRAM vs SRAM 종합 비교

| 비교 항목 | DRAM | SRAM |
|---|---|---|
| **저장 원리** | 커패시터 전하 | 쌍안정 래치(플립플롭) |
| **셀 구성** | 1T + 1C (소형) | 6T (대형) |
| **리프레시** | 필요 (수십 ms 주기) | 불필요 |
| **접근 시간** | 40~100ns (느림) | 1~10ns (빠름) |
| **집적도** | 높음 (대용량) | 낮음 (소용량) |
| **비용** | 저렴 | 비쌈 |
| **전력 소비** | 중간 (리프레시 전력) | 낮음 (정적 상태) |
| **주요 용도** | 주기억장치(RAM) | CPU 캐시, 레지스터 |

#### 메모리 계층에서의 역할

```
┌──────────────────────────────────────────────────────────────┐
│                    메모리 계층 구조                           │
│                                                               │
│  CPU 레지스터  ← SRAM 기반, 수 사이클, 수 KB               │
│       │                                                       │
│  L1 캐시      ← SRAM 기반, ~1ns, 32~64KB                   │
│       │                                                       │
│  L2 캐시      ← SRAM 기반, ~5ns, 256KB~1MB                 │
│       │                                                       │
│  L3 캐시      ← SRAM 기반, ~20ns, 8~64MB                   │
│       │                                                       │
│  주기억장치   ← DRAM 기반, ~100ns, 16~512GB                 │
│       │                                                       │
│  SSD/HDD      ← NAND Flash/자기 디스크, ms 단위             │
└──────────────────────────────────────────────────────────────┘
```

### 3. 핵심 키워드 (Must-Have)

- **DRAM:** 커패시터 전하 저장, 리프레시 필요, 고집적, 저비용, 주기억장치
- **SRAM:** 플립플롭 저장, 리프레시 불필요, 빠름, 비쌈, CPU 캐시
- **리프레시(Refresh):** DRAM이 전하 손실을 보충하는 주기적 재충전
- **1T1C vs 6T:** DRAM 단순 셀 vs SRAM 복잡 셀
- **HBM(High Bandwidth Memory):** 3D 스택 DRAM
- **메모리 계층:** 레지스터→L1/L2/L3(SRAM)→RAM(DRAM)→스토리지
- **ECC(Error Correcting Code):** 서버 DRAM의 오류 정정 기능

### 4. 맥락과 비교

**DRAM 진화의 역사:**

| 세대 | 규격 | 최대 속도 |
|---|---|---|
| DDR | DDR400 | 3.2 GB/s (채널당) |
| DDR2 | DDR2-800 | 6.4 GB/s |
| DDR3 | DDR3-2133 | 17 GB/s |

| DDR5 | DDR5-6400 | 51.2 GB/s |
| HBM3 | HBM3e | ~1.2 TB/s (스택당) |

### 5. 실무 제언

**AI 가속기와 차세대 메모리 기술**

- **챌린지:** LLM 추론에서 수백 GB의 모델 가중치를 빠르게 로드하려면 메모리 대역폭이 핵심 병목입니다. 기존 GDDR6 기반 GPU는 대역폭 한계로 대형 모델 추론 효율이 낮습니다.
- **제언:** HBM(High Bandwidth Memory)은 여러 DRAM 다이를 수직으로 스택하고 수천 개의 TSV(Through-Silicon Via)로 연결하여 기존 DRAM 대비 10~20배 대역폭을 달성합니다. NVIDIA H100의 HBM3은 3.35 TB/s 대역폭으로 A100 대비 추론 처리량이 2~3배 향상됩니다. SRAM/DRAM의 기술적 이해가 AI 인프라 하드웨어 선택의 근거가 됩니다.
