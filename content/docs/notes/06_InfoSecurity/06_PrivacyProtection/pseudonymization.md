## 개요

가명처리(Pseudonymization)는 개인정보의 일부를 삭제하거나 대체하여 추가 정보 없이는 특정 개인을 식별할 수 없도록 하는 처리 기법이다. 「개인정보 보호법」상 가명정보는 통계작성·과학적 연구·공익적 기록보존에 동의 없이 활용 가능하다.

## 핵심 내용

### 가명처리 기법 분류

| 기법 | 설명 | 예시 |
|------|------|------|
| **가명화 (Pseudonymization)** | 식별자를 가명으로 대체 | 홍길동 → A001 |
| **총계처리 (Aggregation)** | 개인 데이터를 집계 통계로 변환 | 개별 나이 → 30대 비율 |
| **데이터 삭제 (Suppression)** | 민감 항목 전체 삭제 | 주민번호 컬럼 제거 |
| **데이터 범주화 (Generalization)** | 구체적 값을 범위·범주로 치환 | 1990-03-15 → 1990년대 |
| **데이터 마스킹 (Masking)** | 일부 문자를 식별 불가 문자로 대체 | 010-1234-5678 → 010-****-5678 |
| **잡음 추가 (Noise Addition)** | 수치 데이터에 랜덤 노이즈 추가 | 급여 300만원 → 295~305만원 |
| **데이터 교환 (Swapping)** | 레코드 간 속성 값을 교환 | 지역 필드를 무작위 교환 |
| **토큰화 (Tokenization)** | 민감 데이터를 토큰으로 대체, 매핑 테이블 별도 보관 | 카드번호 → 임의 토큰 |

### 가명처리 vs 익명처리

| 구분 | 가명처리 | 익명처리(완전 비식별화) |
|------|---------|----------------------|
| 재식별 가능성 | 추가 정보 결합 시 가능 | 이론적으로 불가 |
| 법적 지위 | 가명정보 (별도 규정 적용) | 개인정보 아님 |
| 데이터 활용성 | 높음 | 낮음 |
| 안전성 | 상대적으로 낮음 | 높음 |

### 가명정보 활용 시 안전조치 (개보법 §28의4)
- 가명정보와 추가 정보를 분리 보관
- 재식별 금지 의무
- 결합 전문기관(결합기관) 통해 데이터 결합

## 실무 적용 예시

병원이 의료 AI 연구를 위해 환자 데이터를 가명처리(환자 ID 토큰화, 생년월일 범주화, 희귀 질환명 삭제)한 후 연구 기관에 제공하여 개인정보 동의 없이 연구 목적으로 활용한다.
