## 개요

AI 학습용 데이터 품질관리는 기계학습 모델의 성능을 좌우하는 훈련 데이터의 정확성·완전성·일관성·대표성을 체계적으로 확보하는 활동이다. "Garbage In, Garbage Out" 원칙에 따라 데이터 품질이 AI 모델 성능을 직접 결정한다.

## 핵심 내용

### 가. AI 학습용 데이터 특성

| 특성 | 설명 |
|------|------|
| **대규모성** | 일반 DB 대비 수십~수억 건의 대용량 데이터 필요 |
| **다양성** | 다양한 분포·환경·시나리오를 포함해야 일반화 가능 |
| **대표성** | 실제 배포 환경의 데이터 분포를 반영해야 함 |
| **레이블 정확성** | 어노테이션(레이블)의 오류가 모델 학습에 직접 영향 |
| **균형성** | 클래스 불균형(Imbalance) 시 특정 클래스 과소 학습 |
| **시의성** | 개념 드리프트(Concept Drift) 대응을 위한 최신성 유지 |

### 나. AI 학습용 데이터 품질관리 지표

| 지표 | 측정 방법 |
|------|----------|
| **정확성 (Accuracy)** | 레이블 오류율, 어노테이터 간 일치도(Inter-Annotator Agreement) |
| **완전성 (Completeness)** | 결측값 비율, 필수 속성 누락률 |
| **일관성 (Consistency)** | 동일 데이터에 대한 레이블 일치율 |
| **다양성 (Diversity)** | 클래스 분포, 인구통계학적 분포 |
| **균형성 (Balance)** | 클래스별 샘플 수 편차, 불균형 비율 |
| **최신성 (Timeliness)** | 데이터 수집 시점 vs. 배포 환경 격차 |

### 다. AI 학습용 데이터 품질관리 활동

**수집 단계:**
- 데이터 수집 계획 및 수집 기준 정의
- 개인정보·저작권 검토, 동의 확보

**전처리 단계:**
- 결측값 처리(삭제·대체·보간)
- 이상치 탐지 및 제거
- 중복 데이터 제거
- 정규화·스케일링

**어노테이션 단계:**
- 레이블링 가이드라인 표준화
- 다중 어노테이터(복수 검토) + 불일치 중재
- 어노테이터 간 일치도(Cohen's Kappa 등) 측정

**검증 단계:**
- 훈련·검증·테스트 셋 분리(누수 방지)
- 데이터 드리프트(Distribution Shift) 모니터링
- 모델 성능 기반 역추적 품질 개선

**관리 단계:**
- 데이터 버전 관리(Data Versioning)
- 데이터 계보(Data Lineage) 추적
- 품질 지표 대시보드 운영
