## 개요

확장성 해싱(Extendible Hashing)은 데이터 증가에 따라 동적으로 해시 테이블을 확장하는 기법이다. 전체 파일을 재해싱(Rehashing)하지 않고 디렉토리와 버킷의 분할로 점진적 확장이 가능하다.

## 핵심 내용

### 개념 및 구성 요소

| 구성 요소 | 설명 |
|----------|------|
| **디렉토리(Directory)** | 전역 깊이(Global Depth)를 가진 포인터 배열. 버킷을 가리킴 |
| **버킷(Bucket)** | 실제 레코드를 저장하는 블록. 지역 깊이(Local Depth) 보유 |
| **전역 깊이(Global Depth, d)** | 디렉토리 크기 = 2^d |
| **지역 깊이(Local Depth, d')** | 버킷이 구분에 사용하는 해시 비트 수 |
| **해시 함수** | 키의 하위 d비트를 디렉토리 인덱스로 사용 |

### 동작 원리

**삽입 시 버킷 오버플로우 처리**:

1. **지역 깊이 < 전역 깊이**: 버킷만 분할, 디렉토리 변경 없음
2. **지역 깊이 = 전역 깊이**: 전역 깊이 증가 (디렉토리 크기 2배) + 버킷 분할

```
예시: d=2 (디렉토리 4개 엔트리)
버킷 오버플로우 → 지역 깊이 증가 → 해시 비트 재분배
필요 시 d=3으로 확장 → 디렉토리 8개 엔트리
```

### 충돌회피 기법

| 기법 | 설명 |
|------|------|
| **버킷 분할(Bucket Split)** | 오버플로우 버킷을 두 개로 분할, 레코드 재분배 |
| **디렉토리 확장(Directory Doubling)** | 전역 깊이 증가, 기존 포인터를 새 위치로 복사 |
| **오버플로우 체이닝** | 분할 대신 오버플로우 버킷 연결 (확장성 감소) |

### 정적 해싱 vs 확장성 해싱

| 구분 | 정적 해싱 | 확장성 해싱 |
|------|---------|-----------|
| 파일 크기 | 고정 | 동적 확장 |
| 재해싱 | 전체 재해싱 필요 | 부분 분할만 수행 |
| 공간 효율 | 낮음 (예약 공간) | 높음 |
| 성능 | 오버플로우 시 저하 | 일정 수준 유지 |

## 출제 포인트

- 전역 깊이·지역 깊이의 의미와 관계
- 버킷 분할 vs 디렉토리 확장 결정 조건
- 정적 해싱 대비 확장성 해싱의 장점: 재해싱 없는 동적 확장
