## 개요

멀티클라우드(Multicloud)는 2개 이상의 클라우드 공급자(AWS, Azure, GCP 등)의 서비스를 조합하여 단일 아키텍처로 운영하는 전략이다. 특정 벤더 종속(Vendor Lock-in) 방지, 비용 최적화, 가용성 향상을 목적으로 한다.

## 핵심 내용

### 멀티클라우드 vs 하이브리드 클라우드

| 구분 | 멀티클라우드 | 하이브리드 클라우드 |
|------|-------------|------------------|
| **정의** | 복수의 퍼블릭 클라우드 조합 | 온프레미스 + 퍼블릭 클라우드 연계 |
| **목적** | 벤더 다양화, 비용 최적화 | 레거시 연동, 규정 준수 |
| **데이터 위치** | 복수 퍼블릭 클라우드 | 사내 + 클라우드 혼재 |
| **관리 복잡성** | 높음 | 중간 |

### 멀티클라우드 도입 이유

| 이유 | 설명 |
|------|------|
| **벤더 Lock-in 방지** | 특정 CSP 의존도 제거, 협상력 향상 |
| **비용 최적화** | 워크로드별 가장 저렴한 CSP 선택 |
| **가용성 향상** | CSP 전체 장애 시에도 서비스 유지 |
| **규제 준수** | 데이터 주권 요건에 따른 지역 CSP 선택 |
| **최적 서비스 활용** | AI(GCP), 엔터프라이즈(Azure), 범용(AWS) 혼합 |

### 멀티클라우드 관리 도전과제

- **데이터 이동 비용**: 클라우드 간 데이터 전송 비용(Egress)
- **일관된 보안 정책**: CSP별 상이한 IAM·네트워크 정책 통합 관리
- **통합 모니터링**: 단일 뷰의 비용·성능·보안 대시보드
- **네트워크 레이턴시**: 클라우드 간 통신 지연

### 멀티클라우드 관리 도구

| 도구 유형 | 예시 |
|----------|------|
| 컨테이너 오케스트레이션 | Kubernetes (멀티클러스터) |
| 인프라 코드화 | Terraform (CSP 독립적 IaC) |
| 서비스 메시 | Istio, Anthos |
| 통합 모니터링 | Datadog, Grafana |

## 실무 적용 예시

금융기관: 핵심 거래 시스템은 AWS, AI·분석 워크로드는 GCP, 오피스365는 Azure로 분리 운영.

## 출제 포인트

- 멀티클라우드와 하이브리드 클라우드 차이점
- 도입 이유: 벤더 Lock-in 방지, 비용 최적화, 가용성
- 도전과제: 보안 통합, 비용 관리, 데이터 거버넌스
- 관리 도구: Terraform, Kubernetes 멀티클러스터
