## 개요

멀티클라우드(Multicloud)는 둘 이상의 퍼블릭 클라우드 서비스 제공사(AWS, Azure, GCP 등)의 서비스를 조합하여 단일 아키텍처로 운영하는 전략이다. 특정 벤더 종속(Vendor Lock-in)을 방지하고, 비용 최적화 및 시스템 가용성을 극대화하는 것이 핵심 목적이다.

## 핵심 내용

### 1. 개념 및 도입 필요성

| 필요성 | 설명 |
|--------|------|
| **벤더 Lock-in 방지** | 특정 CSP 의존도 제거, 가격 협상력 및 클라우드 전환 유연성 확보 |
| **서비스 최적화** | AWS AI, Azure AD, GCP BigQuery 등 각 CSP별 강점 서비스 선택 활용 |
| **가용성 및 DR 향상** | 특정 클라우드 전체 장애 시에도 타 클라우드로 서비스 연속성 유지 |
| **규제 준수** | 국가별 데이터 주권(Data Sovereignty) 요건 충족 및 지역 CSP 선택 |
| **비용 최적화** | 워크로드 특성 및 단가에 따라 가장 비용 효율적인 클라우드 선택 |

### 2. 멀티클라우드 vs 하이브리드 클라우드

| 구분 | 멀티클라우드 | 하이브리드 클라우드 |
|------|-------------|------------------|
| **정의** | 복수의 퍼블릭 클라우드 조합 | 온프레미스(프라이빗) + 퍼블릭 클라우드 연계 |
| **목적** | 벤더 다양화, 최적 서비스 활용, 비용 최적화 | 레거시 자산 연동, 규정 준수, 기밀 데이터 자체 보관 |
| **데이터 위치** | 복수 퍼블릭 클라우드 간 분산 | 사내 서버(온프레미스) + 퍼블릭 클라우드 혼재 |
| **관리 복잡성** | 높음 (이기종 퍼블릭 클라우드 연동) | 중간~높음 (사내망과 외부 클라우드 연동) |

### 3. 시스템 요구사항 및 도전과제

#### 시스템 요구사항
| 요구사항 | 내용 |
|---------|------|
| **통합 관리 플레인** | 단일 콘솔 또는 플랫폼에서 다중 클라우드 자원을 일관되게 관리 |
| **네트워크 연결성** | 클라우드 간 전용 네트워크(Direct Connect, ExpressRoute) 또는 VPN 구축 |
| **ID/접근 관리 (IAM)** | 클라우드 간 통합 ID 관리 체계 및 SSO(Single Sign-On) 적용 |
| **데이터 이동성** | 표준 API 및 포맷 활용으로 CSP 간 데이터 이식성 확보 |
| **보안 정책 일관성** | 개별 CSP에 독립적인 일관된 글로벌 보안 정책 표준 수립 |
| **통합 모니터링** | 단일 가시성 플랫폼에서 전체 멀티클라우드 비용 및 성능 모니터링 |

#### 관리적 도전과제
* **데이터 이동 비용 (Egress Cost)**: 클라우드 외부로 데이터를 보낼 때 발생하는 과도한 네트워크 비용
* **일관된 보안 통제**: CSP별로 상이한 IAM 권한 체계와 방화벽 설정을 통합 제어해야 하는 부담
* **통합 모니터링(Observability)**: 단일 뷰로 비용, 가용성, 보안 상태를 실시간 가시화해야 함
* **네트워크 레이턴시**: 이기종 클라우드 서비스 간 통신 시 발생하는 지연 시간 문제

### 4. 주요 기술 및 관리 도구

| 기술 영역 | 설명 | 도구 예시 |
|------|------|---------|
| **컨테이너 오케스트레이션** | 클라우드 인프라에 종속되지 않는 독립적 배포 환경 제공 | Kubernetes (멀티클러스터), OpenShift |
| **서비스 메시 (Service Mesh)** | 멀티클라우드 환경 내 마이크로서비스 간 통신, 보안, 모니터링 통합 | Istio, Consul, Anthos |
| **클라우드 관리 플랫폼 (CMP) / IaC** | 다중 클라우드 자원의 통합 관리 및 CSP 독립적인 인프라 코드화 | Terraform, Pulumi |
| **데이터 패브릭 (Data Fabric)** | 이기종 클라우드에 분산된 데이터의 통합 및 가상화 | Apache Kafka, Delta Lake |
| **보안 및 접근 제어** | 클라우드 이용 상태 모니터링 및 접근 보안 제어 | CASB (Microsoft Defender for Cloud Apps), 통합 IAM/SSO |
| **통합 관제 및 모니터링** | 전체 CSP의 성능 및 비용 지표 통합 수집 | Datadog, Grafana |

## 실무 적용 예시

* **금융기관**: 핵심 원장 거래 시스템은 온프레미스/AWS로 구성하고, AI 및 빅데이터 분석 워크로드는 GCP, 전사 협업 및 SaaS(Office 365)는 Azure를 적용하여 가용성을 극대화하고 벤더 종속을 분산시킴.

## 출제 포인트

* 멀티클라우드와 하이브리드 클라우드의 명확한 차이점 비교
* 멀티클라우드 도입의 5대 필요성 및 직면하는 도전과제 (Egress 비용, 일관된 보안 설정)
* Kubernetes와 Terraform이 멀티클라우드 아키텍처에서 갖는 클라우드 중립적 역할
