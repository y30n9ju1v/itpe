## 개요

공공기관이 SaaS를 안전하게 이용하기 위해 과기정통부·행안부가 발표한 가이드라인으로, 클라우드 서비스 도입 시 위험관리·보안대책·SLA 수립 절차를 규정한다. 클라우드 보안인증(CSAP)과 연계하여 공공 정보화 사업의 클라우드 전환을 지원한다.

## 핵심 내용

### 가. 클라우드 서비스 위험 관리원칙 및 기준

**위험 관리 원칙**

| 원칙 | 내용 |
|------|------|
| **책임 공유 모델** | 클라우드 제공자 책임 + 기관 책임 명확히 구분 |
| **위험 기반 접근** | 처리 데이터 민감도에 따라 차등 보안 적용 |
| **지속적 모니터링** | 서비스 이용 기간 중 지속적 위험 평가 |
| **투명성** | 클라우드 제공자의 보안 현황 공개 요구 |

**위험 평가 기준**

| 등급 | 데이터 유형 | SaaS 허용 여부 |
|------|-------------|---------------|
| 기밀 | 국가기밀, 개인정보 다량 | 제한적 허용 (민간 전용 클라우드 권장) |
| 민감 | 개인정보, 업무 기밀 | CSAP 인증 SaaS만 허용 |
| 공개 | 공개 업무 정보 | 일반 퍼블릭 SaaS 허용 |

### 나. 보안대책 수립 및 보안성 검토

**보안대책 수립 절차**
1. 정보자산 분류 및 민감도 평가
2. 클라우드 제공자 보안 현황 검토 (CSAP 인증, ISO 27001)
3. 책임 공유 모델에 따른 기관 측 보안 통제 도출
4. 보안 요구사항을 계약서·SLA에 반영

**주요 보안 통제**

| 영역 | 기관 책임 | 제공자 책임 |
|------|-----------|-------------|
| 접근통제 | IAM 정책, MFA 설정 | 인프라 물리 보안 |
| 데이터 암호화 | 암호화 키 관리 | 전송·저장 암호화 인프라 |
| 로그·감사 | 감사 로그 보관·분석 | 로그 수집 인프라 제공 |
| 사고 대응 | 침해사고 대응 절차 | 사고 알림·초기 격리 |

**보안성 검토 항목**
- CSAP(클라우드 보안인증) 취득 여부 확인
- 개인정보 처리방침·위탁 계약 적정성
- 데이터 국외 이전 여부 (국내 서버 위치 확인)
- 취약점 점검 결과 공개 여부

### 다. 서비스 수준 협약 (SLA)

**SLA 주요 항목**

| 항목 | 측정 지표 | 공공기관 권장 수준 |
|------|-----------|------------------|
| **가용성** | 월간 가동률(Uptime) | 99.9% 이상 (월 43.8분 미만 장애) |
| **성능** | 응답시간, 처리량 | 업무 특성별 정의 |
| **보안사고 대응** | 침해사고 통보 시간 | 1시간 이내 |
| **데이터 복구** | RTO/RPO | 중요 시스템 RTO 4시간, RPO 1시간 |
| **데이터 이식성** | 서비스 종료 시 데이터 반환 | 90일 이내 표준 포맷 제공 |

**Lock-in 방지 조항**
- 데이터 포맷 표준화 (CSV, JSON 등 개방형)
- 서비스 종료 시 데이터 이전 지원 의무화
- 멀티 클라우드 전략 검토

## 출제 포인트

- 책임 공유 모델의 기관/제공자 책임 구분
- 데이터 민감도 3등급에 따른 차등 보안
- SLA 가용성 지표(99.9%, 월 43.8분)와 RTO/RPO
- CSAP 인증의 공공기관 클라우드 이용 조건으로서의 역할
