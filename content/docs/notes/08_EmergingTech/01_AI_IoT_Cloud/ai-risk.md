## 개요

AI 리스크(AI Risk)는 AI 시스템의 개발·배포·운영 과정에서 발생할 수 있는 부정적 결과 또는 잠재적 위협이다. NIST AI RMF, EU AI Act 등에서 AI 리스크를 보안·프라이버시·사회적 차원으로 구분하여 관리한다.

## 핵심 내용

### 가. AI 리스크 개념

- **정의**: AI 시스템이 의도치 않거나 악의적으로 활용될 때 발생하는 피해 가능성
- **특성**: 불투명성(블랙박스), 자율성, 확장성, 편향 전파 등으로 인해 일반 IT 리스크보다 복잡
- **관리 프레임워크**: NIST AI RMF(측정·관리·거버넌스), EU AI Act(위험 기반 분류)

### 나. AI 보안 리스크

| 유형 | 설명 |
|------|------|
| **적대적 공격 (Adversarial Attack)** | 입력 데이터에 미세한 노이즈를 추가하여 AI 오분류 유도 |
| **모델 역전 공격 (Model Inversion)** | 모델 출력에서 훈련 데이터(개인정보) 추출 |
| **데이터 포이즈닝 (Data Poisoning)** | 훈련 데이터를 오염시켜 모델 동작 조작 |
| **프롬프트 인젝션 (Prompt Injection)** | 악의적 입력으로 LLM의 의도된 제약을 우회 |
| **모델 탈취 (Model Stealing)** | 반복 쿼리로 모델 구조·가중치 추출 |

### 다. AI 프라이버시 리스크

| 유형 | 설명 |
|------|------|
| **개인정보 침해** | 훈련 데이터에 포함된 개인정보가 모델 출력에 노출 |
| **추론 공격 (Inference Attack)** | 모델 출력으로 훈련 셋 포함 여부 추론 (멤버십 추론) |
| **연결 공격** | 비식별 데이터와 외부 정보를 결합한 재식별 |
| **동의 없는 데이터 활용** | 공개 데이터 스크래핑으로 동의 없이 학습 데이터 수집 |

### 라. AI 사회적 리스크

| 유형 | 설명 |
|------|------|
| **알고리즘 편향 (Algorithmic Bias)** | 편향된 훈련 데이터로 인한 차별적 의사결정 |
| **딥페이크·허위정보** | 생성 AI를 이용한 가짜 콘텐츠 대량 생산 |
| **일자리 대체** | 자동화로 인한 특정 직군 고용 감소 |
| **자율 무기 리스크** | 인간 통제 없는 AI 무기 시스템의 윤리적 문제 |
| **설명 불가능성** | 블랙박스 AI의 의사결정 근거 부재로 책임 소재 불명확 |

## 출제 포인트

- 4가지(개념·보안·프라이버시·사회적) 분류별 구체적 유형 서술
- AI RMF, EU AI Act 등 글로벌 리스크 관리 프레임워크 연계
- 적대적 공격, 프롬프트 인젝션, 알고리즘 편향이 각 리스크 범주에서 핵심 사례
