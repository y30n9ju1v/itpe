## 개요

소프트웨어 영향평가는 대기업 또는 공공기관이 SW 사업에 참여하거나 SW 제품을 직접 개발할 때, 중소 SW 기업에 미치는 영향을 사전에 평가하는 제도다. SW산업 진흥법에 근거하며, 대기업·공공기관의 SW 시장 직접 참여로 인한 경쟁 왜곡을 방지한다.

## 핵심 내용

### 영향평가 대상 기관

- **대기업**: SW 사업 직접 수행 또는 계열사 참여 시
- **공공기관**: 민간 SW 시장에서 직접 SW 개발·판매 시
- **국가·지자체**: 민간과 경쟁하는 SW 서비스 개발 시

※ 공공기관의 내부 행정 목적 SW 개발은 해당 없음

### 소프트웨어사업 영향평가 체계

```
영향평가 신청 → 과기정통부 접수 → 영향평가위원회 심의 
→ 현장조사·의견수렴 → 평가 결과 통보 → 조건부 허용/불허
```

- **주관기관**: 과학기술정보통신부
- **심의기구**: SW 영향평가위원회 (중소기업·학계·전문가)

### 평가 항목

| 항목 | 내용 |
|------|------|
| **시장 경쟁 영향** | 해당 SW 분야의 중소기업 시장 점유율 변화 |
| **기술 대체 가능성** | 민간 SW로 대체 가능한지 여부 |
| **공공 목적 필요성** | 공익·보안·특수 목적 달성 불가피성 |
| **고용 영향** | 관련 SW 기업 고용 감소 영향 |
| **혁신 저해 여부** | 민간 SW 기술 개발 의욕 감소 가능성 |

### 기대 효과

- 대기업·공공기관의 SW 시장 잠식 방지
- 중소 SW 기업 생태계 보호
- 공정한 SW 시장 경쟁 환경 조성
