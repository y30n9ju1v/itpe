## 개요

소프트웨어 품질성능 평가시험은 SW 제품이 기능적·비기능적 품질 요구사항을 충족하는지 시험·인증하는 제도다. 국내에서는 TTA(한국정보통신기술협회)의 GS인증(Good Software)이 대표적이며, ISO/IEC 25010 품질 모델을 기반으로 한다.

## 핵심 내용

### SW 품질 평가 체계

| 구분 | 내용 |
|------|------|
| **국제 표준** | ISO/IEC 25010 (SQuaRE: 시스템 및 SW 품질 모델) |
| **국내 인증** | GS인증 (TTA, Good Software) |
| **법적 근거** | 소프트웨어 진흥법 제20조 |
| **목적** | 공공기관 SW 조달 품질 보증, 소비자 선택 정보 제공 |

### ISO/IEC 25010 품질 특성 (8개)

| 품질 특성 | 설명 |
|---------|------|
| **기능 적합성** | 명시된 기능 요구사항 충족 |
| **성능 효율성** | 자원 사용 대비 성능 (응답시간, 처리량) |
| **호환성** | 다른 시스템·환경과 공존·상호운용 |
| **사용성** | 사용자가 목표 달성에 필요한 노력 |
| **신뢰성** | 명시 조건에서 기능 유지 능력 |
| **보안성** | 비인가 접근 방지, 데이터 보호 |
| **유지보수성** | 수정·개선의 용이성 |
| **이식성** | 다른 환경으로 이전 용이성 |

### GS인증 절차

```
신청 (SW 개발사)
    ↓
접수 및 심사 계획 (TTA)
    ↓
기능 시험 + 성능 시험 + 보안 시험
    ↓
결함 수정 → 재시험 (필요 시)
    ↓
심의위원회 → 인증 결정
    ↓
GS 1등급 / GS 2등급 인증서 발급
```

### GS인증 등급

| 등급 | 기준 |
|------|------|
| **1등급** | 기능·성능·보안 전항목 통과 |
| **2등급** | 기능 항목 통과 (성능·보안 일부 제외) |

### 공공기관 SW 조달 우대

- GS인증 제품: 나라장터 조달 시 가산점
- 「SW진흥법」: 공공기관 GS인증 SW 우선 구매 권고
