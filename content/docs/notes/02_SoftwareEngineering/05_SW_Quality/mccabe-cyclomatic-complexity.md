## 개요

맥케이브 순환복잡도(Cyclomatic Complexity, CC)는 소프트웨어 모듈의 복잡성을 정량적으로 측정하는 지표로, 프로그램의 제어 흐름 그래프(CFG)에서 독립적 실행 경로의 수를 나타낸다.

## 핵심 내용

### 계산 방법 (3대 계산 기법)

순환복잡도를 계산하는 공식적인 방법은 크게 3가지가 있으며, 시험장에서 교차 검증을 통해 실수를 예방해야 한다.

#### 1. 제어 흐름 그래프(CFG) 기반 계산식
*   **일반 그래프 공식 (출구에서 입구로의 가상이 없는 경우)**:
    ```
    V(G) = E - N + 2P
    ```
    - $E$: 간선(Edge) 수, $N$: 노드(Node) 수, $P$: 연결된 성분 수 (단일 모듈/함수 시 $P=1$이므로 $V(G) = E - N + 2$)
*   **강결합 그래프 공식 (출구에서 입구로의 가상 간선 1개를 추가한 경우)**:
    ```
    V(G) = E - N + P
    ```
    - 단일 모듈/함수 시 $P=1$이므로 $V(G) = E - N + 1$

#### 2. 영역(Region) 계산법 (가장 직관적이고 빠름)
제어 흐름 그래프를 교차하는 간선이 없는 **평면 그래프(Planar Graph)**로 그렸을 때의 영역 수로 계산한다.
```
V(G) = 평면 그래프의 닫힌 영역(면)의 수 + 1 (외부 영역)
```

#### 3. 분기 노드(Predicate Node, 결정 노드) 수 기반 계산식
*   **공식**:
    ```
    V(G) = d + 1
    ```
    - $d$: 분기 노드(Out-degree가 2 이상인 노드, 예: `if`, `while`)의 개수
*   **🚨 주의 사항 (복합 조건문)**:
    - 소스 코드에 `if (A && B)` 또는 `if (A || B)`와 같은 복합 조건문이 있는 경우, 컴파일러 관점의 CFG에서는 **각각의 조건(A, B)이 독립적인 분기 노드로 분할**되므로 복잡도에 각각 +1씩 반영되어 $d=2$가 됨.

### 복잡도 기준

| 복잡도 | 위험도 | 의미 |
|--------|--------|------|
| 1 ~ 10 | 낮음 | 단순 모듈, 높은 테스트 용이성 |
| 11 ~ 20 | 보통 | 중간 복잡도, 주의 필요 |
| 21 ~ 50 | 높음 | 리팩토링 권고 |
| 50 초과 | 매우 높음 | 즉시 리팩토링 필요 |

### 활용

- **테스트 케이스 하한선**: 최소 V(G)개의 독립 경로를 테스트해야 기본 경로 커버리지 달성
- **리팩토링 기준**: 복잡도 높은 모듈 우선 리팩토링 대상 선정
- **코드 리뷰**: 신규 코드의 복잡도 상한선 기준으로 활용

## 실무 적용 예시

CI/CD 파이프라인에서 SonarQube 같은 정적 분석 도구로 모든 커밋의 순환복잡도를 자동 측정하고, 임계값(예: 15)을 초과하면 빌드를 실패 처리하여 코드 품질을 자동으로 관리한다.

## 출제 포인트

- **순환복잡도 3대 계산식**: $E-N+2P$ 공식, **평면 영역 수 + 1** 기법, **분기 노드($d$) + 1** 기법 적용 능력
- **강결합 그래프(Strongly Connected)** 개념 적용 시 가상 간선 처리와 공식 변화($E-N+P$) 설명
- 복합 조건문(`&&`, `||`)이 있을 때 분기 노드가 어떻게 분할되어 복잡도에 영향을 주는지 예시 CFG로 매핑 및 계산 능력
- **화이트박스 테스트와의 관계**: 순환복잡도는 테스트해야 하는 **기본 경로(Basis Path) 커버리지의 하한선(최소 갯수)**을 의미함 명시
