## 개요

소프트웨어 프로세스(SP) 품질인증은 「소프트웨어 프로세스 품질인증 운영에 관한 지침」(과기정통부)에 근거하여 SW 개발 프로세스의 성숙도를 평가·인증하는 제도다. TTA(한국정보통신기술협회)가 운영하며, CMMI·ISO/IEC 15504(SPICE) 기반의 국내 프로세스 품질 인증이다.

## 핵심 내용

### 가. 소프트웨어 프로세스 품질인증 기준

**인증 기준 체계**:

| 영역 | 내용 |
|------|------|
| **프로세스 관리** | 프로세스 정의·문서화·개선 체계 |
| **프로젝트 관리** | 일정·비용·리스크·품질 관리 |
| **개발 프로세스** | 요구사항·설계·구현·시험 수행 방법 |
| **형상 관리** | 산출물 버전 관리, 변경 통제 |
| **품질 보증** | 프로세스·제품 품질 검토, 감사 |
| **위험 관리** | 위험 식별·분석·대응 계획 |

**심사 방식**:
- 심사원이 프로세스 문서·수행 기록·면담으로 이행 수준 평가
- 각 프로세스 영역별 이행 수준(0~5단계) 판정

### 나. 소프트웨어 프로세스 인증등급 기준

**SP 인증 등급 체계**:

| 등급 | 기준 | 의미 |
|------|------|------|
| **SP 1등급** | 핵심 프로세스 전항목 Level 3 이상 | 정의된 프로세스가 조직 전체에 표준화 |
| **SP 2등급** | 핵심 프로세스 전항목 Level 2 이상 | 프로세스가 계획·추적되고 관리됨 |

**SPICE/CMMI 성숙도 레벨 대응**:

| 수준 | 명칭 | 특징 |
|------|------|------|
| Level 0 | 불완전 | 프로세스 미수행 또는 목적 달성 실패 |
| Level 1 | 수행 | 프로세스 수행, 산출물 존재 |
| Level 2 | 관리 | 계획·모니터링·추적, 산출물 관리 |
| Level 3 | 정의 | 표준 프로세스 정의, 조직 전체 적용 |
| Level 4 | 정량 관리 | 측정 기반 관리, 통계적 분석 |
| Level 5 | 최적화 | 지속적 개선, 혁신 |

### SP 인증 vs GS 인증 비교

| 구분 | SP 인증 | GS 인증 |
|------|--------|--------|
| **대상** | SW 개발 프로세스 | SW 제품 |
| **평가 내용** | 프로세스 성숙도 | 기능·성능·보안 |
| **표준** | ISO 15504(SPICE) | ISO 25010 |
| **운영** | TTA | TTA |

## 출제 포인트

- SP 인증 기준: 6개 프로세스 영역 (프로세스·프로젝트·개발·형상·품질보증·위험 관리)
- 인증 등급: 1등급(Level 3 이상) vs 2등급(Level 2 이상)
- SPICE Level 0~5: 불완전→수행→관리→정의→정량→최적화
