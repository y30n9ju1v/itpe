## 개요

플랫폼 엔지니어링(Platform Engineering)은 개발팀이 소프트웨어를 빠르고 안정적으로 개발·배포할 수 있도록 내부 개발자 플랫폼(IDP: Internal Developer Platform)을 설계·구축·운영하는 엔지니어링 분야다. Gartner 선정 2023-2024 주목 기술 트렌드로, DevOps의 인지 부하(Cognitive Load) 문제를 해결한다.

## 핵심 내용

### 등장 배경

- DevOps 도입 후 개발자가 CI/CD, 클라우드, 모니터링, 보안까지 모두 관리 → **인지 부하 과중**
- 팀마다 중복 인프라 구축 → 비효율
- 플랫폼 엔지니어링: 공통 플랫폼을 제공해 개발자가 비즈니스 로직에 집중하게 함

### IDP (Internal Developer Platform) 구성요소

| 구성요소 | 내용 |
|---------|------|
| **셀프서비스 포털** | 개발자가 UI/CLI로 인프라·서비스 자동 프로비저닝 |
| **황금 경로 (Golden Path)** | 검증된 개발·배포 표준 경로 제공 |
| **CI/CD 파이프라인** | 빌드·테스트·배포 자동화 |
| **관측 가능성** | 로그·메트릭·트레이스 통합 대시보드 |
| **보안·컴플라이언스** | 정책을 코드로(Policy as Code), 자동 보안 스캔 |

### 플랫폼 엔지니어링 vs DevOps

| 구분 | DevOps | 플랫폼 엔지니어링 |
|------|--------|----------------|
| **접근** | 개발+운영 문화·협업 | 내부 플랫폼 제품화 |
| **사용자** | 개발팀 전체 | 개발자 (내부 고객) |
| **목표** | 빠른 배포 | 개발자 자율성 + 인지 부하 감소 |
| **제품** | 없음 (문화) | IDP (소프트웨어 제품) |

### Backstage (오픈소스 IDP 대표)

- Spotify가 개발, CNCF 프로젝트
- 서비스 카탈로그, 기술 문서, 스캐폴딩(템플릿) 통합

## 출제 포인트

- IDP 개념: 내부 개발자 플랫폼 셀프서비스
- 등장 배경: DevOps 인지 부하 문제 해결
- 황금 경로(Golden Path): 표준화된 개발 경로
