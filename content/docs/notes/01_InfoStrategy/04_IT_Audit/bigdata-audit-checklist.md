## 개요

「지능정보기술 감리 실무 가이드」(한국IT감리·컨설팅협회)는 빅데이터, AI, 클라우드 등 지능정보기술 기반 정보화 사업에 대한 감리 수행 점검항목을 제시한다. 기존 일반 감리와 달리 데이터 분석·클라우드 전환의 특수성을 반영한 체계적 점검 기준을 제공한다.

## 핵심 내용

### 가. 빅데이터 분석단계 점검항목

빅데이터 사업의 분석단계(데이터 수집→처리→분석→시각화)별 감리 점검사항:

| 단계 | 점검항목 |
|------|---------|
| **데이터 수집** | 수집 데이터 적법성(개인정보 동의), 출처 데이터 품질 기준, 실시간/배치 수집 설계 적절성 |
| **데이터 저장** | 데이터 레이크 설계(원본 보존), 데이터 분류·보안 레벨, 스토리지 용량 계획 |
| **데이터 처리** | 전처리(정제·변환) 로직 정확성, 데이터 파이프라인 신뢰성, 처리 성능 목표 달성 |
| **데이터 분석** | 분석 알고리즘 적절성, 결과 검증 방법, 편향(Bias) 방지 조치 |
| **시각화·서비스** | 대시보드 정확성, 비전문가 이해 가능성, 보안 접근 통제 |

**공통 점검항목**:
- 개인정보·민감 데이터 비식별화 조치
- 데이터 거버넌스 체계(소유자·책임자 지정)
- 데이터 품질 측정 지표 설정 여부

### 나. 클라우드 계획수립 점검항목

공공기관 클라우드 전환 계획수립 단계 감리 점검항목:

| 분야 | 점검항목 |
|------|---------|
| **전략 수립** | 클라우드 전환 목표·범위 명확성, ROI 분석, 단계별 전환 로드맵 |
| **보안 계획** | CSAP 인증 여부 확인, 책임 공유 모델 명확화, 데이터 분류 기준 |
| **서비스 모델** | IaaS/PaaS/SaaS 선택 근거, 멀티클라우드·하이브리드 전략 |
| **데이터 관리** | 데이터 주권(소재지), 백업·복구 계획, 데이터 이동성 보장 |
| **운영 계획** | SLA 기준 설정, 모니터링 체계, 장애 대응 절차 |
| **비용 관리** | 비용 예측 모델, 사용량 최적화 방안, 예산 통제 방법 |

## 출제 포인트

- 빅데이터 분석 5단계: 수집→저장→처리→분석→시각화 단계별 점검항목
- 클라우드 점검 6개 분야: 전략·보안·서비스모델·데이터·운영·비용
- 개인정보 비식별화와 CSAP 인증이 핵심 점검사항
