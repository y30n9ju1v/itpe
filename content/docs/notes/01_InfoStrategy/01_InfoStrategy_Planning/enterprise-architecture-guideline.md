## 개요

「정보기술 아키텍처(EA: Enterprise Architecture) 도입·운영 지침」(행안부)은 공공기관이 범정부 EA를 도입·운영할 때 준수해야 할 기준을 규정한다. EA는 조직의 업무·정보·기술·응용 자원을 체계적으로 관리하여 IT 투자 효율화와 상호운용성을 확보하는 관리 도구다.

## 핵심 내용

### EA(Enterprise Architecture) 개요

- **정의**: 조직의 목표와 IT 자원(업무·정보·응용·기술) 간의 관계를 명확히 기술하는 청사진
- **목적**: IT 자원의 효율적 관리, 정보시스템 상호운용성, 중복 투자 방지

### 가. 범정부 정보기술 아키텍처 메타모델

메타모델은 EA를 구성하는 요소들의 분류 체계와 관계를 정의한 모델이다.

**5대 아키텍처 영역**:

| 영역 | 약자 | 내용 |
|------|------|------|
| **성과 아키텍처** | PA (Performance Architecture) | 비전·목표·성과지표 |
| **업무 아키텍처** | BA (Business Architecture) | 업무 기능·프로세스·조직 |
| **정보 아키텍처** | IA (Information Architecture) | 데이터 흐름·정보 모델 |
| **응용 아키텍처** | AA (Application Architecture) | 정보시스템·애플리케이션 |
| **기술 아키텍처** | TA (Technology Architecture) | 인프라·HW·SW·네트워크 |

### 나. 범정부 정보기술 아키텍처 참조모형

참조모형(Reference Model)은 아키텍처의 표준 분류 체계를 제공한다.

| 참조모형 | 약자 | 내용 |
|---------|------|------|
| **성과 참조모형** | PRM | 정보화 사업 성과 측정 체계 |
| **업무 참조모형** | BRM | 정부 업무 기능 분류 체계 |
| **서비스 참조모형** | SRM | 정보서비스 분류 체계 |
| **데이터 참조모형** | DRM | 공통 데이터 구조·표준 |
| **기술 참조모형** | TRM | 기술 구성요소·표준 목록 |

### 다. 범정부 정보기술 아키텍처 성숙도 모델

EA 성숙도는 조직의 EA 도입·활용 수준을 5단계로 평가한다.

| 단계 | 명칭 | 특징 |
|------|------|------|
| **0단계** | 부재 | EA 개념 인식 없음 |
| **1단계** | 착수 | EA 수립 시작, 문서화 초기 |
| **2단계** | 개선 | 일부 영역 EA 활용 |
| **3단계** | 정착 | 전사 EA 활용, IT 투자 연계 |
| **4단계** | 관리 | EA 기반 의사결정, 지속적 개선 |
| **5단계** | 최적화 | EA와 조직 전략 완전 통합 |

## 출제 포인트

- EA 5대 영역: 성과·업무·정보·응용·기술 (PA·BA·IA·AA·TA)
- 참조모형 5종: PRM·BRM·SRM·DRM·TRM
- 성숙도 5단계 (0~5단계 또는 1~5단계) 흐름 서술
