## 개요

IT 거버넌스는 이사회와 경영진이 IT 전략·투자·위험을 경영 목표와 일치시키고, IT 자원을 책임 있게 통제·감독하는 프레임워크다. ISO/IEC 38500이 국제 표준이며, 기업 거버넌스(Corporate Governance)의 필수 구성요소로 인식된다.

## 핵심 내용

### 가. IT 거버넌스 구성요소

**ISO/IEC 38500 기반 6대 원칙**

| 원칙 | 내용 |
|------|------|
| **책임(Responsibility)** | IT 사용·공급에 대한 책임 명확화 |
| **전략(Strategy)** | IT 계획이 비즈니스 전략과 정렬 |
| **획득(Acquisition)** | IT 투자의 합리적 이유와 타당성 |
| **성과(Performance)** | IT가 비즈니스에 기여하는 성과 확인 |
| **적합성(Conformance)** | 법규·규정 준수 |
| **인간행동(Human Behavior)** | IT 정책이 인간 행동 변화 고려 |

**COBIT(Control Objectives for IT) 프레임워크**

| 도메인 | 내용 |
|--------|------|
| **계획 및 조직** | IT 전략·아키텍처·방향 설정 |
| **획득 및 구현** | IT 솔루션 도입·변경관리 |
| **인도 및 지원** | IT 서비스 제공·보안·연속성 |
| **모니터링 및 평가** | 성과 측정·감사·컴플라이언스 |

### 나. IT 거버넌스 효과 측정 지표

**재무적 지표**

| 지표 | 설명 |
|------|------|
| **IT ROI** | IT 투자 대비 수익률 |
| **IT 비용 비율** | IT 비용 / 전체 매출 |
| **TCO(총소유비용)** | 시스템 라이프사이클 전체 비용 |
| **IT Value Realization** | 계획 대비 실제 비즈니스 가치 달성율 |

**운영 지표**

| 지표 | 설명 |
|------|------|
| **시스템 가용성** | 업타임, SLA 준수율 |
| **IT 프로젝트 완수율** | 예산·일정·품질 목표 달성 비율 |
| **변경 성공률** | 장애 없는 변경 비율 (CAB 승인 후) |
| **보안 사고 건수** | 침해사고·데이터 유출 발생 빈도 |

**전략 정렬 지표**

| 지표 | 설명 |
|------|------|
| **IT 전략 목표 달성율** | 디지털 전환 KPI 달성 정도 |
| **비즈니스 프로세스 자동화율** | RPA·AI 도입으로 자동화된 업무 비율 |

### 다. IT 거버넌스 효과 측정 방법론

**BSC (Balanced Scorecard) 적용**

| 관점 | IT 거버넌스 지표 예시 |
|------|---------------------|
| 재무 | IT ROI, TCO 절감 |
| 고객 | 사용자 만족도, 디지털 서비스 접근성 |
| 내부 프로세스 | 변경 성공률, ITIL 프로세스 성숙도 |
| 학습·성장 | IT 인력 역량, 신기술 도입 속도 |

**CMMI (Capability Maturity Model Integration)**
- 프로세스 성숙도 5단계(초기→관리→정의→정량적관리→최적화)로 IT 거버넌스 수준 측정

**IT 감사 (IT Audit)**
- 내부 감사: COBIT 통제 목적 기준으로 IT 통제 적정성 검토
- 외부 감사: ISO 27001, SOC 2 인증 취득으로 외부 검증

## 출제 포인트

- ISO/IEC 38500 6대 원칙 암기
- COBIT 4대 도메인 구조
- 재무·운영·전략 정렬 지표 분류
- BSC 4가지 관점에서의 IT 거버넌스 지표 매핑
