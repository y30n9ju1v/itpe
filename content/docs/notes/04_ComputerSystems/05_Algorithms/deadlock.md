## 개요

데드락(Deadlock)은 두 개 이상의 프로세스가 서로 상대방이 점유한 자원을 기다리며 영원히 진행되지 않는 상태다. 운영체제 자원 관리에서 반드시 해결해야 하는 고전적 문제다.

## 핵심 내용

### 데드락 발생 4가지 필요충분 조건 (Coffman 조건)

데드락은 아래 4가지 조건이 **동시에** 성립할 때만 발생한다.

| 조건 | 설명 | 예시 |
|------|------|------|
| **상호 배제 (Mutual Exclusion)** | 자원은 한 번에 하나의 프로세스만 사용 가능 | 프린터, 파일 잠금 |
| **점유와 대기 (Hold and Wait)** | 자원을 점유한 상태에서 다른 자원을 추가 요청하며 대기 | A가 R1 점유, R2 대기 |
| **비선점 (No Preemption)** | 다른 프로세스가 점유 중인 자원을 강제로 빼앗을 수 없음 | CPU와 달리 I/O는 선점 불가 |
| **환형 대기 (Circular Wait)** | 프로세스들이 원형 사슬로 서로의 자원을 대기 | A→B→C→A 순환 |

### 데드락 처리 방법 4가지

| 방법 | 설명 | 장단점 |
|------|------|--------|
| **예방 (Prevention)** | 4가지 조건 중 하나를 원천 제거 | 자원 낭비, 처리율 저하 |
| **회피 (Avoidance)** | 안전 상태 유지 (은행가 알고리즘) | 자원 최대 요구량 사전 파악 필요 |
| **탐지 (Detection)** | 데드락 발생 허용 후 탐지 | 탐지 비용, 복구 필요 |
| **무시 (Ignorance)** | 데드락 발생 시 재부팅 (Ostrich 알고리즘) | 단순, 사용자에게 책임 전가 |

### 예방 방법 구체화

- **상호 배제 제거**: 불가능한 경우가 많음 (프린터는 반드시 상호 배제)
- **점유와 대기 제거**: 실행 전 모든 자원 한꺼번에 요청 (자원 낭비)
- **비선점 제거**: CPU는 선점 가능, I/O는 어려움
- **환형 대기 제거**: 자원에 순서 번호 부여, 오름차순으로만 요청

### 은행가 알고리즘 (Banker's Algorithm)

안전 상태(Safe State): 모든 프로세스가 최종적으로 완료될 수 있는 자원 할당 순서가 존재하는 상태

```
자원 요청 발생 시:
1. 요청 자원을 임시로 할당
2. 안전 상태인지 검사
3. 안전 상태 → 할당 확정
4. 불안전 상태 → 할당 거부, 원래 상태로 복구
```

### 데드락 탐지

- **자원 할당 그래프(RAG)**: 프로세스-자원 간 요청/할당 관계 그래프
- **사이클 존재 = 데드락** (단일 인스턴스 자원의 경우)
- 복수 인스턴스: 은행가 알고리즘 변형으로 탐지
