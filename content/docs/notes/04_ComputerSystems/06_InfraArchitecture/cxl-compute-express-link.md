## 개요

CXL(Compute Express Link)은 CPU·GPU·FPGA·메모리 등 이종 디바이스 간 고속·저지연 캐시 코히런트 통신을 지원하는 개방형 인터커넥트 표준이다. PCIe 물리 계층 위에서 동작하며, AI·빅데이터 워크로드에서 메모리 대역폭 병목을 해소하는 핵심 기술로 주목받는다.

## 핵심 내용

**CXL 개요**
- 발표: Intel 주도, 2019년 CXL 1.0 (현재 CXL 3.1)
- 기반: PCIe 6.0 물리 계층
- 목적: CPU와 가속기·메모리 간 캐시 코히런스 유지하며 고속 데이터 공유

**CXL 3가지 프로토콜**

| 프로토콜 | 용도 | 설명 |
|---------|------|------|
| **CXL.io** | I/O 및 설정 | PCIe 기반 표준 I/O 프로토콜. 디바이스 발견, 링크 구성, 레거시 register 접근 등 (필수 탑재) |
| **CXL.cache** | 가속기 캐시 | 가속기(GPU/NPU)가 호스트(CPU)의 시스템 메모리를 동적 캐싱하여 지연 시간 없이 상호 공유 |
| **CXL.mem** | 호스트 메모리 접근 | 호스트(CPU)가 가속기나 확장 메모리에 마치 로컬 메모리(DRAM)처럼 직접 메모리 맵(load/store) 접근 |

**CXL 버전별 진화 단계 및 핵심 차별점 (고득점 필수 테이블)**

| 구분 | CXL 1.0 / 1.1 | CXL 2.0 | CXL 3.0 / 3.1 |
|------|---------------|---------|---------------|
| **물리 계층** | PCIe 5.0 (32GT/s) | PCIe 5.0 (32GT/s) | **PCIe 6.0 (64GT/s) / PAM4** |
| **연결 토폴로지** | Point-to-Point (단일 1:1) | **CXL Switch 도입** (단일 스위치) | **Fabric 지원** (Multi-stage 스위치, Mesh 등) |
| **메모리 기술** | 메모리 확장만 가능 | **메모리 풀링 (Memory Pooling)** | **메모리 공유 (Memory Sharing)** |
| **보안 기술** | 없음 | 링크 암호화 (IDE) | IDE 강화, 가상화 보안 |
| **상태 관리** | 정적 할당 | 동적 할당 (DCD 지원 시작) | **DCD(Dynamic Capacity Device)** 고도화 |

**메모리 풀링(Pooling) vs 메모리 공유(Sharing) 구분**
*   **메모리 풀링 (CXL 2.0)**: 메모리 풀에서 다수의 호스트에 독립적인 메모리 영역을 **논리적으로 분할하여 할당**함. 호스트 간 데이터를 직접 실시간 공유할 수는 없고 동적 재할당만 가능 (Static/Dynamic Partitioning).
*   **메모리 공유 (CXL 3.0)**: **여러 호스트가 동일한 물리적 메모리 영역에 캐시 일관성(Coherency)을 유지하면서 동시 접근 및 실시간 데이터 공유** 가능. 거대 AI 분산 학습의 데이터 병목 제거 극대화.

**주요 활용 사례**

| 시나리오 | CXL 역할 및 장점 |
|----------|-----------------|
| **메모리 풀링** | 메모리 유휴 자원(Stranded Memory)을 회수하여 전체 인프라의 TCO(총소유비용)를 획득하고 효율성 극대화 |
| **AI 가속기 연결** | 초대형 가속기 군집 환경에서 GPU HBM 메모리 부족 시 CXL 메모리 버퍼로 무한 확장 지원 |
| **메모리 공유** | 대규모 클러스터에서 노드 간 복사(Serialization/Deserialization) 없이 초고속 메모리 데이터 공유 수행 |
| **이종 컴퓨팅** | CPU-GPU-FPGA가 단일 메모리 풀 공간에서 Zero-copy 데이터 파이프라인 형성 |

**기존 기술과의 비교**

| 구분 | PCIe | NVLink | CXL |
|------|------|--------|-----|
| 캐시 코히런스 | 없음 | 있음 (NVIDIA only) | 있음 (개방 표준) |
| 메모리 공유 | 제한적 | GPU간 | CPU-가속기-메모리 |
| 표준화 | 개방 | 독점 | 개방 |

## 실무 적용 예시

대형 LLM 추론 서버에서 GPU의 HBM 메모리 한계를 CXL-attached DRAM으로 투명하게 확장하여, 더 큰 모델을 단일 노드에서 실행하는 구성이 확산되고 있다.

## 출제 포인트

- **CXL의 3가지 서브 프로토콜**: `.io`, `.cache`, `.mem` 의 동작 원리와 기능 결합성
- **CXL 세대별 발전 동향**: PCIe 5.0 ➡️ 6.0 진화 및 스위치, 패브릭 토폴로지 확장
- **메모리 풀링(CXL 2.0의 분할 사용) vs 메모리 공유(CXL 3.0의 동시 동시공유)**의 공학적 차이점 비교
- AI 데이터센터에서의 TCO 절감, HBM 용량 한계 극복 대안으로서의 CXL-attached Memory 적용 구조
