## 개요

「정보시스템 하드웨어 규모산정 지침(TTAK.KO-10.0292/R3)」은 공공기관 정보시스템 구축 시 CPU·메모리·스토리지 등 하드웨어 규모를 산정하는 TTA 단체표준이다. 3가지 방법으로 적정 HW 규모를 도출한다.

## 핵심 내용

### 3가지 HW 규모산정 방법

#### 1. 기준 성능법 (Benchmark-based Sizing)

- **방법**: 표준 벤치마크(TPC-C, SPEC CPU 등)로 측정된 기준 성능치 활용
- **절차**: 업무 트랜잭션 수 산출 → 기준 TPS로 환산 → 벤치마크 성능 비교
- **장점**: 객관적 측정 기반, 비교 가능
- **단점**: 실제 워크로드와 벤치마크 환경 차이 존재

#### 2. 유사 시스템 기준법 (Reference-based Sizing)

- **방법**: 기 구축된 유사 규모·업무의 시스템 성능 데이터를 참조
- **절차**: 유사 시스템 식별 → 규모 차이 보정 계수 적용 → 규모 산정
- **장점**: 실제 운영 데이터 기반, 현실적
- **단점**: 적합한 참조 시스템 확보 어려움

#### 3. 용량 계획법 (Capacity Planning)

- **방법**: 업무량(트랜잭션, 사용자, 데이터량)을 분석하여 자원 소요량 계산
- **절차**: 업무 분석 → 피크 부하 산정 → 자원 요구사항 계산 → 여유율 적용
- **주요 공식**:
  ```
  CPU 코어 수 = (최대 TPS × 평균 CPU 사용 시간) / (1 - 여유율)
  메모리 = 동시 사용자 수 × 1인당 메모리 + OS·미들웨어 고정 메모리
  스토리지 = (일 데이터 증가량 × 보존 기간) × (1 + RAID 오버헤드) × 여유율
  ```
- **여유율**: 일반 30%, 고부하 40~50% 적용

### HW 규모산정 시 고려 사항

| 요소 | 내용 |
|------|------|
| **피크 부하** | 일최대 동시접속자, 최대 TPS |
| **성장률** | 연간 데이터·트랜잭션 증가율 |
| **가용성 요건** | HA 이중화 구성 시 자원 추가 |
| **여유율** | CPU 30~40%, 메모리 30%, 스토리지 20~30% |

## 출제 포인트

- 3가지 방법: 기준 성능법, 유사 시스템 기준법, 용량 계획법
- 각 방법의 특징·장단점 비교
- 여유율과 피크 부하 개념
- 스토리지 산정 시 RAID 오버헤드 포함
