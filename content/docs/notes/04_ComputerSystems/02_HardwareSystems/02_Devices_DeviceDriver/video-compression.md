## 개요

영상압축은 멀티미디어 데이터의 저장·전송 효율을 높이기 위해 데이터 크기를 줄이는 기법이다. 원본 복원 가능 여부에 따라 무손실(Lossless)·손실(Lossy)·혼합(Hybrid) 압축으로 분류된다.

## 핵심 내용

### 무손실 압축기법

원본 데이터를 완벽히 복원 가능. 압축률은 낮으나 품질 손실 없음.

| 기법 | 원리 | 예시 |
|------|------|------|
| **런-렝스 인코딩(RLE)** | 반복 패턴을 횟수+값으로 표현 | BMP, TIFF |
| **허프만 코딩** | 빈도 높은 문자에 짧은 코드 할당 | ZIP, PNG |
| **LZW** | 반복 문자열 딕셔너리 압축 | GIF, TIFF |
| **산술 코딩** | 확률 모델 기반 범위 인코딩 | JPEG 2000 일부 |

### 손실 압축기법

일부 데이터를 버려 높은 압축률. 사람의 지각 특성(HVS) 활용. 복원 불가.

| 기법 | 원리 | 예시 |
|------|------|------|
| **DCT** (이산코사인변환) | 공간 → 주파수 변환, 고주파 성분 제거 | JPEG, MPEG |
| **웨이블릿 변환** | 다해상도 주파수 분석 | JPEG 2000 |
| **예측 부호화** | 인접 픽셀과의 차이만 저장 | H.264 인트라 |
| **모션 보상** | 이전 프레임 대비 움직임 벡터 저장 | H.264/HEVC 인터 |

### 혼합 압축기법

무손실과 손실을 결합하여 품질·압축률을 균형있게 달성.

| 표준 | 특징 | 사용 |
|------|------|------|
| **H.264 (AVC)** | DCT + 모션보상 + CABAC(무손실 엔트로피) | 스트리밍, Blu-ray |
| **H.265 (HEVC)** | H.264 대비 50% 압축률 향상, 4K/8K | OTT, UHD 방송 |
| **AV1** | 오픈소스, HEVC보다 20% 효율 | YouTube, Netflix |
| **VVC (H.266)** | 8K·HDR 목표, HEVC 대비 50% 향상 | 차세대 표준 |

### 영상 압축 원리 (I/P/B 프레임)

- **I 프레임(Intra)**: 독립적 완전 영상 (무손실 기준점)
- **P 프레임(Predicted)**: 이전 프레임 대비 변화만 저장
- **B 프레임(Bidirectional)**: 앞·뒤 프레임 참조, 최고 압축률

## 출제 포인트

- 무손실·손실·혼합 압축의 정의와 대표 알고리즘
- DCT + 모션보상이 혼합 압축의 핵심
- H.264/HEVC/AV1 세대별 압축 효율 발전
