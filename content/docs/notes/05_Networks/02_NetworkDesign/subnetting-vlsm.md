## 개요

서브네팅(Subnetting)은 하나의 네트워크 주소를 여러 작은 서브네트워크로 분할하는 기법이며, 슈퍼네팅(Supernetting)은 연속된 여러 네트워크를 하나로 통합하는 기법이다. VLSM(Variable Length Subnet Mask)은 서브넷마다 다른 크기의 마스크를 적용하여 IP 주소를 효율적으로 할당한다.

## 핵심 내용

### 가. 슈퍼네팅과 서브네팅 개념

| 구분 | 서브네팅 | 슈퍼네팅 |
|------|---------|---------|
| 방향 | 큰 네트워크 → 작은 서브넷 분할 | 작은 네트워크들 → 큰 네트워크 통합 |
| 마스크 | 프리픽스 길이 증가 (/24 → /26) | 프리픽스 길이 감소 (/24 → /22) |
| 목적 | IP 주소 낭비 최소화, 보안 격리 | 라우팅 테이블 축소 (CIDR) |
| 활용 | LAN 분할, 부서별 네트워크 | ISP 경로 집약 |

### 나. VLSM 적용 예제

**조건**: 192.168.0.0/22 (총 IP: 1024개, 192.168.0.0 ~ 192.168.3.255)
- 개발팀 500명, 영업팀 250명, 인사팀 125명, 보안팀 125명

**VLSM 분할 원칙**: 가장 큰 서브넷 먼저 할당

| 팀 | 필요 호스트 | 필요 블록 | 서브넷 주소 | 서브넷 마스크 | 할당 가능 호스트 IP |
|----|-----------|---------|------------|-------------|------------------|
| 개발팀 | 500 | 512 (2⁹) | 192.168.0.0 | /23 (255.255.254.0) | 192.168.0.1 ~ 192.168.1.254 |
| 영업팀 | 250 | 256 (2⁸) | 192.168.2.0 | /24 (255.255.255.0) | 192.168.2.1 ~ 192.168.2.254 |
| 인사팀 | 125 | 128 (2⁷) | 192.168.3.0 | /25 (255.255.255.128) | 192.168.3.1 ~ 192.168.3.126 |
| 보안팀 | 125 | 128 (2⁷) | 192.168.3.128 | /25 (255.255.255.128) | 192.168.3.129 ~ 192.168.3.254 |

> 각 서브넷에서 첫 번째 주소(네트워크)와 마지막 주소(브로드캐스트)는 호스트에 할당 불가

### VLSM 계산 공식

- **필요 호스트 수** → 2ⁿ ≥ (호스트 수 + 2) 인 최소 n 결정
- **서브넷 마스크 프리픽스** = 32 - n
- **서브넷 크기** = 2ⁿ
