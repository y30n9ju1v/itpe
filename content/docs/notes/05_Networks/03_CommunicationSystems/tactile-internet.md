## 개요

촉각 인터넷(Tactile Internet)은 촉각(촉감·힘·진동)을 포함한 실시간 원격 상호작용을 지원하는 네트워크 패러다임이다. ITU-T에서 정의하며, 인간의 반응 시간(1ms 이하 지연)을 목표로 한다. 원격 수술, 원격 제어, 자율주행 등 물리적 세계와 디지털 세계의 실시간 결합을 실현한다.

## 핵심 내용

### 촉각 인터넷 정의 (ITU-T)

- **지연**: 편도(one-way) 1ms 이하 (인간 촉각 반응 시간: ~1ms)
- **신뢰성**: 99.9999% (six nines)
- **가용성**: 99.999%
- **보안**: 종단간 보안 필수

### 촉각 인터넷 vs 기존 인터넷

| 구분 | 현재 인터넷 | 촉각 인터넷 |
|------|-----------|-----------|
| **지연** | 수십~수백ms | 1ms 이하 |
| **전송 내용** | 데이터, 음성, 영상 | 데이터 + 촉각(힘·진동·온도) |
| **적용** | 통신, 엔터테인먼트 | 원격 제어, 수술, XR |
| **핵심 HW** | 서버, 라우터 | 햅틱 기기, 엣지 서버 |
| **요구 기술** | TCP/IP | 6G/5G, MEC, 햅틱 코덱 |

### 핵심 구성 요소

| 구성요소 | 역할 |
|---------|------|
| **햅틱(Haptic) 기기** | 촉각 신호 생성·수신 (글러브, 수트, 로봇 팔) |
| **햅틱 코덱** | 촉각 데이터 압축·전송 |
| **MEC (모바일 엣지 컴퓨팅)** | 지연 최소화를 위한 기지국 근처 처리 |
| **5G/6G 네트워크** | 초저지연 통신 인프라 |
| **AI 예측 제어** | 네트워크 지연을 AI가 예측·보정 |

### 응용 분야

| 분야 | 활용 예시 |
|------|---------|
| **원격 의료** | 의사가 원격에서 수술 도구 조작 |
| **자율주행** | 원격 운전자 개입, 실시간 촉각 피드백 |
| **교육·훈련** | 가상 실험실에서 물리적 감촉 체험 |
| **제조** | 원격 정밀 조립·검사 |
| **XR** | 메타버스 내 물리적 상호작용 |

## 출제 포인트

- 촉각 인터넷의 1ms 지연 요구: 인간 반응 시간 기반
- 6G/5G + MEC + 햅틱 기기 3대 요소
- 현재 인터넷 대비 차별점: 촉각 데이터 + 초저지연
