## 개요

네트워크 프로토콜(Protocol)은 통신 개체들이 데이터를 주고받기 위해 합의한 규칙과 약속의 집합이다. 프로토콜은 이기종 시스템 간 표준화된 통신을 가능하게 하며, 신택스·시맨틱스·타이밍의 3요소로 구성된다.

## 핵심 내용

### 가. 프로토콜의 개념

- **정의**: 컴퓨터 간 또는 네트워크 개체 간 데이터 교환을 위한 규칙과 절차의 집합
- **필요성**: 서로 다른 제조사·OS의 시스템 간 통신 표준화
- **표준화 기관**: ISO (OSI 모델), IETF (TCP/IP), IEEE (유무선 LAN), ITU-T (통신)

### 나. 프로토콜의 3가지 요소

| 요소 | 영문 | 내용 | 예시 |
|------|------|------|------|
| **구문** | Syntax | 데이터의 형식·구조·부호화 방식 | 헤더 필드 배치, 비트 순서, 인코딩(ASCII, Binary) |
| **의미** | Semantics | 각 비트·필드의 의미, 오류 제어·흐름 제어 방법 | ACK=수신 확인, SYN=연결 요청, 오류 시 재전송 규칙 |
| **타이밍** | Timing | 데이터 전송 속도·순서·동기화 | 흐름 제어(전송 속도 조절), 재전송 타임아웃(RTO) |

### 다. 프로토콜의 기능

| 기능 | 설명 |
|------|------|
| **캡슐화 (Encapsulation)** | 상위 계층 데이터에 헤더(주소·제어정보) 추가 |
| **단편화·재조립** | 큰 데이터를 패킷으로 분할, 수신 측에서 재조립 |
| **오류 제어** | CRC 검사, ARQ(자동 재전송 요청)로 신뢰성 확보 |
| **흐름 제어** | 슬라이딩 윈도우로 송수신 속도 조절 |
| **혼잡 제어** | 네트워크 과부하 방지 (TCP 혼잡 윈도우 조절) |
| **주소 지정** | MAC·IP·포트 번호로 송수신 개체 식별 |
| **다중화·역다중화** | 여러 통신 흐름을 하나의 물리 채널 공유 |
| **동기화** | 송수신 간 비트·블록·프레임 타이밍 일치 |

### 대표 프로토콜과 OSI 계층 매핑

| 계층 | 프로토콜 |
|------|---------|
| 7 (응용) | HTTP/HTTPS, FTP, SMTP, DNS, SNMP |
| 4 (전송) | TCP, UDP |
| 3 (네트워크) | IP, ICMP, OSPF, BGP |
| 2 (데이터링크) | Ethernet, PPP, HDLC |

## 출제 포인트

- 프로토콜 3요소: 구문(Syntax)·의미(Semantics)·타이밍(Timing) 각각의 의미 설명
- 주요 기능 8가지: 캡슐화·단편화·오류·흐름·혼잡·주소·다중화·동기화
- 흐름 제어 vs 혼잡 제어 차이
