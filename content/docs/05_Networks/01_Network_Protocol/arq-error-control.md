## 개요

오류 제어(Error Control)는 데이터 통신에서 전송 오류를 검출하고 정정하여 신뢰성 있는 전송을 보장하는 기술이다. ARQ(Automatic Repeat reQuest)는 수신 측의 오류 검출 후 재전송을 요청하는 대표적 오류 제어 방식이다.

## 핵심 내용

### 오류 검출 vs 오류 정정

| 구분 | 오류 검출(Error Detection) | 오류 정정(Error Correction) |
|------|---------------------------|----------------------------|
| 개념 | 오류 발생 여부 판단 | 오류를 직접 수정 |
| 방식 | CRC, 패리티 비트, 체크섬 | 해밍 코드, Reed-Solomon |
| 추가 비트 | 적음 | 많음 |
| 재전송 | 필요 | 불필요 (FEC) |
| 적용 환경 | 유선 통신 | 위성, 무선, CD/DVD |

**FEC(Forward Error Correction)**: 재전송 없이 수신 측에서 직접 정정. 위성·우주 통신에서 재전송 지연이 클 때 사용.

### 정지-대기 ARQ (Stop-and-Wait ARQ)

**개념**: 송신 측이 한 프레임을 전송한 후 ACK/NAK 수신까지 대기

**수행 절차**:
```
송신 → [Frame 0] → 수신
       대기...
수신 → [ACK 0] → 송신
송신 → [Frame 1] → 수신
수신 → [NAK 1] → 송신  (오류 발생)
송신 → [Frame 1 재전송] → 수신
```

**특징**:
- 구현이 단순, 한 번에 하나의 프레임만 전송
- 전파 지연이 큰 환경에서 효율 매우 낮음
- 효율 = 1 / (1 + 2a), a = 전파지연/전송시간

### Go-Back-N ARQ

**개념**: 슬라이딩 윈도우(Sliding Window)로 여러 프레임을 연속 전송하다 오류 프레임부터 **모두 재전송**

**수행 절차**:
```
송신 → [0][1][2][3][4]... (윈도우 크기만큼 연속 전송)
수신 → ACK 0, ACK 1, NAK 2 (프레임 2에서 오류)
송신 → [2][3][4] 재전송 (2 이후 모두 재전송)
```

**특징**:
- 윈도우 크기 = 2^n - 1 (n비트 순서 번호)
- 오류 하나에 윈도우 내 모든 프레임 재전송 → 대역폭 낭비
- Selective Repeat ARQ: 오류 프레임만 재전송 (버퍼 필요)

### Stop-and-Wait vs Go-Back-N 비교

| 구분 | Stop-and-Wait | Go-Back-N |
|------|--------------|-----------|
| 전송 방식 | 1 프레임씩 | 윈도우 크기만큼 |
| 재전송 범위 | 해당 프레임만 | 오류 이후 모두 |
| 효율 | 낮음 | 높음 |
| 구현 복잡도 | 단순 | 복잡 |
| 수신 버퍼 | 1 프레임 | 윈도우 크기 |
| 적합 환경 | 단순 통신 | 고속·위성 통신 |

## 출제 포인트

- 오류 검출과 오류 정정의 차이(재전송 필요 여부) 명확히
- Stop-and-Wait의 수행 절차를 타임라인 형태로 서술
- Go-Back-N의 핵심: 슬라이딩 윈도우 + 오류 이후 전부 재전송
- 두 방식의 효율·복잡도·버퍼 비교표 작성
