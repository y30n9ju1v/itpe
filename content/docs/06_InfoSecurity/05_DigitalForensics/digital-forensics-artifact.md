## 개요

디지털 포렌식에서 아티팩트(Artifact)는 사용자의 행위나 시스템 동작 과정에서 자동으로 생성·저장된 디지털 흔적(증거 데이터)이다. 악의적 행위 분석, 사고 원인 규명, 법적 증거 수집에 활용된다.

## 핵심 내용

### 아티팩트의 특징

- 사용자가 의도적으로 생성하지 않고 OS·애플리케이션이 자동 생성
- 삭제 후에도 저장 매체에 잔존할 수 있음
- 타임스탬프를 포함하여 행위 시간 순서 재구성 가능

### 주요 아티팩트 유형 (Windows 기준)

| 분류 | 아티팩트 | 포함 정보 |
|------|----------|----------|
| **레지스트리** | NTUSER.DAT, SAM, SYSTEM | 사용자 설정, 최근 실행 파일, 자동 시작 목록 |
| **이벤트 로그** | Security.evtx, System.evtx | 로그인 기록, 프로세스 실행, 정책 변경 |
| **파일 시스템** | $MFT (NTFS 마스터 파일 테이블) | 파일 생성·수정·삭제 기록 |
| **웹 브라우저** | 히스토리, 캐시, 쿠키 | 방문 URL, 다운로드 파일, 세션 정보 |
| **프리패치** | *.pf 파일 | 최근 실행된 프로그램 목록 |
| **링크 파일** | *.lnk | 최근 열람 파일 경로 |
| **페이지 파일** | pagefile.sys | 메모리 스왑 내용 (악성코드 흔적) |
| **섀도 복사본** | VSS | 이전 파일 버전 복원 |

### 휘발성 아티팩트 vs 비휘발성 아티팩트

| 구분 | 휘발성 | 비휘발성 |
|------|--------|---------|
| 저장 위치 | RAM, 캐시 | 디스크, 레지스트리 |
| 수명 | 전원 차단 시 소멸 | 삭제 전까지 유지 |
| 수집 우선순위 | 높음 (먼저 수집) | 낮음 |
| 예시 | 실행 중 프로세스, 네트워크 연결 | 로그 파일, 레지스트리 |

### 포렌식 수집 원칙

- **무결성 보존**: 수집 시 원본 변경 금지 → 쓰기 방지 장치(Write Blocker) 사용
- **휘발성 순서**: RFC 3227 기준 — RAM → 스왑 → 네트워크 → 디스크 순
- **해시 검증**: MD5/SHA-256으로 수집 전후 동일성 증명
- **연계 보관(Chain of Custody)**: 증거 이동·접근 이력 기록

## 출제 포인트

- 아티팩트의 정의(자동 생성 디지털 흔적)를 명확히 서술
- 휘발성·비휘발성 구분과 수집 우선순위
- 대표 아티팩트 5가지 이상 예시
- 무결성 보존 원칙(Write Blocker, 해시 검증) 포함
