## 개요

랜섬웨어(Ransomware)는 피해자 시스템의 데이터를 암호화하고 복호화 키를 담보로 금전(주로 암호화폐)을 요구하는 악성코드다. RaaS(Ransomware as a Service)는 랜섬웨어 개발자가 서비스 형태로 공격 도구를 제공하여 기술력 없는 공격자도 랜섬웨어를 활용할 수 있게 한 비즈니스 모델이다.

## 핵심 내용

### 랜섬웨어 vs RaaS 비교

| 구분 | 랜섬웨어 | RaaS |
|------|----------|------|
| 정의 | 데이터 암호화·금전 요구 악성코드 | 랜섬웨어를 서비스로 제공하는 모델 |
| 공격자 | 개발·운영을 직접 수행 | 개발자(Developer)와 운영자(Affiliate) 분리 |
| 진입 장벽 | 높음 (기술 필요) | 낮음 (구독·수익 배분) |
| 수익 구조 | 단독 수익 | 개발자 20~30% + 운영자 70~80% |
| 사례 | WannaCry, NotPetya | LockBit, REvil, BlackCat |

### 주요 감염 경로

1. **피싱 이메일**: 악성 첨부파일·링크 클릭
2. **취약점 익스플로잇**: 패치 안 된 시스템의 CVE 악용 (MS17-010)
3. **RDP 무차별 대입**: 약한 RDP 계정 탈취
4. **공급망 공격**: 신뢰할 수 있는 SW 업데이트에 악성코드 삽입
5. **악성 광고(Malvertising)**: 웹 광고를 통한 드라이브-바이 다운로드

### 암호화 동작 원리

```
1. 감염 시작
2. C2 서버 접속 → 공개키 수신
3. 대칭키(AES) 생성 → 파일 암호화
4. 대칭키를 공개키(RSA)로 암호화 → 로컬 저장
5. 원본 대칭키 삭제
6. 랜섬노트 표시 → 암호화폐 결제 요구
7. 결제 확인 → 개인키 전송 → 복호화 가능
```

비대칭+대칭 하이브리드 암호화: 성능(AES) + 보안(RSA) 장점 결합

### 피해 예방 및 사고 대응 절차

**예방**:
- 정기적 오프라인 백업 (3-2-1 원칙: 3개 복사본, 2개 미디어, 1개 오프사이트)
- 패치 관리 체계 (취약점 신속 패치)
- 이메일 필터링, 첨부파일 샌드박스 분석
- 네트워크 세분화 (측면 이동 차단)
- EDR 솔루션 도입

**사고 대응**:
1. **격리**: 감염 시스템 즉시 네트워크 분리
2. **신고**: KISA(한국인터넷진흥원) 118 신고
3. **증거 보존**: 포렌식 이미징 (전원 차단 전)
4. **피해 범위 파악**: 감염 확산 여부 확인
5. **복구**: 깨끗한 백업으로 복원
6. **재발 방지**: 감염 경로 분석, 보안 강화

## 출제 포인트

- 랜섬웨어와 RaaS 개념 비교 (진입장벽, 역할 분리)
- 암호화 원리: 하이브리드(AES+RSA) 방식 순서 서술
- 감염 경로 3~5개 열거
- 대응 절차 6단계 서술
