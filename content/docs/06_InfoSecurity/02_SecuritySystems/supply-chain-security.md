## 개요

공급망 보안(Supply Chain Security)은 소프트웨어·하드웨어·서비스의 개발·납품·운영 전 과정에서 악의적 삽입·변조·취약점 유입을 차단하는 보안 전략이다. SolarWinds·Log4Shell 사건 이후 소프트웨어 공급망 보안이 국가 안보 수준의 이슈로 부상했다.

## 핵심 내용

**공급망 보안 위협 유형**
- 소프트웨어 공급망 공격: 개발 도구·오픈소스·업데이트 배포 경로 오염
- 하드웨어 공급망 공격: 칩·부품 제조 단계에서 백도어 삽입
- 서비스 공급망 공격: 협력업체·MSP(관리형 서비스)를 경유한 침투
- 오픈소스 취약점: 의존성 패키지의 알려진·제로데이 취약점

**SBOM (Software Bill of Materials)**
- 소프트웨어를 구성하는 모든 컴포넌트·의존성·라이선스 목록
- 공급망 투명성 확보의 핵심 수단
- 미국 행정명령 14028(2021)에서 연방 조달 소프트웨어에 SBOM 의무화

**제로트러스트 기반 공급망 보안 아키텍처**

| 원칙 | 공급망 적용 |
|------|------------|
| 명시적 검증 (Verify Explicitly) | 모든 공급업체·컴포넌트·사용자 지속 인증 |
| 최소 권한 (Least Privilege) | 납품업체 접근 권한 최소화, JIT(Just-in-Time) 접근 |
| 침해 가정 (Assume Breach) | 내부자·협력사도 잠재적 위협으로 간주, 세그멘테이션 |

**보안 대응 체계**
1. 공급업체 보안 평가·인증 (보안 요건 계약화)
2. SBOM 생성·관리·취약점 추적
3. 코드 서명 및 무결성 검증
4. 빌드 파이프라인 보안 (CI/CD 보안, Secure Supply Chain)
5. 지속적 모니터링 및 이상 탐지

## 실무 적용 예시

미국 NIST SP 800-161은 공급망 리스크 관리 프레임워크를 제시하며, 국내 주요 기반시설 운영기관은 협력업체 보안 감사와 SW 납품 시 SBOM 제출을 요구하기 시작했다.
