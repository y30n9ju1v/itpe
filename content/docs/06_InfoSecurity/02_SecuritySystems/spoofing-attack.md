## 개요

스푸핑(Spoofing)은 공격자가 자신의 정체(IP·MAC·도메인 등)를 신뢰할 수 있는 대상으로 위장하여 시스템이나 사용자를 속이는 공격이다. 네트워크의 신뢰 관계를 악용하며, ARP·IP·DNS 스푸핑이 대표적이다.

## 핵심 내용

### 가. 스푸핑 공격 개념

- **정의**: 합법적인 사용자·시스템인 것처럼 위장하여 불법 접근·도청·조작을 수행하는 공격
- **공통 원리**: 네트워크 프로토콜의 인증 부재 또는 약한 인증 악용
- **목적**: 도청, 세션 하이재킹, 중간자 공격(MITM), 서비스 방해

### 나. ARP 스푸핑

**공격 방법**:
- ARP(Address Resolution Protocol)는 IP → MAC 주소 변환 시 인증 없이 응답 수락
- 공격자가 가짜 ARP Reply를 전송하여 피해자의 ARP 캐시 조작
- 피해자의 트래픽이 공격자를 통해 흐르는 MITM 구성

```
정상: 호스트A → [ARP 요청: 192.168.1.1은?] → 게이트웨이(MAC: AA)
공격: 공격자 → [ARP Reply: 192.168.1.1 = 내 MAC(BB)] → 호스트A 캐시 오염
결과: 호스트A의 트래픽 → 공격자 경유
```

**보안 대책**:
- **Dynamic ARP Inspection(DAI)**: 스위치에서 ARP 패킷 검증
- **정적 ARP 항목**: 중요 서버 MAC 고정
- **VPN/암호화**: 도청해도 내용 보호

### 다. IP 스푸핑

**공격 방법**:
- IP 패킷의 출발지 IP 주소를 신뢰 가능한 주소로 위조
- 서버의 IP 기반 접근 통제 우회
- DDoS 증폭 공격에서 반사 서버의 응답을 피해자에게 유도

**보안 대책**:
- **패킷 필터링**: ISP 레벨에서 출발지 IP 검증 (BCP38)
- **역방향 경로 필터링(RPF)**: 입력 인터페이스와 라우팅 테이블 일치 검증
- **암호화 인증**: IP 주소 대신 PKI 기반 인증

### 라. DNS 스푸핑

**공격 방법**:
- DNS 응답을 위조하여 정상 도메인 → 공격자 IP로 변환
- 캐시 포이즈닝(Cache Poisoning): DNS 서버 캐시에 가짜 레코드 삽입

```
정상: 브라우저 → DNS 요청(bank.com) → 정상 IP(1.2.3.4)
공격: 공격자가 빠른 가짜 응답 → bank.com = 공격자 IP(5.6.7.8)
결과: 피싱 사이트로 유도
```

**보안 대책**:
- **DNSSEC**: DNS 응답에 디지털 서명 추가
- **DoH(DNS over HTTPS)**: DNS 쿼리 암호화
- **DNS 캐시 TTL 최소화**: 오염 지속 시간 단축
- **임의 포트·거래 ID**: Kaminsky 공격 방어

### 스푸핑 유형 비교

| 구분 | ARP 스푸핑 | IP 스푸핑 | DNS 스푸핑 |
|------|----------|---------|----------|
| **조작 대상** | MAC 주소 | 출발지 IP | 도메인→IP 매핑 |
| **영향 범위** | 로컬 네트워크 | 인터넷 광범위 | 전체 인터넷 |
| **목적** | MITM, 도청 | DDoS, 인증 우회 | 피싱, 리다이렉션 |
| **핵심 대책** | DAI | BCP38/RPF | DNSSEC |
