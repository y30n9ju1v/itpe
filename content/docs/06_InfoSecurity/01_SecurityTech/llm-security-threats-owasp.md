## 개요

OWASP LLM Top 10은 대규모 언어 모델 기반 애플리케이션의 주요 보안 취약점 10가지를 정리한 가이드라인이다. LLM이 서버·클라우드·엣지 환경에 배포되면서 기존 소프트웨어 보안 위협과는 다른 AI 특유의 새로운 공격 표면이 등장했다.

## 핵심 내용

### OWASP LLM Top 10 (2023)

| 순위 | 위협 | 설명 |
|------|------|------|
| **LLM01** | 프롬프트 인젝션 | 악의적 입력으로 모델 동작 조작 |
| **LLM02** | 안전하지 않은 출력 처리 | LLM 출력을 검증 없이 실행 (XSS, RCE 등) |
| **LLM03** | 학습 데이터 오염 | 학습 데이터에 백도어 삽입 |
| **LLM04** | 모델 DoS | 자원 소진 공격 (TTFT/TPOT 급격히 증가) |
| **LLM05** | 공급망 취약점 | 사전 학습 모델·플러그인의 악성 소프트웨어 |
| **LLM06** | 민감 정보 노출 | 학습 데이터 기억으로 개인정보 유출 |
| **LLM07** | 안전하지 않은 플러그인 | 플러그인의 과도한 권한 및 검증 부재 |
| **LLM08** | 과도한 에이전트 권한 | LLM 에이전트의 불필요한 권한 부여 |
| **LLM09** | 과신 (Overreliance) | LLM 출력을 검증 없이 신뢰 |
| **LLM10** | 모델 도난 | 모델 가중치·지식 탈취 |

### LLM이 컴퓨터 시스템 자원에 미치는 영향

| 자원 | 영향 |
|------|------|
| **GPU 메모리** | 수십~수백 GB 점유 (모델 가중치 + KV 캐시) |
| **CPU** | 토큰화, 전/후처리 연산 부하 |
| **메모리 (RAM)** | KV 캐시 오프로딩, 컨텍스트 관리 |
| **네트워크 대역폭** | 스트리밍 응답, 분산 추론 통신 |
| **스토리지 I/O** | 모델 가중치 로딩, RAG 데이터 검색 |

### 프롬프트 인젝션 유형

- **직접 인젝션**: 사용자 입력으로 시스템 프롬프트 무력화
- **간접 인젝션**: 웹페이지·문서 등 외부 콘텐츠에 악성 지시 삽입 → LLM이 처리 시 실행

### 시스템 수준 대응 방안

1. **입력 검증 및 샌드박싱**: 프롬프트 필터링, 최대 토큰 제한
2. **최소 권한 원칙**: LLM 에이전트에 필요한 최소 API/도구 권한만 부여
3. **출력 검증**: LLM 응답을 코드 실행 전 검증 (화이트리스트)
4. **모델 분리**: 신뢰 수준별 LLM 분리 운영
5. **Rate Limiting**: API 요청 수 제한으로 Model DoS 방지
6. **모니터링**: 비정상적 토큰 사용량·응답 패턴 탐지

## 출제 포인트

- OWASP LLM Top 10 중 LLM01(프롬프트 인젝션), LLM04(Model DoS), LLM06(민감정보) 중점 설명
- LLM이 GPU·CPU·메모리·네트워크에 미치는 영향
- 직접·간접 프롬프트 인젝션 구분
- 대응 방안: 최소 권한 + 입력/출력 검증 + Rate Limiting
