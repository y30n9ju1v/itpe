## 개요

양자 컴퓨터의 등장은 RSA·ECC 등 현행 공개키 암호를 위협한다. 이에 대응하는 두 가지 접근이 QKD(양자 키 분배)와 PQC(양자 내성 암호)다. QKD는 양자 물리학으로 키를 안전하게 분배하고, PQC는 수학적 난제를 활용하여 양자 컴퓨터도 풀기 어려운 알고리즘을 사용한다.

## 핵심 내용

### 양자키분배 (QKD: Quantum Key Distribution)

- **원리**: 양자 역학의 하이젠베르크 불확정성 원리·광자 편광 활용
- **동작**: 송신자(Alice)와 수신자(Bob)가 광자 기반 키를 교환
- **도청 탐지**: 도청자(Eve)가 측정하면 양자 상태가 변화 → 도청 즉시 탐지 가능
- **대표 프로토콜**: BB84 (Bennett-Brassard 1984)

**BB84 동작 원리**:
```
Alice: 무작위 비트 + 무작위 편광 기저로 광자 전송
Bob: 무작위 기저로 측정
채널 공개: 기저 일치 비트만 키로 사용
도청 시: 오류율 증가 → 탐지
```

**QKD 특징**:
- 정보이론적 안전성 (무조건 안전)
- 전용 광학 인프라(광섬유) 필요
- 전송 거리 제한 (~수백 km), 중계기(Quantum Repeater) 필요

### 양자내성암호 (PQC: Post-Quantum Cryptography)

- **원리**: 양자 컴퓨터도 효율적으로 풀 수 없는 수학적 난제 기반
- **NIST 표준화**: 2024년 FIPS 203(ML-KEM), FIPS 204(ML-DSA), FIPS 205(SLH-DSA) 발표

**PQC 알고리즘 유형**:

| 유형 | 기반 난제 | NIST 표준 |
|------|---------|---------|
| **격자 기반(Lattice)** | LWE, NTRU 문제 | ML-KEM(Kyber), ML-DSA(Dilithium) |
| **해시 기반(Hash)** | 해시함수 충돌 저항성 | SLH-DSA(SPHINCS+) |
| **코드 기반(Code)** | 선형 코드 디코딩 난제 | (후보) |
| **다변수 기반** | 다변수 다항방정식 | (후보) |

### QKD vs PQC 비교

| 구분 | QKD | PQC |
|------|-----|-----|
| 안전성 근거 | 양자 물리학 법칙 | 수학적 난제 |
| 안전 수준 | 무조건적 (정보이론적) | 계산론적 (가정 기반) |
| 인프라 | 전용 양자 채널 필요 | 기존 네트워크 사용 가능 |
| 도입 비용 | 높음 | 낮음 (소프트웨어 교체) |
| 전송 거리 | 제한적 | 무제한 |
| 표준화 현황 | ITU-T Y.3800 시리즈 | NIST FIPS 203/204/205 |
| 적합 환경 | 금융·국방 핵심 구간 | 범용 인터넷 통신 |

## 출제 포인트

- QKD: BB84 프로토콜, 도청 탐지 원리, 정보이론적 안전성
- PQC: NIST 표준 3종 (ML-KEM, ML-DSA, SLH-DSA), 격자 기반
- 비교: 인프라 요건·안전성 근거·비용의 차이
