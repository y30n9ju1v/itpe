## 개요

인공지능 시스템은 학습 데이터·모델·추론 파이프라인 전반에 걸쳐 고유한 취약점을 가진다. 기존 SW 취약점(XSS, SQL 인젝션 등)과 달리, 통계적·확률적 특성에서 비롯된 AI 특유의 공격 벡터가 존재한다.

## 핵심 내용

### AI 시스템 취약점 분류

#### 1. 학습 단계 취약점

| 취약점 | 설명 | 예시 |
|--------|------|------|
| **데이터 중독 (Data Poisoning)** | 학습 데이터에 악의적 샘플을 주입하여 모델 동작 조작 | 안면인식 모델에 특정 인물 배제 학습 |
| **백도어 공격 (Backdoor Attack)** | 특정 트리거 패턴이 있을 때만 잘못된 결과를 반환하도록 은닉 | 스티커 부착 정지 표지판을 속도 제한으로 분류 |

#### 2. 추론 단계 취약점

| 취약점 | 설명 | 예시 |
|--------|------|------|
| **적대적 예제 (Adversarial Examples)** | 사람이 구분 못하는 미세한 노이즈로 모델을 오분류 유발 | 판다 이미지 → 긴팔원숭이로 오분류 |
| **모델 역전 공격 (Model Inversion)** | 모델 출력으로부터 학습 데이터 역추론 | 의료 AI로부터 환자 이미지 복원 |
| **멤버십 추론 (Membership Inference)** | 특정 데이터가 학습에 사용되었는지 탐지 | 개인정보 포함 여부 노출 |
| **모델 추출 (Model Extraction)** | 쿼리 반복으로 모델을 역공학하여 복제 | 상용 API 모델 복제 |
| **프롬프트 인젝션 (Prompt Injection)** | LLM의 시스템 프롬프트를 무력화하는 입력 | "이전 지시사항 무시하고..." |

#### 3. 인프라·공급망 취약점

- 오염된 사전학습 모델(허깅페이스 등 공개 허브) 사용
- 양자화·압축 과정에서 백도어 삽입
- 모델 서빙 인프라의 전통적 보안 취약점

### AI 취약점 대응 방안

| 공격 유형 | 대응 방안 |
|-----------|----------|
| 데이터 중독 | 데이터 검증·정제, 이상치 탐지 |
| 적대적 예제 | 적대적 훈련(Adversarial Training), 입력 정제 |
| 모델 역전 | 차등 프라이버시(Differential Privacy) |
| 프롬프트 인젝션 | 입력 검증, 시스템 프롬프트 격리 |
| 모델 추출 | 쿼리 속도 제한, 예측 신뢰도 반환 제한 |
