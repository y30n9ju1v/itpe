## 개요

국가 망 보안체계(N²SF: National Network Security Framework)는 국가정보원이 2025년 1월 발표한 공공기관 대상 망 보안 가이드라인이다. 기존 물리적 망 분리 정책을 보완하여 업무 효율성과 보안을 균형 있게 달성하는 것이 목적이다.

## 핵심 내용

### N²SF 개념

- **기존 망 분리**: 인터넷망과 업무망을 물리적으로 완전 분리 → 보안 강하나 업무 비효율
- **N²SF**: 업무 데이터의 중요도와 위협 수준에 따라 **차등화된 보안 수준** 적용
- **핵심 패러다임 전환**: "모든 것을 차단"에서 "위험 기반 차등 보안"으로

### N²SF 3가지 보안 구역

| 구역 | 대상 | 보안 수준 |
|------|------|---------|
| **고보안 구역** | 국가기밀, 개인정보 등 민감 업무 | 완전 망 분리 유지 |
| **일반 구역** | 일반 행정 업무 | 논리적 망 분리 + 보안 솔루션 |
| **인터넷 접근 구역** | 대민 서비스, 인터넷 필요 업무 | 제어된 인터넷 접근 허용 |

### N²SF 적용 절차

1. **업무 분류**: 취급 정보의 중요도·민감도 등급 분류
2. **위험 평가**: 각 업무의 위협·취약점·영향도 분석
3. **보안 구역 지정**: 업무별 적절한 보안 구역 배치
4. **보안 통제 적용**: 구역별 접근제어, 암호화, 모니터링 등 보안 수단 구현
5. **검증 및 승인**: 국가정보원 검토·승인
6. **운영 및 모니터링**: 지속적 보안 상태 점검

### N²SF 고려사항

| 고려사항 | 내용 |
|----------|------|
| **업무 연속성** | 보안 강화 시 업무 효율성 저하 최소화 |
| **제로 트러스트 연계** | 사용자·단말·접근 경로 모두 신뢰 검증 |
| **클라우드 적용** | 공공 클라우드 이용 시 N²SF 적용 방안 |
| **공급망 보안** | 제3자 소프트웨어·장비 보안 검증 |
| **보안 인시던트 대응** | 구역 침해 시 격리·복구 절차 |
| **법적 준거성** | 개인정보보호법, 정보통신망법 등 준수 |

### 기존 망 분리 정책과의 비교

| 구분 | 기존 물리적 망 분리 | N²SF |
|------|------------------|------|
| 방식 | 모든 업무망·인터넷망 분리 | 업무 중요도별 차등 적용 |
| 유연성 | 낮음 | 높음 |
| 비용 | 이중 인프라 비용 높음 | 합리적 비용 |
| 보안 수준 | 일률적 | 위험 기반 맞춤형 |
