## 개요

경계 기반 보안(Perimeter Security)은 내부·외부를 물리적 경계로 구분하는 전통 보안 모델이다. 제로 트러스트(Zero Trust)는 "절대 신뢰하지 말고, 항상 검증하라(Never Trust, Always Verify)"는 원칙으로, 내부 위협과 클라우드·원격근무 환경에 대응한다. CISA의 ZT 성숙도 모델 2.0은 조직의 ZTA 구현 수준을 5단계로 평가한다.

## 핵심 내용

### 경계 기반 보안 vs 제로 트러스트 비교

| 구분 | 경계 기반 보안 | 제로 트러스트 |
|------|-------------|------------|
| 신뢰 모델 | 내부 = 신뢰, 외부 = 불신 | 위치에 관계없이 항상 검증 |
| 인증 | 경계 진입 시 1회 | 매 세션·리소스 접근 시마다 |
| 동-서 트래픽 | 내부 자유 통신 허용 | 세분화(Micro-segmentation) |
| 클라우드 적합성 | 낮음 | 높음 |
| 위협 대응 | 내부 침투 후 탐지 어려움 | 내부 측면 이동 차단 |

### 제로 트러스트 5대 원칙 (NIST SP 800-207)

1. 모든 데이터·서비스를 리소스로 간주
2. 네트워크 위치와 무관하게 모든 통신 보호
3. 리소스 접근은 세션 단위로 허가
4. 접근 정책은 클라이언트 ID·애플리케이션·상태로 동적 결정
5. 모든 자산의 무결성·보안 상태 모니터링

### 제로 트러스트 성숙도 모델 2.0 (CISA, 2023)

**5개 핵심 기둥 × 4단계 성숙도**:

| 기둥 | 내용 |
|------|------|
| **ID(Identity)** | 사용자·기기·서비스의 강력한 인증 |
| **디바이스(Device)** | 기기 상태·컴플라이언스 지속 검증 |
| **네트워크(Network)** | 마이크로 세분화, 암호화 통신 |
| **애플리케이션·워크로드** | 앱별 접근 제어, CI/CD 보안 |
| **데이터(Data)** | 데이터 분류·암호화·DLP |

**4단계 성숙도**:
1. **전통(Traditional)**: 경계 기반, 수동 운영
2. **초기(Initial)**: ZT 일부 도입, 사용자 접근 제어 시작
3. **고급(Advanced)**: 전사 ZT 정책, 자동화 탐지
4. **최적(Optimal)**: 완전 자동화, AI 기반 동적 정책

### ZTA 도입 시 고려사항

| 항목 | 고려사항 |
|------|---------|
| **기술** | IAM 강화(MFA·RBAC), SASE 도입, 마이크로 세분화 구현 |
| **조직** | ZT 거버넌스 체계, 임원 스폰서십 확보 |
| **점진적 전환** | 기존 인프라와의 공존 전략, 단계별 로드맵 |
| **가시성** | SIEM·UEBA·XDR 연계, 지속적 모니터링 |
| **사용자 경험** | 과도한 인증 요구로 인한 생산성 저하 방지 |

## 출제 포인트

- "Never Trust, Always Verify": 제로 트러스트 핵심 원칙
- CISA 5개 기둥: ID·디바이스·네트워크·앱워크로드·데이터
- 4단계 성숙도: 전통→초기→고급→최적
