## 개요


AI 기술이 도입되면서 데이터 처리 방식이 대규모화·자동화되어 기존 개인정보보호 체계로 다루기 어려운 새로운 프라이버시 이슈가 발생하고 있다. AI 생명주기(데이터 수집 → 학습 → 추론 → 배포) 각 단계에서 서로 다른 위험이 나타난다.

## 핵심 내용

### AI 데이터 처리방식의 변화와 프라이버시 이슈

| 변화 | 프라이버시 이슈 |
|------|--------------|
| **대규모 데이터 수집** | 동의 없는 개인정보 수집, 목적 외 이용 |
| **자동화된 의사결정** | 설명 불가능한 AI 결정(대출·채용 거부), 차별 발생 |
| **모델의 기억** | 학습 데이터 기억(Memorization) → 추론으로 개인정보 노출 |
| **다중 데이터 결합** | 개별로 무해한 데이터도 결합 시 개인 식별 가능 (재식별) |
| **국경 없는 처리** | 글로벌 클라우드 AI 사용 시 국외 이전 문제 |
| **지속적 프로파일링** | IoT·행동 데이터로 장기 행동 패턴 분석 |

### AI 단계별 데이터 처리기준, 위험요건, 대응방안

#### 1단계: 데이터 수집

| 항목 | 내용 |
|------|------|
| **처리기준** | 수집 목적 명시, 최소 수집 원칙, 동의 기반 수집 |
| **위험요건** | 과도한 수집, 민감정보 미구분, 제3자 데이터 결합 |
| **대응방안** | 데이터 최소화(Data Minimization), 수집 목적 명확화, 민감정보 별도 동의 |

#### 2단계: 학습 (Training)

| 항목 | 내용 |
|------|------|
| **처리기준** | 가명처리 후 학습, 학습 데이터 출처 관리 |
| **위험요건** | 모델 기억(Memorization), 학습 데이터 편향, 역추론(Inversion Attack) |
| **대응방안** | 차분 프라이버시(DP) 적용, 연합학습(FL), 학습 데이터 검증 |

#### 3단계: 추론·서비스 (Inference)

| 항목 | 내용 |
|------|------|
| **처리기준** | 자동화 의사결정 고지·설명 제공, 이의신청 절차 |
| **위험요건** | 모델 역추론 공격, 멤버십 추론 공격, 차별적 결과 |
| **대응방안** | XAI(설명가능AI) 적용, 공정성(Fairness) 지표 모니터링, API 접근 제한 |

#### 4단계: 배포·폐기

| 항목 | 내용 |
|------|------|
| **처리기준** | 데이터 보유기간 준수, 삭제 요청 대응(잊힐 권리) |
| **위험요건** | 모델에 잔류한 개인정보 삭제 불가, 모델 가중치 유출 |
| **대응방안** | Machine Unlearning 기술 적용, 모델 접근 로그 관리 |

### Machine Unlearning

- 특정 개인의 데이터가 모델에 미친 영향을 제거하는 기술
- **잊힐 권리(Right to Erasure, GDPR Article 17)**와 연계
- 현재 기술적으로 완전 구현 어려움 → 근사적 제거 또는 재학습

## 출제 포인트

- AI 데이터 처리 변화 3가지: 대규모 수집, 자동화 결정, 모델 기억
- AI 생명주기 4단계별 위험요건과 대응방안 서술
- 차분 프라이버시, 연합학습 등 PET와의 연계
- Machine Unlearning: 잊힐 권리의 기술적 구현
