## 개요

안면인식 결제는 카메라로 촬영한 얼굴 이미지를 딥러닝 모델로 분석하여 신원을 인증하고 결제를 처리하는 서비스다. 중국의 Alipay, 국내 일부 편의점 등에서 도입되었으며, 편의성과 개인정보 침해 우려가 공존한다.

## 핵심 내용

### 기술 구조

1. **얼굴 감지(Detection)**: 영상에서 얼굴 영역 추출 (MTCNN 등)
2. **특징 추출(Feature Extraction)**: 딥러닝으로 얼굴 임베딩 벡터 생성 (FaceNet, ArcFace)
3. **매칭(Matching)**: 등록된 특징 벡터와 코사인 유사도 비교
4. **인증(Authentication)**: 임계값 이상 시 신원 확인 → 결제 승인

### 개인정보보호 쟁점

| 쟁점 | 내용 |
|------|------|
| 민감정보 처리 | 생체정보는 개인정보보호법상 민감정보, 별도 동의 필요 |
| 동의 없는 수집 | 불특정 다수 촬영 시 정보주체 동의 확보 곤란 |
| 오인식 위험 | FAR(False Accept Rate), FRR(False Reject Rate) 관리 필요 |
| 데이터 유출 | 해시 불가능한 생체정보 특성상 유출 시 회복 불가 |

### 개인정보보호법 적용

- 생체인식정보: 민감정보(제23조) — 명시적 동의 필수
- 영상정보처리기기: 제25조 규제 적용
- 해외 전송 시 별도 고지·동의 필요

## 출제 포인트

- 기술 구조(감지→추출→매칭→인증) 단계 서술
- 개인정보보호법상 민감정보로서의 규제 설명
- 오인식률(FAR/FRR) Trade-off 언급
- PET 기술(차분 프라이버시, 연합학습)과의 연계
