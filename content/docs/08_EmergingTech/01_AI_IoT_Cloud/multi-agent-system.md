## 개요

다중 에이전트 시스템(MAS)은 여러 AI 에이전트가 협력·경쟁·조율을 통해 복잡한 목표를 달성하는 분산 AI 아키텍처다. 각 에이전트는 자율적으로 인식(Perceive)·추론(Reason)·행동(Act)하며, 단일 에이전트가 처리하기 어려운 대규모·복합 업무를 분업·협업으로 해결한다.

## 핵심 내용

### 가. 정의 및 필요성

- **정의**: 자율적 행동 능력을 가진 복수의 에이전트가 공유 환경에서 상호작용하여 목표를 달성하는 시스템
- **필요성**:
  - 단일 LLM의 컨텍스트 한계·전문성 한계 극복
  - 병렬 처리로 복합 업무의 효율화
  - 역할 분리(기획·실행·검증)로 오류 감소
  - 기업 업무 자동화(AX) 실현

### 나. 단일 에이전트 vs 다중 에이전트 시스템 비교

| 구분 | 단일 에이전트 | 다중 에이전트 |
|------|------------|------------|
| 구조 | 하나의 LLM + 도구 | 복수 LLM/에이전트 협력 |
| 처리 방식 | 순차적 | 병렬·분산 가능 |
| 전문성 | 범용 | 역할별 전문 에이전트 |
| 복잡 업무 | 컨텍스트 한계 봉착 | 분할 처리 가능 |
| 오류 검증 | 자기 검증 어려움 | 에이전트 간 상호 검증 |
| 구현 복잡도 | 낮음 | 높음 |

### 에이전트 역할 분류 (예시)

- **오케스트레이터(Orchestrator)**: 전체 목표를 하위 태스크로 분해·할당·통합
- **전문 에이전트(Specialist Agent)**: 코딩, 검색, 분석 등 특화 역할 수행
- **검증 에이전트(Critic Agent)**: 다른 에이전트의 출력 품질 평가
- **실행 에이전트(Executor Agent)**: 도구(API, DB, 코드 실행) 직접 호출

### 다. 효과와 고려사항

**효과:**
- 업무 처리 속도 향상 (병렬 실행)
- 품질 향상 (다중 검증)
- 복잡한 워크플로우 자동화 (ERP, SCM 연계)

**고려사항:**
- **조율 오버헤드**: 에이전트 간 통신·컨텍스트 전달 비용
- **일관성**: 에이전트 간 정보 불일치 가능성
- **보안**: 프롬프트 인젝션 확산 위험
- **책임 소재**: 오류 발생 시 책임 에이전트 추적 어려움
- **비용**: 다수 LLM 호출로 API 비용 증가

## 실무 적용 예시

기업 AI 전환(AX) 프로젝트에서 '요구사항 분석 에이전트 → 코드 생성 에이전트 → 테스트 에이전트 → 문서화 에이전트'로 구성된 MAS로 소프트웨어 개발 파이프라인을 자동화한다.

## 출제 포인트

- MAS의 정의·필요성과 단일 에이전트 비교 테이블
- 에이전트 역할 분류(오케스트레이터·전문·검증)
- 고려사항: 조율 오버헤드, 일관성, 보안(프롬프트 인젝션)
