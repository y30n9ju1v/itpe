## 개요

차원의 저주(Curse of Dimensionality)는 데이터의 차원(특징 수)이 증가할수록 동일한 밀도의 데이터를 유지하는 데 필요한 데이터 샘플 수가 기하급수적으로 증가하며, 거리 기반 알고리즘의 성능이 급격히 저하되는 현상이다. Richard Bellman이 1957년 처음 제시했다.

## 핵심 내용

### 발생 원인

**공간 희소성(Sparsity)**: d차원 단위 초입방체에서 중심으로부터 거리 r 이내의 데이터 비율

```
비율 = r^d

- d=1, r=0.1 → 10% 포함
- d=10, r=0.1 → 0.1^10 = 10^-10 ≈ 0% (거의 없음)
```

→ 고차원에서는 모든 데이터 포인트가 서로 동등하게 먼 거리에 위치

### 주요 문제 현상

| 현상 | 설명 |
|------|------|
| **거리 무의미화** | 유클리드 거리의 최대/최소 비율이 1에 수렴 (KNN 성능 저하) |
| **데이터 희소성** | 동일 통계적 신뢰도 유지에 지수적 샘플 필요 |
| **과적합** | 특징 수 > 샘플 수이면 모델이 노이즈까지 학습 |
| **계산 복잡성** | 고차원 탐색·최적화 비용 지수적 증가 |

### 해결 방법

| 방법 | 기법 | 설명 |
|------|------|------|
| **차원 축소** | PCA | 분산이 큰 주성분만 선택 |
| | t-SNE, UMAP | 비선형 차원 축소, 시각화 |
| | Autoencoder | 딥러닝 기반 압축 표현 학습 |
| **특징 선택** | 필터 방법 | 상관계수·분산으로 중요 특징 선택 |
| | 래퍼 방법 | 모델 성능 기준 특징 조합 탐색 |
| | 임베디드 방법 | L1 정규화(Lasso)로 불필요 특징 제거 |
| **데이터 증강** | 오버샘플링 | SMOTE로 소수 클래스 합성 |
| **정규화** | L1/L2 | 과적합 방지 |

## 출제 포인트

- 차원 증가 시 공간이 희소해지는 수학적 직관 설명
- AI 성능 저하 맥락: KNN, SVM 등 거리 기반 알고리즘에서 심각
- 해결책: PCA(선형), t-SNE(비선형), 특징 선택
- Hughes Phenomenon: 샘플 고정 시 특징 수 증가 → 성능 선증가 후 감소
