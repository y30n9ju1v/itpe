## 개요

ISO/PAS 8800:2024는 도로 차량에 탑재된 AI 기반 시스템의 안전을 확보하기 위한 국제 표준(공개 표준 초안)이다. 기존 기능 안전 표준 ISO 26262(하드웨어/소프트웨어 결함)와 SOTIF(ISO 21448, 의도된 기능의 안전성)를 AI의 불확실성·예측 불가능성에 맞게 확장한다.

## 핵심 내용

### 등장 배경

| 기존 표준 | 한계 |
|-----------|------|
| ISO 26262 | 하드웨어/소프트웨어 결함 대응, AI 불확실성 미포함 |
| ISO 21448 (SOTIF) | 비결함 기반 안전 (성능 한계), AI 데이터 편향 미다룸 |
| **ISO/PAS 8800** | AI 편향, 데이터 품질, 학습 결함 등 AI 특유 위험 추가 |

### 생명주기 관점의 주요 요구사항

| 단계 | 요구사항 |
|------|----------|
| **설계** | AI 안전 목표(ASIL 수준) 정의, 데이터 편향 분석 |
| **데이터 수집** | 학습 데이터 품질 기준: 다양성·대표성·정확성 |
| **학습/개발** | 모델 설명가능성(XAI) 확보, 과적합 방지 |
| **검증/시험** | 시뮬레이션 + 실도로 테스트, 코너 케이스 검증 |
| **배포** | 운영 설계 도메인(ODD) 정의, 안전 모니터링 |
| **유지보수** | 모델 재학습 시 안전 재평가, 인시던트 보고 |

### ISO 26262 및 ISO 21448과의 연계 관계

```
ISO 26262 (기능 안전)
  → 하드웨어/소프트웨어 결함(Fault) 방지
  → ASIL(A~D) 안전 무결성 수준

ISO 21448/SOTIF (의도된 기능의 안전성)
  → 결함이 없어도 성능 한계로 발생하는 위험
  → 엣지 케이스, 센서 한계

ISO/PAS 8800 (AI 안전)
  → 위 두 표준을 포함하며 AI 특유 위험 추가:
     · 학습 데이터 편향
     · 배포 후 모델 드리프트
     · 블랙박스 AI의 설명불가능성
     · 적대적 공격(Adversarial Attack)
```

### 자율주행 적용 사례

- **자동 긴급 제동(AEB)**: AI 비전 모델이 보행자 인식 → 8800 기준으로 학습 데이터 다양성·야간·악천후 조건 검증 필요
- **차선 유지 보조(LKAS)**: 불명확한 차선 마킹에서의 ODD 정의 및 fallback 로직 요구
- **L3 조건부 자율주행**: 운전자 핸드오프 전환 시 AI 신뢰도 임계값 및 최소 위험 상태(MRC) 정의

## 출제 포인트

- ISO 26262(결함), SOTIF(성능 한계), ISO/PAS 8800(AI 불확실성) 3단계 확장 관계
- 생명주기 6단계별 요구사항 서술
- AI 특유 위험 3가지: 데이터 편향, 모델 드리프트, 설명불가능성
- ODD(운영 설계 도메인)의 역할
