## 개요

스마트시티 데이터 거버넌스는 도시에서 수집·처리·공유되는 방대한 데이터(교통·에너지·환경·시민 데이터)의 품질·보안·개인정보·공유를 체계적으로 관리하는 정책·조직·기술 프레임워크다. 데이터를 도시의 핵심 공공 자산으로 관리한다.

## 핵심 내용

### 스마트시티 데이터 거버넌스 구성요소

| 구성요소 | 내용 |
|----------|------|
| **데이터 정책** | 데이터 수집·사용·공유·폐기 원칙 및 규정 |
| **데이터 조직** | 데이터 책임관(CDO), 데이터 관리자 역할 정의 |
| **데이터 품질** | 정확성·완전성·최신성 관리 프로세스 |
| **데이터 보안** | 접근통제, 암호화, 개인정보 비식별화 |
| **데이터 공유** | 공공·민간·시민 간 데이터 공유 플랫폼 |
| **데이터 윤리** | AI 알고리즘 편향, 감시 사회 방지 |

### 핵심 거버넌스 원칙 (FAIR 원칙)

| 원칙 | 내용 |
|------|------|
| **F**indable | 메타데이터로 데이터를 쉽게 발견 가능 |
| **A**ccessible | 표준 프로토콜로 접근 가능 |
| **I**nteroperable | 다른 시스템·도시 간 상호운용 가능 |
| **R**eusable | 명확한 라이선스로 재사용 가능 |

### 데이터 공유 모델

| 모델 | 설명 | 사례 |
|------|------|------|
| **오픈 데이터** | 공공 데이터 무조건 공개 | 교통·날씨·공기질 |
| **선택적 공유** | 계약·동의 기반 민간 데이터 공유 | 통신사 유동인구 |
| **데이터 신탁** | 제3 기관이 시민 대신 데이터 관리 | 개인 건강 데이터 |
| **데이터 공유 협약** | 공공-민간 파트너십 협약 | 교통카드사 ↔ 지자체 |

### 개인정보보호 요건

- **비식별화**: 이동 경로, 카메라 영상 등 개인 특정 가능 데이터 처리
- **동의 관리**: 스마트 서비스 이용 시 데이터 수집 동의
- **데이터 최소화**: 서비스 목적에 필요한 최소 데이터만 수집
- **CCTV·IoT 규제**: 공공장소 감시 데이터 활용 범위 제한
