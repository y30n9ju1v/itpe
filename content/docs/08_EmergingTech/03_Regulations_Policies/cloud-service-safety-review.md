## 개요

「행정기관 및 공공기관의 클라우드 컴퓨팅 서비스 이용 기준 및 안전성 확보 등에 관한 고시」(과기정통부)는 공공기관이 클라우드 서비스 도입 시 준수해야 할 보안 기준을 규정한다. 서비스 운영 분야는 클라우드 서비스 운영 전반의 안전성을 평가하는 항목이다.

## 핵심 내용

### 클라우드 보안인증(CSAP) 개요

- **목적**: 공공기관 클라우드 서비스 이용의 보안성 확보
- **근거**: 클라우드컴퓨팅법 제23조, 과기정통부 고시
- **인증 등급**:
  - **상**: 중요 정보 처리 (국가안보, 기밀)
  - **중**: 일반 공공 업무
  - **하**: 민감도 낮은 공공 서비스

### 서비스 운영 분야 안전성 검토 항목

| 대분류 | 세부 항목 |
|-------|---------|
| **가용성 관리** | 서비스 연속성 계획, BCP/DRP 수립·훈련 |
| **용량 관리** | 자원 모니터링, 용량 계획, 임계값 경보 |
| **변경 관리** | 변경 승인 절차, 긴급 변경 프로세스, 테스트 후 배포 |
| **인시던트 관리** | 장애 탐지·보고·대응·사후 분석 절차 |
| **서비스 수준 관리** | SLA 정의·측정·보고, SLA 위반 시 대응 |
| **백업·복구** | 데이터 정기 백업, 복구 절차 검증, RTO/RPO 설정 |
| **로그 관리** | 접근 로그 수집·보관(최소 6개월), 무결성 보장 |
| **패치 관리** | 취약점 패치 정책, 긴급 패치 절차, 패치 이력 관리 |

### 안전성 검토 절차

```
클라우드 서비스 선정
    ↓
안전성 검토 계획 수립 (담당 부서)
    ↓
CSAP 인증 여부 확인 (필수)
    ↓
서비스 운영 항목별 심사
    ↓
결과 보고 → 기관장 승인
    ↓
주기적 사후 점검 (연 1회 이상)
```

### 공공 클라우드 이용 원칙

1. **CSAP 인증 서비스 우선**: 미인증 서비스 사용 시 별도 승인
2. **정보 등급 분류**: 민감도에 따른 클라우드 등급 적용
3. **보안 책임 공유**: CSP(클라우드 사업자)와 기관의 역할 명확화
4. **감사 추적**: 모든 접근·변경 이력 보관

## 출제 포인트

- 고시의 목적: 공공기관 클라우드 보안 기준 규정
- CSAP 3등급(상·중·하) 구분
- 서비스 운영 분야 8대 항목: 가용성·용량·변경·인시던트·SLA·백업·로그·패치
