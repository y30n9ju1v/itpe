## 개요

공공 마이데이터는 행정·공공기관이 보유한 개인정보를 정보주체(개인)의 요구에 따라 본인 또는 제3자에게 제공하는 제도다. 개인정보보호법·전자정부법 개정으로 데이터 이동권이 명시되었으며, 금융 마이데이터에 이어 공공 분야로 확장 중이다.

## 핵심 내용

### 공공 마이데이터 개념

- **마이데이터(MyData)**: 정보 주체가 자신의 데이터 열람·전송·활용을 직접 통제하는 권리
- **데이터 이동권**: 기관이 보유한 나의 데이터를 다른 기관·서비스에 전송 요구 가능
- **공공 마이데이터 플랫폼**: 행정안전부 '정부24' 마이데이터 서비스

### 주요 활용 방안

| 분야 | 활용 내용 |
|------|----------|
| **금융** | 소득·납세 증명 자동 전송으로 대출 심사 자동화 |
| **복지** | 생애주기별 복지 자격 자동 확인·신청 (기초생활, 장애인 지원) |
| **의료** | 예방접종·처방 이력 공유로 중복 검사 방지 |
| **교육** | 성적·자격증 정보를 취업 플랫폼에 직접 전송 |
| **부동산** | 등기·세금 데이터 기반 부동산 서비스 |

### 구현 기술 체계

- **표준 API**: 행정기관 마이데이터 제공 표준 (OAuth 2.0 기반 인증)
- **마이데이터 지갑**: 개인이 데이터를 수집·관리하는 앱(정부24, 네이버, 카카오)
- **동의·철회 관리**: 세분화된 데이터 항목별 동의·철회
- **데이터 유통 이력**: 언제 어디에 제공했는지 로그 보관

### 추진 과제 및 한계

- 기관 간 데이터 표준화 부족
- 고령층·디지털 취약계층 접근성
- 개인정보 재식별 위험 (결합 시 특정 가능)
- 동의 피로(Consent Fatigue) — 복잡한 동의 절차

## 출제 포인트

- 공공 마이데이터의 데이터 이동권 기반 설명
- 5개 분야(금융·복지·의료·교육·부동산) 활용 방안
- 구현의 기술적 기반(OAuth, 표준 API)과 한계
