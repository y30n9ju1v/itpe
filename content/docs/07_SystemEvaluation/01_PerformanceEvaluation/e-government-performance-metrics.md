## 개요

전자정부 정보시스템 성과 측정지표는 공공기관이 정보화 사업의 효과를 정량·정성적으로 평가하기 위한 지표 체계다. 「전자정부법」 및 「정보화사업 성과관리 지침」에 근거하며, 투자 대비 효과(ROI)를 측정하고 사업의 지속·개선·종료를 결정하는 데 활용된다.

## 핵심 내용

### 성과 측정 체계

| 구분 | 설명 |
|------|------|
| **법적 근거** | 전자정부법 제64조, 정보화사업 성과관리 지침 |
| **측정 시점** | 사업 전(계획)·중(진행)·후(완료) 3단계 |
| **평가 방법** | 정량지표(수치) + 정성지표(설문) 병행 |

### 정보시스템 운영성과평가 모델 (행정안전부 고시 핵심)

공공 정보화 사업의 성과관리는 일반 BSC 외에 **행정안전부의 「정보시스템 운영성과평가 지침」**을 따르는 것이 기술사 시험 고득점의 절대적 요건이다.

#### 1. 운영성과평가 2대 차원 (평가 영역)

| 평가 차원 | 핵심 내용 | 세부 지표 예시 |
|-----------|-----------|----------------|
| **업무 적합성** (Business Suitability) | 시스템이 현행 비즈니스 요건을 얼마나 잘 지원하며 사용자에게 유용한지 검증 | 업무 지원도, 시스템 이용률, 사용자 만족도 |
| **운영 효율성** (Operational Efficiency) | 시스템 운영·유지보수가 비용 및 기술적 관점에서 효율적으로 이루어지는지 검증 | 유지관리비 효율성, 시스템 품질(신뢰성, 가용성), 자원 효율성 |

#### 2. 성과 측정 포트폴리오 분석 (4분면 매트릭스 모형)

측정 결과(업무 적합성 vs 유지관리 비용)를 바탕으로 4분면 분석을 실시하여 정보시스템의 차기 진로(유지, 폐기 등)를 결정한다.

```
                  [업무 적합성 (High)]
          ▲
          │   (2사분면) 기능재구축 / 효율화  │     (1사분면) 지속 유지
          │   - 업무적합도 높으나 비용 높음   │   - 업무적합도 높고 비용 낮음
          │   - 기능 보완, 가비지 자원 회수  │   - 현행 상태 보존 및 고도화
          │                                  │
[유지관리 ├──────────────────────────────────┼──────────────────────────────────▶
비용 High]│                                  │                   [유지관리 비용 Low]
          │   (3사분면) 폐기 / 재개발        │     (4사분면) 기능 고도화 / 통폐합
          │   - 업무적합도 낮고 비용 높음     │   - 비용 낮으나 업무 지원 미흡
          │   - 폐기, 아웃소싱, 재설계 전환   │   - 타 시스템과 통폐합, 기능 고도화
          │                                  │
          ▼
                  [업무 적합성 (Low)]
```

### 성과 측정지표 분류 (BSC 관점)

| BSC 관점 | 지표 예시 |
|---------|---------|
| **재무 관점** | IT 투자 대비 절감액, 비용 절감률, ROI |
| **고객 관점** | 민원 처리 시간 단축률, 서비스 만족도, 온라인 이용률 |
| **내부 프로세스** | 업무 처리 속도 향상, 오류율 감소, 자동화율 |
| **학습·성장** | 직원 IT 역량 향상, 시스템 안정성, 장애 복구 시간 |

### 주요 성과 지표 유형 (전통적 분류)

| 유형 | 설명 | 예시 |
|------|------|------|
| **효율성 지표** | 동일 자원으로 더 많은 성과 | 민원 처리 건수/인력 |
| **효과성 지표** | 목표 달성 정도 | 온라인 민원 전환율 |
| **접근성 지표** | 서비스 이용 편의성 | 모바일 접근 가능 서비스 비율 |
| **신뢰성 지표** | 시스템 안정성 | 가용성(SLA), MTBF |
| **보안 지표** | 정보보호 수준 | 보안 사고 발생 건수, 취약점 조치율 |

### 성과 측정 프로세스

```
사업 계획 수립 → 성과 지표(KPI) 정의
    ↓
기준값(Baseline) 설정
    ↓
사업 수행 중 중간 측정
    ↓
사업 완료 후 최종 측정
    ↓
목표 대비 달성도 분석
    ↓
차기 사업 계획 반영 (피드백)
```

## 출제 포인트

- 행정안전부 「정보시스템 운영성과평가 지침」의 2대 차원: 업무 적합성, 운영 효율성
- 성과 측정 4분면 포트폴리오 분석 모형과 분면별 차기 의사결정 방향 (유지/폐기/기능재구축/통폐합)
- BSC 4관점으로 전자정부 성과지표 분류 및 측정 시점 3단계 (사전·중간·사후)
