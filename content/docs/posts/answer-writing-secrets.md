# 기술사 답안 비기(祕機)

> 남이 엿볼 수 없는 아주 비밀스러운 기회(답안 스킬)  
> — KPC 중급반 공쌤·이쌤

---

## 1. 원칙 (Do & Don't)

### ✅ Do: 일관성, 일관성, 일관성

- **글씨 크기·굵기** 를 처음부터 끝까지 동일하게 유지한다.  
- **공간 배분(여백·들여쓰기)** 을 일정하게 맞춘다.  
- **흐름(1단락→2단락→3단락)** 의 패턴을 반복한다.  
- 제목줄 표기 방식도 통일 (`I.` 또는 `1.` 중 하나로 고정).

### ❌ Don't

| 금지 항목 | 이유 |
|---|---|
| 수정액(화이트) 사용 | 가독성 저하, 중급반 기준 **−1점** |
| 취소선 | 마찬가지로 감점 요인 |
| 해설지 보고 베끼기 | 실력 향상 없음; **미리 요약한 키워드로 글짓기** 연습이 핵심 |

---

## 2. 1교시형 (단답형 13문항)

1교시는 **1단락 → 2단락 → 3단락** 3개 섹션으로 구성한다.

### 1단락 — 리드문 & 정의

- 첫 줄은 **리드문(제목 요약)** 으로 시작 — 채점관에게 친절한 한 스푼.
- **줄글 정의는 2줄**, 표 안에 쓰는 정의는 4줄이 기준.
  - 정의만 쓰면 2줄, 개념도·특징·등장배경까지 추가하면 4줄 표로 확장.
- **키워드 밑줄은 1~2개** — 너무 많으면 오히려 가독성 저하.

#### 1단락 템플릿 유형

| 유형 | 특징 |
|---|---|
| 기본형 | 제목 + "개념" 행에 2줄 줄글 정의 |
| 표형①  | 정의 열 + 예시 열 2단표 |
| 표형②  | 그림(개념도) + 정의 2단표 |
| 표형③  | 개념 행 + 특징 세부 행으로 분리 |

### 2단락 — 구성도/표

- **구성도(개념도)** 가 핵심. 절차·스택·비교 등 다양한 형태 가능.
  - 절차형: 번호(①②③) 추가로 순서 부여
  - 스택형: 계층 구조 표현
  - 구간 분리: 좌우 영역 나눈 뒤 그룹핑 활용
- 도형 종류는 **2개 이상** 혼용하면 답안이 풍성해짐.
- **도형 옆 라벨** 배치 → 키워드 표현 + 밸런스 확보.
- 배치가 비면 글씨 또는 선으로 채워 균형을 맞춘다.

#### 표 공간 가이드

| 구성 | 권장 |
|---|---|
| 행당 내용량 | 2줄씩 3~4개 항목 (3단표), 내용이 많으면 3줄씩 2개도 OK |
| 표 안 개념도 | 축소 추상화로 공간 절약 |
| 구성도 없이 표만 | 1개 큰 표(5개 항목)도 가능 |

- 3단표 키워드: 너무 긴 설명보다 **관련 키워드 1개가 더 득점**. 밑줄은 2~3줄 정도.

### 3단락 — 활용/비교/발전흐름

| 상황 | 3단락 유형 |
|---|---|
| 네거티브 토픽(문제점·위협) | **해결방안** 무조건 추가 |
| 두 개념 비교 요청 | **퓨전도** 또는 **장단점 비교표** |
| 비슷한 개념 1개만 물어봄 | 유사 토픽 가져와 비교 가능 |
| 버전/릴리즈가 있는 기술 | **발전흐름도** (화살표로 연결) |

### 문제 수에 따른 템플릿 (5·4·3 법칙)

3단락을 다 쓰고 나면 다음 문제 공간 설정이 막히는 경우 — **5·4·3 법칙** 으로 미리 틀을 잡는다.

- 3문제 물어볼 때: 각 문제를 `1.` `2.` `3.` 으로 분리, 각각 1단락+2단락 구조
- 2개 비교 문제: 1단락에 정의 비교표, 2단락에 상세 비교표
- 2개 비교 아닌 경우(Case 1): 2단락을 `1)` `2)` 소단락으로 분리
- 2개 비교 아닌 경우(Case 2): 각 개념을 독립된 대단락으로 구성

## 3. 2교시형 (서술형 2문항)

2교시는 **1단락 → 2단락 → 3단락 → 간글 → 4단락** 흐름.

### 1단락 — 부각 배경 & 답안 요약

- **부각 배경**: 등장 배경·발전 연혁이 있으면 1단락에서 그래프나 흐름도로 표현.  
  → 숫자는 지어내도 OK, 그래프 형태만 맞으면 신뢰도 상승.
- **답안 요약**: 이후 2·3단락에서 다룰 내용을 미리 마인드맵/표로 요약.

### 2·3단락 — 1도 1표 (고득점 비결)

- **1개 그림 + 1개 표** 조합이 기본.
- 그룹핑으로 2개를 연결하면 고득점 구조.
- 1개만 물어볼 때: 2단락=개념도, 3단락=절차 또는 요소 상세.

### 간글 (단락 사이 연결문)

- 앞 단락을 한 줄로 요약 + 뒤 단락의 필요성을 제시.
- 단락 간 흐름을 자연스럽게 이어주는 역할.

### 4단락 — 토픽 융합 & 거버넌스

- 앞 내용을 기반으로 **확장·고려사항·시사점** 제시.
- **토픽 융합**: 두 기술의 단점을 서로 보완하는 융합 방안.
- **거버넌스 패턴**: 데이터·AI·클라우드·보안·경영 모두 동일한 **집(House) 모양** 구조로 표현 가능.

  ```
          ╱  관리 Vision  ╲
         ╱  품질 향상 / 관리 ╲
        ├──────────────────────┤  ← 거버넌스 요소
        │ Data조직 │ MDM │ Process │
        ├──────────────────────┤  ← 기반 요소
        │    표준 / 원칙 / 정책    │
  ```

  - **지붕**: 관리 Vision (목표)
  - **중간층**: 거버넌스 요소 (조직·프로세스·도구)
  - **기반**: 표준·원칙·정책

  2교시 4단락에서 "이 기술을 조직에서 잘 운영하려면?" 류의 질문이 나올 때 꺼내 쓰는 만능 템플릿. 데이터·AI·클라우드·보안 등 도메인만 바꿔 재활용 가능.

---

## 핵심 요약

```
1교시형:  1단락(정의) → 2단락(구성도+표) → 3단락(활용/비교/발전)
2교시형:  1단락(배경/요약) → 2단락(개념도) → 3단락(상세표) → 간글 → 4단락(융합/거버넌스)
```

**한 줄 원칙**: 일관성 있는 글씨, 일관성 있는 공간, 일관성 있는 흐름.
