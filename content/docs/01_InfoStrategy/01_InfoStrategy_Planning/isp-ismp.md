## 개요

ISP(Information Strategy Planning)는 조직의 경영전략과 연계하여 정보화 방향과 추진과제를 도출하는 중장기 전략 계획이다. ISMP(Information System Master Plan)는 ISP를 기반으로 구체적인 정보시스템 구축·개선을 위한 실행 계획을 수립하는 후속 단계다.

## 핵심 내용

### ISP 정의 및 목적

- **정의**: 조직의 비전·전략과 연계하여 정보시스템의 목표 모델, 이행 로드맵, 우선순위를 결정하는 전략 계획 활동
- **목적**: 정보화 투자 효율화, 중복 투자 방지, 조직 전략과 IT의 정렬(Strategic Alignment)

### ISP 수행 방법론 체계 및 절차

| 단계 | 활동 | 산출물 |
|------|------|--------|
| 1. 환경 분석 | 내외부 환경, 현행 시스템 분석 (AS-IS) | 현황 분석서 |
| 2. 방향 설정 | 비전·목표·원칙 수립 | IT 비전 체계도 |
| 3. 목표 모델 설계 | TO-BE 비즈니스·정보·기술 아키텍처 | 목표 아키텍처 |
| 4. 이행 계획 수립 | 과제 우선순위, 투자 계획, 로드맵 | 이행 로드맵 |

분석 기법: SWOT, 7S, 5-Force, 벤치마킹, JAD(Joint Application Design)

### ISP vs ISMP 비교

| 구분 | ISP | ISMP |
|------|-----|------|
| 목적 | 정보화 방향·전략 수립 | 시스템 구축 실행 계획 |
| 산출물 | 이행 로드맵, 아키텍처 방향 | 기능 요건, RFP 초안 |
| 기간 | 3~6개월 | 3~4개월 |
| 깊이 | 전략적·개념적 | 구체적·실행적 |
| 근거 법령 | 전자정부법 | 전자정부법, 정보화사업 지침 |
| 관계 | ISP → ISMP → 사업 발주 순서 |

### EA(Enterprise Architecture)와의 관계

EA는 조직 전체의 업무·정보·기술·응용 아키텍처의 청사진. ISP는 EA 수립의 선행 또는 병행 활동으로 수행.
