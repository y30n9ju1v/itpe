## 개요

NIA(한국지능정보사회진흥원)의 클라우드 정보화사업 감리 점검가이드는 공공 정보화사업에서 클라우드 서비스를 활용할 때 적용하는 감리 기준이다. 클라우드 도입 유형별로 사업 특성에 맞는 단계별 감리 활동을 정의한다.

## 핵심 내용

### 클라우드 정보화사업 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **신규 개발** | 클라우드 환경에서 새로운 정보시스템 구축 | AWS 위에 공공 포털 신규 개발 |
| **기존 시스템 이관** | 온프레미스 시스템을 클라우드로 마이그레이션 | 레거시 전산시스템 → GovCloud 이관 |
| **클라우드 서비스 활용** | 기존 SaaS·PaaS·IaaS 서비스를 구독·적용 | M365, 줌 같은 SaaS 도입 사업 |
| **하이브리드** | 온프레미스와 클라우드 혼합 운영 | 민감 데이터는 온프레미스, 나머지는 퍼블릭 |

### 클라우드 서비스 활용 사업의 단계별 감리 활동

#### 착수 단계 감리

| 점검 항목 | 내용 |
|----------|------|
| **클라우드 서비스 적정성** | 선택한 CSP·서비스 유형이 사업 요구에 적합한지 |
| **보안 인증 확인** | CSP의 CSAP(클라우드 보안인증) 등급 적합 여부 |
| **계약 검토** | SLA(서비스 수준 협약) 내용·책임 범위 명확성 |
| **공급망 위험** | 종속성(Vendor Lock-in) 위험 및 탈출 전략 |

#### 중간 단계 감리

| 점검 항목 | 내용 |
|----------|------|
| **데이터 주권·위치** | 국내 데이터 저장 요건 준수 여부 |
| **접근 통제** | IAM(Identity and Access Management) 설정 적정성 |
| **보안 설정** | 네트워크 보안 그룹, 암호화 설정 검토 |
| **비용 관리** | 예산 초과 위험, 리소스 최적화 여부 |

#### 종료 단계 감리

| 점검 항목 | 내용 |
|----------|------|
| **성능 검증** | SLO 달성 여부 (가용성, 응답시간) |
| **백업·복구** | 백업 정책 및 DR(재해복구) 계획 수립 |
| **운영 이관** | 운영 매뉴얼, 모니터링 체계 인계 |
| **개인정보** | 클라우드 내 개인정보 처리 적정성 |

### 클라우드 감리의 특수 고려사항

1. **CSAP(클라우드 보안인증)**: 공공기관은 CSAP 인증된 CSP 사용 의무 (표준·고급 등급)
2. **공동 책임 모델**: CSP와 기관 간 보안 책임 경계 명확화
   - IaaS: 인프라는 CSP, 그 위 모든 것은 기관
   - SaaS: 데이터·접근관리는 기관, 나머지는 CSP
3. **전자정부법**: 공공 클라우드는 전자정부법 기반 행안부 클라우드 이용 지침 준수
