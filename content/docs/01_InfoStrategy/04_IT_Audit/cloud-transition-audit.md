## 개요

공공기관의 클라우드 전환 사업은 기존 On-Premise 시스템을 퍼블릭·프라이빗·하이브리드 클라우드로 이전하는 대형 정보화 사업이다. 각 단계별 감리를 통해 이전 완결성, 보안, 성능, 서비스 연속성을 검증한다.

## 핵심 내용

### 클라우드 전환 단계별 감리 방법 및 검토 항목

**1단계: 계획·설계 단계 감리**

| 검토 항목 | 세부 내용 |
|---------|---------|
| **이전 전략 적정성** | 6R 전략(Rehost·Replatform·Repurchase·Refactor·Retire·Retain) 선택 근거 |
| **클라우드 서비스 선정** | CSAP 인증 여부, SLA 수준, 벤더 종속(Lock-in) 위험 |
| **보안 아키텍처** | 책임 공유 모델 기반 보안 통제 설계 |
| **네트워크 설계** | 전용선·VPN·전용 인터넷 망 연결 방식 적정성 |
| **데이터 분류** | 민감도별 데이터의 클라우드 저장 가능 여부 |

**2단계: 이전·구현 단계 감리**

| 검토 항목 | 세부 내용 |
|---------|---------|
| **데이터 이전 완결성** | 원본 vs 이전 데이터 비교 검증 (Hash 비교, 레코드 수 검증) |
| **기능 완전성** | 이전 전후 기능 동작 동등성 테스트 |
| **서비스 연속성** | 이전 중 서비스 중단 최소화 계획 (블루/그린, 점진적 이전) |
| **형상 관리** | 클라우드 인프라를 코드로 관리 (IaC: Terraform, CloudFormation) |
| **보안 설정** | IAM 정책·보안 그룹·암호화 설정 검토 |

**3단계: 운영 안정화 단계 감리**

| 검토 항목 | 세부 내용 |
|---------|---------|
| **성능 기준 충족** | 응답시간·처리량이 이전 수준 이상 달성 여부 |
| **SLA 준수** | 가용성 99.9% 이상, RTO/RPO 목표치 달성 |
| **비용 최적화** | 클라우드 비용 모니터링, 비효율 자원 식별 |
| **백업·복구** | 자동 백업, 복구 테스트 주기적 수행 여부 |
| **모니터링 체계** | CloudWatch, Azure Monitor 등 통합 모니터링 구축 |

### 감리 중점 위험 영역

**보안 위험**
- 클라우드 설정 오류(Misconfiguration): S3 퍼블릭 노출, 보안 그룹 과도 개방
- 권한 과다 부여: 최소 권한 원칙 위반
- 암호화 키 관리: KMS 설정 적정성

**성능·가용성 위험**
- 클라우드 지역(Region) 선택 부적절 → 지연 증가
- 단일 가용 영역(AZ) 구성 → 가용성 저하

**비용 위험**
- 사이징 과대 → 비용 낭비
- 데이터 전송 비용(Egress) 미고려

## 출제 포인트

- 3단계(계획설계·이전구현·운영안정화) 감리 구조
- 각 단계별 핵심 검토 항목 (특히 데이터 이전 완결성, SLA)
- 6R 이전 전략 개념 (Rehost·Replatform·Refactor 등)
- CSAP 인증의 공공 클라우드 전환 기준으로서의 역할
