## 개요

WBS는 프로젝트 목표를 달성하기 위해 필요한 작업 전체를 계층적으로 분해한 구조물이다. 프로젝트 범위를 정의하고 관리 가능한 단위로 나눠 일정·비용·자원 계획의 기초를 제공한다.

## 핵심 내용

### 작성 원칙
| 원칙 | 설명 |
|------|------|
| 100% 규칙 | 상위 요소의 작업 100%가 하위 요소에 포함되어야 함 |
| 상호배타성 | 각 작업 패키지는 중복 없이 독립적으로 정의 |
| 결과물 지향 | 행위(동사)가 아닌 산출물(명사) 중심으로 분해 |
| 분해 수준 | 80시간 규칙: 단말 작업(Work Package)은 80시간 이내 |

### 작성 절차
1. **범위 기술서(Scope Statement) 검토** — 주요 인도물·제약사항 파악
2. **1수준 식별** — 프로젝트 전체 단계 또는 주요 인도물 도출
3. **하향식 분해** — 관리 가능한 수준까지 반복 분해
4. **Work Package 정의** — 최하위 단위에 WBS 코드(ID) 부여
5. **WBS 사전 작성** — 각 요소의 범위·책임자·인수기준 문서화

### WBS 구성 요소
- **WBS 사전(Dictionary)**: 각 Work Package의 상세 정의서
- **제어 계정(Control Account)**: 비용·일정 통합 관리 지점
- **계획 패키지(Planning Package)**: 세부 계획 미확정 작업

## 실무 적용 예시

공공 정보시스템 구축 프로젝트에서 WBS 1수준을 '요구분석 → 설계 → 개발 → 시험 → 전환'으로 구성하고, 2수준에서 기능별 모듈로 분해하여 PMO가 진척 관리에 활용한다.

## 출제 포인트

- 작성 원칙(100% 규칙, 상호배타성) 암기 필수
- WBS와 OBS(조직분류체계) 연계, RAM(책임배정표) 작성으로 확장 출제
- 분해 기준(단계 기반 vs. 산출물 기반)의 차이 비교
