## 개요

IT 프로젝트 관리는 정보기술 프로젝트의 목표 달성을 위해 범위·일정·비용·품질·자원·위험을 계획·실행·통제하는 체계적 활동이다. PMBOK 기반의 프로세스 체계가 국제 표준으로 활용된다.

## 핵심 내용

### IT 프로젝트 관리의 개념

- **프로젝트**: 고유한 산출물을 만들기 위한 한시적(Temporary) 활동
- **IT 프로젝트 관리**: 5대 프로세스 그룹 × 10대 지식 영역으로 구성 (PMBOK 7판에서는 성과 도메인 중심으로 전환)

**10대 지식 영역 (PMBOK)**:
통합·범위·일정·비용·품질·자원·의사소통·위험·조달·이해관계자 관리

### IT 프로젝트 관리 프로세스

| 프로세스 그룹 | 주요 활동 |
|-------------|---------|
| **착수(Initiating)** | 프로젝트 헌장 승인, 이해관계자 식별 |
| **기획(Planning)** | WBS 작성, 일정·비용·위험 계획 수립 |
| **실행(Executing)** | 팀 관리, 품질 보증, 의사소통 |
| **감시·통제(Monitoring & Controlling)** | 실적 추적, 변경 통제, EVM 분석 |
| **종료(Closing)** | 인수·인계, 교훈 문서화, 계약 종료 |

### IT 프로젝트 관리 vs 프로그램 관리 vs 포트폴리오 관리

| 구분 | 프로젝트 관리 | 프로그램 관리 | 포트폴리오 관리 |
|------|------------|------------|--------------|
| 범위 | 단일 프로젝트 | 연관 프로젝트 군 | 전략적 투자 묶음 |
| 목표 | 산출물 납기·예산 준수 | 시너지·편익 실현 | 전략 목표 달성 극대화 |
| 기간 | 한시적 | 중장기 | 지속적 |
| 관리 초점 | 실행 효율성 | 상호의존성 조율 | ROI·우선순위 결정 |
| 주관 | PM(Project Manager) | PgM(Program Manager) | PMO·경영진 |

### EVM (Earned Value Management)

프로젝트 일정·비용 성과를 통합 측정:
- **PV(Planned Value)**: 계획 기준 완료 가치
- **EV(Earned Value)**: 실제 완료된 작업의 계획 가치
- **AC(Actual Cost)**: 실제 발생 비용
- **SV = EV - PV** (일정 차이), **CV = EV - AC** (비용 차이)

## 출제 포인트

- 프로젝트·프로그램·포트폴리오 관리의 3계층 비교
- PMBOK 5대 프로세스 그룹 순서
- EVM 지표: PV·EV·AC와 SV·CV 계산
