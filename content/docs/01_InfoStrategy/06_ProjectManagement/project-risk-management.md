## 개요

프로젝트 위험관리는 프로젝트 목표 달성에 영향을 미칠 수 있는 불확실한 사건이나 조건을 체계적으로 식별·분석·대응하는 프로세스다. PMI PMBOK 기준 6개 프로세스로 구성된다.

## 핵심 내용

### 위험관리 프로세스 (PMBOK 6판)

| 프로세스 | 내용 |
|----------|------|
| 1. 위험관리 계획 | 위험 관리 접근법·활동 정의, 위험 등록부 양식 수립 |
| 2. 위험 식별 | 브레인스토밍, 체크리스트, SWOT 분석으로 위험 도출 |
| 3. 정성적 위험 분석 | 발생 확률×영향도 매트릭스로 우선순위화 |
| 4. 정량적 위험 분석 | 몬테카를로 시뮬레이션, 민감도 분석 |
| 5. 위험 대응 계획 | 회피·전가·완화·수용 전략 수립 |
| 6. 위험 감시 및 통제 | 잔존 위험 모니터링, 새로운 위험 식별 |

### 위험 대응 전략 — 위협 vs 기회

**위협(Threat) 대응 전략**

| 전략 | 설명 | 예시 |
|------|------|------|
| **회피(Avoid)** | 위험 원인 제거 | 일정 변경, 범위 축소, 불확실한 기술 제거 |
| **전가(Transfer)** | 보험·계약으로 제3자 이전 | 보험 가입, 외주 계약 시 패널티 조항 |
| **완화(Mitigate)** | 발생 확률·영향도 감소 | 프로토타입 제작, 추가 테스트, 백업 |
| **수용(Accept)** | 능동적(예비비) 또는 수동적(발생 시 대응) | 비상예비비 설정, 변경 요청 절차 |

**기회(Opportunity) 대응 전략**

| 전략 | 설명 | 예시 |
|------|------|------|
| **활용(Exploit)** | 기회 실현을 확실히 함 | 더 능력 있는 팀원 배치로 일정 단축 |
| **공유(Share)** | 파트너와 기회 공동 추구 | 조인트벤처, 파트너십 |
| **강화(Enhance)** | 기회 발생 확률·영향도 증가 | 우수 자원 추가 투입 |
| **수용(Accept)** | 기회 발생 시 활용 | 별도 조치 없이 기회 발생 시 혜택 수용 |

**대응 계획 수립 절차**
1. 위험 식별 및 등록부 작성
2. 정성적/정량적 분석으로 우선순위화
3. 위협/기회 구분 후 적합한 전략 선정
4. 담당자(Risk Owner) 지정
5. 잔존 위험·2차 위험 식별 및 모니터링

### 위험 등록부 (Risk Register)

위험 ID, 설명, 발생 확률, 영향도, 위험 점수(P×I), 대응 전략, 담당자, 잔존 위험을 기록하는 핵심 문서.

## 출제 포인트

- 6단계 프로세스 순서와 각 단계의 산출물 서술
- 정성적 vs 정량적 분석 차이 설명
- 대응 전략 4가지(회피·전가·완화·수용) 예시와 함께 서술

## 연관 개념

- ISP — 정보전략계획 수립 시 위험관리 통합
