## 개요

정보시스템 감리는 전자정부법 제57조에 근거하여 정보시스템의 효율성·안전성을 확보하기 위해 제3자(감리법인)가 개발·운영·유지보수 전반을 점검하는 제도다. 시스템 운영 감리는 운영 중인 시스템의 서비스 품질과 보안을 점검하고, 유지보수 감리는 변경·개선 작업의 적정성을 확인한다.

## 핵심 내용

### 시스템 운영 감리 개념

운영 중인 정보시스템이 설계된 목적대로 안전하게 동작하며, 서비스 수준 목표(SLO)를 달성하고 있는지 정기적으로 점검하는 감리 활동.

### 시스템 운영 감리 점검분야

| 점검분야 | 주요 점검 항목 |
|----------|--------------|
| **서비스 수준 관리** | SLA 준수 여부, 가용성·응답시간·처리량 지표 |
| **변경 관리** | 변경 요청 절차 준수, 영향도 분석 수행 여부 |
| **문제 관리** | 장애 이력 관리, 근본 원인 분석, 재발 방지 |
| **보안 관리** | 접근 제어, 취약점 관리, 보안 패치 적용 |
| **용량 관리** | HW 자원 사용률, 확장 계획 적정성 |
| **성능 관리** | 응답시간, 처리량, 병목 구간 |
| **백업·복구** | 백업 정책 준수, 복구 테스트 수행 여부 |

### 유지보수 감리 개념

운영 중 발생하는 결함 수정, 환경 변화 대응, 기능 개선 등 유지보수 작업이 적절한 절차에 따라 수행되는지 점검.

### 유지보수 감리 점검분야

| 점검분야 | 주요 점검 항목 |
|----------|--------------|
| **요구사항 관리** | 변경 요구사항의 명확한 정의·추적 |
| **유지보수 계획** | 우선순위·일정·자원 배분 적정성 |
| **영향도 분석** | 변경이 기존 기능에 미치는 영향 분석 |
| **테스트 수행** | 회귀 테스트, 성능 테스트 수행 여부 |
| **형상 관리** | 버전 관리, 변경 이력 추적 |
| **이관 절차** | 운영 이관 절차 준수, 롤백 계획 |
| **문서화** | 변경 내용 문서 갱신 여부 |

### 감리 법적 근거

- 전자정부법 제57조: 행정기관 등의 정보시스템 감리 의무
- 40억 원 이상 정보화 사업은 감리 의무

## 출제 포인트

- 운영 감리와 유지보수 감리의 개념 구분
- 운영 감리 점검분야 5~7개 서술
- 유지보수 감리 점검분야 5~7개 서술
- 법적 근거(전자정부법 제57조) 언급

## 연관 개념

- [SLA]({{< relref "sla-service-level-agreement" >}}) — 운영 감리의 핵심 평가 기준
- [ISP/ISMP]({{< relref "isp-ismp" >}}) — 정보화 사업 전 단계 계획
- SW 품질 — 감리의 품질 보증 역할
