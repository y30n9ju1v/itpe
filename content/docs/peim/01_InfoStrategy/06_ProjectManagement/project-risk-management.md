## 개요

프로젝트 위험관리는 프로젝트 목표 달성에 영향을 미칠 수 있는 불확실한 사건이나 조건을 체계적으로 식별·분석·대응하는 프로세스다. PMI PMBOK 기준 6개 프로세스로 구성된다.

## 핵심 내용

### 위험관리 프로세스 (PMBOK 6판)

| 프로세스 | 내용 |
|----------|------|
| 1. 위험관리 계획 | 위험 관리 접근법·활동 정의, 위험 등록부 양식 수립 |
| 2. 위험 식별 | 브레인스토밍, 체크리스트, SWOT 분석으로 위험 도출 |
| 3. 정성적 위험 분석 | 발생 확률×영향도 매트릭스로 우선순위화 |
| 4. 정량적 위험 분석 | 몬테카를로 시뮬레이션, 민감도 분석 |
| 5. 위험 대응 계획 | 회피·전가·완화·수용 전략 수립 |
| 6. 위험 감시 및 통제 | 잔존 위험 모니터링, 새로운 위험 식별 |

### 위험 대응 전략 4가지

- **회피(Avoid)**: 위험 원인 제거 (일정 변경, 범위 축소)
- **전가(Transfer)**: 보험, 계약으로 제3자에게 이전
- **완화(Mitigate)**: 발생 확률 또는 영향도 감소
- **수용(Accept)**: 능동적(예비비 설정) 또는 수동적(발생 시 대응)

### 위험 등록부 (Risk Register)

위험 ID, 설명, 발생 확률, 영향도, 위험 점수(P×I), 대응 전략, 담당자, 잔존 위험을 기록하는 핵심 문서.

## 출제 포인트

- 6단계 프로세스 순서와 각 단계의 산출물 서술
- 정성적 vs 정량적 분석 차이 설명
- 대응 전략 4가지(회피·전가·완화·수용) 예시와 함께 서술

## 연관 개념

- 은행가 알고리즘 — 자원 교착 상태 위험 회피 알고리즘
- ISP — 정보전략계획 수립 시 위험관리 통합
