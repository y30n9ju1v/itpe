## 개요

프로그램 실행 시 운영체제는 프로세스 메모리를 코드(Code)·데이터(Data)·힙(Heap)·스택(Stack) 4개 영역으로 나누어 관리한다. 각 영역은 저장 대상, 크기 결정 시점, 접근 방식이 다르며, 이를 이해하는 것은 메모리 버그 진단과 보안 취약점 분석의 기초다.

## 핵심 내용

### 메모리 레이아웃 (낮은 주소 → 높은 주소)

```
+------------------+  높은 주소
|      Stack       |  ↓ (grows down)
+------------------+
|        ↕         |  (미사용 공간)
+------------------+
|       Heap       |  ↑ (grows up)
+------------------+
|       Data       |  BSS + Data Segment
+------------------+
|       Code       |  Text Segment
+------------------+  낮은 주소
```

### 각 영역 상세

| 영역 | 저장 내용 | 크기 | 특징 |
|------|-----------|------|------|
| **Code (Text)** | 실행 가능한 기계어 코드 | 컴파일 시 고정 | 읽기 전용(Read-only), 쓰기 방지로 코드 변조 차단 |
| **Data** | 전역변수, 정적변수 | 컴파일 시 고정 | 초기화된 변수(Data Segment) + 미초기화 변수(BSS Segment) |
| **Heap** | 동적 할당 메모리 (malloc, new) | 런타임 가변 | 프로그래머가 직접 관리, 누수(Memory Leak) 발생 가능 |
| **Stack** | 지역변수, 함수 매개변수, 반환주소 | 런타임 가변 | LIFO 구조, 함수 호출마다 스택 프레임 생성·소멸 |

### 영역 간 동작 메커니즘

**함수 호출 시 스택 동작**:
```
함수 호출 → 스택 프레임 생성 (반환주소 + 매개변수 + 지역변수)
함수 반환 → 스택 프레임 소멸 (SP 복원)
```

**힙 vs 스택 비교**:
- 스택: 빠른 접근(SP 이동만으로 할당/해제), 크기 제한(Stack Overflow)
- 힙: 유연한 크기, 느린 접근(포인터 역참조), 단편화 발생

### 보안 관련 취약점

- **Stack Buffer Overflow**: 스택 버퍼 초과 입력으로 반환주소 덮어쓰기 (ROP 공격)
- **Heap Overflow**: 힙 영역 덮어쓰기 (Use-After-Free)
- **메모리 누수**: 힙 해제 누락으로 가용 메모리 고갈
