## 개요

멀티클라우드(MultiCloud)는 둘 이상의 퍼블릭 클라우드 서비스 제공사(AWS, Azure, GCP 등)를 동시에 활용하는 전략이다. 특정 벤더 종속(Vendor Lock-in)을 방지하고 최적 서비스를 선택적으로 활용하는 것이 핵심 목적이다.

## 핵심 내용

### 개념 및 필요성

| 필요성 | 설명 |
|--------|------|
| **벤더 종속 방지** | 단일 클라우드 의존 시 가격 협상력·전환 유연성 약화 |
| **서비스 최적화** | AWS AI, Azure AD, GCP BigQuery 등 강점 서비스 선택 활용 |
| **가용성 향상** | 특정 클라우드 장애 시 다른 클라우드로 서비스 연속성 유지 |
| **규제 준수** | 국가별 데이터 주권(Data Sovereignty) 요건 충족 |
| **비용 최적화** | 워크로드별 비용 효율적인 클라우드 선택 |

**하이브리드 클라우드와의 차이**:
- 하이브리드: 프라이빗 + 퍼블릭 클라우드 연계
- 멀티클라우드: 복수의 퍼블릭 클라우드 연계

### 시스템 요구사항

| 요구사항 | 내용 |
|---------|------|
| **통합 관리 플레인** | 단일 콘솔에서 다중 클라우드 자원 관리 |
| **네트워크 연결성** | 클라우드 간 전용 네트워크(Direct Connect, ExpressRoute) |
| **ID/접근 관리** | 클라우드 간 통합 IAM, SSO |
| **데이터 이동성** | 표준 API·포맷으로 데이터 이식성 확보 |
| **보안 일관성** | 클라우드별 보안 정책 통합·표준화 |
| **모니터링 통합** | 단일 가시성 플랫폼에서 전체 클라우드 모니터링 |

### 주요 기술

| 기술 | 설명 | 도구 예시 |
|------|------|---------|
| **컨테이너 오케스트레이션** | Kubernetes로 클라우드 중립적 배포 | Kubernetes, OpenShift |
| **서비스 메시** | 마이크로서비스 간 통신·보안 통합 | Istio, Consul |
| **클라우드 관리 플랫폼(CMP)** | 다중 클라우드 자원 통합 관리 | Terraform, Pulumi |
| **데이터 패브릭** | 이기종 클라우드 데이터 통합 | Apache Kafka, Delta Lake |
| **CASB** | 클라우드 접근 보안 브로커 | Microsoft Defender for Cloud Apps |
