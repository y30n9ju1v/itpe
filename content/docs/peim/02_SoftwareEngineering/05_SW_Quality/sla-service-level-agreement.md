## 개요

SLA(Service Level Agreement)는 서비스 제공자와 고객 사이에 서비스의 수준·품질·책임을 정량적으로 명시한 공식 협약이다. IT 서비스의 품질 기준을 객관화하고 분쟁 시 판단 근거로 활용되며, 정보시스템 운영 전환 시 필수적으로 수립된다.

## 핵심 내용

### SLA 구성요소

| 구성요소 | 내용 |
|----------|------|
| **서비스 정의** | 제공 서비스 범위·내용 명확화 |
| **성능 지표(SLI)** | 측정 가능한 서비스 품질 지표 |
| **목표 수준(SLO)** | 각 SLI의 목표값 (예: 가용성 99.9%) |
| **측정 방법** | 지표 수집 방법, 도구, 주기 |
| **보고 체계** | 정기 보고 주기·형식·담당자 |
| **위반 시 조치** | 패널티, 서비스 크레딧, 에스컬레이션 |
| **예외 조항** | 불가항력, 계획된 다운타임 |
| **검토·갱신 주기** | 정기 SLA 재협상 절차 |

### SLA 수립 절차

1. **서비스 현황 분석**: 현행 서비스 수준 측정 (AS-IS)
2. **요구사항 수집**: 고객 비즈니스 요구·기술 환경 파악
3. **지표 선정**: HW·SW·네트워크 영역별 측정 지표 결정
4. **목표 수준 협의**: 기술적 달성 가능 수준과 비용 균형 협상
5. **SLA 문서화**: 협약서 작성·검토·서명
6. **모니터링 체계 구축**: 자동 측정·보고 도구 설정
7. **정기 검토**: 분기/반기 성과 검토 및 갱신

### 전자상거래 시스템 영역별 SLA 평가지표 예시

**하드웨어**:
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 서버 가용성 | 99.9% | 월간 업타임 모니터링 |
| CPU 사용률 | 평균 80% 이하 | 5분 간격 수집, 월평균 |
| 스토리지 가용 용량 | 70% 이하 유지 | 일별 용량 모니터링 |

**소프트웨어**:
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 응답 시간 | 95% 요청 2초 이내 | APM 도구 (New Relic 등) |
| 장애 복구 시간(MTTR) | 4시간 이내 | 장애 티켓 기록 |
| 배포 성공률 | 99% 이상 | CI/CD 파이프라인 로그 |

**네트워크**:
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 네트워크 가용성 | 99.95% | SNMP 모니터링 |
| 지연 시간(Latency) | 50ms 이하 | 핑 테스트 |
| 패킷 손실률 | 0.1% 이하 | 네트워크 모니터링 툴 |

### SLI → SLO → SLA 관계

- **SLI** (Service Level Indicator): 측정 지표 (가용성 %)
- **SLO** (Service Level Objective): 내부 목표 (99.95%)
- **SLA** (Service Level Agreement): 고객과의 공식 협약 (99.9%)
- SLO > SLA: 내부 목표를 더 엄격하게 설정하여 SLA 위반 방지
