## 개요

A/B 테스팅은 두 가지 버전(A: 대조군, B: 실험군)을 동시에 사용자에게 무작위 노출하여 어느 버전이 목표 지표(전환율, 클릭률 등) 개선에 효과적인지 통계적으로 검증하는 실험 기법이다. 데이터 기반 의사결정(DDDM)의 핵심 도구다.

## 핵심 내용

### A/B 테스팅 절차

1. **가설 수립**: "버튼 색상을 파란색에서 빨간색으로 바꾸면 클릭률이 향상된다"
2. **지표 선정**: 주요 지표(Primary Metric) + 가드레일 지표 설정
3. **샘플 크기 계산**: 통계적 검정력(Power ≥ 80%)·유의수준(α = 0.05) 기반
4. **무작위 배분**: 사용자를 A군·B군에 무작위 할당
5. **실험 실행**: 충분한 기간 동안 데이터 수집
6. **통계 분석**: t-검정, z-검정, p-value 계산
7. **의사결정**: p < 0.05이면 유의미한 차이로 판단 → 우수 버전 채택

### 핵심 통계 개념

| 개념 | 설명 |
|------|------|
| **p-value** | 귀무가설(차이 없음)이 참일 때 관측 결과가 나올 확률. p < 0.05이면 귀무가설 기각 |
| **신뢰구간** | 모수가 포함될 범위 (95% CI) |
| **검정력(Power)** | 실제 차이가 있을 때 탐지할 확률 (1 - β) |
| **MDE** (Minimum Detectable Effect) | 탐지 가능한 최소 효과 크기 |

### 주의사항

- **피킹(Peeking) 문제**: 중간에 결과 확인하며 조기 종료 → 위양성 증가
- **신기효과(Novelty Effect)**: 새로운 것에 대한 일시적 관심 → 충분한 기간 필요
- **샘플 오염**: A군 사용자가 B군 기능에 노출 (SUTVA 위반)
- **다중 검정 문제**: 여러 지표 동시 검정 시 Bonferroni 보정 필요

### 멀티바리에이트 테스트 (MVT)

A/B 테스팅의 확장판. 여러 요소를 동시에 변경하여 최적 조합 탐색.

## 실무 적용 예시

Netflix는 썸네일 이미지 A/B 테스트로 클릭률 20~30% 향상. Amazon은 버튼 위치·색상·문구를 지속적으로 A/B 테스트하여 구매 전환율을 최적화.
