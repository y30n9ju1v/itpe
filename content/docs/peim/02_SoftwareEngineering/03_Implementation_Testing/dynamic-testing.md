## 개요

동적 테스트(Dynamic Testing)는 소프트웨어를 실제 실행하여 결함을 발견하는 테스트 기법이다. 코드 실행 없이 검사하는 정적 테스트(코드 리뷰, 인스펙션)와 달리, 입력값을 제공하고 출력값을 확인하는 방식이다.

## 핵심 내용

### 명세 기반 테스트 vs 구조 기반 테스트

| 구분 | 명세 기반 테스트 (블랙박스) | 구조 기반 테스트 (화이트박스) |
|------|--------------------------|--------------------------|
| **관점** | 외부 기능 명세 기반 | 내부 코드 구조 기반 |
| **코드 가시성** | 불필요 | 필요 |
| **목적** | 명세 충족 여부 확인 | 코드 커버리지 확인 |
| **수행자** | 테스터, 사용자 | 개발자 |
| **기법** | 동등 분할, 경계값 분석, 분류 트리 | 구문 커버리지, 분기 커버리지, 경로 커버리지 |

### 명세 기반 테스트 기법

**① 동등 분할 (Equivalence Partitioning)**

입력 값의 범위를 동일하게 처리되는 그룹(동등 클래스)으로 분할, 각 클래스에서 대표값 1개 선택.

**[프로그램 명세 적용]**
- 입력: 정수
- 출력: 10 미만 또는 100 초과 → "Fail", 10~100 → "Pass", 정수 아닌 값 → "Invalid Input"

| 동등 클래스 | 유형 | 대표 테스트값 | 예상 출력 |
|------------|------|-------------|---------|
| EC1: 정수, 10 미만 | 유효 | 5 | Fail |
| EC2: 정수, 10~100 | 유효 | 50 | Pass |
| EC3: 정수, 100 초과 | 유효 | 150 | Fail |
| EC4: 정수가 아닌 값 | 유효 | "abc" | Invalid Input |

**② 분류 트리 기법 (Classification Tree Method)**

입력을 분류 기준(Aspect)으로 계층적으로 분류하고 조합으로 테스트 케이스 생성.

```
[입력값]
├── 데이터 타입
│   ├── 정수
│   └── 비정수 (문자, 실수 등)
└── 정수 범위 (데이터 타입이 정수인 경우)
    ├── 10 미만
    ├── 10~100
    └── 100 초과
```

| 테스트 케이스 | 데이터 타입 | 범위 | 입력값 | 예상 출력 |
|-------------|-----------|------|--------|---------|
| TC1 | 정수 | 10 미만 | 5 | Fail |
| TC2 | 정수 | 10~100 | 55 | Pass |
| TC3 | 정수 | 100 초과 | 200 | Fail |
| TC4 | 비정수 | - | "abc" | Invalid Input |

### 경계값 분석 (Boundary Value Analysis)

동등 분할 경계에서 오류가 많이 발생하므로 경계값 선택:
- 10 미만 경계: 9, 10
- 100 이하 경계: 100, 101

## 출제 포인트

- 명세 기반(블랙박스) vs 구조 기반(화이트박스) 비교표 작성
- 동등 분할의 4개 클래스(EC1~EC4) 직접 도출
- 분류 트리를 계층 구조로 그리고 테스트 케이스 작성
- 경계값 분석과의 차이 언급
