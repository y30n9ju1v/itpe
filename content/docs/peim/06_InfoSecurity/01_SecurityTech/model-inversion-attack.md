## 개요

모델 전도 공격(Model Inversion Attack)은 AI 모델의 출력(예측 결과, 신뢰도 점수)을 역방향으로 분석하여 학습 데이터의 민감정보를 재구성하는 공격이다. 안면인식 모델에서 학습 데이터의 얼굴 이미지를 복원하는 것이 대표적 사례다.

## 핵심 내용

### 공격 원리

```
공격자 → [쿼리: 다양한 입력] → AI 모델 → [출력: 예측/신뢰도 반환]
공격자 ← [역추론: 출력을 분석하여 학습 데이터 복원]
```

모델이 학습 데이터의 특성을 "기억"하는 성질(Memorization)을 악용.

### 공격 유형

| 유형 | 설명 |
|------|------|
| **화이트박스 공격** | 모델 파라미터에 직접 접근 가능한 경우 |
| **블랙박스 공격** | API를 통한 예측 결과만 이용 (실제 공격 시나리오) |
| **멤버십 추론** | 특정 데이터가 학습 데이터에 포함되었는지 판단 |
| **속성 추론** | 학습 데이터의 특정 속성(민감 정보) 추론 |

### 실제 공격 사례

- 안면인식 분류기에 최적화 기법을 적용하여 학습 이미지 복원 (Fredrikson et al., 2015)
- 의료 AI 모델에서 환자 개인정보 재구성

### 대응 방안

- **차분 프라이버시(Differential Privacy)**: 학습 시 노이즈 추가로 개인 기여도 은닉
- **연합학습(Federated Learning)**: 원본 데이터 공유 없이 분산 학습
- **쿼리 제한**: 모델에 대한 쿼리 횟수·속도 제한
- **모델 워터마킹**: 공격 여부 추적
- **출력 정보 최소화**: 신뢰도 점수 대신 최상위 클래스만 반환

## 출제 포인트

- 공격 원리(모델 출력 역추론)를 명확히 서술
- 화이트박스 vs 블랙박스 구분
- PET 기술(차분 프라이버시, 연합학습)과 연계한 대응 방안
