## 개요

ISMS(Information Security Management System)는 정보보호 관리체계 인증 제도로, 조직의 정보보호 수준을 체계적으로 관리하고 법적 의무를 이행하기 위한 국내 인증이다. ISMS-P는 ISMS에 개인정보 보호(Privacy) 관리체계를 통합한 확장 인증이다.

## 핵심 내용

### ISMS 인증 영역과 법적 근거

**법적 근거**: 정보통신망법 제47조, 개인정보보호법 제32조의2

**의무 인증 대상**:
- 연간 매출 100억 원 이상 정보통신서비스 제공자
- 직전 년도 말 기준 이용자 수 100만 명 이상
- 정보통신망 서비스 제공 ISP

**인증 영역 (ISMS 16개 + 개인정보 9개 통제항목)**:
- ISMS: 관리체계 수립·운영 + 보호대책 요구사항(총 80개 통제항목)
- ISMS-P: ISMS 80개 + 개인정보 처리 단계별 19개 = 총 102개

### ISMS 관리과정 (PDCA 기반 5단계)

| 단계 | 내용 |
|------|------|
| **정보보호 정책 수립** | 경영진 의지, 정보보호 정책 문서화 |
| **범위 설정** | 인증 범위(시스템·서비스·조직) 결정 |
| **위험 관리** | 위험 식별·분석·평가·처리 |
| **구현** | 보호대책 실행, 교육·훈련 |
| **사후 관리** | 모니터링, 내부 심사, 경영 검토 |

### ISMS-P 간편인증 제도

소규모 기업의 인증 부담 완화를 위해 도입된 간소화 심사 제도:
- **대상**: 중소기업, 스타트업 등 소규모 정보통신서비스
- **특징**: 통제항목 축소 적용, 심사 기간·비용 단축
- **목적**: 인증 사각지대 해소, 보안 수준 향상

### ISMS-P 제도 강화 방안

1. **의무 대상 확대**: 클라우드·핀테크 등 신산업 분야 포함
2. **실효성 강화**: 형식적 문서 심사 → 기술적 취약점 점검 병행
3. **갱신 주기 개선**: 3년 갱신 → 중간 사후 점검 강화
4. **처벌 강화**: 인증 위반 시 과징금 상향
5. **국제 표준 연계**: ISO 27001과 상호인정 체계 구축
