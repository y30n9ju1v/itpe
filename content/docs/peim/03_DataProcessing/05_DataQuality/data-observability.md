## 개요

데이터 관측가능성(Data Observability)은 데이터 파이프라인 전반에서 데이터의 상태, 품질, 흐름을 실시간으로 이해하고 이상을 탐지·진단하는 능력이다. 소프트웨어 관측가능성(로그·메트릭·트레이스) 개념을 데이터 영역에 적용한 것으로, 데이터 다운타임(Data Downtime)을 최소화하는 것이 목적이다.

## 핵심 내용

### 5가지 핵심 지표 (Five Pillars)

| 지표 | 설명 |
|------|------|
| **Freshness (신선도)** | 데이터가 얼마나 최신인가? 마지막 갱신 시점 |
| **Distribution (분포)** | 데이터 값의 범위·분포가 정상인가? |
| **Volume (볼륨)** | 예상 데이터 양이 수집되고 있는가? |
| **Schema (스키마)** | 테이블·컬럼 구조가 변경되지 않았는가? |
| **Lineage (계보)** | 데이터가 어디서 생성되어 어떻게 변환·전달되는가? |

### 데이터 다운타임 (Data Downtime)

데이터가 누락·부정확·오래된 상태로 있는 기간. 비즈니스 의사결정 품질에 직접 영향.

### 데이터 관측가능성 vs 데이터 품질

| 구분 | 데이터 품질 관리 | 데이터 관측가능성 |
|------|-----------------|-----------------|
| 시점 | 정기 점검 | 실시간 모니터링 |
| 범위 | 저장된 데이터 | 파이프라인 전체 |
| 목적 | 품질 측정 | 이상 탐지·근본원인 분석 |

### 주요 도구

Monte Carlo Data, Great Expectations, dbt, Apache Atlas (데이터 계보)

## 실무 적용 예시

대형 e-커머스에서 구매 데이터 파이프라인 중간에 스키마 변경이 발생할 경우, 데이터 관측가능성 도구가 자동으로 Lineage를 추적하여 영향받는 하위 대시보드를 즉시 식별하고 알림 발송.

## 출제 포인트

- 5가지 핵심 지표(Freshness, Distribution, Volume, Schema, Lineage) 명칭과 의미
- 데이터 다운타임 개념 포함
- 기존 데이터 품질 관리와의 차이점 설명

## 연관 개념

- 데이터 품질관리 — 관측가능성의 상위 목표
- 데이터 계보(Data Lineage) — 관측가능성의 핵심 요소
- 빅데이터 파이프라인 — 관측가능성 적용 대상
