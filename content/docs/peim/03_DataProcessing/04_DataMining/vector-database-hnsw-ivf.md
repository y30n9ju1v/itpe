## 개요

벡터 데이터베이스(Vector Database)는 AI 임베딩(고차원 벡터)을 저장하고 의미적 유사도 기반으로 검색하는 전용 데이터베이스다. LLM의 RAG(Retrieval-Augmented Generation)와 추천 시스템에서 핵심 인프라로 부상했다. HNSW와 IVF는 고차원 벡터의 효율적 근사 최근접 이웃(ANN) 검색 알고리즘이다.

## 핵심 내용

### 벡터 데이터베이스 개념

- **저장 대상**: 텍스트·이미지·음성의 임베딩 벡터 (수백~수천 차원)
- **검색 방식**: 코사인 유사도, 유클리드 거리 기반 유사 벡터 탐색
- **대표 DB**: Pinecone, Weaviate, Milvus, Chroma, pgvector

**정확 탐색 vs 근사 탐색**:
- 완전 탐색(KNN): 모든 벡터와 비교 → O(n), 수백만 개에서 느림
- 근사 최근접 이웃(ANN): 약간의 정확도 희생으로 O(log n) 탐색

### HNSW (Hierarchical Navigable Small World)

**개념**: 소셜 네트워크의 Small World 특성을 계층적 그래프 구조에 적용한 ANN 알고리즘.

**구조**:
```
레이어 2 (희소): 장거리 고속 도약
레이어 1 (중간): 중간 거리 탐색
레이어 0 (밀집): 정밀 근접 탐색
```

**검색 과정**:
1. 최상위 레이어에서 쿼리 벡터와 가장 가까운 진입점 탐색
2. 하위 레이어로 이동하며 점점 정밀하게 탐색
3. 레이어 0에서 최종 k-NN 결과 반환

**특징**:
- 검색 속도: O(log n), 매우 빠름
- 메모리: 그래프 저장으로 많은 메모리 필요
- 실시간 삽입 지원
- 업계 표준 (Faiss, Qdrant 등 사용)

### IVF (Inverted File Index)

**개념**: 벡터 공간을 클러스터로 분할하고, 쿼리와 가까운 클러스터만 탐색하는 알고리즘.

**구조**:
```
[K-Means 클러스터링]
  클러스터 1: [v1, v3, v7, ...]
  클러스터 2: [v2, v5, v9, ...]
  ...
  클러스터 n: [v4, v6, v8, ...]
```

**검색 과정**:
1. 쿼리 벡터와 가장 가까운 nprobe개 클러스터 선택
2. 선택된 클러스터 내 벡터만 완전 탐색
3. k-NN 결과 반환

**특징**:
- 학습(Train) 필요: K-Means로 사전 클러스터링
- 메모리 효율: HNSW보다 적은 메모리
- nprobe 조정으로 속도·정확도 트레이드오프
- 대규모 정적 데이터셋에 적합

### HNSW vs IVF 비교

| 구분 | HNSW | IVF |
|------|------|-----|
| 구조 | 계층적 그래프 | 클러스터+역파일 |
| 검색 속도 | 매우 빠름 | 빠름 (nprobe 의존) |
| 메모리 | 많음 | 적음 |
| 사전 학습 | 불필요 | 필요 (K-Means) |
| 실시간 삽입 | 가능 | 어려움 |
| 정확도 | 높음 | nprobe에 따라 조정 |
