## 개요

이진 탐색 트리(BST: Binary Search Tree)는 각 노드에 대해 왼쪽 서브트리의 모든 값이 해당 노드보다 작고, 오른쪽 서브트리의 모든 값이 해당 노드보다 큰 이진 트리다. 탐색·삽입·삭제 연산을 평균 O(log n)에 수행한다.

## 핵심 내용

### BST 속성

```
        8
       / \
      3   10
     / \    \
    1   6    14
       / \   /
      4   7 13
```

- 왼쪽 자식 < 부모 < 오른쪽 자식
- 중위 순회(Inorder: 좌→루트→우) 시 정렬된 순서 출력

### 주요 연산

**탐색 (Search)**: 루트에서 시작, 목표값 < 현재 → 좌, > 현재 → 우 반복
- 평균: O(log n), 최악(편향 트리): O(n)

**삽입 (Insert)**: 탐색으로 적절한 위치 찾아 리프 노드로 삽입
- 중복 허용 여부는 설계에 따라 결정

**삭제 (Delete)**: 3가지 경우
1. 리프 노드: 그냥 제거
2. 자식 1개: 자식으로 대체
3. 자식 2개: 오른쪽 서브트리의 최솟값(후계자)으로 대체

### 균형 이진 탐색 트리

편향 트리(Skewed Tree) 발생 시 모든 연산 O(n)으로 저하 → 자동 균형 유지 트리 필요

| 트리 | 균형 조건 | 특징 |
|------|----------|------|
| **AVL 트리** | |좌·우 높이 차| ≤ 1 | 엄격한 균형, 회전 많음 |
| **Red-Black 트리** | 루트~리프 흑색 노드 수 동일 | 느슨한 균형, Java TreeMap |
| **B-트리** | 모든 리프 동일 깊이 | 디스크 기반 DB 인덱스 |

### 순회 방법

| 순회 | 방법 | 출력 순서 (위 예시) |
|------|------|------------------|
| 전위(Preorder) | 루트→좌→우 | 8, 3, 1, 6, 4, 7, 10, 14, 13 |
| 중위(Inorder) | 좌→루트→우 | 1, 3, 4, 6, 7, 8, 10, 13, 14 |
| 후위(Postorder) | 좌→우→루트 | 1, 4, 7, 6, 3, 13, 14, 10, 8 |

## 출제 포인트

- BST 속성(좌 < 부모 < 우) 정의
- 삭제 연산 3가지 경우(리프·자식1·자식2) 서술
- 편향 트리 문제와 AVL·Red-Black 트리로의 해결
- 중위 순회 = 정렬된 출력 강조
