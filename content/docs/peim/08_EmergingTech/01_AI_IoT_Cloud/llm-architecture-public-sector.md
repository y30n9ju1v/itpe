## 개요

공공기관의 거대 언어 모델(LLM) 도입은 효율성 향상 기회를 제공하지만, 보안·신뢰성·법적 요건 등 고유한 제약이 존재한다. 올바른 고려사항과 아키텍처 선택이 성공적 도입의 전제다.

## 핵심 내용

### LLM 적용을 위한 5가지 고려사항

| 고려사항 | 설명 |
|---------|------|
| **1. 데이터 보안 및 프라이버시** | 개인정보·기밀정보의 외부 LLM 전송 금지, 온프레미스/전용 클라우드 검토 |
| **2. 환각(Hallucination) 대응** | LLM의 허위 정보 생성 위험 → 사실 검증 체계(RAG, 출처 표시) 구축 |
| **3. 편향·공정성** | 학습 데이터 편향으로 인한 차별적 산출물 위험 → 편향 감사 체계 |
| **4. 법적·규제 준수** | 개인정보보호법·AI 기본법 준수, 저작권 리스크 관리 |
| **5. 운영·유지보수 비용** | GPU 인프라·파인튜닝·모니터링 비용의 지속성 확보 |

### 현재 구현 가능한 5가지 LLM 아키텍처

| 아키텍처 | 특징 | 적합 용도 |
|---------|------|---------|
| **1. 클로즈드-소스 API** | OpenAI GPT, Claude API 활용 | 빠른 프로토타이핑, 일반 업무 |
| **2. 오픈소스 자체 구축** | LLaMA, Mistral 온프레미스 배포 | 데이터 보안 중요 기관 |
| **3. RAG(Retrieval-Augmented Generation)** | 내부 문서 DB + LLM 연동 | 지식 기반 Q&A, 민원 응대 |
| **4. 파인튜닝(Fine-tuning)** | 기관 특화 데이터로 모델 재학습 | 도메인 특화 업무(법령, 의료) |
| **5. 에이전틱 AI(Agentic AI)** | LLM + 도구 호출 + 멀티에이전트 | 복합 업무 자동화, 워크플로우 |

### RAG 아키텍처 상세

```
사용자 질의
    ↓
벡터 DB 검색 (내부 문서 임베딩)
    ↓
관련 문서 청크 검색
    ↓
LLM 프롬프트 = 질의 + 검색 문서
    ↓
답변 생성 (출처 포함)
```
