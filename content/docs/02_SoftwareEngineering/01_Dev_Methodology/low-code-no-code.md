## 개요

로우코드(Low Code)는 시각적 드래그앤드롭 인터페이스로 애플리케이션을 개발하되, 필요 시 코드 작성도 허용하는 개발 플랫폼이다. 노코드(No Code)는 코드 없이 비개발자가 완전한 앱을 구축하는 플랫폼으로, 시민개발자(Citizen Developer) 시대를 이끌고 있다.

## 핵심 내용

### 로우코드 주요 특징

- **시각적 개발(Visual Development)**: WYSIWYG 방식 UI 설계
- **컴포넌트 기반**: 미리 만들어진 UI·로직 블록 재사용
- **통합 용이성**: API 연동, DB 연결 내장
- **자동 코드 생성**: 플랫폼이 실행 코드 자동 생성
- **빠른 프로토타이핑**: 개발 속도 5~10배 향상 (가트너 추정)

### 노코드 vs 로우코드 비교

| 구분 | 노코드 (No Code) | 로우코드 (Low Code) |
|------|-----------------|-------------------|
| 대상 | 비개발자, 시민개발자 | 개발자 + 시민개발자 |
| 코드 작성 | 불필요 | 필요 시 허용 |
| 유연성 | 낮음 | 중간~높음 |
| 복잡도 | 단순 업무 자동화 | 엔터프라이즈급 가능 |
| 대표 도구 | Bubble, Airtable, Notion | OutSystems, Mendix, Microsoft PowerApps |
| 거버넌스 | 관리 어려움 | 거버넌스 체계 존재 |

### 로우코드 적용 시 한계점

1. **벤더 종속성(Vendor Lock-in)**: 플랫폼 의존으로 마이그레이션 비용 높음
2. **성능 한계**: 자동 생성 코드의 비효율로 고성능 시스템에 부적합
3. **보안 위험**: 시민개발자의 보안 인식 부족, 섀도우 IT 발생
4. **커스터마이징 한계**: 복잡한 비즈니스 로직 구현에 한계
5. **기술 부채**: 플랫폼 업데이트 시 호환성 문제

## 출제 포인트

- 로우코드 주요 특징 3~5가지 서술
- 노코드 vs 로우코드 비교표 (대상·코드·유연성·복잡도)
- 한계점을 벤더종속·성능·보안·기술부채 측면으로 분류
