## 개요

테일러링(Tailoring)은 표준 소프트웨어 개발방법론을 특정 프로젝트의 규모·특성·환경에 맞게 조정하는 활동이다. 과도한 프로세스 적용으로 인한 낭비를 줄이고 프로젝트 성공률을 높이기 위해 수행한다.

## 핵심 내용

### 테일러링 필요성

- 표준 방법론은 모든 프로젝트에 동일 적용 불가
- 소규모·긴급·혁신 프로젝트에서 과도한 절차가 오히려 장애
- 선도개발(Pilot/Prototype) 프로젝트는 불확실성이 높아 유연한 프로세스 필요

### 테일러링 기준 (판단 요소)

| 기준 | 설명 |
|------|------|
| 프로젝트 규모 | 소규모 프로젝트 → 문서 간소화 |
| 위험도 | 고위험 → 검토·승인 단계 강화 |
| 기술 성숙도 | 신기술 적용 → 프로토타이핑 추가 |
| 납기 긴급성 | 긴급 → 일부 단계 병행 또는 생략 |
| 법적 규제 | 의료·금융 → 규정 준수 프로세스 유지 |

### 테일러링 절차

1. **기준 방법론 선정** — 폭포수, 애자일, RUP 등
2. **프로젝트 특성 분석** — 규모, 유형, 제약사항
3. **조정 항목 식별** — 추가·생략·변경할 프로세스·산출물 결정
4. **테일러링 방법론 문서화** — 조정 근거와 결과 기록
5. **승인 및 적용** — 프로젝트 관리자·PMO 승인

### 선도개발과 테일러링

선도개발(Spike/Pilot)에서는 핵심 기능만 빠르게 구현하여 기술 검증 후 본 개발로 전환. 이 단계에서는 설계 문서보다 실행 가능한 프로토타입 산출이 우선.

## 출제 포인트

- 테일러링의 정의와 필요성을 선도개발 맥락에서 서술
- 5단계 절차 순서대로 설명
- 구체적 조정 예시(산출물 간소화, 단계 병행) 포함

## 연관 개념

- 로우코드(Low Code) — 테일러링의 구현 수단 중 하나
- 애자일 방법론 — 테일러링의 대표적 결과물
- 프로젝트 위험관리 — 테일러링 시 위험 고려
