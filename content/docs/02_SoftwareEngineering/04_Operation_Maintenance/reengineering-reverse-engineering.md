## 개요

재공학(Re-engineering)은 기존 소프트웨어를 분석하여 동일한 기능을 유지하면서 품질·유지보수성을 개선하는 활동이다. 역공학(Reverse-engineering)은 완성된 시스템에서 설계·요구사항 등 상위 수준 추상화 정보를 도출하는 기법으로, 재공학의 선행 단계다.

## 핵심 내용

### 소프트웨어 유지보수 유형 (IEEE 1219)

| 유형 | 설명 |
|------|------|
| 수정(Corrective) | 결함 수정 |
| 적응(Adaptive) | 환경 변화 대응 |
| 완전(Perfective) | 기능 향상 |
| 예방(Preventive) | 미래 유지보수성 향상 → **재공학** |

### 재공학 (Re-engineering)

**개념**: 기존 시스템의 기능은 보존하면서 내부 구조·코드를 개선하는 예방적 유지보수

**목적**: 기술 부채 해소, 레거시 현대화, 유지보수성·확장성 향상

**재공학 절차**:
1. **역공학(분석)**: 기존 코드에서 설계·요구사항 추출
2. **재구조화(Restructuring)**: 코드 품질 개선 (리팩토링)
3. **데이터 재공학**: 데이터 구조 개선 (정규화, 마이그레이션)
4. **순공학(Forward Engineering)**: 새로운 설계로 구현

```
[기존 시스템] → 역공학 → [설계 문서] → 순공학 → [새 시스템]
```

### 역공학 (Reverse-engineering)

**개념**: 완성된 제품·코드로부터 설계·명세·요구사항을 도출하는 분석 활동

**활용 방안**:
- 문서가 없는 레거시 시스템 분석
- 인터페이스 호환성 분석
- 보안 취약점 분석 (악성코드 분석, 취약점 탐지)
- 경쟁사 제품 분석 (법적 한계 주의)
- 클라우드 마이그레이션을 위한 의존성 분석

### 재공학 vs 재개발 비교

| 구분 | 재공학 | 재개발 |
|------|--------|--------|
| 위험 | 낮음 (기존 기능 보존) | 높음 (새로운 시스템) |
| 비용 | 중간 | 높음 |
| 기간 | 단기~중기 | 장기 |
| 적합 | 기능은 유효하나 품질 저하 | 기능 자체가 부적합 |

## 출제 포인트

- 재공학과 역공학의 관계(역공학이 재공학의 첫 단계) 명확히 서술
- 재공학 4단계 절차 순서대로 서술
- 역공학 활용 방안(레거시 분석, 보안 분석) 2~3개 예시
- 재공학 vs 재개발 비교
