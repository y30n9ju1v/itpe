## 개요

요구사항 추적표(RTM, Requirement Traceability Matrix)는 소프트웨어 요구사항과 설계·구현·테스트 산출물 간의 연결 관계를 문서화한 표다. 요구사항의 완전 구현 여부와 변경 영향을 추적하는 핵심 도구다.

## 핵심 내용

### RTM의 목적

- **완전성 검증**: 모든 요구사항이 설계·구현·테스트에 반영되었는지 확인
- **변경 영향 분석**: 요구사항 변경 시 영향받는 산출물 파악
- **품질 보증**: 테스트 케이스와 요구사항 간 커버리지 측정

### 추적 방향

| 유형 | 방향 | 목적 |
|------|------|------|
| 순방향 추적 | 요구사항 → 설계/구현/테스트 | 요구사항 구현 여부 확인 |
| 역방향 추적 | 설계/테스트 → 요구사항 | 불필요한 기능 제거 (황금도금 방지) |
| 양방향 추적 | 상호 연결 | 완전성 + 불필요 기능 동시 점검 |

### RTM 구성 요소

| 컬럼 | 내용 |
|------|------|
| 요구사항 ID | 고유 식별자 (예: REQ-001) |
| 요구사항 명 | 기능·비기능 요구사항 설명 |
| 설계 산출물 | 연결된 설계 문서·컴포넌트 |
| 소스코드 모듈 | 구현 모듈·클래스 |
| 테스트 케이스 ID | 검증 테스트 |
| 상태 | 완료/진행 중/미구현 |

### 활용 시점

- 요구사항 분석 완료 후 초안 작성
- 설계·구현·테스트 단계에서 지속 갱신
- 변경 요청(CR) 처리 시 영향 범위 분석

## 실무 적용 예시

공공 SI 프로젝트에서 감리 수검 시 RTM을 통해 요구사항 충족률을 증빙하고, 테스트 커버리지 부족 항목을 보완 조치한다.

## 출제 포인트

- RTM의 순방향·역방향·양방향 추적 구분
- 황금도금(Gold Plating) 방지 수단으로서의 역방향 추적
- 감리·품질 보증에서의 활용
