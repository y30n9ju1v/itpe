## 개요

소프트웨어 품질보증(SQA, Software Quality Assurance)은 소프트웨어 개발 프로세스와 산출물이 정해진 품질 기준을 만족하는지 체계적으로 확인하는 활동이다. 인스펙션(Inspection)은 SQA의 핵심 정적 검토 기법으로, 결함을 조기에 발견해 수정 비용을 최소화한다.

## 핵심 내용

**소프트웨어 품질의 의미**
- ISO/IEC 25010 기준: 기능적합성·성능효율성·호환성·사용성·신뢰성·보안성·유지보수성·이식성
- 프로세스 품질 + 제품 품질 + 사용 품질의 통합

**SQA 목적과 기능**
- 목적: 결함 예방, 조기 발견, 개발 프로세스 개선
- 기능: 품질 계획 수립, 표준 적용 검토, 감사, 메트릭 수집·분석, 개선 권고

**인스펙션(Fagan Inspection)**
- 정의: 개발 산출물(요구사항·설계·코드)을 구조화된 절차로 검토하는 정형 리뷰
- 목적: 테스트 전 조기 결함 제거 (비용 효과: 테스트 단계 결함 수정 비용의 1/10)

**인스펙션 프로세스 6단계**

| 단계 | 설명 |
|------|------|
| 1. 계획 (Planning) | 인스펙션 목표·범위·참여자 결정 |
| 2. 개요 설명 (Overview) | 작성자가 산출물 배경 설명 |
| 3. 개별 준비 (Preparation) | 각 참여자가 개별 검토·결함 목록 작성 |
| 4. 인스펙션 회의 (Meeting) | 결함 식별·기록 (단, 해결 X) |
| 5. 재작업 (Rework) | 작성자가 식별된 결함 수정 |
| 6. 후속 조치 (Follow-up) | 수정 완료 여부 확인 |

**참여 역할**
- 중재자(Moderator): 회의 진행, 프로세스 준수 관리
- 작성자(Author): 산출물 작성자
- 검토자(Reviewer/Inspector): 결함 발견
- 기록자(Recorder): 결함 목록 기록
