## 개요

ISO 26262는 자동차 전기·전자 시스템의 기능안전(Functional Safety) 국제 표준이다. 시스템 오작동으로 인한 인명 피해를 방지하기 위해 개발 프로세스 전체(개념→설계→구현→검증→운용)에 안전 요구사항을 부과한다. ASIL(Automotive Safety Integrity Level)은 위험도 기반 안전 무결성 수준이다.

## 핵심 내용

### 가. ISO 26262 구성요소

**ISO 26262:2018 (2판) 구조 (12개 파트)**:

| 파트 | 내용 |
|------|------|
| Part 1 | 용어 정의 |
| Part 2 | 기능안전 관리 (개발 조직, 프로세스) |
| Part 3 | 개념 단계 (위험 분석·HARA, 기능안전 목표) |
| Part 4 | 시스템 레벨 제품 개발 |
| Part 5 | HW 레벨 제품 개발 |
| Part 6 | SW 레벨 제품 개발 |
| Part 7 | 양산 및 운용 |
| Part 8 | 지원 프로세스 (형상·변경·문서 관리) |
| Part 9 | ASIL 기반 분석 |
| Part 10 | ISO 26262 가이드라인 |
| Part 11 | 반도체 적용 가이드라인 |
| Part 12 | 이륜차 |

**핵심 개발 흐름 (V-모델)**:
```
개념 단계 (HARA)
    ↓
시스템 설계 (안전 요구사항 할당)
    ↓          ↑
HW 설계 → HW 통합 시험
    ↓          ↑
SW 설계 → SW 통합 시험
    ↓          ↑
SW 구현 → SW 단위 시험
```

### 나. ASIL (Automotive Safety Integrity Level)

**ASIL 정의**: 위험 분석·위험 평가(HARA)를 통해 결정되는 4단계 안전 무결성 수준

| ASIL | 위험도 | 대표 시스템 | 요구 수준 |
|------|--------|-----------|---------|
| ASIL A | 낮음 | 비상등, 와이퍼 | 기본 안전 기법 |
| ASIL B | 중간 | 에어백, ABS | 진단 커버리지(DC) 60% |
| ASIL C | 높음 | 조향 어시스트(EPS) | DC 90% |
| ASIL D | 매우 높음 | 전자식 스티어링, 브레이크 | DC 99%, PMHF ≤ 10⁻⁸/h |
| QM | 해당 없음 | 오락·편의 시스템 | 품질관리만 적용 |

**ASIL 결정 요소 (HARA)**:

| 요소 | 등급 | 설명 |
|------|------|------|
| **심각도 (S)** | S0~S3 | 사고 발생 시 상해 정도 |
| **노출도 (E)** | E0~E4 | 위험 상황 노출 빈도 |
| **제어가능성 (C)** | C0~C3 | 운전자가 사고를 회피할 수 있는 능력 |

**ASIL = f(S, E, C)**: 세 요소의 조합으로 ASIL 등급 결정

## 실무 적용 예시

- 자율주행 차량: ADAS의 각 기능(긴급 제동, 차선 유지)에 ASIL 등급 부여 → 등급에 맞는 HW·SW 개발
- ISO 26262 인증: 차량 ECU 개발사가 Tier 1으로 완성차 납품 시 인증 필요

## 출제 포인트

- ISO 26262 파트 구조: Part 3(개념/HARA), 5(HW), 6(SW)
- ASIL 4단계: A(낮음) → D(매우 높음) + QM
- ASIL 결정 3요소: 심각도(S)·노출도(E)·제어가능성(C)
