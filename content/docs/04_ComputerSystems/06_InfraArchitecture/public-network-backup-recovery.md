## 개요

공공망(통신·전력·교통 등) 인프라는 배터리 화재·자연재해·사이버 공격 등으로 서비스 중단이 발생할 수 있다. 인프라 관리, 백업복구, 이중화 관점에서 사전 대비와 신속 복구 체계가 필수적이다.

## 핵심 내용

### 공공망 장애 발생 문제점 분석

**인프라 관리 관점:**
- 배터리(ESS/UPS) 노후화 상태 파악 부재
- 배터리 모니터링 시스템 미흡 (온도·충전상태 실시간 감시 부재)
- 화재 예방 설비(할론소화설비, 온도센서) 미설치

**백업복구 관점:**
- 단일 공급 경로(단점): 주 전원 차단 시 백업 절체 실패
- RTO(Recovery Time Objective)·RPO(Recovery Point Objective) 미정의
- 복구 절차 문서화 부재 및 정기 훈련 미실시

**이중화 관점:**
- Active-Active 이중화 미구성 → 단일 장애점(SPOF) 존재
- 지리적 분산(Geographic Redundancy) 부재
- 대역폭 확보 없어 장애 시 우회 경로 없음

### 보완 대응 방안

#### 인프라 관리

| 방안 | 내용 |
|------|------|
| **ESS/배터리 모니터링** | BMS(Battery Management System) 도입, 온도·충전량 실시간 모니터링 |
| **예방 점검** | 정기 절연 저항 측정, 배터리 교체 주기 관리 |
| **화재 예방** | 온도 임계값 경보, 자동 소화 시스템, 방화구획 강화 |

#### 백업복구

| 방안 | 내용 |
|------|------|
| **RTO/RPO 설정** | 서비스별 목표 복구 시간·복구 시점 설정 |
| **다중 전원 공급** | 한전 전력 + 자체 발전기 + UPS 삼중화 |
| **BCP/DR 훈련** | 연 1회 이상 장애 시나리오 실전 훈련 |
| **백업 회선** | 유선·무선·위성 다경로 통신 확보 |

#### 이중화

| 방안 | 내용 |
|------|------|
| **Active-Active 구성** | 모든 노드가 동시 서비스, 장애 시 즉시 전환 |
| **지리적 이중화** | 데이터센터 분산 (최소 50km 이상) |
| **SDN 기반 자동 우회** | 장애 감지 즉시 대체 경로 자동 전환 |
| **N+1 이중화** | 최소 여분 장비 1대 이상 확보 |

### 핵심 개념: 3-2-1 백업 원칙

```
3개의 백업 복사본
2가지 다른 저장 매체 (SSD + 테이프 등)
1개는 오프사이트 (원격지 보관)
```

### RTO와 RPO

| 지표 | 의미 | 예시 |
|------|------|------|
| **RTO** (Recovery Time Objective) | 장애 후 서비스 복구까지 최대 허용 시간 | 4시간 내 복구 |
| **RPO** (Recovery Point Objective) | 복구 시 허용 가능한 데이터 손실 시간 | 최대 1시간 전 데이터까지 허용 |
