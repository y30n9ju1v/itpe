## 개요

기아현상(Starvation)은 특정 프로세스가 우선순위·자원 경쟁에서 계속 밀려 CPU 또는 자원을 무한정 할당받지 못하는 현상이다. 교착상태(Deadlock)와 혼동되나, 기아는 이론적으로 자원 할당이 가능하지만 실제로 받지 못하는 상태이고, 교착상태는 순환 대기로 어떤 프로세스도 진행 불가한 상태다.

## 핵심 내용

### 기아현상의 정의 및 발생 조건

**정의**: 자원을 할당받을 수 있음에도 불구하고, 우선순위 또는 자원 할당 정책으로 인해 특정 프로세스가 무한히 대기하는 상태.

**발생 조건**
1. 우선순위 기반 스케줄링: 고우선순위 프로세스가 계속 유입되어 저우선순위 프로세스 대기
2. 자원 경쟁: 특정 자원을 다른 프로세스가 독점
3. 불공정한 할당 정책: LIFO 큐, 부족한 공정성 보장

### 기아현상 해결 방안 3가지

| 방안 | 설명 |
|------|------|
| **에이징(Aging)** | 대기 시간이 길어질수록 우선순위를 점진적으로 높여 결국 CPU 획득 |
| **공정 큐잉(Fair Queuing)** | 라운드 로빈·다단계 피드백 큐로 모든 프로세스에 순환 기회 부여 |
| **타임아웃 기반 자원 반환** | 자원 보유 최대 시간 제한, 자동 반환으로 독점 방지 |

### 교착상태와 기아현상 비교

| 구분 | 교착상태 (Deadlock) | 기아현상 (Starvation) |
|------|-------------------|--------------------|
| **정의** | 프로세스들이 서로 자원을 기다려 무한 대기 | 특정 프로세스만 무한 대기 |
| **원인** | 순환 대기 (4가지 필요조건) | 불공정한 스케줄링·할당 |
| **범위** | 관련 프로세스 모두 멈춤 | 일부 프로세스만 영향 |
| **해결** | 예방/회피/탐지·복구 | 에이징, 공정 큐잉 |
| **OS 감지** | Banker's Algorithm 등 | 대기 시간 모니터링 |
| **비유** | 교착된 교차로 (모두 멈춤) | 횡단보도 우선권 없는 보행자 |

### 교착상태 4가지 필요 조건 (Coffman 조건)

상호 배제, 점유 대기, 비선점, 순환 대기 — 하나라도 제거하면 교착상태 불가.
