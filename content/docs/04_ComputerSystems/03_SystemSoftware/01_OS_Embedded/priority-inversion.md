## 개요

우선순위 역전(Priority Inversion)은 실시간 시스템에서 낮은 우선순위 프로세스가 자원을 점유한 채 높은 우선순위 프로세스를 간접적으로 차단하고, 중간 우선순위 프로세스가 먼저 실행되는 현상이다. 1997년 Mars Pathfinder 임무에서 실제로 발생하여 시스템 리셋을 유발한 사례로 유명하다.

## 핵심 내용

### 우선순위 역전 발생 메커니즘

```
시나리오: 우선순위 H > M > L

1. L이 공유 자원(mutex) 획득
2. H가 깨어나 자원 요청 → 차단(L 대기)
3. M이 실행 가능 → H보다 먼저 실행 (우선순위 역전!)
4. M 실행 중 L은 대기 → H는 더 오래 차단
```

**발생 원인**: 뮤텍스(Mutex)/세마포어 사용 시 낮은 우선순위 프로세스가 자원 해제 전에 선점당하는 상황.

### PIP vs PCP 비교

| 구분 | PIP (Priority Inheritance Protocol) | PCP (Priority Ceiling Protocol) |
|------|------------------------------------|---------------------------------|
| **개념** | 자원 보유 프로세스가 대기 중인 최고 우선순위를 임시로 상속 | 자원에 최대 우선순위 상한값(Ceiling) 사전 설정, 획득 시 즉시 상승 |
| **동작** | H가 자원 대기 → L이 H의 우선순위로 실행 | 자원 획득 순간 Ceiling 우선순위로 설정 |
| **교착상태 방지** | 미방지 (우선순위 역전만 해결) | 방지 가능 |
| **구현 복잡도** | 단순 | 복잡 (사전에 Ceiling 값 결정 필요) |
| **지연 예측** | 어려움 | 상한 예측 가능 (WCET 분석 용이) |
| **사용 환경** | 일반 RTOS | 항공·자동차 안전 시스템 |

### PIP 동작 예시

```
L 우선순위: 1 (낮음), H 우선순위: 10 (높음)

PIP 적용:
1. L이 mutex 획득 (우선순위 1)
2. H가 mutex 요청 → L의 우선순위를 10으로 상승
3. M(우선순위 5)이 깨어나도 L(임시 10)이 먼저 실행
4. L이 mutex 해제 → 원래 우선순위(1) 복귀 → H 실행
```

## 실무 적용 예시

- Mars Pathfinder(1997): VxWorks RTOS에서 우선순위 역전 발생 → 시스템 리셋. 원격으로 PIP(우선순위 상속) 활성화하여 해결.
- 자동차 ECU: AUTOSAR OS에서 PCP 적용으로 안전 임계 태스크 응답 시간 보장.

## 출제 포인트

- 우선순위 역전의 발생 시나리오 (H→자원대기→M이 선점)
- PIP: 동적 우선순위 상속; PCP: 정적 Ceiling 사전 설정
- PCP가 PIP보다 교착상태 방지에 유리한 이유
