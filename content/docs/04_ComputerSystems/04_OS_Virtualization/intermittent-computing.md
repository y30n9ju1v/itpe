## 개요

인터미턴트 컴퓨팅(Intermittent Computing)은 태양광·RF·진동 등 주변 환경에서 수확한 에너지(Energy Harvesting)로 간헐적으로 동작하는 컴퓨팅 패러다임이다. 배터리가 없거나 충전이 불가능한 극한 환경의 IoT 센서에 적용되며, 전원 중단 후 재개 시 일관성 있는 연산 지속이 핵심 과제다.

## 핵심 내용

### 등장 배경

- 배터리 교체가 불가능한 환경: 체내 삽입 센서, 구조물 내부, 우주·심해
- 초소형 IoT 기기의 에너지 자급 필요성
- 전통적 저전력 컴퓨팅의 한계 (배터리 수명 여전히 유한)

### 핵심 특성

| 특성 | 설명 |
|------|------|
| **간헐적 실행** | 에너지가 충분할 때만 실행, 부족 시 전원 차단 |
| **비휘발성 메모리 활용** | FRAM, MRAM 등을 사용해 전원 차단 후 상태 보존 |
| **체크포인팅** | 실행 진행 상태를 주기적으로 저장, 재개 시 복원 |
| **실행 비결정성** | 전원 차단 시점 예측 불가 → 트랜잭셔널 보장 어려움 |

### 주요 기술 과제

**1. 상태 일관성 (State Consistency)**
- 전원 중단 시 연산이 절반만 완료된 상태 발생
- WAR(Write-After-Read) 해저드: 읽기 후 쓰기 중 전원 차단 시 불일치
- 해결: 트랜잭셔널 메모리, 멱등성(Idempotent) 연산 설계

**2. 체크포인팅 오버헤드**
- 너무 자주 저장 → 에너지 낭비
- 너무 드물게 저장 → 재실행 비용 증가
- 해결: 적응형 체크포인팅(배터리 잔량 기반 동적 조정)

**3. 에너지 예측**
- 하베스팅 에너지의 가용성 예측 어려움
- 해결: Task-based 프로그래밍 모델 (에너지 단위로 작업 분할)

### 에너지 하베스팅 소스

| 소스 | 전력 밀도 | 적용 환경 |
|------|-----------|-----------|
| 태양광 | ~10mW/cm² (실외) | 야외 센서 |
| RF | ~1μW~1mW | 무선 충전, NFC |
| 진동/압전 | ~0.1mW/cm² | 기계 진동 환경 |
| 열전 | ~40μW/cm² | 체온, 산업 폐열 |

### 대표 연구 및 플랫폼

- **Mementos** (MIT): MSP430 기반 체크포인팅 런타임
- **TICS** / **Alpaca**: 태스크 기반 인터미턴트 프로그래밍 모델
- **nRF-Harvester**: Nordic Semiconductor의 에너지 하베스팅 플랫폼

## 출제 포인트

- 4가지 에너지 하베스팅 소스와 전력 밀도
