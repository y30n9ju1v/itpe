## 개요

병렬 컴퓨팅(Parallel Computing)은 하나의 문제를 여러 프로세서·코어가 동시에 처리하여 성능을 향상시키는 컴퓨팅 패러다임이다. 단일 프로세서의 물리적 속도 한계를 극복하고, AI·빅데이터·과학 계산의 고성능 처리 요구에 대응한다.

## 핵심 내용

### 가. 병렬 컴퓨팅 개념

- **정의**: 여러 프로세서 또는 컴퓨터가 협력하여 하나의 문제를 동시에 분할 처리
- **등장 배경**: 무어의 법칙 한계(단일 코어 주파수 정체), AI·빅데이터의 고성능 처리 수요
- **암달의 법칙(Amdahl's Law)**: 직렬 부분 비율이 병렬화 속도 향상의 한계를 결정
  - S = 1 / (P + (1-P)/N) — P: 병렬화 불가 비율, N: 프로세서 수

### 나. 병렬 프로세서의 분류 (Flynn's Taxonomy)

| 분류 | 명칭 | 명령어 스트림 | 데이터 스트림 | 예시 |
|------|------|------------|------------|------|
| **SISD** | 단일 명령·단일 데이터 | 1 | 1 | 기존 단일 코어 CPU |
| **SIMD** | 단일 명령·다중 데이터 | 1 | 多 | GPU, 벡터 프로세서, MMX/SSE |
| **MISD** | 다중 명령·단일 데이터 | 多 | 1 | 이론적 분류, 실용 예시 거의 없음 |
| **MIMD** | 다중 명령·다중 데이터 | 多 | 多 | 멀티코어 CPU, 클러스터 |

### 병렬 처리 구현 방식

| 방식 | 설명 | 예시 |
|------|------|------|
| **비트 수준 병렬** | 워드 크기 확대 (8→16→32→64bit) | 64비트 CPU |
| **명령어 수준 병렬(ILP)** | 파이프라인, 슈퍼스칼라, OOO 실행 | 현대 CPU |
| **작업 수준 병렬(TLP)** | 여러 스레드·프로세스 동시 실행 | 멀티코어, 하이퍼스레딩 |
| **데이터 병렬** | 동일 연산을 다른 데이터에 동시 적용 | GPU AI 학습 |

### 메모리 구조에 따른 분류

| 구분 | 공유 메모리 | 분산 메모리 |
|------|-----------|-----------|
| **구조** | 모든 프로세서가 단일 메모리 공유 | 각 프로세서 전용 메모리 |
| **통신** | 공유 변수 직접 접근 | 메시지 패싱 (MPI) |
| **일관성** | 캐시 일관성 문제 | 없음 |
| **확장성** | 낮음 | 높음 (수천 노드 가능) |
| **예시** | SMP (대칭 멀티프로세서) | 클러스터, 슈퍼컴퓨터 |

## 출제 포인트

- Flynn 분류 4가지: SISD·SIMD·MISD·MIMD와 각 예시
- 암달의 법칙: 직렬 부분이 병렬화 한계 결정
- SIMD = GPU의 기본 동작 원리 (벡터 연산)
- 공유 메모리 vs 분산 메모리 아키텍처
