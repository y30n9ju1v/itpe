## 개요

시스템 버스(System Bus)는 CPU, 메모리, I/O 장치 간 데이터·주소·제어 신호를 전달하는 공유 통신 경로다. 버스는 공유 자원이므로 동시 접근 충돌을 방지하기 위한 버스 중재(Bus Arbitration) 메커니즘이 필수다.

## 핵심 내용

### 시스템 버스 구성

| 버스 종류 | 신호 방향 | 역할 |
|----------|----------|------|
| **주소 버스** | 단방향 (CPU → 장치) | 메모리·I/O 주소 전송 |
| **데이터 버스** | 양방향 | 실제 데이터 전송 (8/16/32/64비트) |
| **제어 버스** | 양방향 | 읽기/쓰기, 인터럽트, 버스 요청 신호 |

### 버스 중재 방식

#### 1. 집중식 중재 (Centralized Arbitration)

**데이지 체인(Daisy Chain)**
```
CPU(중재기) → 장치1 → 장치2 → 장치3 (직렬 연결)
```
- 장치1이 Bus Grant 받으면 사용, 미사용 시 장치2로 전달
- 장점: 단순, 배선 적음
- 단점: 장치1에 우선권 고정, 장치3 기아 가능

**폴링(Polling) 방식**
- 중재기가 순서대로 각 장치에 버스 요청 확인
- 유연하지만 폴링 오버헤드 발생

**독립 요청(Independent Request)**
- 각 장치가 별도의 요청·허가 신호선 보유
- 중재기가 우선순위 결정
- 빠르지만 배선 많음

#### 2. 분산식 중재 (Distributed Arbitration)

- 각 장치가 자체적으로 버스 접근 결정 (중재기 없음)
- 자기 선택 방식: 장치 ID 비교로 최고 우선순위 결정
- 예: SPI 버스의 기본 개념, IEEE 1394 (FireWire)

### 버스 성능 지표

- **버스 대역폭** = 버스 폭(bit) × 버스 클럭(Hz) / 8
- **버스 점유율**: 사용 시간 / 전체 시간
