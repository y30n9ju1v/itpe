## 개요

세그멘테이션 오류(Segmentation Fault, Segfault)는 프로세스가 접근 권한이 없는 메모리 영역에 접근하거나, 잘못된 메모리 주소를 참조할 때 운영체제가 해당 프로세스를 강제 종료시키는 오류다. 유닉스/리눅스 계열에서 SIGSEGV 시그널로 처리되며, 주로 C/C++ 포인터 오류에서 발생한다.

## 핵심 내용

**발생 원인**

| 원인 | 설명 | 예시 |
|------|------|------|
| NULL 포인터 역참조 | NULL이 가리키는 주소 접근 | `int *p = NULL; *p = 5;` |
| 댕글링 포인터 | 해제된 메모리 재접근 | free() 후 포인터 사용 |
| 버퍼 오버플로우 | 배열 범위 초과 접근 | `arr[10]` (크기 5인 배열) |
| 스택 오버플로우 | 무한 재귀 등으로 스택 한계 초과 | 종료 조건 없는 재귀 |
| 읽기전용 영역 쓰기 | 코드·상수 영역에 쓰기 시도 | 문자열 리터럴 수정 |

**OS의 메모리 보호 메커니즘**
- 메모리 세그멘테이션: 각 세그먼트(코드·데이터·스택·힙)에 접근 권한(R/W/X) 설정
- 가상 메모리: 프로세스별 독립된 주소 공간 제공
- MMU(Memory Management Unit): 가상→물리 주소 변환 및 접근 권한 검사
- 잘못된 접근 감지 시 OS가 SIGSEGV 시그널 발생 → 프로세스 종료

**보안적 함의**
- 버퍼 오버플로우는 스택/힙 스매싱 공격의 기반
- 제어 흐름 하이재킹(Return Address 덮어쓰기) → 임의 코드 실행
- 대응: ASLR(주소 공간 랜덤화), Stack Canary, DEP/NX(실행 방지)

**디버깅 방법**
- GDB, Valgrind (메모리 오류 탐지)
- AddressSanitizer (컴파일러 수준 메모리 오류 탐지)

## 출제 포인트

- 세그멘테이션 오류의 정의와 발생 원인 분류
- OS의 메모리 보호 메커니즘 (MMU·가상 메모리)
- 보안 취약점으로서의 버퍼 오버플로우와 대응 기법
