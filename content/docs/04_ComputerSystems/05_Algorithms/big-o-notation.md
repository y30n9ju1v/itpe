## 개요

알고리즘 복잡도(Algorithm Complexity)는 입력 크기 n에 따른 연산 횟수(시간복잡도) 또는 메모리 사용량(공간복잡도)을 수학적으로 표현한다. O-Notation(Big-O 표기법)은 최악의 경우 성능 상한선을 나타내는 표기법으로 알고리즘 비교의 표준이다.

## 핵심 내용

### 복잡도의 종류

| 구분 | 의미 | 측정 지표 |
|------|------|----------|
| **시간복잡도** | 연산 횟수의 입력 크기 의존성 | 비교·대입 연산 횟수 |
| **공간복잡도** | 메모리 사용량의 입력 크기 의존성 | 사용 변수·자료구조 크기 |

### O-Notation (점근 표기법)

| 표기 | 명칭 | 의미 |
|------|------|------|
| **O(f(n))** | Big-O | 최악 경우 상한선 (주로 사용) |
| **Ω(f(n))** | Big-Omega | 최선 경우 하한선 |
| **Θ(f(n))** | Big-Theta | 평균 경우 (상·하한 동일) |

### 주요 복잡도 유형과 연산 시간

| O-Notation | 명칭 | 대표 알고리즘 | n=1000 연산 수 |
|------------|------|-------------|--------------|
| **O(1)** | 상수 | 배열 인덱스 접근, 해시 테이블 조회 | 1 |
| **O(log n)** | 로그 | 이진 탐색, AVL 트리 조회 | ~10 |
| **O(n)** | 선형 | 순차 탐색, 배열 순회 | 1,000 |
| **O(n log n)** | 선형 로그 | 퀵정렬(평균), 병합정렬, 힙정렬 | ~10,000 |
| **O(n²)** | 이차 | 버블·선택·삽입 정렬, 이중 반복문 | 1,000,000 |
| **O(n³)** | 삼차 | 행렬 곱셈(기본), Floyd-Warshall | 10⁹ |
| **O(2ⁿ)** | 지수 | 부분집합 생성, 배낭 문제(완전탐색) | 10³⁰⁰ |
| **O(n!)** | 팩토리얼 | 순열 생성, TSP 완전탐색 | ∞ |

**성능 비교**: O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

### 정렬 알고리즘 복잡도 비교

| 알고리즘 | 최선 | 평균 | 최악 | 공간 | 안정성 |
|----------|------|------|------|------|--------|
| 버블 정렬 | O(n) | O(n²) | O(n²) | O(1) | 안정 |
| 선택 정렬 | O(n²) | O(n²) | O(n²) | O(1) | 불안정 |
| 삽입 정렬 | O(n) | O(n²) | O(n²) | O(1) | 안정 |
| 병합 정렬 | O(n log n) | O(n log n) | O(n log n) | O(n) | 안정 |
| 퀵 정렬 | O(n log n) | O(n log n) | O(n²) | O(log n) | 불안정 |
| 힙 정렬 | O(n log n) | O(n log n) | O(n log n) | O(1) | 불안정 |

### 복잡도 계산 규칙

- **상수 제거**: O(3n) → O(n)
- **낮은 차수 제거**: O(n² + n) → O(n²)
- **최고 차수 항 유지**: O(n³ + n² + n) → O(n³)
- **반복문 중첩**: 이중 반복문 → O(n²), 삼중 → O(n³)
- **반으로 줄이는 분할**: O(log n)

## 출제 포인트

- O-Notation의 3가지 표기법(O·Ω·Θ) 의미 구분
- 8가지 복잡도 유형을 성능 순서로 나열
- 대표 정렬 알고리즘의 시간·공간 복잡도 표
- 퀵정렬 최악 O(n²) 이유 (피벗 선택 시 이미 정렬된 배열)
